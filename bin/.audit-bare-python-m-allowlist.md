# Bare `python3 -m cortex_command.*` allowlist

Callsites retained on the `python3 -m cortex_command.*` form post-R14 promotion, with rationale.
Each entry must name the file, line range, module, and one of the accepted categories below.

**Categories**:
- `module-execution-pathway-test` — the callsite is specifically exercising the module-execution
  path (i.e. `python3 -m`) as a distinct dispatch vector under test. Replacing it with the
  console-script name would change what the test measures.
- `document-content-invariant` — the callsite is a string literal or docstring that pins
  the invocation text found in a companion document (`implement.md`, skill reference, etc.).
  Updating the string here without updating the companion document would break the invariant
  the test is enforcing; companion document updates are tracked separately.
- `wrapper-script-r14-pending` — the callsite is inside a bin/ shell wrapper that serves as
  the console-script entry point itself. Branch (a) logic (`python3 -m`) becomes redundant once
  `[project.scripts]` entries are installed; cleanup deferred to R14.
