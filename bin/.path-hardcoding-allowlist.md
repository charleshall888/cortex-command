# Path-hardcoding allowlist

Allowlist for `bin/cortex-check-path-hardcoding` (lifecycle 203). Entries here suppress the `PH001` violation for source lines whose bare-prefix path literal is legitimate — typically because the code operates on pre-relocation archived data, the bare prefix appears in narrative prose, or a one-shot migration tooling references the old layout intentionally.

The linter parses this file at load time. **Fail-open**: if this file is absent, the gate runs in strict mode (every detected violation is reported); a missing allowlist is not a failure condition. This matches the `bin/.parity-exceptions.md` precedent.

## Schema (markdown table, 6 columns)

| Column | Constraint |
| --- | --- |
| `file` | Path to the source file, relative to repo root (e.g. `bin/cortex-archive-rewrite-paths`). Bare path, no leading `./`. |
| `line_pattern` | Regex matched against the offending line via `re.search`. Authors who want a literal substring match should escape regex metacharacters (e.g. `Path\("lifecycle"\)`). Unescaped `.` matches any character. |
| `category` | One of `archive-rewriter`, `docstring-narrative`, `migration-script`. |
| `rationale` | ≥30 characters after trim. Forbidden substrings (case-insensitive): `internal`, `misc`, `tbd`, `n/a`, `pending`, `temporary`. Be specific. |
| `lifecycle_id` | Lifecycle ticket / backlog id that introduced the entry. |
| `added_date` | `YYYY-MM-DD`. |

## Category enum

- `archive-rewriter` — operates on pre-relocation archived data; the bare prefix is the FROM side of a rewrite, not a path the code constructs in new work.
- `docstring-narrative` — bare prefix appears in narrative prose inside a docstring, help message, or comment — not in path-construction code.
- `migration-script` — one-shot migration tooling that references the old layout intentionally because that is its operational scope.

Adding an entry is a small architectural decision. Reviewers: confirm `category` enum is correct and `rationale` is specific (not 'internal' or 'misc' — those literals are linter-rejected). Prefer refactoring the offending line to use `Path("cortex/<prefix>")` style over allowlisting; allowlist only when the bare prefix is genuinely load-bearing.
