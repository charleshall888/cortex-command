# Events registry

Source of truth for the `bin/cortex-check-events-registry` static gate (spec R5/R7 of the events.log discipline ticket). Every `"event": "<name>"` literal that appears in skill prompts or orchestrator-round-style prompt corpus (gate-enforced scan surface) must be declared here. Python emission sites are out-of-scan (`scan_coverage: manual`) and listed here for human reference only.

The schema is documented in `docs/internals/events-registry.md`. Required column order matches spec R5: `event_name | target | scan_coverage | producers | consumers | category | added_date | deprecation_date | rationale | owner`.

Adding an entry is an architectural commitment: each row promises a documented consumer. `deprecated-pending-removal` rows are pruned by a follow-up cleanup PR after the 30-day grace window (the `--audit` recipe surfaces stale rows; `owner` identifies bump authority).
