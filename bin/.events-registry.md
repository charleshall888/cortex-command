# Events registry

Source of truth for the `bin/cortex-check-events-registry` static gate (spec R5/R7 of the events.log discipline ticket). Every `"event": "<name>"` literal that appears in skill prompts or orchestrator-round-style prompt corpus (gate-enforced scan surface) must be declared here. Python emission sites are out-of-scan (`scan_coverage: manual`) and listed here for human reference only.

The schema is documented in `docs/internals/events-registry.md`. Required column order matches spec R5: `event_name | target | scan_coverage | producers | consumers | category | added_date | deprecation_date | rationale | owner`.

Adding an entry is an architectural commitment: each row promises a documented consumer. `deprecated-pending-removal` rows are pruned by a follow-up cleanup PR after the 30-day grace window (the `--audit` recipe surfaces stale rows; `owner` identifies bump authority).

## Field-additive schema extensions

Field-additive extensions extend an event's schema without minting a new event-name row. Optional fields are declared here under their owning event; emitters omit the field from the JSON when its value is `None`. Consumers tolerate the absence and treat the field as `None`/missing.

### `sentinel_absence` (Req 7/8, 2026-05-20)

- `source_path:` (string, optional) — original caller-supplied path when the artifact was ad-hoc-snapshotted via `validate_artifact_path(..., allow_adhoc=True)`. Preserved verbatim post-NUL/surrogate validation; ASCII control characters (including newlines) are legal and are JSON-escaped on write by `json.dumps` in `append_event`. Omitted from the emitted JSON when the artifact resolved under `cortex/lifecycle/` or `cortex/research/` (non-adhoc path).
- `snapshot_sha:` (string, optional) — full hex SHA-256 of the ad-hoc snapshot content. Consumed by `cortex clean --adhoc` (Requirement 9) as the retention-pin token: any snapshot whose computed SHA appears in any `events.log` `snapshot_sha:` value is retained regardless of mtime. Omitted from the emitted JSON when the artifact resolved under `cortex/lifecycle/` or `cortex/research/` (non-adhoc path).

### `dispatch_error` (#309 diagnostics, 2026-06-17)

- `cwd:` (string, optional) — the dispatch worktree working directory (`str(worktree_path)`) at the point of failure, added so a failed dispatch's location is diagnosable from the event log alongside the pre-existing `child_stderr`/`exit_code` fields. Emitted only from the two exception error branches in `dispatch_task` (`cortex_command/pipeline/dispatch.py`); omitted on the success and budget-exhausted returns.

### `task_output` (#309 diagnostics, 2026-06-17)

Field-additive extension carrying the final failed dispatch attempt's diagnostics as **distinct** keys (NOT folded into `output`, which is empty on a worker crash), sourced from `RetryResult.last_dispatch_diagnostics`. Emitters omit each field when the bundle is `None` (the success path); the consumer (`render_failed_features` via `_read_last_task_diagnostics`, `cortex_command/overnight/report.py`) tolerates absence.

- `child_stderr:` (string, optional) — redacted, byte-capped stderr tail of the final failed attempt. Omitted on success.
- `exit_code:` (int, optional) — exit code of the final failed attempt; absent for `CLIConnectionError`/timeout (rendered `unknown` by consumers) and on success.
- `cwd:` (string, optional) — dispatch worktree working directory of the final failed attempt. Omitted on success.

### `feature_deferred` (overnight-review-gate-reverts-a-clean Phase 2, 2026-06-23)

Field-additive extension to the review-failure deferral details emitted by `outcome_router.py`. These keys split a *could-not-run review* (the review agent completed, `success=True`, but produced no parseable verdict — the merge is **preserved** on the integration branch for human re-review) from a *genuine dispatch crash* (`success=False` / a raised exception — the merge is **reverted** and the systemic breaker is tripped as a crash). All keys are optional; consumers (`render_deferred_questions` / `render_merged_to_integration` in `cortex_command/overnight/report.py`) read each with `.get(key, default)` for archived-log tolerance.

- `could_not_run:` (bool, optional) — `True` ONLY when the review agent completed but produced no usable verdict (resolved verdict `ERROR` sentinel); the merge is preserved (`merge_reverted=False`). This is the **positive** discriminator the report and integration-PR surfaces key on — never inferred from `merge_reverted=False`, which a genuine crash with a failed revert (or a repair-path `_reset_ff_merge` returning `False`) also produces. Defaults to `False` / absent for `APPROVED`/`REJECTED`/`CHANGES_REQUESTED` and for the genuine-crash path.
- `merge_reverted:` (bool, optional) — whether the live merge commit was reverted. `False` on the preserved could-not-run path; the authoritative revert outcome (`revert_merge(...).success` or `_reset_ff_merge(...)`) on the crash path. A could-not-run preserve and a crash with a failed revert both record `False`, which is why the report disambiguates on `could_not_run`, not on this field.
- `review_dispatch_crashed:` (bool, optional) — set `True` ONLY on a genuine dispatch crash (`success=False` / raised exception). The could-not-run path no longer co-sets this flag (it was previously co-set with the ERROR verdict); `review_dispatch_crashed` now denotes a crash exclusively (overnight-review-gate-reverts-a-clean R6).

### `pipeline_systemic_failure` (overnight-review-gate-reverts-a-clean Phase 2, 2026-06-23)

- `cause_class:` (list[str]) — the trailing-window systemic-failure class labels at the trip. Now admits the new `review_no_artifact` value (could-not-run review) alongside the existing `review_dispatch_crash` (genuine crash). A mixed crash + no-artifact batch surfaces BOTH labels for diagnosis; the threshold counts the aggregate (`SYSTEMIC_FAILURE_THRESHOLD = 3`), the labels distinguish the kinds. Both values are members of `_SYSTEMIC_ERROR_TYPES` so the threshold derivation recognizes them.
