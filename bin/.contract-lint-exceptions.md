# Contract lint exception ledger

Allowlist for `cortex-check-contract` (lifecycle 253). Entries here suppress contract-lint violations for `cortex-*` binaries whose argv parsing is not AST-extractable. Categorical whole-binary exemptions use `*` for both `flag-or-subcommand` and `path-glob`.

The linter parses this file at load time. Schema (markdown table, 7 columns):

| Column | Constraint |
| --- | --- |
| `binary` | Matches regex `cortex-[a-z][a-z0-9-]*`. Bare token, no `bin/` prefix. |
| `flag-or-subcommand` | Flag token (e.g. `--status`), subcommand name, or `*` for whole-binary categorical exemption. |
| `path-glob` | `fnmatch`-style glob against the violation's file path, or `*` for all paths. |
| `category` | Closed enum: `migration-note`, `template-fragment`, `intentional-omission`, or the no-argparse variant (see categories in ledger rows below). |
| `rationale` | ≥30 characters after trim. Forbidden literals (case-insensitive): `internal`, `misc`, `tbd`, `n/a`, `pending`, `temporary`. Be specific. |
| `lifecycle_id` | Lifecycle ticket / backlog id that introduced the entry. |
| `added_date` | `YYYY-MM-DD`. |
