# Specification: Lifecycle and hook hygiene one-offs

## Problem Statement

The lifecycle/discovery token-waste audit originally surfaced four hygiene gaps in this bundle. Critical review of the spec discovered that one of the four (Sub-item 2, scan-lifecycle metrics regen gating) was based on a misread of the hook — the cited targets (L417/L446) are already gated, the proposed predicate (`status:` field in `index.md`) does not exist anywhere in the corpus (0/28 sampled), and the optimization target (all-complete lifecycle dirs) is unreachable because the archive sweep already migrates completed features out of the scanned tree. Sub-item 2 is therefore dropped from this ticket; the surviving three are: (1) lifecycle SKILL.md Step 2 globs the backlog four times for the same feature slug; (3) discovery's no-topic auto-scan (`/cortex-core:discovery` with no argument) has zero invocation evidence across 6 git commits and 33 lifecycle event logs; (4) skill-edit-advisor runs all 5 sub-suites of `just test-skills` on every Edit/Write to any SKILL.md, only 2 of which can regress from a SKILL.md edit. Sub-items 1 and 3 are token-waste cuts that fit epic #187's framing directly; Sub-item 4 is mixed (context-bloat reduction = token-cost-shaped + latency reduction = wall-clock-shaped), bundled for execution efficiency rather than pure token math — the spec, commits, and PR description acknowledge this categorization explicitly. Sub-item numbering preserves the original ticket's labels (1, 3, 4) for traceability; Requirements are renumbered 1–4.

## Edge Cases

- **Filename ambiguity in Sub-item 1**: A `$ARGUMENTS` value like `scan-lifecycle` substring-matches both `170-fix-pre-existing-scan-lifecycle-test-failures-in-tests-test-hookssh.md` and any future `200-test-feature.md` whose title also contains "scan-lifecycle". Expected behavior: the existing resolver returns exit-2 with the candidate list on stderr; Step 1 prose presents the candidates to the user and halts pending selection. This is a behavior improvement over the current silent first-glob-wins.
- **Empty `$ARGUMENTS` in Sub-item 1**: Skip the resolver call entirely; fall through to the existing incomplete-dirs scan at `skills/lifecycle/SKILL.md:48`. Step 1 prose explicitly checks for empty `$ARGUMENTS` before invoking the resolver.
- **Phase-gated branches in Sub-item 1's Step 2**: The four sub-sections need disjoint frontmatter fields (Backlog Status Check reads `status`; Create index.md reads `uuid`+`tags`; Backlog Write-Back needs no frontmatter; Discovery Bootstrap reads `discovery_source`/`research`). A single Step-2-entry frontmatter read pulls all fields each call may use; sub-sections that never run on a given phase simply ignore the loaded fields. If a long session's intervening tool calls cause Claude to drop the parsed frontmatter from working memory, the prose explicitly permits a single targeted re-Read on the resolved file path — savings degrade gracefully rather than failing.
- **Early-exit on Closed lifecycle in Sub-item 1**: When Backlog Status Check returns `status: complete` and the user chooses "Close lifecycle", Step 2 exits before Create index.md / Backlog Write-Back / Discovery Bootstrap can run. The single-read contract is trivially satisfied (frontmatter is read once before the exit).
- **Concurrent `/cortex-core:discovery` invocations in Sub-item 3**: Post-DELETE, calling `/cortex-core:discovery` with no argument errors out at SKILL.md's Step 1 (argument-required validation). Users who recover the auto-scan reference via the `deprecated-auto-scan-2026-05-11` git tag can run the old flow but it is no longer reachable via the skill router; recovery requires manual file restoration + manual skill-router re-wiring + manual fixture restoration (this is a backup, not a one-command rollback — documented in the Rollback Playbook).
- **Pre-existing prose collisions in citation scanner (Sub-item 4)**: After backtick-wrapping `lifecycle/discovery/overnight flows` at `review.md:39`, a live-tree grep confirms no other bare slash-paths of the form `lifecycle/<slug>` exist in `lifecycle/` or `research/` directories (excluding `NON_FEATURE_SUBDIRS` suppression of `references`). Verify by running `uv run pytest tests/test_lifecycle_references_resolve.py` in the verification step.
- **Advisor stdout failure-mode bloat (Sub-item 4)**: When `test-skill-design` or `test-skill-contracts` fails, the captured stderr can exceed 500 characters. The cap truncates with an explicit `... [truncated; run \`just test-skill-design test-skill-contracts\` for full output]` suffix so the agent has the failure signal without the noise.
- **Future citation-grammar drift (Sub-item 4)**: A future skill edit's prose may introduce a phrase like `lifecycle/foo/bar` that re-triggers the same citation-extractor false-positive. The scoped advisor will fail at edit time; the author backtick-wraps the phrase to resolve. If this pattern recurs, file a follow-up to tighten the extractor grammar (out of scope here).
