# Specification: fix-archive-predicate-and-sweep-lifecycle-and-research-dirs

> Background context lives in `research/repo-spring-cleaning/research.md` (epic discovery) and `lifecycle/fix-archive-predicate-and-sweep-lifecycle-and-research-dirs/research.md` (this ticket's research). The discovery covers epic #165 scope across all four children; this spec is scoped strictly to #169.

## Problem Statement

The `lifecycle-archive` recipe at `justfile:212` greps for the JSON-quoted token `"feature_complete"` only, silently skipping ~11 modern lifecycle dirs that emit YAML-form events (`event: feature_complete`). Combined with stale top-level `lifecycle/` and `research/` directories accumulated since the discovery's "37 dirs" baseline, the cortex-command repo currently renders ~44 disposition-eligible lifecycle dirs and 33 research dirs at root — a clutter surface visible to potential installers browsing the GitHub repo before deciding to `uv tool install` cortex. The fix is a predicate update + bounded sweep that protects live cross-references, preserves historical-record integrity in archived research artifacts, and adds a regression-resistant verification gate so future archive sweeps stay low-anxiety.

## Slug-and-citation grammar (used by multiple requirements)

A `<slug>` matches `re.fullmatch(r"[a-z0-9][a-z0-9-]*", slug)` (canonical definition; referenced by R3 and R10).

R10's citation extraction recognizes exactly five forms:

- **Slash-path body or frontmatter path field**: `lifecycle/<slug>/<path>` — anchored as `(?:^|[^A-Za-z0-9_/-])lifecycle/(?P<slug>[a-z0-9][a-z0-9-]*)(?:/[\w./-]+)?` (matches `spec: lifecycle/foo/spec.md`, `discovery_source: lifecycle/foo/research.md`, body-text `lifecycle/foo`).
- **Wikilink with path**: `\[\[lifecycle/(?P<slug>[a-z0-9][a-z0-9-]*)(?:/[^\]\|#]+)(?:#[^\]\|]+)?(?:\|[^\]]+)?\]\]` (matches `[[lifecycle/foo/spec]]`, `[[lifecycle/foo/spec#section]]`, `[[lifecycle/foo/spec|alias]]`).
- **Wikilink slug-only**: `\[\[lifecycle/(?P<slug>[a-z0-9][a-z0-9-]*)(?:#[^\]\|]+)?(?:\|[^\]]+)?\]\]` (matches `[[lifecycle/foo]]`, `[[lifecycle/foo#section]]`, `[[lifecycle/foo|alias]]`).
- **Wikilink with `.md` suffix**: `\[\[lifecycle/(?P<slug>[a-z0-9][a-z0-9-]*)\.md(?:#[^\]\|]+)?(?:\|[^\]]+)?\]\]` (matches `[[lifecycle/foo.md]]`).
- **Bare-slug frontmatter field `lifecycle_slug:`**: extracted only when the field is a non-empty scalar string matching `<slug>`. Field values of `null`, empty string, or absent are skipped (no reference). Field values that are lists or non-string scalars cause the test to fail with a `malformed lifecycle_slug:` message naming the file path.

Wikilinks of the form `[[<slug>]]` (no `lifecycle/` prefix) are out of scope — cortex's wikilink convention prefixes with `lifecycle/` per existing artifact patterns.

## Requirements

1. **R1 — Archive predicate matches JSON-form and YAML-form events**: `justfile:212`'s grep predicate is replaced with `grep -qE '"event":[[:space:]]*"feature_complete"|^[[:space:]]*event:[[:space:]]*feature_complete[[:space:]]*$'`, and the immediately-preceding justfile line is a comment naming both event formats. Acceptance: `grep -nE '"event":\[\[:space:\]\]\*"feature_complete"\|\^\[\[:space:\]\]\*event:\[\[:space:\]\]\*feature_complete\[\[:space:\]\]\*\$' justfile` returns exactly 1 line at line 212; the immediately-preceding line contains a comment naming both event formats.

2. **R2 — Helper accepts `--exclude-dir`**: `bin/cortex-archive-rewrite-paths` exposes a repeatable `--exclude-dir` argparse flag. Each value is normalized to a root-relative `Path` (handling trailing slash, leading `./`, and absolute paths under `--root`); out-of-root paths are rejected with a clear stderr message and exit code 2. The normalized values are merged into the existing `EXCLUDED_REL_PREFIXES` set at runtime. Acceptance: `python3 bin/cortex-archive-rewrite-paths --slug foo --exclude-dir research/repo-spring-cleaning --exclude-dir ./research/opus-4-7-harness-adaptation/ --dry-run --root .` exits 0 with NDJSON stdout listing zero rewrites in either excluded dir; `python3 bin/cortex-archive-rewrite-paths --slug foo --exclude-dir /tmp/outside --dry-run --root .` exits 2 with stderr matching `out-of-root`.

3. **R3 — Helper rejects malformed slugs**: `bin/cortex-archive-rewrite-paths` validates each `--slug` value with `re.fullmatch(r"[a-z0-9][a-z0-9-]*", slug)`; non-matching values cause exit 2 with a stderr message naming the offending slug before any file walk runs. Acceptance: `python3 bin/cortex-archive-rewrite-paths --slug ../etc --dry-run --root .` exits 2 with stderr matching `invalid slug`; the helper does not open any file.

4. **R4 — `research/archive/` is a recognized exclusion prefix**: When F-10 creates `research/archive/`, the helper's `EXCLUDED_REL_PREFIXES` tuple is extended to include `Path("research/archive")` as a paired change in commit 1. Acceptance: `grep -n "research/archive" bin/cortex-archive-rewrite-paths` returns at least 1 line referencing the EXCLUDED_REL_PREFIXES tuple.

5. **R5 — Unified disposition table + externally-verifiable F-9c re-validation**: F-9b produces a single committed disposition manifest at `lifecycle/fix-archive-predicate-and-sweep-lifecycle-and-research-dirs/disposition.md` covering BOTH lifecycle dirs (for F-9c) AND research dirs (for F-10). The manifest's body is a markdown table with frontmatter `manifest_sha256: <sha256>` recording the sha256 of the table content (excluding frontmatter). Before F-9c executes, the executor (a) runs `just lifecycle-archive --dry-run`, captures stdout to a tempfile, and computes its sha256; (b) reads the manifest's `manifest_sha256` and re-computes the manifest body's current sha256; (c) extracts the lifecycle-partition candidate set from both the dry-run stdout and the manifest, asserts they are equal as sets. Both the dry-run stdout sha256 and the manifest body sha256 are written to events.log. Any divergence (set inequality, manifest sha256 mismatch, or empty dry-run output) aborts execution and forces a manifest refresh + re-commit before retry. **R5 is externally verifiable**: a reviewer reads the events.log entry, re-runs the dry-run on the same commit, and recomputes the sha256s — they must match. Acceptance: events.log entry for F-9c executor contains BOTH `dry_run_stdout_sha256: <hex>` AND `manifest_body_sha256: <hex>` AND `dry_run_match: true`; running `just lifecycle-archive --dry-run | shasum -a 256` on the F-9c commit's parent matches the stored `dry_run_stdout_sha256`.

6. **R6 — F-9c sweep executes**: After R5 passes, the predicate-eligible lifecycle dirs (~30 under the new predicate) are moved to `lifecycle/archive/<slug>/` via `just lifecycle-archive`; the recipe invokes `bin/cortex-archive-rewrite-paths` with the consolidated `--exclude-dir` set from R7. Plus 4 manually-archived dirs (named in the ticket: `add-playwright-htmx-test-patterns-to-dev-toolchain`, `define-evaluation-rubric-update-lifecycle-spec-template-create-dashboard-context-md`, `run-claude-api-migrate-to-opus-4-7-on-throwaway-branch-and-report-diff`, `clean-up-active-sessionjson-when-overnight-session-transitions-to-phasecomplete`) are moved via `git mv lifecycle/<slug>/ lifecycle/archive/<slug>/` followed by a single targeted `bin/cortex-archive-rewrite-paths --slug <slug>` call per dir. `lifecycle/feat-a/` is removed via `rm -rf`. Acceptance: `ls lifecycle/feat-a/ 2>&1` reports "No such file or directory"; each of the 4 manual dirs and each predicate-eligible dir resolves to an existing directory under `lifecycle/archive/`; `git status` shows the moves staged.

7. **R7 — Unified manifest binds F-9c exclude list to F-10 archive set; both abort on divergence**: The manifest produced by F-9b (per R5) lists the F-10 research-archive set as a separate partition. F-9c's `--exclude-dir` argument set is constructed from this manifest's research partition (plus the two named live dirs `research/repo-spring-cleaning` and `research/opus-4-7-harness-adaptation`). F-10's executor performs the same R5-style re-validation against the same manifest before running its sweep: it re-validates the research partition against the current research/ directory state and asserts set equality. **Binding rule**: if F-10's re-validation diverges from the manifest, F-10 aborts AND the executor halts the sweep with a stderr message instructing the operator to revert F-9c (if already committed), refresh the manifest, re-commit, and replay from F-9c. If F-10 has not yet diverged but the operator detects an issue post-F-9c-pre-F-10, the same revert+refresh+replay cycle applies. Acceptance: events.log entry for F-10 executor contains `manifest_body_sha256: <hex>` matching R5's stored value; `f10_research_set_match: true`; the recorded `--exclude-dir` argument list (also written to events.log) when sorted equals the manifest's research partition (when sorted) plus the two named live dirs; for each excluded dir, `git diff <commit-before-sweep> <commit-after-sweep> -- research/<dir>/research.md` shows zero modifications to `lifecycle/<slug>/...` citation lines.

8. **R8 — F-10 creates `research/archive/` and sweeps stale dirs**: F-10 (a) runs `mkdir research/archive/`, (b) `git mv research/<slug>/ research/archive/<slug>/` for every research dir in the manifest's research partition (per R7's re-validation), and (c) keeps `research/repo-spring-cleaning/` and `research/opus-4-7-harness-adaptation/` at top level. Acceptance: `test -d research/archive/`; `ls research/archive/ | wc -l` ≥ 25; `test -d research/repo-spring-cleaning/ && test -d research/opus-4-7-harness-adaptation/`.

9. **R9 — `cortex init` registers `research/` in `sandbox.filesystem.allowWrite`**: `cortex init` is extended to additionally register the repo's `research/` path. Implementation details:
   - **Target derivation**: in handler step 2, after the existing `lifecycle_target = scaffold.check_symlink_safety(repo_root)` call, derive `research_target = str(repo_root / "research") + "/"` directly (no symlink-safety gate; cortex does not scaffold or own `research/`, so the gate's "future-created from templates" assumption does not apply).
   - **Registration**: in handler step 7, after the existing `settings_merge.register(repo_root, lifecycle_target, home=home)` call, add `settings_merge.register(repo_root, research_target, home=home)`. The two register calls are independent — if the second fails after the first succeeds, the half-registered state persists; `cortex init --update` is idempotent and recovers (existing pattern at `cortex_command/init/handler.py:18–19` continues to apply).
   - **Unregister**: handler step 0 (the `--unregister` branch at `cortex_command/init/handler.py:124–138`) gets a third `settings_merge.unregister` call for `research_target`. The function is no-op on absent entries (per `cortex_command/init/settings_merge.py:281–282`), so a pre-R9 install with no `research/` entry unregisters cleanly.

   Acceptance: after `cortex init` runs in a fresh repo, `~/.claude/settings.local.json`'s `sandbox.filesystem.allowWrite` array contains BOTH the `lifecycle/` and `research/` paths in that order (trailing slashes preserved); `cortex init --unregister` removes both entries; the 8 enumerated test behaviors above each have a passing dual-registration test in `test_settings_merge.py`.

10. **R10 — Integration test asserts every `lifecycle/<slug>` reference resolves**: A new pytest test at `tests/test_lifecycle_references_resolve.py` enumerates every git-tracked `*.md` via `git ls-files '*.md'` (no filesystem walk; no separate exclusion list), and applies the five citation-form regexes defined in the "Slug-and-citation grammar" section above to extract `<slug>` references. For each extracted reference, the test asserts the slug exists at either `lifecycle/<slug>/` or `lifecycle/archive/<slug>/`. **Negative-case fixture**: the test loads a sentinel fixture at `tests/fixtures/lifecycle_references/broken-citation.md` containing one deliberately-broken citation (e.g., `lifecycle/this-slug-does-not-exist`); a parametrized test variant runs the same extraction over the fixture and asserts the test detects the broken citation (test exits non-zero with a message naming the file:line:slug). **Coverage assertion**: the test reports the total number of resolved references; the test's assertions include `total_resolved >= 50` (cortex repo currently has ≥50 such references; this prevents the test from passing with zero file matches due to a regex bug). **Form-coverage assertion**: the test logs a per-form match count (slash-path, wikilink-with-path, wikilink-slug-only, wikilink-md-suffix, lifecycle_slug-frontmatter); each of the five forms must have ≥1 match in the cortex repo's tracked `*.md`. Acceptance: `pytest tests/test_lifecycle_references_resolve.py` exits 0 after F-9c + F-10 sweeps land; the test's stdout (or pytest report) shows `total_resolved >= 50` and ≥1 match per form; the negative-case parametrized variant runs and proves the test detects the sentinel broken citation.

## Edge Cases

- **macOS BSD `grep -E` portability**: the predicate uses `[[:space:]]*` and `^...$` anchors; `\b` is intentionally avoided per the existing helper convention at `bin/cortex-archive-rewrite-paths:14–19`.
- **events.log emitter writes `feature_complete` with extra spacing** (e.g., `"event" : "feature_complete"`): the regex requires `"event":[[:space:]]*` (no allowance before the colon). This pattern is not currently emitted by any cortex-command code path; if a future emitter introduces it, the predicate must be widened (out of scope here, tracked as a follow-up).
- **events.log line containing prose that mentions `feature_complete`** (e.g., a `findings:` array element string): the regex's left alternative requires the `"event":` key prefix and the right alternative anchors `^...$`; both reject prose mentions.
- **Active worktrees**: `git worktree list` output is cross-checked at F-9c execution; any candidate slug with an active worktree is skipped (per the existing recipe behavior at `justfile:198–209`).
- **A backlog ticket renamed/archived between F-9b commit and F-9c execution**: R5's re-validation step catches this; the sweep aborts and the manifest is refreshed.
- **F-10 disposition drift between F-9b commit and F-10 execution** (e.g., a research dir gains a live cross-ref or is reclassified): R7's binding-rule re-validation catches this; F-10 aborts and the operator reverts F-9c, refreshes the manifest, re-commits, and replays.
- **Sandbox `allowWrite` for `research/` not yet registered when F-10 runs**: F-10's executor reads `~/.claude/settings.local.json` (or the equivalent home location) and asserts the `research/` entry is present BEFORE running `mkdir research/archive`. If absent, F-10 aborts with a stderr message instructing the operator to run `cortex init --update` and retry. The executor does not silently re-run `cortex init` on the operator's behalf.
- **Pre-commit hook auto-mirror failure on commit 1**: if the hook fails to regenerate the plugin mirror, commit 1 fails atomically; partial state is impossible at the helper-binary level. Operator re-runs after fixing the hook issue.
- **Trailing slash, leading `./`, absolute path in `--exclude-dir`**: R2 normalizes all three to root-relative `Path` objects.
- **Backlog ticket body still cites a now-archived slug after the sweep**: R10's integration test catches this; the rewrite recipe handled body-text and frontmatter `spec:`/`discovery_source:` fields.
- **#166 lands but is reverted before #169 runs**: R12's externally-verifiable precondition gate catches the regression — `cli.py:268` reverts to the old string, the captured `cli_py_268_content` reflects the revert, the substring check fails, F-9c/F-10 abort.
- **R9 partial-failure during register**: lifecycle/ register succeeds, research/ register fails — half-registered state persists; `cortex init --update` is idempotent and adds the missing `research/` entry on retry.
- **Pre-R9 install + plain `cortex init`**: refuses via existing R6 marker-decline gate. User must run `cortex init --update` to add the `research/` entry (no auto-migration on plain init).
- **lifecycle_slug field is `null`, empty string, or absent in a backlog file**: R10's test treats as no-reference (skip); does not flag as broken.
- **lifecycle_slug field is a list or non-string scalar**: R10's test fails with a `malformed lifecycle_slug:` message naming the file path.
