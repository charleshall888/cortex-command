# Specification: Instrument orchestrator-round subprocess with token-cost telemetry

## Problem Statement

Ticket 104's per-skill aggregator (`compute_skill_tier_dispatch_aggregates` in `cortex_command/pipeline/metrics.py`) groups dispatch records by `skill` to surface per-skill token costs, but the orchestrator-round subprocess — the top-level `claude -p` invocation that hosts each round's orchestrator session — emits no dispatch records, so the aggregator never produces an `orchestrator-round` bucket. The orchestrator subprocess's `claude -p` token costs are invisible to the per-skill rollup, making prompt-template regressions undetectable and forcing prior baseline measurements (ticket 111's R11) to fall back to static character-count estimates. This feature instruments `_spawn_orchestrator` to capture token usage from `claude -p --output-format=json`, emits `dispatch_start` plus either `dispatch_complete` (success-shaped envelope) or `dispatch_error` (error-shaped envelope or non-zero exit) records into the per-session `pipeline-events.log`, and extends the `Skill` Literal so the existing aggregator surfaces an `orchestrator-round,<tier>` bucket without any aggregator-side change.

## Requirements

1. **`_spawn_orchestrator` redirects subprocess stdout to a session-scoped file**: Replace `stdout=subprocess.PIPE` with `stdout=open(<session_dir>/orchestrator-round-{round_num}.stdout.json, "wb")` and add `--output-format=json` to the argv. Acceptance: (a) `grep -A 30 "def _spawn_orchestrator" cortex_command/overnight/runner.py | grep -c "subprocess.PIPE"` = 0 (PIPE removed from the orchestrator spawn site); (b) `grep -A 30 "def _spawn_orchestrator" cortex_command/overnight/runner.py | grep -c "\-\-output-format=json"` ≥ 1 (output-format flag present within the function body); (c) `grep -c "orchestrator-round-{round_num}" cortex_command/overnight/runner.py` ≥ 1 (file path uses the round-number variable).

2. **Emit `dispatch_start` between the dry-run gate and `_spawn_orchestrator`**: The `pipeline_log_event` call MUST be placed AFTER the `if dry_run: ... continue` branch at runner.py:1612-1617 and BEFORE `_spawn_orchestrator(...)` at runner.py:1626 — placing it before the dry-run gate would silently start writing per-session telemetry during every dry-run and violate R8. Use `from cortex_command.pipeline.state import log_event as pipeline_log_event` (precedent at runner.py:1301-1318); write to `<session_dir>/pipeline-events.log`. Required fields: `event="dispatch_start"`, `feature="<orchestrator-round-{round_num}>"`, `skill="orchestrator-round"`, `complexity=<tier>`, `criticality="medium"`, `model=null` (model is unknown at dispatch time; the orchestrator subprocess uses whatever the user's `claude` CLI is configured to use, which the runner does not query upfront), `attempt=1`. Acceptance: a unit test in `cortex_command/overnight/tests/test_orchestrator_round_telemetry.py` that invokes the round loop with a mocked subprocess and asserts a `dispatch_start` record with `skill=="orchestrator-round"` and `model` is `None` (or absent) is appended to the per-session `pipeline-events.log`. Pass: `pytest cortex_command/overnight/tests/test_orchestrator_round_telemetry.py -k dispatch_start` exit code = 0. A second assertion in the same test verifies that with `dry_run=True`, no records are appended to `pipeline-events.log`.

3. **Emit `dispatch_complete` (success envelope) or `dispatch_error` (error envelope or non-zero exit) after `_poll_subprocess` returns**: After `_poll_subprocess` returns, parse the stdout file via `json.loads(file.read())`. Branch:
   - **Success envelope**: when `_poll_subprocess` returned 0 AND the parsed envelope is a dict AND `envelope.get("is_error")` is falsy AND `envelope.get("subtype", "").startswith("error_") is False`, emit `event="dispatch_complete"`.
   - **Error envelope or non-zero exit**: otherwise (non-zero exit, error-shaped envelope, or partial parse), emit `event="dispatch_error"` (a known event type — `pair_dispatch_events` handles dispatch_error pairing per metrics.py:439).
   - **Parse failure**: if `json.loads` raises, emit `event="dispatch_error"` with `details={"reason": "parse_failure"}` (fire-and-forget) — the orphan dispatch_start would otherwise sit unmatched, and dispatch_error pairs cleanly.

   Field extraction (defensive, mirroring `pressure_runner.py`'s `data.get(...)` pattern): use `.get()` chains throughout, not rigid `envelope["usage"]["input_tokens"]`. Required `dispatch_complete` fields: `event`, `feature="<orchestrator-round-{round_num}>"`, `cost_usd=envelope.get("total_cost_usd")`, `duration_ms=envelope.get("duration_ms")`, `num_turns=envelope.get("num_turns")`, `model=envelope.get("model") or envelope.get("model_id")`, plus token fields extracted from `envelope.get("usage", {})`: `input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`. Cache fields may be absent for non-cached sessions; persist `None` when missing.

   `dispatch_error` carries the same field set with whatever values are extractable; missing fields are `None`.

   Acceptance: unit tests assert (a) `dispatch_complete` is emitted with `input_tokens` and `output_tokens` populated and cache fields populated-or-None for the success fixture (`cortex_command/overnight/tests/fixtures/orchestrator_envelope_success.json`); (b) `dispatch_error` is emitted with `is_error` source recorded for the error fixture (`cortex_command/overnight/tests/fixtures/orchestrator_envelope_error.json`); (c) `dispatch_error` is emitted on a malformed-JSON fixture with `details["reason"] == "parse_failure"`. Pass: `pytest cortex_command/overnight/tests/test_orchestrator_round_telemetry.py -k "dispatch_complete or dispatch_error or parse_failure"` exit code = 0.

4. **`Skill` Literal extended with `"orchestrator-round"` plus inline documentation comment**: Add `"orchestrator-round"` to the closed Literal at `cortex_command/pipeline/dispatch.py:156`. An inline comment immediately above or beside the value must read `# documentation-only; emitted via pipeline.state.log_event from runner.py and never passed to dispatch_task` (or wording with the same content). The aggregator does not gate on Literal membership (`metrics.py:668` buckets unknown skills as `"legacy"`), so this entry is canonical-vocabulary documentation. Acceptance: (a) `python -c "from cortex_command.pipeline.dispatch import Skill; from typing import get_args; assert 'orchestrator-round' in get_args(Skill)"` exits 0; (b) `grep -A 1 "orchestrator-round" cortex_command/pipeline/dispatch.py | grep -c "documentation-only"` ≥ 1.

5. **Aggregator surfaces an `orchestrator-round,<tier>` bucket end-to-end and the report formatter renders it**: Integration test feeds a fixture per-session `pipeline-events.log` containing one paired `dispatch_start`/`dispatch_complete` for the orchestrator-round through `discover_pipeline_event_logs` → `pair_dispatch_events` → `compute_skill_tier_dispatch_aggregates`, asserts the returned dict has a key matching `orchestrator-round,<tier>` with non-null `cost_usd` and `num_turns`. A second assertion calls `_format_skill_tier_dispatch_report` (metrics.py:1196) on the resulting aggregate and asserts the rendered string contains the substring `orchestrator-round` (presence-check; column-alignment is not byte-asserted). Acceptance: `pytest cortex_command/overnight/tests/test_orchestrator_round_telemetry.py -k aggregator_bucket` exit code = 0.

6. **Fire-and-forget telemetry contract**: Any exception during stdout-file open, JSON parse, or `pipeline_log_event` call is caught and logged via `print(..., file=sys.stderr, flush=True)` with a `[telemetry] ` prefix, and does not propagate or abort the round loop. Acceptance: unit test that injects a malformed JSON envelope and asserts the round loop completes with exit_code reflecting only the subprocess result (not the parse failure). Pass: `pytest cortex_command/overnight/tests/test_orchestrator_round_telemetry.py -k fire_and_forget` exit code = 0.

7. **Stalled rounds do not poison subsequent rounds' pairing within a single session**: Because `feature` is per-round-unique (`<orchestrator-round-{round_num}>`), a `dispatch_start` without a matching `dispatch_complete` (stalled round) leaves a single orphan in the FIFO queue keyed to that round only — subsequent rounds' starts and completes pair correctly *within the same session log*. Cross-session safety relies on `pair_dispatch_events` being called per-log at metrics.py:1403 (per-file FIFO isolation), not on feature-name uniqueness — see Technical Constraints. Acceptance: integration test constructs `[start_R1 (feature="orchestrator-round-1"), start_R2 (feature="orchestrator-round-2"), complete_R2 (feature="orchestrator-round-2")]`, calls `pair_dispatch_events(events)`, asserts `len(result) == 1` and `result[0]["feature"] == "orchestrator-round-2"`, and asserts `capsys.readouterr().err` contains no `"orphan"` substring (orphan `dispatch_start` records silently stay in the FIFO queue; only orphan `dispatch_complete` and `dispatch_error` emit warnings at metrics.py:405 and metrics.py:439). Pass: `pytest cortex_command/overnight/tests/test_orchestrator_round_telemetry.py -k stalled_round_isolation` exit code = 0.

8. **Dry-run mode emits no telemetry**: When `dry_run=True` (runner.py:1612-1617), no stdout file is created and no `dispatch_start`/`dispatch_complete`/`dispatch_error` records are written. Acceptance: existing dry-run snapshot test (`tests/test_runner_pr_gating.py` filtering to `DRY-RUN ` prefixed lines and applying path/SHA normalization) continues to pass — the test asserts filtered+normalized line equality against `tests/fixtures/dry_run_reference.txt`, not raw stdout byte-equality. Pass: `pytest tests/test_runner_pr_gating.py` exit code = 0. Additional assertion: a unit test exercises the dry-run path explicitly and asserts `<session_dir>/pipeline-events.log` is absent or contains no orchestrator-round dispatch records.

## Non-Requirements

- Stall-failure event emission in the watchdog stall branch (runner.py:1636-1653). The pre-existing behavior is unchanged; per-round-unique feature names make stalled rounds isolate cleanly within a session log without it. File a separate ticket if orphan-start accumulation becomes operationally noisy.
- Updating ticket 111's `verification.md` with a live R11/R12 baseline. The R11 static-estimate-vs-live-measurement comparison is method-divergent and warrants its own follow-up ticket framing the divergence; out of scope for 153.
- Threading cache-token fields (`cache_read_input_tokens`, `cache_creation_input_tokens`) through `pair_dispatch_events`'s output dict. The aggregator's pair output stays uniform across skills (`cost_usd`, `num_turns` only); cache-aware analyses read raw events directly.
- Editing `requirements/pipeline.md:27` ("stdout remains clean as the orchestrator agent's input channel"). The phrasing refers to the runner's own stdout when invoked by an orchestrator agent, not the orchestrator subprocess that the runner spawns.
- Token-cost optimization or model-tier reshuffling of the orchestrator-round itself.
- Instrumentation of the `cortex-batch-runner` subprocess.
- Historical backfill of past sessions.
- Surfacing the new bucket in the dashboard or morning report. The existing aggregator output is enough; downstream consumers can pick it up incrementally.
- Repairing `dry_run_echo`'s pre-existing synthesized argv divergence. The echo at runner.py:1613 already omits `--dangerously-skip-permissions` and `--max-turns` from the real argv; 153 marginally widens the divergence by adding `--output-format=json` to the real argv but does not introduce or fix this design. File a separate ticket if dry-run echo accuracy ever becomes load-bearing.
- Guarding against orchestrator-round events being written to the repo-root `lifecycle/pipeline-events.log`. No emitter exists today that would do this; the spec mandates per-session writes only. A defensive de-dup in `compute_skill_tier_dispatch_aggregates` is scope creep for a hypothetical misconfig.

## Edge Cases

- **JSON parse failure on stdout file**: Catch the exception; emit a `[telemetry]` stderr breadcrumb; emit `dispatch_error` with `details={"reason": "parse_failure"}` so the dispatch_start is paired (no orphan accumulation). Round loop continues normally.
- **Subscription-billed sessions**: `total_cost_usd` may be `0`, `null`, or absent from the envelope. `.get()` chain extracts `None` when absent; the aggregator already tolerates missing cost.
- **Cache fields absent (non-cached session)**: `usage.cache_creation_input_tokens` and `usage.cache_read_input_tokens` may be absent for sessions with no cached blocks. `.get()` extracts `None` when absent; acceptance R3 explicitly tolerates this.
- **Watchdog kill mid-stream**: subprocess SIGKILL'd before envelope flush. The stdout file may be empty or truncated; JSON parse fails; `dispatch_error` with `parse_failure` is emitted (fire-and-forget surrounding the parse).
- **Non-zero clean exit**: existing `events.ORCHESTRATOR_FAILED` event is still emitted at runner.py:1659. The stdout file may contain a partial or error-shaped envelope. Always emit `dispatch_error` (not `dispatch_complete`) on non-zero exit; persist whatever fields are extractable.
- **Error-shaped success envelope** (`is_error: true` or `subtype: "error_*"` with exit code 0): emit `dispatch_error`, not `dispatch_complete`. Prevents pollution of the `orchestrator-round,<tier>` bucket with zero-token records that represent failures.
- **Stdout file already exists** (resume after a crash): overwrite via `open(..., "wb")` mode; do not preserve prior content.
- **Round number 0 or negative**: not possible — `round_num` is initialized from `state.current_round` which is positive in healthy state. Defensive coding not required.
- **Concurrent rounds**: not possible — `_spawn_orchestrator` is called once per round in a strict serial loop. FIFO pairing within one feature key is safe.
- **Round 1 of a fresh session**: no prior records exist; the new emission is the first `dispatch_start` written to the session's `pipeline-events.log`. `pipeline_log_event` already creates parent dirs (atomic write).
- **Envelope top-level not a dict** (list, scalar, NDJSON drift): `isinstance(envelope, dict)` guard before `.get()` calls; on type mismatch, emit a `[telemetry]` stderr breadcrumb naming the actual top-level type and emit `dispatch_error` with `details={"reason": "envelope_shape_drift", "top_level_type": <type>}`.
- **Envelope model field name drift**: `model` may be `model` or `model_id`; extract via `.get("model") or .get("model_id")`; fall back to `None` if neither is present.

Caller closes the file after `_poll_subprocess` returns. Existing return tuple `(proc, wctx, watchdog)` unchanged.
- **MODIFIED: round loop at runner.py:1622-1631** — emits `dispatch_start` via `pipeline_log_event` AFTER the dry-run gate (1612-1617) and BEFORE `_spawn_orchestrator`; emits `dispatch_complete` (success envelope) or `dispatch_error` (non-zero exit, error envelope, or parse failure) after `_poll_subprocess` returns. Per-round-unique `feature` ensures stalled rounds' orphan starts do not poison subsequent rounds' pairing within the same session log.
- **ADDED: `Skill` Literal at dispatch.py:156** — gains `"orchestrator-round"` value with an inline comment marking it documentation-only and never passed to `dispatch_task`.

## Technical Constraints

- Emit telemetry via `from cortex_command.pipeline.state import log_event as pipeline_log_event` (existing precedent at runner.py:1301-1318). Do not route through `cortex_command/pipeline/dispatch.py` — its closed-Literal `Skill` runtime guard at dispatch.py:432 fires only inside `dispatch.py` functions; emitting raw event dicts via `pipeline.state.log_event` bypasses the guard cleanly while still benefiting from atomic JSONL append.
- Write to `<session_dir>/pipeline-events.log` (per-session), not the repo-root `lifecycle/pipeline-events.log`. `discover_pipeline_event_logs` at metrics.py:270 already aggregates both paths; per-session is the convention `feature_executor.py` uses (`config.pipeline_events_path`).
- `feature` field MUST be per-round-unique (`<orchestrator-round-{round_num}>`) to keep `pair_dispatch_events` FIFO matching robust to stalls *within a single session log*. Single-sentinel forms (`<orchestrator-round>`, `None`, empty) cause stalled rounds to mispair subsequent rounds' completes against orphan starts.
- **Cross-session FIFO isolation depends on per-file pairing, not feature-name uniqueness.** `round_num` resets to 1 each session, so the feature-name `<orchestrator-round-1>` is NOT globally unique across sessions. The current aggregator (`metrics.py:1397-1404`) calls `pair_dispatch_events(filtered)` once per log file with that file's events only and a function-local `unmatched_starts` defaultdict (metrics.py:371) — only the paired *result list* is concatenated into `all_paired`. This per-file FIFO isolation is load-bearing for cross-session correctness; a future refactor that concatenates events across logs before pairing would silently break it. Any change to that aggregator call shape requires re-evaluating session-scoped feature naming.
- Field extraction from the JSON envelope MUST use defensive `.get()` chains (`envelope.get("usage", {}).get("input_tokens")`), not rigid attribute or index access. Mirrors the in-repo precedent at `pressure_runner.py` (`data.get('result')`, `.get('text')`, `.get('content')`).
- Envelope shape drift detection: `isinstance(envelope, dict)` guard before any `.get()` calls. On type mismatch, emit `[telemetry] envelope-shape-drift` breadcrumb identifying the actual top-level type, then emit `dispatch_error` with the drift reason recorded in `details`.
- The `--output-format=json` envelope is buffered in full until subprocess exit. Redirecting subprocess stdout to a file (not PIPE) eliminates the OS pipe-buffer-fill deadlock that would otherwise hang long sessions whose envelope exceeds ~64 KB.
- All telemetry I/O is fire-and-forget per `docs/overnight-operations.md`'s `_write_activity_event` contract: swallow exceptions, never block the round loop.
- Watchdog kill semantics are unchanged. The OS closes the stdout file on subprocess death; runner reads whatever was flushed (or treats parse failure as fire-and-forget).
- The subprocess invocation already passes `CORTEX_RUNNER_CHILD=1` (R28); preserve this in the modified argv path.
- No new dependencies. Standard library `json`, `pathlib`, existing `pipeline.state.log_event` and `pipeline.dispatch.Skill`.
- Aggregator side requires no change. `compute_skill_tier_dispatch_aggregates` at metrics.py:624 already buckets by `(skill, tier)`; new records form the `orchestrator-round,<tier>` bucket automatically. `metrics.py:668` falls back to `"legacy"` for unknown skills, so the Skill-Literal extension at R4 is documentation-only — the aggregator works whether or not the new value is in the Literal.
- `dispatch_start` emits `model=null` because the orchestrator subprocess model is unknown before the subprocess runs. On `dispatch_complete` and `dispatch_error`, extract the model from the JSON envelope using `.get("model") or .get("model_id")` to tolerate field-name variations across Claude CLI versions; persist `None` if neither key is present.
- Pinned envelope fixtures (`orchestrator_envelope_success.json`, `orchestrator_envelope_error.json`) capture canonical CLI output verbatim with a comment-header CLI version. Tests assert against these pinned shapes; on shape drift, refresh the fixture (and its comment header) and re-run.
