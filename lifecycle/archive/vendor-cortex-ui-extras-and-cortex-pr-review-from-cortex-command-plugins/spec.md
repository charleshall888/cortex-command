## Problem Statement

The sibling repo `cortex-command-plugins` currently hosts `cortex-ui-extras`, `cortex-pr-review`, and `android-dev-extras`. The first two are useful to most cortex-command users and should live alongside `cortex-interactive` under `plugins/` in this repo. `android-dev-extras` stays external. Because the two vendored plugins never had a top-level source tree (unlike `cortex-interactive`), the existing `just build-plugin` / `.githooks/pre-commit` / drift-test machinery — which assumes every plugin is build-output from top-level sources — must be parameterized to distinguish **build-output** plugins (regenerated from top-level via rsync) from **hand-maintained** plugins (plugin tree is the source of truth; no drift check). After vendoring lands, the external repo's copies of the two plugins will be deleted in a follow-up commit (land-then-delete ordering to avoid a gap window) — tracked in the Follow-ups section, not as a plan requirement.

## Requirements

1. **`plugins/cortex-ui-extras/` exists in-tree with all skills copied verbatim from the external repo.** Acceptance: every path present in `~/Workspaces/cortex-command-plugins/plugins/cortex-ui-extras/` (excluding `.DS_Store` and any `.git*` files) appears identically under `plugins/cortex-ui-extras/`, verified via `rsync -rcn --delete --exclude='.DS_Store' --exclude='.git*' ~/Workspaces/cortex-command-plugins/plugins/cortex-ui-extras/ plugins/cortex-ui-extras/ 2>&1 | grep -vE "^(sending|sent |total |$|plugin\.json|README\.md)" | wc -l` = 0 (only the plugin.json enrichment per R3 and any README additions per R12 differ). Additionally, all six skills are present: `for s in ui-a11y ui-brief ui-check ui-judge ui-lint ui-setup; do test -f plugins/cortex-ui-extras/skills/$s/SKILL.md || exit 1; done`.

2. **`plugins/cortex-pr-review/` exists in-tree; `evidence-ground.sh` preserves the 755 mode bit.** Acceptance: `[ -f plugins/cortex-pr-review/skills/pr-review/SKILL.md ]` AND `[ -f plugins/cortex-pr-review/skills/pr-review/scripts/evidence-ground.sh ]` AND `stat -f "%Lp" plugins/cortex-pr-review/skills/pr-review/scripts/evidence-ground.sh` (macOS) or `stat -c "%a" plugins/cortex-pr-review/skills/pr-review/scripts/evidence-ground.sh` (Linux) outputs exactly `755`. `%Lp` / `%a` return permission-only bits, without the file-type prefix and without a trailing newline ambiguity.

3. **Both vendored plugin.json files carry `name`, `description`, `author` fields matching cortex-interactive's shape; `cortex-ui-extras` additionally carries `"experimental": true`.** Acceptance: `jq -e '.name == "cortex-ui-extras" and (.description | length > 0) and (.author | length > 0) and .experimental == true' plugins/cortex-ui-extras/.claude-plugin/plugin.json` exits 0; `jq -e '.name == "cortex-pr-review" and (.description | length > 0) and (.author | length > 0) and (.experimental // false) == false' plugins/cortex-pr-review/.claude-plugin/plugin.json` exits 0.

4. **`justfile` declares two readonly plugin-policy arrays near the existing SKILLS declaration, and two helper recipes emit each list one name per line.** Acceptance: `just _list-build-output-plugins | sort -u` outputs exactly `cortex-interactive\ncortex-overnight-integration`; `just _list-hand-maintained-plugins | sort -u` outputs exactly `cortex-pr-review\ncortex-ui-extras`. Order within each list is not acceptance-checked; presence of exactly these names (via `sort -u`) is.

5. **`just build-plugin` regenerates only build-output plugins whose plugin directory exists; absent directories are skipped explicitly, not inferred from rsync behavior.** Acceptance: the recipe guards each iteration with an explicit directory check (e.g., `[[ -d plugins/$p/.claude-plugin ]] || { echo "build-plugin: skipping $p (not yet materialized)" >&2; continue; }`); after `just build-plugin`, `[ ! -d plugins/cortex-overnight-integration ]` remains true (no ghost tree materialized) AND `just build-plugin 2>&1 | grep -q "skipping cortex-overnight-integration"` exits 0 AND the recipe's overall exit code is 0. `git status --porcelain plugins/cortex-ui-extras/ plugins/cortex-pr-review/` remains empty.

6. **`.githooks/pre-commit` fails the commit on drift in any build-output plugin tree; does NOT fail on edits to hand-maintained plugin trees.** Acceptance: with a staged edit to `plugins/cortex-interactive/skills/commit/SKILL.md`, `.githooks/pre-commit` exits 1 AND `stderr | grep -q "plugins/cortex-interactive/skills/commit/SKILL.md"` matches; with a staged edit to `plugins/cortex-ui-extras/skills/ui-lint/SKILL.md` (and no other changes), `.githooks/pre-commit` exits 0. The hook iterates the build-output list and runs `git diff --quiet plugins/{name}/` per build-output plugin.

7. **`.githooks/pre-commit` fails-closed when any plugin directory (`plugins/*/.claude-plugin/plugin.json`) is not classified in either policy array.** Acceptance: with a staged new file `plugins/cortex-unclassified/.claude-plugin/plugin.json` containing `{"name":"cortex-unclassified"}`, `.githooks/pre-commit` exits 1 AND `stderr | grep -qE "cortex-unclassified.*(not classified|unclassified|BUILD_OUTPUT_PLUGINS|HAND_MAINTAINED_PLUGINS)"` matches. Directories without `.claude-plugin/plugin.json` are silently skipped (they don't look like plugins).

8. **`.githooks/pre-commit` validates every `plugins/*/.claude-plugin/plugin.json` has a non-empty `name` key, regardless of classification.** Acceptance: with `plugins/cortex-ui-extras/.claude-plugin/plugin.json` staged with its `name` key removed or blanked, `.githooks/pre-commit` exits 1 AND `stderr | grep -q "plugins/cortex-ui-extras/.claude-plugin/plugin.json"` matches AND `stderr | grep -qi "name"` matches.

9. **`.githooks/pre-commit` short-circuits `just build-plugin` when no top-level source paths are staged.** Acceptance: instrument-free verification — with only `backlog/some-file.md` staged and `plugins/cortex-interactive/` unmodified, capture `git diff --stat plugins/cortex-interactive/` before running the hook, run the hook, capture the diff-stat after; the two outputs are byte-identical AND the hook exits 0. The hook inspects `git diff --cached --name-only` and invokes `just build-plugin` only when at least one staged path matches `^skills/`, `^bin/cortex-`, or `^hooks/cortex-validate-commit\.sh$`. The drift-check loop still runs regardless of short-circuit (cheap `git diff --quiet` per plugin).

10. **`tests/test_drift_enforcement.sh` gains subtests C/D/E for hand-maintained plugins and the unclassified guard; existing subtests A and B continue to pass.** Acceptance: running the test script exits 0 with verbose output showing at least five distinct subtest passes (labeled A through E at minimum). (A) build-output drift (top-level `skills/commit/SKILL.md` edit propagated, plugin-tree diff detected) — asserts hook exits 1. (B) build-output drift on hooks (`hooks/cortex-validate-commit.sh` edit) — asserts hook exits 1. (C) hand-maintained edit to `plugins/cortex-ui-extras/skills/ui-lint/SKILL.md` — asserts hook exits 0. (D) hand-maintained edit to `plugins/cortex-pr-review/skills/pr-review/SKILL.md` — asserts hook exits 0. (E) unclassified plugin directory creation — asserts hook exits 1 AND stderr mentions the plugin name. Cleanup uses `git stash push -u -- <paths>` (with `-u` for untracked files in subtest E) followed by `git stash pop`, or equivalent snapshot-restore; unconditional `git restore` is explicitly disallowed. Subtest E's new untracked file `plugins/cortex-unclassified/.claude-plugin/plugin.json` is additionally cleaned via `rm -rf plugins/cortex-unclassified/` in the trap handler, in case stash-with-untracked-flag fails to restore cleanly.

11. **`README.md` documents the four shippable plugins with core/extras framing and marks `cortex-ui-extras` as experimental.** Acceptance: `grep -oE "cortex-(interactive|overnight-integration|ui-extras|pr-review)" README.md | sort -u | wc -l` = 4 (four distinct plugins named) AND `grep -iE "experimental.*cortex-ui-extras|cortex-ui-extras.*experimental" README.md | wc -l` ≥ 1 (experimental marker co-located with ui-extras in the README) AND the README contains either an explicit table header, a subsection heading, or a prose sentence distinguishing core plugins from extras (verified by `grep -iE "core|extras|optional" README.md | head -20` producing at least one match in the plugin section).

- **Do not implement `cortex-overnight-integration`.** Ticket 121 owns it. This spec pre-registers the name in `BUILD_OUTPUT_PLUGINS` so 121's build path works when it lands; it does not create `plugins/cortex-overnight-integration/`.
- **Do not write a user migration guide.** Ticket 124 owns it.

## Edge Cases

- **Staged commit touches only `plugins/` files (not top-level sources).** R9's short-circuit skips the `just build-plugin` invocation. The drift-check loop still runs — since no build-output plugin changed, `git diff --quiet` passes per plugin. Hand-maintained edits in the staged commit pass through unimpeded.
- **Commit stages both a top-level source edit and a hand-maintained plugin edit.** Short-circuit does not fire (top-level sources are staged). `just build-plugin` runs. Drift check fires on build-output plugins only; the hand-maintained edit is not in the diff scope.
- **A new file under `plugins/foo/` but no `plugins/foo/.claude-plugin/plugin.json`.** The fail-closed guard (R7) enumerates `plugins/*/.claude-plugin/plugin.json` (glob). Directories without a plugin.json are silently skipped (they don't look like plugins). A stray README or scratch file under `plugins/` does not trip the guard.
- **`cortex-overnight-integration` is in BUILD_OUTPUT_PLUGINS but the directory doesn't exist yet.** R5's explicit directory guard prevents rsync from creating a ghost tree. The recipe emits `build-plugin: skipping cortex-overnight-integration (not yet materialized)` to stderr and continues.
- **Conflict between DR-2 and ticket 121 on `morning-review` assignment.** DR-2 in the epic research lists `morning-review` under `cortex-interactive`; ticket 121's Scope lists it under `cortex-overnight-integration`. This spec does NOT resolve the conflict — 121's own refine phase will. The pre-registration in R4/R5 only commits to the plugin NAME being in BUILD_OUTPUT_PLUGINS; which skills that plugin owns is 121's call. If 121 lands as hand-maintained instead, 121 will move the name to HAND_MAINTAINED_PLUGINS.
- **`evidence-ground.sh` mode bit after `rsync -a` across a filesystem that doesn't preserve modes.** Unlikely on macOS/Linux local disks. Mitigation: R2 verifies mode bit post-vendor; if it ever fails, add a one-shot `chmod 755` after rsync.
- **Subtest E cleanup fails to restore cleanly.** Primary mechanism is `git stash push -u`. Secondary cleanup in the test trap: `rm -rf plugins/cortex-unclassified/`. If both fail, the next test run's seeding will error loudly, surfacing the residue.
- **Pre-commit sub-recipe invocation cost.** Each `just _list-*-plugins` call re-parses the justfile (~100ms overhead x2 calls per commit). R9's short-circuit mitigates for non-source commits. If measured real-world cost becomes a concern, refactor to a single bash helper reading the arrays via a sourced `.env`-style file — explicit follow-up, not in 144's scope.

## Changes to Existing Behavior

- **MODIFIED**: `just build-plugin` — previously hardcoded to `cortex-interactive` via a single rsync loop. Now iterates `BUILD_OUTPUT_PLUGINS` (two entries: `cortex-interactive`, `cortex-overnight-integration`), with an explicit `[[ -d plugins/$p/.claude-plugin ]] || continue` guard to skip absent directories.
- **MODIFIED**: `.githooks/pre-commit` — previously unconditional `git diff --quiet plugins/cortex-interactive/`. Now: (a) short-circuits `just build-plugin` when no top-level source paths are staged, (b) iterates BUILD_OUTPUT_PLUGINS for per-plugin drift checks, (c) adds fail-closed guard for unclassified plugins (via enumeration of `plugins/*/.claude-plugin/plugin.json`), (d) adds plugin.json `name`-key validation via `jq -e`.
- **MODIFIED**: `tests/test_drift_enforcement.sh` — previously two positive subtests (A, B). Adds three subtests (C, D, E). Cleanup mechanism changes from unconditional `git restore` to `git stash push -u` + secondary `rm -rf plugins/cortex-unclassified/` in the trap.
- **ADDED**: `plugins/cortex-ui-extras/` and `plugins/cortex-pr-review/` directories (~15 files total, preserving `evidence-ground.sh` 755 mode).
- **ADDED**: Two readonly bash arrays in `justfile` (`BUILD_OUTPUT_PLUGINS`, `HAND_MAINTAINED_PLUGINS`) plus two helper recipes (`_list-build-output-plugins`, `_list-hand-maintained-plugins`).
- **ADDED**: `"experimental": true` custom key in `plugins/cortex-ui-extras/.claude-plugin/plugin.json`. Claude Code ignores unknown keys; safe.
- **ADDED**: README documentation of the four-plugin roster with core/extras framing and experimental marker for ui-extras.

## Technical Constraints

- **`rsync -a` is the canonical vendor mechanism**, matching the existing `just build-plugin` recipe's rsync usage. Preserves file modes (critical for `evidence-ground.sh` 755).
- **Policy arrays are bash-only** (not POSIX sh). Both `justfile` recipes and `.githooks/pre-commit` use `#!/bin/bash` explicitly.
- **Dual-source drift enforcement contract from ticket 120 Task 15 is preserved for build-output plugins.** Top-level `skills/`, `bin/cortex-*`, `hooks/cortex-validate-commit.sh` remain canonical; `plugins/cortex-interactive/` remains build-output. This spec does not reopen that decision.
- **`${CLAUDE_PLUGIN_ROOT}` is the correct path variable in plugin-internal hook manifests.** Not relevant to this spec (no new plugin hook registrations).
- **Plugin.json proposed text**:
  - `cortex-ui-extras`: `description` = `"Experimental UI design skills (brief, setup, lint, a11y, judge, check) for Claude Code interactive workflows"`; `author` = `"Charlie Hall <charliemhall@gmail.com>"`.
  - `cortex-pr-review`: `description` = `"Multi-agent GitHub pull request review pipeline for Claude Code"`; `author` = `"Charlie Hall <charliemhall@gmail.com>"`.
- **Pre-commit hook short-circuit filter**: `git diff --cached --name-only --diff-filter=ACMR` piped through `awk '/^(skills\/|bin\/cortex-|hooks\/cortex-validate-commit\.sh$)/'`. If any line matches, run the build.
- **`cortex-overnight-integration` classification coordination note for ticket 121**: this spec pre-registers the name in `BUILD_OUTPUT_PLUGINS`. Ticket 121's refine phase owns the final classification. If 121 specs as hand-maintained (e.g., due to `.mcp.json` + runner-only hooks lacking a canonical top-level source), 121 moves the entry to `HAND_MAINTAINED_PLUGINS`. This is a cheap two-line edit in 121 and does not invalidate 144's implementation.
- **macOS/Linux stat portability**: R2 uses `stat -f "%Lp"` (macOS, permission bits only) or `stat -c "%a"` (Linux); tests and documentation should show both invocations where acceptance criteria are platform-sensitive.
