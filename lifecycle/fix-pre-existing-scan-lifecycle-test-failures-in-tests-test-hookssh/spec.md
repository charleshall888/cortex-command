# Specification: fix-pre-existing-scan-lifecycle-test-failures-in-tests-test-hookssh

## Problem Statement

`hooks/cortex-scan-lifecycle.sh` regressed at commit `ea51d0c` (2026-04-01, "Remove Cursor and Gemini support"): the auto-detection block that set `AGENT="claude"` from input JSON was deleted, but the `if [[ "${AGENT:-}" == "claude" ]]` output-shape gate at lines 460–471 was left intact. Commit `fde7dc6` then papered over the resulting `set -u` failure with `${AGENT:-}` instead of removing the now-dead branch. Since no live caller sets `AGENT` (verified: zero hits across settings files, plugin manifests, CI scripts, the overnight runner, and the status line — only a defensive write in `tests/test_lifecycle_phase_parity.py`), the hook's default branch runs every time and emits `{"additional_context": "..."}` — a key Claude Code does not consume per the canonical hooks schema at code.claude.com/docs/en/hooks. Result: every real Claude Code SessionStart since 2026-04-01 has received no lifecycle context injection from this hook, breaking the "session continuity across `/clear`" guarantee documented in `docs/agentic-layer.md` line 246. Two tests in `tests/test_hooks.sh` (`scan-lifecycle/single-incomplete-feature` and `scan-lifecycle/claude-output-format`) catch this regression by asserting the canonical Claude Code shape; they have been red since at least `c019e97`. Fixing the hook restores SessionStart context injection and turns the suite green.

## Requirements

1. **Hook emits canonical Claude SessionStart shape unconditionally when context is non-empty**: After the change, when `hooks/cortex-scan-lifecycle.sh` has computed a non-empty `$context`, it emits exactly:
   ```json
   {"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":"<context>"}}
   ```
   No env-var branching. No `additional_context` (snake_case) shape. Acceptance: `AGENT= bash hooks/cortex-scan-lifecycle.sh <<<'{"hook_event_name":"SessionStart","session_id":"x","cwd":"/tmp"}' | jq -r 'has("hookSpecificOutput")'` returns `true` from a working directory whose `lifecycle/` contains at least one incomplete feature.

2. **Both failing scan-lifecycle tests pass without modification to assertions**: `tests/test_hooks.sh` lines 114–143 are kept as-is; the hook change alone makes them green. Acceptance: `bash tests/test_hooks.sh` exits 0 and reports `14 passed, 0 failed`.

3. **Empty-context path preserves current behavior**: When `$context` is empty, the hook continues to emit nothing (no JSON, no stderr, exit 0). Acceptance: `bash hooks/cortex-scan-lifecycle.sh <<<'{"hook_event_name":"SessionStart","session_id":"x","cwd":"/tmp"}' | wc -c` returns `0` from a working directory whose `lifecycle/` is empty or only contains completed features.

4. **Plugin mirror regenerated by the pre-commit drift hook, not by hand**: `plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh` is updated only through the standard regeneration path (the pre-commit drift hook installed by `just setup-githooks`). The canonical source is the only file edited directly. Acceptance (three sub-checks, all must pass): (a) `git diff --name-only HEAD` includes both `hooks/cortex-scan-lifecycle.sh` and `plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh` after the commit lands; (b) `diff <(awk '/jq -n/,/^}$/' hooks/cortex-scan-lifecycle.sh) <(awk '/jq -n/,/^}$/' plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh)` produces no output (the `jq -n` emission block is byte-equal across canonical and mirror); (c) `grep -c 'additional_context' plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh` returns `0` (mirror does not retain the snake_case shape).

5. **Dead defensive code in `tests/test_lifecycle_phase_parity.py` removed**: Three precise edits — (a) delete the `env["AGENT"] = "claude"` assignment at line 511; (b) delete the snake_case fallback branch in the parser (the `else: ctx = payload.get("additional_context", "")` arm of the `if "hookSpecificOutput" in payload:` block around line 545), collapsing the parser to the Claude-shape access only; (c) update the docstring at line 490 to replace `additional_context` with `additionalContext` so the file no longer references the dead snake_case key. Per project.md's "Workflow trimming" principle, hard-deletion is preferred over leaving dead code when the surface has zero runtime consumers. Acceptance (three precise sub-checks, all must pass): (a) `grep -nE '^[[:space:]]*env\["AGENT"\][[:space:]]*=' tests/test_lifecycle_phase_parity.py` returns no matches; (b) `grep -nE 'payload\.get\("additional_context"' tests/test_lifecycle_phase_parity.py` returns no matches; (c) `pytest tests/test_lifecycle_phase_parity.py` exits 0.

6. **No regression in the targeted pytest module or the bash hook suite**: `bash tests/test_hooks.sh` AND `pytest tests/test_lifecycle_phase_parity.py` AND `pytest tests/test_hooks*` (if such files exist; otherwise this conjunct is vacuously satisfied) all exit 0 after the change. Acceptance: each command runs and exits 0 in the verification step. **Out of scope**: a pre-existing failure of `tests/test_lifecycle_references_resolve.py::test_every_lifecycle_reference_resolves` exists on `main` independent of this fix (stale citation in `lifecycle/migrate-gate-1-researchspecify-open-questions-escalation-to-python-hook-remove-gate-2-entirely/plan.md`); the broader `pytest tests/` sweep is therefore NOT a gate for this spec. If the broader sweep is desired, the unrelated lifecycle citation drift must be repaired in a separate ticket.

## Non-Requirements

- **Do NOT add `AGENT=claude` to test invocations in `tests/test_hooks.sh`.** That would entrench a phantom env-var contract Claude Code does not inject; this was the rejected Alternative B in research.
- **Do NOT delete `hooks/cortex-scan-lifecycle.sh` or any scan-lifecycle test.** The hook is load-bearing live infrastructure (overnight runner depends on `LIFECYCLE_SESSION_ID` injection; SessionStart continuity across `/clear`).
- **Do NOT change `tests/fixtures/hooks/scan-lifecycle/*.json` fixtures.** They already use the original Claude discriminator (`hook_event_name`) and are correctly shaped.
- **Do NOT touch the `claude-agent.json` fixture's name** (it conveys test intent, not a runtime contract).
- **Do NOT introduce a runtime mechanism for selecting alternate hook output shapes** (per-Cursor, per-Gemini, etc.). Cursor and Gemini support were deleted in `ea51d0c`; reintroducing branching would re-create the bug class.
- **Do NOT update the backlog ticket's "all 16 PASS" line.** The current real test count is 14 (post-#168/#171 cleanup); the acceptance criterion that matters is "zero failures," and editing the ticket body is out of scope.
- **Do NOT modify the documented hook contract in `docs/agentic-layer.md`.** The docs already describe the canonical shape the hook will now emit.

## Edge Cases

- **`$context` is empty (no incomplete features in `lifecycle/`)**: Hook emits nothing and exits 0 — unchanged. The `if [[ -n "$context" ]]` outer guard at line 459 remains.
- **`jq` is unavailable on $PATH**: Pre-existing hook behavior (the hook already invokes `jq` elsewhere); the fix does not change `jq` dependency. No new failure mode.
- **`$context` contains characters that need JSON-escaping (newlines, quotes, backslashes)**: `jq -n --arg ctx "$context" '{ hookSpecificOutput: { hookEventName: "SessionStart", additionalContext: $ctx } }'` handles escaping via `--arg`. Unchanged from current Claude-branch behavior.
- **Hook invoked by something other than Claude Code (e.g., a maintainer running it manually for debugging)**: The hook now emits `hookSpecificOutput`-wrapped JSON unconditionally. Any external consumer that was relying on the snake_case shape is broken — but research confirmed there are zero runtime consumers (the only remaining grep hits for `additional_context` outside the hook are: the parity test's fallback parser branch (deleted by R5), the parity test's docstring at line 490 (rewritten by R5), and immutable archive lifecycle artifacts; none parse or react to the hook output at runtime).
- **A future test re-uses `tests/test_lifecycle_phase_parity.py`'s defensive pattern**: With the snake_case branch and `AGENT=claude` line removed, the next maintainer reads only the Claude-shape parser and is not tempted to perpetuate the dead contract. Mitigated by Requirement 5.
- **The pre-commit drift hook fails on commit**: The fix bundles a canonical-source change; if the drift hook's `just build-plugin` regeneration produces a non-trivial diff in the plugin mirror, the maintainer must `git add` the regenerated mirror before retrying. This is the standard dual-source flow; no special handling required.

## Changes to Existing Behavior

- **MODIFIED: `hooks/cortex-scan-lifecycle.sh` output shape** → The hook no longer branches on `${AGENT:-}`. When `$context` is non-empty, it emits the canonical `hookSpecificOutput.additionalContext` shape every time. Previously emitted `additional_context` (snake_case) by default; now this shape is unreachable.
- **REMOVED: agent-specific output routing in `cortex-scan-lifecycle.sh`** → The `if [[ "${AGENT:-}" == "claude" ]]` / `else` block at lines 460–471 is replaced by a single unconditional `jq -n` invocation.
- **REMOVED: `env["AGENT"] = "claude"` defensive write in `tests/test_lifecycle_phase_parity.py`** (line 511) → No longer needed; the hook emits the Claude shape unconditionally.
- **REMOVED: `additional_context` fallback parser branch in `tests/test_lifecycle_phase_parity.py`** (line 545) → The hook can no longer emit this shape, so the fallback is dead code.
- **MODIFIED: docstring at `tests/test_lifecycle_phase_parity.py` line 490** → The `additional_context` reference is rewritten to `additionalContext` so the file no longer mentions the dead snake_case key.
- **MODIFIED: `plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh`** → Regenerated to mirror the canonical source. No human edits.

## Technical Constraints

- **Dual-source enforcement** (CLAUDE.md, requirements/project.md): edit only `hooks/cortex-scan-lifecycle.sh`; let the pre-commit drift hook regenerate `plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh`. Hand-editing the mirror is a pre-commit-blocking failure mode.
- **Canonical Claude Code hooks schema** (code.claude.com/docs/en/hooks): `additionalContext` MUST be nested inside `hookSpecificOutput` for SessionStart events. The hook's new output complies with this contract.
- **Existing repo convention**: every other Claude hook in the repo (`cortex-validate-commit.sh`, `cortex-tool-failure-tracker.sh`, `cortex-skill-edit-advisor.sh`) emits `hookSpecificOutput.additionalContext` unconditionally — the fix aligns with this pattern.
- **`set -euo pipefail` in the hook**: the change must not reintroduce an unbound-variable hazard. Removing the `${AGENT:-}` reference (along with the branch it gated) removes one source of such hazards rather than adding any.
- **Acceptance is "zero failures," not a specific test count**: the backlog ticket's "all 16 PASS" is illustrative and stale; the live count is 14 tests post-#168/#171. The operative criterion is `bash tests/test_hooks.sh` exits 0.
- **Out-of-scope pre-existing red test**: `tests/test_lifecycle_references_resolve.py::test_every_lifecycle_reference_resolves` is red on `main` due to a stale citation in `lifecycle/migrate-gate-1-researchspecify-open-questions-escalation-to-python-hook-remove-gate-2-entirely/plan.md`, unrelated to `hooks/cortex-scan-lifecycle.sh`. This spec does NOT attempt to repair that drift. Verification commands in R6 are scoped to avoid this pre-existing failure.
