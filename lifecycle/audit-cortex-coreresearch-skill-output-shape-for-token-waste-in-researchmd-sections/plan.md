## Veto Surface

- **Single commit vs split-by-phase**: The spec describes Phase 1 (Reqs 1, 2, 3, 9) and Phase 2 (Reqs 4, 5, 6, 7, 8) but its own Edge Cases section (`tests/test_plugin_mirror_parity.py` per-phase commit risk) explicitly recommends a single atomic commit covering canonical edits + mirror regeneration. This plan adopts the single-commit approach. If the user prefers two commits, Task 6 must run between them so the mirror is regenerated before each commit.
- **Whether to update non-line-numbered citations in Task 3**: The strict acceptance grep only catches citations with a `:LINE` suffix, but the spec's prose intent is to update all citations to the deleted file. This plan adopts the broader intent. If the user prefers strict-acceptance-only, drop the bullet about backlog/185 lines 73/86 from Task 3's Context.
- **Skipping the critical-review for tier=complex**: The plan reference §3b mandates a critical-review pass for `tier = complex`. The active complexity tier per backlog frontmatter is `complex`, but `events.log` lacks a `lifecycle_start` event, so state-detection defaults to `simple`. If the user wants to honor the frontmatter and run critical-review on this plan, they should override the tier; otherwise the §3b step is skipped per state-detection rules.
- **Backlog closure timing**: Spec Requirement 8 (close backlog item #185) is excluded from the plan tasks because the `/cortex-core:lifecycle` Complete phase performs that closure automatically. If the user wants explicit visibility, an extra task can be added — but it would duplicate Complete-phase work.
