# Plan: reframe-discovery-to-principal-architect-posture

## Overview

Reframe `/cortex-core:discovery` from finding-mirroring to architectural distillation by landing the spec's 15 requirements as 9 commit-atomic tasks. Ordering preserves R12's pre-implementation gate semantics: Task 1 (Unit A: events registry) → Task 2 (Unit B: scanner) → **Task 9 (R12 re-walk, gates Tasks 3-8)** → Task 3 (Unit C: decompose trim) → Task 5 (Unit E: research Architecture + R15 gate) → Task 6 (Unit F: helper module + R4 gate) → Task 7 (R13: re-run semantics); Task 4 (R2: scope envelope) and Task 8 (Unit D: #195 rewrite) run in parallel after the gate clears. Each Unit lands as one commit so pre-commit gates (`cortex-check-parity`, `cortex-check-events-registry`, new `cortex-check-prescriptive-prose`) never see partial states. The re-walk is positioned after the scanner ships (so (ii-b) is checkable) but before the rule-bearing implementation lands (so (i)/(ii-a)/(iii) failure can still invalidate the spec via single-task amendments — to spec.md plus Task 2's scanner if (ii-a) fails — and halt implementation per R12's "implementation does not proceed" semantics).
