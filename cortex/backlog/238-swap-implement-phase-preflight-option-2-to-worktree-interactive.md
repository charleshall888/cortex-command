## Role

Update the lifecycle implement-phase preflight menu by replacing the current "Implement in autonomous worktree" option (option 2) with a new "Implement on feature branch with worktree" option. The new option triggers worktree creation and routes to the worktree-interactive dispatch path. "Implement on current branch" (option 1, recommended) and bare "Create feature branch" (option 3) stay unchanged.

## Integration

Lives in the lifecycle implement-phase skill prose. Preflight selection of the new option routes to the worktree-creation step. The existing runtime probe for `cortex_command` import remains; on absence, the new worktree-interactive option is hidden (parallel to the current behavior for the autonomous option). Uncommitted-changes guard for option 1 remains unchanged.

## Edges

- Bound by the preflight menu contract: exactly three options when on `main`/`master`; reduce to two when the import probe fails.
- Bound by the option-label contract: each option has a stable label string used by `AskUserQuestion` dispatch logic; renaming option 2 must keep dispatch parity.
- Bound by the menu-action routing contract: each option owns its dispatch path; option 2's path is the worktree-interactive flow rather than the daytime subprocess launch.
