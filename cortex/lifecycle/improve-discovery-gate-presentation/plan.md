## Risks
- **Structural tests don't fully verify runtime behavior**: Tasks 1-3 produce static-prose checks; Task 4's dispatch transcript is the runtime check. If Task 4 reveals the soft-form gate prose isn't routing correctly under Opus 4.7, escalation to MUST is required per CLAUDE.md `MUST-escalation policy` — adding scope to this lifecycle. The probability is real but low; the policy provides a clean escalation path with documented evidence (F-row citation), so the risk is bounded.
- **Subjective re-run is the experiential validation, not gated**: per the spec's `Evidence-thin acceptance` constraint, structural tests + R4's content-bound assertions gate merge, but the experiential signal (user's reaction on the next applicable posture-check topic) is post-merge. If the runtime behavior structurally conforms but the user's subjective reaction still fires, the gate-prose change would need a follow-up. This risk is accepted by spec design.
- **Topic for Task 4's dispatch**: Re-running `artifact-format-evaluation` produces an R13 `-N` slug (`cortex/research/artifact-format-evaluation-2/`). If the user prefers a different posture-check topic, the choice happens at task-start time and the transcript file content reflects whichever topic was chosen. Either path satisfies R4 acceptance.
