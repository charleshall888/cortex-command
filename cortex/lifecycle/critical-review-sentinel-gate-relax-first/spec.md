# Specification: critical-review-sentinel-gate-relax-first

## Problem Statement

The `/cortex-core:critical-review` skill's Step 2c.5 Phase 1 verification gate enforces a "literal first line" rule for each reviewer's `READ_OK: <path> <sha>` sentinel via prose at `skills/critical-review/references/verification-gates.md:37-42` and `skills/critical-review/SKILL.md:60,70` that the orchestrator agent reads and follows. Reviewer agents on Opus 4.7 and Sonnet 4.6 reliably emit short preamble before the sentinel — preface lines like *"I have enough context. Let me write the findings."* — despite emphatic prompt instructions. This is structurally expected: prefilling is deprecated on those models, and Opus 4.7's verbosity calibration produces preamble before substantive analysis. The gate over-fires, excluding reviewers that verifiably read the artifact and emitted `READ_OK` with the correct SHA, just not at line 1. The protocol's documented response ("re-run after resolving concurrent write source") cannot resolve a model-behavior failure mode, so the gate degrades the skill to either single-reviewer synthesis or quiet protocol bypass — both observed in lifecycle 109. This spec relaxes the position check while preserving SHA-match enforcement and absence/mismatch routes, and collapses the prose-driven check into a code-level parser mirroring the existing `verify_synth_output` precedent.

## Phases

- **Phase 1: Code parser + regression fixtures** — introduce `verify_reviewer_output` and its `verify-reviewer-output` subcommand in `cortex_command/critical_review.py`; capture synthetic reviewer-output fixtures in-repo; add unit tests covering the gate's pass / absent / mismatch routes and false-positive defenses.
- **Phase 2: Wire and align** — rewrite the Phase 1 prose in `verification-gates.md` and `SKILL.md` to invoke the new subcommand; align `reviewer-prompt.md` and `fallback-reviewer-prompt.md` instructions with the parser's contract; extend `tests/test_plugin_mirror_parity.py` to cover the critical-review references; update the events-registry rationale row.

## Edge Cases

- **Sentinel inside a reviewer's `evidence_quote` (artifact under review names `READ_OK:`)**: When the artifact under review is itself this lifecycle's spec.md or `verification-gates.md`, reviewers can emit `evidence_quote` strings containing literal `READ_OK: <path> <sha>` patterns. Expected behavior: the first-match-matching-SHA semantics route to the reviewer's actual sentinel (whose `group(2) == expected_sha`) rather than to the quoted artifact text. If the quote happens to contain the same SHA (impossible for SHAs that did not exist when the artifact was written — chicken-and-egg), the gate would route incorrectly; for in-scope artifacts no realistic attack surface exists.
- **Multiple `READ_OK:` lines in the window with different SHAs**: Iterate matches; return the first whose `group(2) == expected_sha`. If none match expected, classify as `mismatch` against the first observed SHA. Covered by test case (g) in Requirement 6.
- **Blockquoted sentinel (`> READ_OK: ...`)**: `^` anchor with `re.MULTILINE` rejects leading `> `. Classified as `absent` if no other valid sentinel appears. Covered by test case (h).
- **Code-fenced sentinel inside ` ``` `**: The current regex would match a sentinel inside a code fence. Documented limitation; deferred per Non-Requirement above. The 64-hex SHA group + first-match-matching-SHA semantics still provide defense for the common case.
- **CRLF line endings (`\r\n`)**: `\s*$` matches `\r` (since `\r ∈ \s`); `re.MULTILINE`'s `$` matches before `\n`, leaving `\r` consumed by the `\s*` group. Covered by test case (j).
- **BOM-prefixed first line (`﻿`)**: When `--input-file` reads a file with a UTF-8 BOM, line 1 starts with `﻿` which fails the `^READ_OK:` anchor. Sentinel on line 2+ still matches. Covered by test case (i).
- **`--input-file` points at a non-existent or unreadable path**: Subcommand exits with a clear error message (use the same exit-2 pattern as `_cmd_record_exclusion`'s OSError handler). The orchestrator surfaces the error verbatim.
- **All four reviewers fail Phase 1 verification**: The existing total-failure path at `verification-gates.md:61-63` ("All reviewers excluded — drift or Read failure detected; critical-review pass invalidated") fires unchanged. The fix does not change the total-failure behavior.
- **`record-exclusion` invocation race**: The new `verify-reviewer-output` subcommand internally invokes `record-exclusion` on exit-3 paths (per Requirement 3). This is a single subprocess from the orchestrator's perspective. The atomic `append_event` at `critical_review.py:223-244` already handles concurrent appends via tempfile + `os.replace`.
- **Reviewer output is empty or whitespace-only**: `_REVIEWER_RE.finditer` returns no matches; classify as `absent`. No special-case needed.
- **Reviewer output exceeds 50 lines but sentinel is at line 51+**: Classified as `absent` (sentinel outside window). The first-match-matching-SHA semantics make window size a secondary defense; 50 lines is the chosen tradeoff between robustness to Opus 4.7 verbosity and false-positive surface.

## Technical Constraints

- **Skill-helper modules constraint** (`cortex/requirements/project.md:31`): the fix collapses the paraphrase-prone Phase 1 prose ceremony into an atomic `cortex_command/critical_review.py` subcommand fusing validation (regex parse) + mutation (`record-exclusion` event emission on exit-3) + telemetry (events.log append). Mirrors the established `_SYNTH_RE` / `verify_synth_output` precedent.
- **Structural separation over prose-only enforcement** (`CLAUDE.md:58`): the existing prose-only gate is exactly the bypass-prone enforcement the principle warns against. Moving to code control flow makes the gate harder to accidentally circumvent.
- **MUST-escalation policy** (`CLAUDE.md:72-80`): the fix introduces no new MUST / CRITICAL / REQUIRED language. The existing MUSTs at `SKILL.md:46` and `verification-gates.md:4-7` are pre-existing, grandfathered, and untouched. No evidence-artifact requirement triggered.
- **Voice anchor preservation** (backlog #082 / #085): four anchors named in Non-Requirements remain untouched.
- **SKILL.md size cap** (`cortex/requirements/project.md:30`): `SKILL.md` is currently 116 lines; this fix changes ~1 line. Headroom unchanged.
- **Dual-source mirror** (`CLAUDE.md`): edit canonical sources only at `skills/critical-review/` and `cortex_command/critical_review.py`. The pre-commit hook regenerates `plugins/cortex-core/skills/critical-review/` via rsync.
- **Events.log schema stability**: the `sentinel_absence` payload's eight fields are a documented contract (`bin/.events-registry.md:112`). No fields added, removed, or renamed.
- **UTF-8 encoding contract**: `verify-reviewer-output` reads `--input-file` with `encoding='utf-8', errors='strict'`. Mixed-encoding outputs fail loud rather than silently corrupting the parse.
- **First-match-matching-SHA semantics** (Adversarial F2): the parser MUST iterate matches and return the first whose `group(2) == expected_sha`, NOT the first match unconditionally. This is the load-bearing false-positive defense against the self-defeating-loop case (reviewer reviewing this fix's spec.md).
