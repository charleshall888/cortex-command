# Specification: pre-commit-gates-silently-skip-deep

## Problem Statement

Four pre-commit checkers silently fail to scan most of the files they claim to cover. Their `--staged` corpus-membership test uses `Path.match(glob)`, which treats `**` as exactly one path segment on every supported Python (3.12+), so depth-1 and depth-≥3 in-scope files are dropped after the bash trigger already fired — a **false-green** (the gate reports pass without ever scanning the staged edit). Measured skip rates on this repo: `cortex-check-parity` 87%, `cortex-check-events-registry` 68%, `cortex-check-prescriptive-prose` 13%. Separately, `cortex-check-bare-python-import` already migrated to `PurePath.full_match` (correct for `**`) but that method only exists in Python 3.13+, so the checker **raises `AttributeError` on 3.12**, the floor of `requires-python`. This fix makes every gate actually scan its deep-nested files on every supported Python, killing the silent-skip class across the family. It benefits the commit path, dual-source enforcement, and the overnight runner. #282 is the explicitly-deferred sibling of #279, which fixes the identical bug in the 5th checker (`cortex-check-contract`).

## Prerequisite (hard dependency)

**#282 is `blocked_by: 279`.** #279 (`cortex-check-contract-false-positives-on`) lands the same `--staged` membership fix in `contract.py` first; its plan already designs the correct matcher (`**`=zero-or-more-segments, 3.12-safe, no `full_match`) and factors it into a private helper. Gating #282 behind #279's completion removes all runtime-coordination ambiguity (no landing-state guessing, no concurrent-worktree collision on `contract.py`/`_globs.py`) and lets #282 perform a clean, full 5-site consolidation: promote the proven matcher into a canonical shared module and migrate every site — including `contract.py` — onto it. The overnight runner honors `blocked_by` (`cortex_command/overnight/backlog.py`): #282 will not be selected until #279 is terminal, or sequenced after #279 within one session.

## Phases

- **Phase 1: Shared matcher + importable checkers** — create the canonical 3.12-safe matcher module and route the three checkers that can import it (`parity_check`, `prescriptive_prose`, `bare_python_import`) through it, deleting their old `Path.match`/`full_match` membership calls.
- **Phase 2: Standalone events-registry + regression lock** — vendor the matcher inline into the standalone `bin/cortex-check-events-registry` (it cannot import the package) guarded by a full-truth-table parity test, regenerate the dual-source mirror, and author the behavioral real-`--staged` regression tests for all four checkers.
- **Phase 3: contract.py consolidation** — migrate `contract.py` (#279's file, already landed per the prerequisite) onto the shared helper so all five sites share one implementation.

## Requirements

1. **Shared membership matcher.** A new stdlib-only module `cortex_command/lint/_globs.py` exposes a glob-membership predicate (e.g. `matches_any_glob(rel_path: str, globs) -> bool`) that decides membership by treating each glob's `**` as **zero-or-more path segments** (admitting depth-1, depth-2, depth-≥3 uniformly) and is identical on Python 3.12 and 3.13+. Membership is specified **directly** (`**` = zero-or-more segments), NOT pinned to `root.glob`'s output (version-split — see Technical Constraints). Acceptance: (a) `pytest tests/test_lint_globs.py -q` exits 0, where the test calls the matcher and asserts **every one of the nine Edge Case truth-table rows** (not a subset); (b) `grep -c "full_match\|glob\.translate" cortex_command/lint/_globs.py` = 0 (no Python-3.13-only API at runtime). **Phase**: Shared matcher + importable checkers

2. **Three importable checkers route `--staged` membership through the helper, old calls deleted.** `cortex_command/parity_check.py`, `cortex_command/lint/prescriptive_prose.py`, and `cortex_command/lint/bare_python_import.py` consult `matches_any_glob` for the `--staged` membership decision (preserving each checker's own `SCAN_GLOBS` and any literal/`*` fast-path), and their previous `Path.match(glob)` / `full_match(glob)` membership call-sites are removed. Acceptance (deletion + behavioral, NOT import-presence): (a) `grep -c "\.match(glob)\|full_match" cortex_command/parity_check.py` = 0 AND same = 0 for `cortex_command/lint/prescriptive_prose.py` AND for `cortex_command/lint/bare_python_import.py` (the old membership calls are gone — mirrors R3/R7); (b) the R5 behavioral staged tests pass, proving the helper is actually consulted on the staged path (import-without-use cannot satisfy R5). **Phase**: Shared matcher + importable checkers

3. **events-registry inline vendor + full-truth-table parity test.** Because `bin/cortex-check-events-registry` is a standalone script with no `cortex_command` import, it carries an **inline copy** of the matcher (replacing its `Path.match` membership). A new test pins the inline copy to the shared helper over a corpus covering **all nine Edge Case shapes** — explicitly including the single-`*` over-scan rejection, the directory-prefix discrimination, and the hidden-file case (not just depth-1/depth-≥3/`**`/exact-name). The test loads the extensionless script via `importlib.util.spec_from_loader(...)` + `importlib.machinery.SourceFileLoader(...)` — **NOT** `spec_from_file_location`, which returns `None` for an extensionless path — and loads it **without** triggering the script's module-load `cortex-log-invocation` subprocess (lines 29–34 run at import; neutralize via monkeypatching `shutil.which` to return `None`, or an env guard, before load). Acceptance: (a) `pytest tests/test_events_registry_glob_parity.py -q` exits 0 with the nine-shape corpus; (b) `grep -c "\.match(glob)" bin/cortex-check-events-registry` = 0. **Phase**: Standalone events-registry + regression lock

4. **Dual-source mirror regenerated.** After editing `bin/cortex-check-events-registry`, its `plugins/cortex-core/bin/` mirror is regenerated (not hand-edited). Acceptance: `just build-plugin` then `git diff --quiet plugins/cortex-core/bin/cortex-check-events-registry` exits 0 (mirror matches canonical). Run `just setup-githooks` first so the drift hook is active; the commit-time hook also auto-builds, so this is the post-build no-drift confirmation. **Phase**: Standalone events-registry + regression lock

5. **Behavioral regression lock — the bug-fix proof.** For **each** of the four checkers, a test builds a real temp git repo, stages an in-scope file carrying a known violation, and runs the checker in `--staged` mode **without** `--root` (the only path that reaches `_matches_scan_glob`), then asserts the checker **exits non-zero and reports the violation in that file**. Depth coverage: a depth-≥3 file is mandatory for all four (the universal failing case); a depth-1 synthetic in-scope file (e.g. `skills/foo.md` against `skills/**/*.md`) is mandatory for every checker whose `SCAN_GLOBS` contains a `**` glob (it exercises the `**`=zero-segments case the bug misses). Acceptance: `pytest tests/test_check_parity.py tests/test_check_prescriptive_prose.py tests/test_check_events_registry.py tests/test_bare_python_import_lint.py -q` exits 0, where each checker's new test asserts a **non-zero checker exit** on the staged deep file (a behavioral assertion — a string-count grep does not satisfy this). This is the criterion that fails if the implementer ships the same bug (bare `Path.match`, bare `fnmatch`, or an untouched `full_match`). **Phase**: Standalone events-registry + regression lock

6. **No collateral damage on the current repo (NOT the regression-lock).** Distinct from R5: this guards only against the corrected matcher surfacing latent findings in this repo's deep files. Research confirmed it lands green. Acceptance: `just test` exits 0 AND the pre-commit gates pass on the actual #282 commit (no new violations introduced; no companion clearing tickets required). Note: R6 alone does NOT prove the bug is fixed — it routes through the already-correct `root.glob`/audit path; R5 is the bug-fix lock. **Phase**: Shared matcher + importable checkers

7. **contract.py consolidation (gated on #279 complete).** With #279 landed (per the Prerequisite), `contract.py` carries the proven matcher. #282 ensures `cortex_command/lint/_globs.py` is the single canonical implementation and migrates `contract.py`'s membership decision to import and use `matches_any_glob` from it, removing any private matcher. Acceptance: (a) `grep -c "from cortex_command.lint._globs import\|_globs import\|matches_any_glob" cortex_command/lint/contract.py` ≥ 1 (contract.py uses the shared helper); (b) `contract.py` retains no independent glob-membership matcher — `grep -c "rel_path\.match(glob)" cortex_command/lint/contract.py` = 0 AND its `--staged` membership routes through the shared helper (verified by `pytest tests/test_check_contract.py -q` exit 0, including #279's depth membership tests, which must still pass after the migration). **Phase**: contract.py consolidation

## Non-Requirements

- **Does NOT fix the `--audit` / full-corpus `root.glob('hooks/**')` 3.12 under-scan.** `bare_python_import.py`'s audit path enumerates `hooks/**` via `root.glob`, which on Python 3.12 returns zero files (a CPython version quirk, not the `Path.match` membership bug this ticket targets). The shared matcher fixes `hooks/**` correctly for the `--staged` membership decision on both versions; the audit-path enumeration quirk is a separate, pre-existing concern left as a potential micro-follow-up.
- **Does NOT add CI enforcement of the parity test.** The R3 parity test and the R5 regression tests are enforced by `just test` (local + pre-commit), not by CI — `validate.yml` runs no pytest. Closing that gap is ticket #283; #282 does not expand into it.
- **Does NOT clear or exclude any existing repo content.** Research confirmed the fix lands green; no inline-clearing, no new hard-exclusions, no deferral tickets.
- **Does NOT import contract.py's hard-exclusion prefixes into the three checkers that never had them.** "Hard-exclusions still apply" means preserve each checker's *existing* exclusion behavior (parity/prose/events-registry have none beyond their glob set; bare-python keeps its `_is_hard_excluded` layered as today).
- **Does NOT change the two-mode gate architecture.** Only the `--staged` membership predicate changes; the `--audit`/`root.glob` enumeration and the `--staged`/`--audit` split are preserved.
- **Does NOT promote `bin/cortex-check-events-registry` into a `cortex_command` module.** That refactor (which would let it import the helper directly) is out of scope; the inline-vendor + parity-test is the chosen treatment.
- **Does NOT add a third-party dependency** (`wcmatch`/`pathspec`). The matcher is stdlib-only.

## Edge Cases

The matcher's truth-table — each row is a required assertion in BOTH the R1 unit test and the R3 events-registry parity corpus:

- **Depth-1 against `dir/**/*.ext`** (e.g. `docs/agentic-layer.md` vs `docs/**/*.md`): **in-scope** (the bug's core miss; `Path.match` and bare `fnmatch` both return False here).
- **Depth-2 against `dir/**/*.ext`** (e.g. `docs/internals/pipeline.md`): in-scope.
- **Depth-≥3 against `dir/**/*.ext`** (e.g. `skills/lifecycle/references/implement.md` vs `skills/**/*.md`): in-scope.
- **Single-`*` over-scan rejection** (e.g. `cortex/backlog/sub/x.md` vs `cortex/backlog/*.md`): **NOT in-scope** (`*` must not cross `/`; bare `fnmatch` and the `fnmatch(...) or fnmatch(..., pat.replace('/**/','/'))` variant both wrongly accept it).
- **Deep literal-prefix single-`*`** (e.g. `cortex_command/overnight/prompts/plan-synthesizer.md` vs `cortex_command/overnight/prompts/*.md` — a 3-segment literal prefix, the real events-registry glob): **in-scope**; a sibling one level deeper (`cortex_command/overnight/prompts/sub/x.md`) is **NOT** in-scope. (Locks interior-segment anchoring for deep prefixes, which no `**`/depth-2 row exercises.)
- **Bare trailing `**`** (e.g. `hooks/cortex-cleanup-session.sh` vs `hooks/**`): **in-scope** at depth-1 and deeper (a naive `**`→`(?:[^/]+/)*` rule matches directories only and would skip these — the trailing `**` must emit a basename group).
- **Exact-name literal** (e.g. `justfile`, `CLAUDE.md`): matches exactly; `sub/CLAUDE.md` does NOT match `CLAUDE.md` (anchored, not tail-match — `Path.match` wrongly tail-matches).
- **Single-`*` with directory prefix** (`hooks/cortex-foo.sh` vs `hooks/cortex-*.sh` in-scope; `claude/hooks/cortex-x.sh` must NOT match `hooks/cortex-*.sh`): correct prefix discrimination.
- **Hidden files match** (e.g. `tests/fixtures/.parity-exceptions.md`-style dotfiles under `tests/**/*.md`): **in-scope** — `Path.glob` is `include_hidden=True`-equivalent on all supported versions, so the matcher must NOT skip dotfiles (research's adversarial pass verified `include_hidden=False` produces 5 real divergences on this repo).

Input contract: the matcher receives a raw posix relative string from git `--name-only` (no `./`, no trailing slash, forward slashes) — not `str(Path(rel))` — for platform stability.

## Technical Constraints

- **`requires-python = ">=3.12"`** (`pyproject.toml:8`). The matcher must run on 3.12: no `PurePath.full_match` (3.13+) and no `glob.translate` (3.13+) at runtime. Implement an inlined `re`-only glob→regex translation: split on `/`; a **non-final** `**` segment → an optionally-empty "(any segments)/" group (`(?:[^/]+/)*`) — it MUST NOT collapse to `.*` (which would cross `/` and mis-match a literal segment placed after a `**`); a **final** `**` → "any, incl. basename" (`(?:[^/]+/)*(?:[^/]+)?`); `*` → `[^/]*`; `?` → `[^/]`; literals `re.escape`d; anchored via `re.fullmatch`. (`glob.translate(pat, recursive=True, include_hidden=True)` is the behavioral oracle for tests on 3.13+, but MUST NOT be called at runtime.)
- **Do not pin the target to `root.glob`'s output.** `root.glob('hooks/**')` is version-split (files on 3.13+, directories-only/zero on 3.12), so "congruent with root.glob" is ill-defined for bare `**`. Specify the membership set directly. (#279's plan reached the same conclusion independently.)
- **stdlib-only** for all checkers and the shared helper (each checker's docstring asserts this).
- **Dual-source**: `cortex_command/` modules are wheel-only (no mirror); `bin/cortex-check-events-registry` IS byte-mirrored to `plugins/cortex-core/bin/` — regenerate via `just build-plugin`; the `.githooks/pre-commit` hook fails closed on drift (run `just setup-githooks` first). Do not hand-edit the mirror.
- **Tests must use real git-staging, not `--root`.** Under a `--root` override, prose/events-registry/parity route through the already-correct `root.glob` path and do not exercise the bug; only the real-git `--staged` path reaches `_matches_scan_glob`. This is why R5 (not R6) is the regression-lock.
- **events-registry import-time side effect**: `bin/cortex-check-events-registry` runs a `cortex-log-invocation` subprocess at module load (lines 29–34, gated on `shutil.which`). The R3 parity test must neutralize this before loading the module (monkeypatch `shutil.which` → `None`, or load via a loader that the test controls).
- **Helper home**: `cortex_command/lint/_globs.py` (`lint/__init__.py` is docstring-only today). The three importable checkers run as `python3 -m cortex_command...`, so importing `cortex_command.lint._globs` is safe; the new `.py` module is not scanned by any checker (they scan `.md`/`.sh`/`justfile`), so it trips no gate.
- **Editing `bin/cortex-*` and these modules requires the lifecycle skill** (CLAUDE.md); commit via `/cortex-core:commit`.
