# Plan: pre-commit-gates-silently-skip-deep

## Overview
Promote #279's already-proven `**`=zero-or-more-segments matcher (`_glob_to_regex` + `_in_scan_scope` in `cortex_command/lint/contract.py:430-482`) into a new stdlib-only, Python-3.12-safe module `cortex_command/lint/_globs.py` exposing `matches_any_glob(rel_path, globs)`, then migrate all five `--staged` membership call-sites onto it (three importable checkers route through it, the standalone events-registry carries an inline vendor pinned by a parity test, and contract.py drops its private matcher). The bug-fix proof is a per-checker real-git `--staged` behavioral regression test that fails if the old `Path.match`/`full_match` semantics return.
**Architectural Pattern**: layered

## Outline

### Phase 1: Shared matcher + importable checkers (tasks: 1, 2)
**Goal**: Land the canonical 3.12-safe matcher and route the three package-importable checkers (`parity_check`, `prescriptive_prose`, `bare_python_import`) through it, deleting their old `Path.match`/`full_match` membership calls.
**Checkpoint**: `pytest tests/test_lint_globs.py -q` green over all nine truth-table rows; the three old membership-call greps return 0; `just test` exits 0. **R6 collateral-damage check (the real one):** run each corrected checker over the full repo corpus in its non-`--staged` audit path (which enumerates deep files via `root.glob`) and confirm exit 0 — this is what proves the corrected matcher surfaces no latent findings in this repo's deep files. NB: the pre-commit gate on this PR's OWN commits is NOT a collateral-damage check — the Task-2 commit stages only `cortex_command/*.py` (and the lifecycle commit stages only `cortex/lifecycle/*`), neither of which matches any checker's `SCAN_GLOBS`, so that gate passes vacuously and verifies nothing about deep-file content.

### Phase 2: Standalone events-registry + regression lock (tasks: 3, 4, 5, 6)
**Goal**: Vendor the matcher inline into the standalone `bin/cortex-check-events-registry` (it cannot import the package), regenerate its dual-source mirror in the same commit, pin the inline copy to the shared helper with a full-truth-table parity test, and author the real-`--staged` behavioral regression tests for all four checkers.
**Checkpoint**: `pytest tests/test_events_registry_glob_parity.py -q` green over the nine-shape corpus (comparing the inline two-arg `matches_any_glob` against the shared helper); the four behavioral regression tests each assert BOTH a non-zero checker exit AND the deep file's path in the reported violation (parity via a staged-blob-only divergence; events-registry with a valid registry materialized in the temp repo); `git diff --quiet plugins/cortex-core/bin/cortex-check-events-registry` exits 0 (mirror in sync).

### Phase 3: contract.py consolidation (task: 7)
**Goal**: Migrate `contract.py` (#279's file, already landed) onto the shared helper so all five sites share one implementation, removing contract.py's now-duplicated private matcher.
**Checkpoint**: `tests/test_check_contract.py` green (including #279's depth membership tests) with contract.py's membership routing through `cortex_command.lint._globs`.

## Tasks

### Task 1: Create the shared `_globs.py` matcher and its full-truth-table unit test
- **Files**: `cortex_command/lint/_globs.py` (new), `tests/test_lint_globs.py` (new)
- **What**: Extract #279's proven matcher into a public, stdlib-only, cached shared module, and lock its behavior with a 9-row truth-table unit test. This is R1 and the canonical implementation every other site consumes.
- **Depends on**: none
- **Complexity**: complex
- **Context**: Promote `_glob_to_regex(glob) -> re.Pattern[str]` from `cortex_command/lint/contract.py:430-462` essentially verbatim (it already treats a non-final `**` as `(?:[^/]+/)*` and a trailing `/**` as `^{prefix}/.+$`, with single-`*` → `[^/]*`, literals `re.escape`d, anchored). Expose the public predicate `matches_any_glob(rel_path: str, globs: Sequence[str]) -> bool` that returns True iff `rel_path` matches any glob. Use `Sequence[str]` (not `Iterable[str]`) so the argument is re-iterable — a one-shot generator would silently yield False on reuse; all callers pass module-level `tuple[str, ...]` constants. Wrap `_glob_to_regex` in `functools.lru_cache` so repeated calls across many staged paths (and contract.py's former import-time pre-compile) stay amortized. Module docstring asserts stdlib-only and Python-3.12-safety (no `PurePath.full_match`, no `glob.translate` at runtime). `lint/__init__.py` is docstring-only (1 line) — no `__init__` edit needed. The test must assert **every one of the nine Edge Case truth-table rows** in spec.md §Edge Cases (depth-1/depth-2/depth-≥3 against `dir/**/*.ext`; single-`*` over-scan rejection; deep literal-prefix single-`*` plus its one-level-deeper sibling rejection; bare trailing `**` at depth-1 and deeper; exact-name literal with anchored non-tail-match; single-`*` with directory-prefix discrimination; hidden-file match). Input contract: matcher receives a raw posix relative string (no `./`, no trailing slash, forward slashes). `glob.translate(pat, recursive=True, include_hidden=True)` MAY be used as the test's behavioral oracle on 3.13+ but MUST NOT be called in `_globs.py` at runtime.
- **Verification**: `pytest tests/test_lint_globs.py -q` — pass if exit 0 with all nine rows asserted; AND `grep -c "full_match\|glob\.translate" cortex_command/lint/_globs.py` = 0 (no Python-3.13-only API at runtime).
- **Status**: [x] done

### Task 5: Behavioral `--staged` regression tests for the three importable checkers
- **Files**: `tests/test_check_parity.py`, `tests/test_check_prescriptive_prose.py`, `tests/test_bare_python_import_lint.py`
- **What**: For each of parity/prose/bare-python, build a real temp git repo, stage an in-scope file carrying a checker-appropriate known violation, run the checker in `--staged` mode WITHOUT `--root`, and assert it exits non-zero and reports the violation. This is the R5 bug-fix proof for the three importable checkers — it fails red if the old `Path.match`/`full_match` semantics return.
- **Depends on**: [2]
- **Complexity**: complex
- **Context**: Tests MUST use real git-staging, not `--root` — under `--root` the checkers route through the already-correct `root.glob` audit path and never reach `_matches_scan_glob` (spec Technical Constraints). Depth coverage: a depth-≥3 in-scope file (e.g. `skills/lifecycle/references/foo.md` vs `skills/**/*.md`) is mandatory for all three; a depth-1 synthetic in-scope file (e.g. `skills/foo.md`) is mandatory for every checker whose globs contain a `**` glob (exercises the `**`=zero-segments case the bug misses). The "known violation" must be tailored to each checker's rule set (parity: an orphan/unreferenced bin-script reference; prescriptive_prose: a prescriptive-prose construct it flags; bare_python_import: a bare `python`/`python3` import-style invocation it flags) so the staged deep file genuinely fails the checker. **Two assertions per test, not one** — (i) a **non-zero checker exit**, AND (ii) the reported violation **names the deep file's path** (e.g. assert the path substring is in stdout/the Violation list). A bare non-zero exit is as weak as the grep R5 forbids: it can come from an unrelated cause, so it must be paired with the path assertion to prove the deep file was actually scanned. **Parity needs a staged-blob-only divergence (critical):** `parity_check.lint()` (parity_check.py:742-758) overlays staged blobs onto a working-tree `gather_scan_files`/`root.glob` enumeration, and `_matches_scan_glob` gates ONLY the overlay — so a violation present in BOTH the staged blob and the on-disk copy fires via the disk path even with the bug present (false lock). The parity test must therefore make the violation reachable ONLY through the staged blob: commit the deep file clean, stage a modification that adds the violation, then restore the working-tree copy to the clean version (index=violation, worktree=clean) — or use a staged-only addition absent from the working tree. With the bug, the deep file is dropped from the overlay and only the clean worktree copy is scanned → checker exits 0 → test goes RED. prose and bare_python_import have NO working-tree fallback (their `--staged` path is purely `_matches_scan_glob`-gated), so simple staging (violation in both staged + worktree) suffices for those two. A shared temp-git-repo + stage-and-run helper across the three tests reduces duplication; reuse each test file's existing fixtures/helpers where present.
- **Verification**: `pytest tests/test_check_parity.py tests/test_check_prescriptive_prose.py tests/test_bare_python_import_lint.py -q` — pass if exit 0, with each checker's new test asserting BOTH a non-zero checker exit AND that the deep file's path appears in the reported violation; the parity test specifically uses a staged-blob-only divergence (worktree copy clean) so it goes red with the bug present.
- **Status**: [x] done

## Risks
- **The matcher is a near-verbatim promotion of #279's proven code**, verified to pass all nine truth-table rows (the critical-review matcher-fidelity pass ran the exact `contract.py:430-462` body against every row, cross-checked against `glob.translate(..., recursive=True, include_hidden=True)` as the 3.13+ oracle — zero failures) — so R1 carries low novel-correctness risk. Task 1's `test_lint_globs.py` re-locks this at runtime. The one deliberate addition is `lru_cache` on `_glob_to_regex` to preserve contract.py's former import-time pre-compile amortization once it stops pre-compiling its own patterns (Task 7).
- **Per-checker `--staged` corpus asymmetry (drives the R5 test design).** `parity_check` is the only checker that overlays staged blobs onto a working-tree `root.glob` enumeration (`parity_check.py:742-758`) — so its membership predicate gates only which staged blob to use, not whether the file is scanned at all; the other three (`prescriptive_prose`, `bare_python_import`, `bin/cortex-check-events-registry`) scan a purely `_matches_scan_glob`-gated staged path with no working-tree fallback. This is exactly why Task 5's parity test needs a staged-blob-only divergence while the others don't, and why parity's deep corpus was already implicitly clean (it was reading it via `root.glob`) while prose/bare-python were genuinely blind to deep files until this fix. The R6 audit-path check covers all four uniformly.
- **Task 2 widens `parity_check`'s scan scope.** Once routed through the fixed matcher, `parity_check` newly brings depth-1 files like the just-added `tests/test_lint_globs.py` into scope via `root.glob`. This is safe (the new test files exercise pure glob logic and carry no `cortex-*` references that would create parity wiring signals), and the R6 full-corpus audit check confirms it — but the implementer should expect the scanned set to grow and verify the new test files pass clean.
- **R5 violation crafting is the trickiest part**: each of the four checkers flags a different construct, so the test author must place a checker-appropriate violation in a deep in-scope file for each. Research confirmed this is achievable; if any one checker proves hard to trigger on a deep file, that is a genuine signal worth surfacing rather than weakening the assertion to a grep.
- **Import-path asymmetry**: `parity_check.py` is at `cortex_command/parity_check.py` (not under `lint/`), so it uses the absolute `from cortex_command.lint._globs import ...` while the two `lint/` siblings use a relative import. (The backlog touch-points listed `cortex_command/lint/parity_check.py` — that path is wrong; the spec R2 path is correct and is what this plan uses.)
- **Out of scope (do not expand)**: the `--audit`/`root.glob('hooks/**')` 3.12 enumeration quirk, clearing/excluding existing repo content, importing contract.py's hard-exclusions into the other checkers, the two-mode architecture, promoting events-registry to a module, CI wiring (#283), and third-party glob deps. The plan touches only the `--staged` membership predicate.
