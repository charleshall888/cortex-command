# Plan: manage-interactive-feature-worktree-lifecycle-creation

## Overview
Restructure the lifecycle Complete phase into a 12-step protocol that pauses mid-phase for PR merge and finalizes only after merge + cleanup, and wire the `interactive/{slug}` worktree creation path that implement.md option 2 will call. Ship as two staged commits inside one PR: Commit 1 lands primitive fixes (`cleanup_worktree` refactor, event-schema extension, metrics segmentation), the morning-review ordering bug fix, and the ADR — independently bisectable. Commit 2 lands the multi-step `complete.md`, the `interactive/` prefix wiring, statusline + kept-pauses + SessionStart-PATH-bootstrap changes, and the new tests.

## Outline

### Phase 1: Foundation (tasks: 1, 2, 3, 4, 5)
**Goal**: Land primitive fixes, event-schema extensions, metrics segmentation, morning-review ordering bug fix, and ADR.
**Checkpoint**: `just test` exits 0 after commit; no behavioral change yet to the Complete phase or implement.md option 2 — all primitive surfaces and consumers have the new shape but the rewrite that uses them comes in Phase 2.

### Phase 2: Multi-step Complete + interactive worktree lifecycle (tasks: 6, 7, 8, 9, 10, 11, 12, 13)
**Goal**: Rewrite complete.md as a 12-step phase, wire implement.md option 2 to create the `interactive/{slug}` worktree, migrate PATH bootstrap to SessionStart, render awaiting-merge sub-state, extend kept-pauses parity, and land routing/schema/cleanup tests.
**Checkpoint**: `just test` exits 0; new lifecycles invoking `/cortex-core:lifecycle complete <slug>` follow the 12-step protocol with mid-phase merge-wait pause; statusline renders "Complete (awaiting merge)" between `pr_opened` and `feature_complete`.
