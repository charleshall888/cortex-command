# Specification: offload-refine-step-2-state-checking

## Problem Statement

`/cortex-core:refine` Step 2 makes the model re-derive deterministic logic on every run: a four-branch resume-point decision tree (which artifacts exist → which phase) and a two-arm "omit `--backlog-slug` on a non-local backend" rule mirrored across `emit-lifecycle-start` (Step 2) and `reconcile-clarify` (Step 5). Per the operator's guiding principle — **the CLI should own deterministic behavior; the LLM should own judgment** — both belong in the `cortex-refine` CLI. Phase 1 moves the deterministic resume classification into a read-only verb, leaving the skill the genuine judgment guards. Phase 2 turns the prose-only "don't read a local backlog file on a non-local backend" rule into a **structural guard with a helpful diagnostic** inside the seed/reconcile verbs — a defense-in-depth fix for a real silent-corruption footgun (a stale local `cortex/backlog/*.md` left after a backend migration). The benefit is correctness-by-construction, not line-count relief (the skill is well under its size cap).

**Scope honesty:** this does NOT remove backend-awareness from the skill wholesale. The skill still resolves the backend (it must, to gate the Step 3/5 `cortex-update-item` write-backs and the §3b critical-review decision), and the Step 3 write-back keeps its own three-arm backend branch — correctly, because its external-tracker arm is LLM-best-effort *judgment*, not deterministic. Phase 2 is scoped to the purely-deterministic seed/reconcile local-file-read invariant only.

## Phases

- **Phase 1: resume-point verb** — move the deterministic resume-state classification into a new read-only `cortex-refine resume-point` subcommand; the skill keeps the genuine judgment guards.
- **Phase 2: backend structural guard** — add a `--backend` flag to `emit-lifecycle-start` and `reconcile-clarify` that structurally enforces (and loudly diagnoses) the deterministic "don't read a local backlog file on a non-local backend" invariant, replacing the skill's two-arm slug-omission prose.

## Priority (MoSCoW)

Both phases are **Must-have** for the ticket. Within them, all requirements are Must-have for a correct, byte-safe implementation, except: the `spec_exists`/`research_exists` booleans in Requirement 1 are **Should-have** (a convenience for a data-driven warn message and a cleaner test surface; the load-bearing field is `resume`). The **Won't-do** set is `## Non-Requirements`.

## Requirements

1. **New `cortex-refine resume-point --lifecycle-slug {slug}` subcommand.** Returns a single-line JSON object to stdout, house style `json.dumps(..., separators=(",",":"))`: `{"resume":"clarify|research|spec|complete","spec_exists":<bool>,"research_exists":<bool>}`. Determination from `cortex/lifecycle/{slug}/{spec,research}.md`: `spec∧research`→`complete`; `spec∧¬research`→`research`; `research∧¬spec`→`spec`; else→`clarify`. Read-only (no writes, no backend, no events). Grounds: new handler in `cortex_command/refine.py` following the `_cmd_*`/`add_parser`/`set_defaults` idiom (existing handlers at refine.py:130,213; parser block 305-368); confirmed resume classification reads neither backlog frontmatter nor events.log. **Acceptance**: a new test in `tests/` drives all four artifact states plus the missing-lifecycle-dir case and asserts the exact `resume` value and both booleans for each; `just test` exits 0. **Phase**: resume-point verb.

2. **resume-point existence semantics are `is_file()` — a deliberate tightening over the current prose's "exists".** A directory named `spec.md`/`research.md` must NOT count (today's prose "spec.md exists" would be read by the model as presence, counting a directory; `is_file()` narrows that — a defensible correctness improvement, NOT a byte-faithful port). An empty-but-present `spec.md` MUST still count (the non-empty check is a separate post-research Step-4 gate, out of scope). Grounds: `skills/refine/SKILL.md` Step 2 (lines 41-53). **Acceptance**: a test asserts an empty `spec.md` yields `spec_exists=true` and a directory named `research.md` yields `research_exists=false`; `just test` exits 0. **Phase**: resume-point verb.

3. **resume-point exit-code contract: exit 0 for all four resume states; exit 2 for usage errors; never 64/70.** All four states are successful determinations; the verb performs no writes, so it has no sandbox/IO-failure path. Grounds: contrast with emit/reconcile exit 64/70 (refine.py:72,200-208,246-254). **Acceptance**: a test asserts exit 0 on a valid slug (every state) and exit 2 when `--lifecycle-slug` is omitted (argparse `required`); `just test` exits 0. **Phase**: resume-point verb.

4. **Step 2 resume-tree pseudo-code replaced by a single `cortex-refine resume-point` call, retaining the judgment guards.** The skill calls the verb, branches on `resume`, and keeps in prose: (a) on `complete` — announce, skip to Step 6, do NOT prompt/offer a menu, re-run only on explicit user request, AND the re-run integrity contract (re-running overwrites the existing spec and resets `status` to `in_progress` until the new spec is approved — currently SKILL.md line 55); (b) on `research` (spec-without-research) — warn that overnight needs both and that Clarify is skipped because intent was established; (c) on `spec` (research-only) — apply the Research Sufficiency Check (`clarify.md §6`). `else→clarify` needs no guard. Grounds: `skills/refine/SKILL.md` Step 2 (41-55) + `clarify.md §6`. **Acceptance** (discriminating): `grep -c 'cortex-refine resume-point' skills/refine/SKILL.md` ≥ 1 AND `grep -c 'research.md does NOT exist' skills/refine/SKILL.md` = 0 (the old pseudo-code block is gone) AND all four guard behaviors — including the re-run status-reset — present in Step 2 prose. Interactive/session-dependent for runtime branching. **Phase**: resume-point verb.

5. **`--backend {value}` flag on `emit-lifecycle-start` and `reconcile-clarify` as a structural guard + diagnostic.** Optional; default `cortex-backlog` when absent; the verb `.strip()`s the value before comparison (robust to a trailing newline from the piped `cortex-read-backlog-backend` output). When the stripped value `!= "cortex-backlog"`: coerce `backlog_slug = None` so no local backlog file is read, AND emit a stderr diagnostic naming the ignored slug and the backend (e.g. `cortex-refine: ignoring --backlog-slug on non-local backend 'github'; seeding from defaults`). For `emit-lifecycle-start` (which has ONLY `--backlog-slug`/`--lifecycle-slug` — no value flags; refine.py:313-325), the coerced path yields seed defaults `("simple","medium")`; for `reconcile-clarify` (refine.py:350-367), the explicit `--complexity`/`--criticality` flags drive. The verbs MUST NOT call `resolve_backlog_backend` or read config themselves — they act only on the passed, stripped flag (a structural guard, not backend resolution). Grounds: `_read_backlog_frontmatter(None)→("simple","medium")` (refine.py:52-53); the verbs are backend-unaware today. **Acceptance**: a test writes `cortex/lifecycle.config.md` with `backlog.backend: github`, places a local `X.md` with non-default values, invokes each verb with `--backend github --backlog-slug X`, and asserts (a) the local file is NOT read (defaults/flags drive), (b) a stderr diagnostic is emitted; plus a test that `--backend $'cortex-backlog\n'` (trailing whitespace) is treated as local; `just test` exits 0. **Phase**: backend structural guard.

6. **Step 2 seed routing and Step 5 reconcile routing each collapse to one unconditional call passing `--backend {resolved}`.** The skill resolves the backend once (existing `cortex-read-backlog-backend`, retained — it still gates the Step 3/5 write-backs and the §3b decision) and passes it as `--backend`; `--backlog-slug` is passed when a local backlog item exists (Context A); `--complexity`/`--criticality` are passed exactly as today (Context B / reconcile). The two-arm "omit `--backlog-slug` on a non-local backend" prose is removed (the verb's guard now owns it). Grounds: `skills/refine/SKILL.md` Step 2 (57-64), Step 5 (145-150). **Acceptance** (discriminating): `grep -c 'emit-lifecycle-start --backend' skills/refine/SKILL.md` ≥ 1 AND `grep -c 'reconcile-clarify --backend' skills/refine/SKILL.md` ≥ 1 (new `--backend` pass-through, absent pre-change) AND `grep -c 'cortex-read-backlog-backend' skills/refine/SKILL.md` ≥ 1 (resolution retained); the explicit non-local slug-omission branch prose for `emit-lifecycle-start` is gone. **Phase**: backend structural guard.

7. **Local `cortex-backlog` arm stays byte-identical, verified against a hardcoded format contract (not self-comparison).** With `--backend` absent OR `cortex-backlog`, the guard never fires and the verbs produce byte-identical `events.log` rows (key order, separators, idempotent short-circuit, read-after-write verify, exit-70 remediation message), with flags-win precedence (refine.py:156-161) unchanged; the skill still passes `--backlog-slug` and no value flags on local Context A, so override `to` values source from frontmatter. **Acceptance** (non-self-sealing, ts-robust): a test runs each verb (i) with no `--backend` and (ii) with `--backend cortex-backlog`, masks the non-deterministic `ts` field, and asserts the serialized row exactly equals a **hardcoded expected literal** pinning key order and separators (e.g. `{"schema_version": 1, "ts": "<TS>", "event": "lifecycle_start", "feature": "f", "tier": "simple", "criticality": "medium", "entry_point": "refine"}`) — the contract is a human-written literal, not captured from the same code; both invocation paths must match it. Existing `tests/test_refine_module.py` / `tests/test_refine_reconcile_clarify.py` default-arm cases also pass unchanged; `just test` exits 0. **Phase**: backend structural guard.

8. **`test_refine_non_local_reconcile_branch_is_value_aware` rewritten to the one-call shape WITHOUT dropping the value-aware negative controls.** The rewrite asserts the one-call-with-`--backend` shape AND preserves the load-bearing #285/#317 negative controls — the SKILL.md reconcile prose must NOT hardcode seed defaults (e.g. the assertion that no `reconcile-clarify ... --complexity simple` literal is emitted); it must pass Clarify's computed values. The wiring/anchor tests (`test_refine_lifecycle_start_wiring.py`, `test_refine_reconcile_wiring.py` incl. the `specify.md`-delegation ordering anchor, `test_refine_skill.py`'s `Complexity/value gate` anchor) must continue to pass. **Acceptance**: the rewritten test retains an assertion that no seed-default literal (`--complexity simple` / `--criticality medium`) appears in the reconcile prose; `just test` exits 0. **Phase**: backend structural guard.

9. **Canonical sources only; mirror regenerated (every commit in both phases).** Edit `cortex_command/refine.py` and `skills/refine/SKILL.md` (canonical); regenerate `plugins/cortex-core/skills/refine/SKILL.md` via `just build-plugin` and commit canonical+mirror together (drift pre-commit hook enforces). `cortex-refine` is a wheel-only console-script — the new subcommand needs no `bin/` wrapper, no parity-allowlist row, no rsync into `plugins/cortex-core/bin/`. **Acceptance**: the drift pre-commit hook passes; `bin/cortex-check-parity` passes. **Phase**: resume-point verb.

## Non-Requirements

- **Thread 4 (refine→research `research-considerations` arg-interface) is OUT of scope** — split into its own follow-up ticket (interface redesign touching the standalone `research/SKILL.md`, with its own stale-state correctness concern; zero dependency on threads 1-3).
- **No fold of resume-point into `cortex-lifecycle-state`** — that module is a pure `events.log` reducer; resume-point is filesystem artifact-stat.
- **No consolidated `begin` verb** — resume-point stays read-only and separate from `emit-lifecycle-start`; merging would couple a pure read to a write that can exit 64/70.
- **The Step 3 `cortex-update-item` write-back backend branch stays in the skill** — its external-tracker arm is LLM-best-effort judgment, so it is correctly skill-side; Phase 2 does not touch it. Phase 2 is NOT a wholesale backend-routing offload.
- **The verbs do not resolve the backend** — they act only on the passed `--backend` flag. Backend resolution stays the skill's job via `cortex-read-backlog-backend`.
- **No new ordering-invariant sentinel.** The seed→reconcile→§3b ordering stays as-is; `--backend` does not change the temporal sequence, and `specify.md §3b`'s existing non-local seed-tier fail-safe already backstops the silent-skip case (no `clarify_reconciled` event, no fail-closed reconcile).
- Does not address the lifecycle's own staleness concern (separate ticket #325).

## Edge Cases

- **Missing lifecycle dir** → `resume-point` returns `{"resume":"clarify","spec_exists":false,"research_exists":false}`, exit 0 — never an exception.
- **Empty `spec.md` present** → counts as existing (`spec_exists=true`); non-empty is a separate Step-4 gate.
- **Directory named `spec.md`/`research.md`** → does NOT count (`is_file()` is False); the one place this diverges from today's prose "exists".
- **`--backend` absent** → treated as `cortex-backlog` (backward-compatible; existing tests/callers unaffected).
- **`--backend` value with trailing whitespace/newline** → `.strip()`'d before comparison, so a piped `cortex-read-backlog-backend` newline does not flip the local arm into the guard path (preserves byte-identity).
- **Non-local backend with a stale local `cortex/backlog/{slug}.md`** → the guard coerces the slug to None (does not read the stale file) and emits a stderr diagnostic; this is the footgun Phase 2 structurally prevents.
- **`complete` state with an explicit user re-run request** → the verb still returns `complete`; the explicit-re-run override (and the status-reset to `in_progress`) is conversational and sits above the CLI in the skill.
- **Non-local resume-to-spec** → `resume=spec` skips Clarify/reconcile, so on a non-local backend the seed tier stays `simple`; `specify.md §3b`'s fail-safe (which stats `research.md` independently of the resume-point verb) fires and runs critical-review.
- **Run from a subdirectory / under `CORTEX_REPO_ROOT`** → resume-point and the verbs use the same bare-cwd-relative `cortex/lifecycle/...` base as today; the `--backend` flag carries the skill's already-resolved value, so no second, divergent resolver runs inside the verb.

## Technical Constraints

- New events register in `bin/.events-registry.md`; resume-point is read-only (no event) and emit/reconcile reuse existing rows → **no registry change** (verified).
- `cortex-refine` binstub/console-script reads the installed wheel; when iterating, use sequential dispatch (not a worktree) for `cortex_command/` edits — `just test` runs the editable install, so a worktree would verify stale code.
- The skill invokes the CLI via the `cortex-refine` console-script (bare-Python `cortex_command` imports in `skills/**/*.md` are prohibited, L201).
- Backlog `grep -c` acceptance tokens (`cortex-refine resume-point`, `emit-lifecycle-start --backend`, `reconcile-clarify --backend`, `cortex-read-backlog-backend`) resolve to literal strings under `skills/`/`cortex_command/` (satisfies `tests/test_backlog_grep_targets_resolve.py`).
- §3b non-local seed-tier fail-safe coverage is the existing `tests/test_critical_review_gate_nonlocal_failsafe.py`; Phase 1 must not change `specify.md §3b`'s own independent `research.md` stat — that test must still pass.
- **ADR-0016 boundary:** the backend-blind rule applies to the backlog-*engine* `cortex-*` CLIs (`cortex-create-backlog-item`, etc.) and to external-tracker adaptation (LLM-best-effort in the skill). A skill-helper verb (`cortex-refine`) acting on a caller-passed `--backend` value as a structural guard — without resolving the backend and without external-adapter logic — is a deliberate, narrow carve-out (see Proposed ADR).

## Open Decisions

None — all design forks resolved at spec time (resume-point home, `--backend` structural-guard design over full internalization, byte-identity verified against a hardcoded contract, School A, thread-4 split, no ordering sentinel, ADR carve-out proposed).

## Proposed ADR

### Proposed ADR: 0017-skill-helper-verb-backend-structural-guard

**Context:** ADR-0016 establishes that the `cortex-*` backlog-engine CLIs gain no backend awareness and that backend branching lives in the consumer skills, with external-tracker handling done LLM-best-effort. This ticket adds a `--backend` flag to the skill-helper verbs `cortex-refine emit-lifecycle-start` / `reconcile-clarify`. **Decision:** A skill-helper verb may act on a *caller-passed* `--backend` value purely as a structural guard — coercing `backlog_slug=None` and emitting a diagnostic on a non-local backend so it never reads a (possibly stale) local backlog file — provided it does NOT resolve the backend itself (the skill does, via `cortex-read-backlog-backend`) and contains no external-tracker adapter logic. **Trade-off:** this brushes ADR-0016's letter ("branching lives in the skills") in exchange for converting a prose-only, model-remembered invariant into a structural, fail-loud guard (CLAUDE.md "prefer structural separation over prose-only enforcement"). It is deliberately bounded to skill-helper verbs over the local `events.log`, distinct from the backend-blind backlog-engine CLIs and from the judgment-bearing external write-back. Whether this is recorded as a standalone ADR-0017 or an amendment to ADR-0016 is for the plan phase.
