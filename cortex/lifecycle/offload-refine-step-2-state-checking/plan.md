# Plan: offload-refine-step-2-state-checking

## Overview

Two independent offloads of deterministic logic out of `skills/refine/SKILL.md` and into the `cortex-refine` CLI. **Phase 1** adds a read-only `resume-point` subcommand that classifies the refine resume state from filesystem artifact-stat, replacing Step 2's four-branch pseudo-code while leaving the three genuine judgment guards in the skill. **Phase 2** adds an optional `--backend {value}` flag to the existing `emit-lifecycle-start` and `reconcile-clarify` verbs that structurally enforces (and loudly diagnoses) the "don't read a local backlog file on a non-local backend" invariant, replacing the skill's two-arm slug-omission prose. The load-bearing constraint throughout is **byte-identity** of the local `cortex-backlog` arm, verified against a hardcoded contract literal rather than self-comparison. The two phases share no *runtime* code path and can be reviewed independently — but they both edit `cortex_command/refine.py` at the file level, so the two refine.py-editing tasks (1 and 4) are serialized by an explicit dependency edge to prevent a same-batch shared-checkout race.

## Outline

### Phase 1: resume-point verb (tasks: 1, 2, 3)
**Goal**: Move Step 2's deterministic resume-state classification into a new read-only `cortex-refine resume-point` subcommand; the skill keeps the three judgment guards.
**Checkpoint**: `cortex-refine resume-point --lifecycle-slug {slug}` returns the correct `{"resume":...,"spec_exists":...,"research_exists":...}` JSON for all four artifact states (verified by a new test); `skills/refine/SKILL.md` Step 2 calls the verb and branches on `resume`; mirror in sync; `just test` green.

### Phase 2: backend structural guard (tasks: 4, 5, 6, 7)
**Goal**: Add a `--backend` flag that structurally drops `--backlog-slug` on a non-local backend (with a stderr diagnostic), collapse the skill's backend two-arm prose to one `--backend {resolved}` call per site, preserve byte-identity of the local arm, and record the ADR-0016 carve-out.
**Checkpoint**: both verbs accept `--backend`; the non-local arm ignores a stale local file and emits a diagnostic; the local arm is byte-identical against a hardcoded literal; Step 2 seed routing and Step 5 reconcile routing each pass `--backend {resolved}` unconditionally; the two-arm prose is gone; the carve-out ADR is landed; mirror in sync; `just test` green.

The implement phase's branch-mode selection should pick the current-branch (trunk) path, and the resume-point/guard tests call `main([...])` in-process to avoid a wheel reinstall. This also argues against worktree dispatch (which is already contraindicated by the editable-install constraint) and for trunk dispatch with disciplined per-task pathspec commits.
- **Shared-file races are prevented by the dependency graph, not by dispatch mode.** The implement phase batches by topological level and launches each batch concurrently — "sequential" mode only removes worktree isolation, which would make a same-batch shared-file collision *worse* (direct shared-checkout clobber, no merge step to surface it). A reviewer expecting full single-line collapse would be wrong to flag the residual Context A/B prose.
- **Byte-identity is the load-bearing regression boundary.** The local `cortex-backlog` arm must stay byte-for-byte identical (key order, separators, idempotent short-circuit, read-after-write verify, exit-70 remediation).
