# Specification: Offload complete.md PR-state routing + shared stage-artifacts to CLI verbs (#331)

## Problem Statement

`complete.md` is ~50-60% deterministic git/gh plumbing the agent hand-executes every Complete run: Step 7's PR-state router (Branches 1-4, sub-cases 4a-4g) is a pure state machine over `events.log` + `pr.json` + `gh pr view` emitting ~10 fixed terminal messages, and Step 11a's finalization staging (enumerated `git add`, resolver-filename lookup, stage-first guard, the "never directory-glob" warning repeated ~5×) is near-identical to `post-refine-commit.md`'s staging. This is exactly the work the CLI exists to absorb — a deterministic outcome does not need an agent, and the narration pollutes runtime context every phase. Moving it into two verbs (`cortex-lifecycle-complete-route`, `cortex-lifecycle-stage-artifacts`) is the offload epic's (#336) highest-leverage, biggest-byte-reduction child. Benefits the lifecycle Complete phase (and refine's post-commit) by collapsing the prose to "run it, act on `route`" while keeping genuine-judgment steps as prose.

**Scope note**: critical-review found that offloading would *entrench* two latent correctness bugs (the Step 11→11a emission-ordering hazard and the `git add -u cortex/backlog/` cross-session sweep) into a shipped, tested verb. Because both sit in the exact lines #331 rewrites, the operator authorized folding **both fixes** into #331 — a deliberate exception to the epic's "offloads are not correctness fixes" carve-out (#329/#335), scoped here to bugs that the offload would otherwise freeze. Release-type stays **minor** (new verbs + bugfixes, no breaking change).

## Phases

- **Phase 1: complete-route classifier** — a `cortex-lifecycle-complete-route <slug>` verb that owns Step 7 routing + the terminal message strings; collapse complete.md Step 7 prose and migrate its routing test. Independent of #330.
- **Phase 2: stage-artifacts staging verb** — a `cortex-lifecycle-stage-artifacts --phase {complete|refine}` verb owning the staging shared by complete.md Step 11a and post-refine-commit.md; collapse both prose sections. Independent of #330.
- **Phase 3: event-emission migration** — migrate complete.md's `pr_opened` (Step 5) and `feature_complete` (Step 11) raw-JSON emission to `cortex-lifecycle-event log --field` calls; update the events-registry producer rows; add field-set+values round-trip tests. **Gated on #330's `--field` extension landing.**

Each phase lands incrementally on trunk; the build/parity/mirror/test gate (Req 19) is a **per-phase** completion gate, not a one-time final step.

## Requirements

**Priority (MoSCoW)**: This is a deterministic refactor that cannot be partially applied, so all numbered requirements are **must-have** — including Req 3 (verb owns the terminal strings), which Req 1's `message`-key contract and Req 8's prose-collapse both depend on, so it is not separable. The **won't-dos** are the explicit boundaries in `## Non-Requirements`; there is no should-have tier because no requirement here is independently droppable without breaking another.

### Phase 1 — complete-route classifier

1. **Routing classifier with bounded side-effects.** `cortex-lifecycle-complete-route <slug>` resolves the slug→lifecycle-dir and classifies the Complete-phase route by reading `events.log`, `pr.json`, and (when present) querying `gh`, emitting a single JSON object on stdout: `{route, terminal, message, pr_state, pr_url, pr_number, head_branch}`. It writes **no** `events.log` rows. Its only side effects are the Branch-3 single-match `pr.json` reconstruction (Req 6) and network `gh` calls — it is **not** git-free/fast and must not be wired into latency-sensitive or speculative surfaces (Req 7a). Acceptance: (a) `cortex-lifecycle-complete-route <slug>` prints one line of valid JSON with keys `route, terminal, message, pr_state` (`python3 -c "import json,sys; d=json.loads(sys.stdin.read()); assert {'route','terminal','message','pr_state'} <= set(d)"` exits 0); (b) a test asserts that across all non-Branch-3 routes the verb appends **zero** new lines to `events.log` (`git diff`/line-count on the fixture's events.log unchanged). **Phase**: complete-route classifier

2. **Strict-order branch state machine preserved.** The verb reproduces today's Step-7 evaluation order and outcomes (first match wins): Branch 1 `feature_wontfix` (precedes all PR checks) → Branch 2 `feature_complete` present (non-terminal short-circuit, no duplicate event/cleanup/pr.json) → Branch 3 `pr.json` absent (orphan probe: zero/one/multiple) → Branch 4 `gh pr view` (4a auth/network → `pr_state: unknown`; 4b not-found; 4c OPEN; 4d MERGED+dirty; 4e MERGED+clean+ancestor → non-terminal continue; 4f MERGED+clean+not-ancestor → refuse cleanup; 4g CLOSED). Branch 2's `feature_complete` detection is **shape-agnostic** (matches the 6-key merge row and the 3-key close row alike). Acceptance: a parametrized golden-route-table test asserts the exact `{route, terminal, pr_state}` tuple for each of the 12 branches, using `tests/fixtures/gh-stub.sh` (`$GH_STUB_SCENARIO`) for the gh states; pass if `.venv/bin/pytest tests/test_*complete_route*.py -q` exits 0. **Phase**: complete-route classifier

3. **Verb owns the routing contract; the existing routing test is migrated in full.** The ~10 verbatim terminal messages (Branch 1, 4a-4g) move from `complete.md` prose into the verb. `tests/test_lifecycle_complete_state_routing.py` currently asserts against `complete.md` **prose** not only the 12 message substrings but ~14 additional structural tokens that Req 8's collapse removes — the `gh pr list`/`--head` probe (test:168), `gh pr view --json state,mergedAt` (176), `git merge-base --is-ancestor` (184), the evaluation-order/precedence language ("strict order":142, "Takes precedence":150), and verbatim messages ("first-run path":241, "retroactively":249, "The PR may have been deleted":208, etc.). **The entire test must be migrated to assert against the verb** — its `{route, message, pr_state}` output for the message/branch assertions, and the verb's module source (or its observable `gh`/`git` invocations under the stub) for the command/order/precedence assertions — not just the 12 message substrings. Acceptance: after migration, `.venv/bin/pytest tests/test_lifecycle_complete_state_routing.py -q` exits 0 **and** every assertion that previously read `complete.md` text now targets the verb (no assertion in the migrated file reads `skills/lifecycle/references/complete.md`, verified by `grep -c "complete.md" tests/test_lifecycle_complete_state_routing.py` reflecting only intentional residual references, if any). **Phase**: complete-route classifier

4. **gh-absent / no-network degrades gracefully.** Missing `gh` binary, `gh auth status` non-zero, or a `gh pr view` network/auth error all route to Branch 4a: `pr_state: unknown`, the verbatim "PR state unknown; gh unauthenticated or network error; retry later. (Worktree retained.)" message, exit 0 — never a non-zero exit or traceback. Detect availability with `shutil.which` and distinguish the three failure bands (missing / non-zero exit / present-but-no-network) per the research. Acceptance: with an empty dir prepended to `PATH` (gh absent), `cortex-lifecycle-complete-route <slug>; echo $?` prints JSON with `"pr_state": "unknown"` and exit 0; covered by a case in the Req-2 table. **Phase**: complete-route classifier

5. **on-main route bypasses the orphan probe.** The verb detects the current branch first; on `main`/`master` it returns an `on_main` route that does not run the Branch-3 `gh pr list` orphan probe (preserving complete.md:21's on-main short-circuit). Acceptance: invoked on a `main` checkout with no `pr.json`, the verb returns `route == "on_main"` and performs no `gh pr list` (assert via the gh-stub recording no invocation). **Phase**: complete-route classifier

6. **Branch-3 recovery and ambiguity preserved as designed.** Single-match orphan → the verb reconstructs `pr.json` atomically (tempfile + `os.replace`) and routes to Branch 4. Multiple-match (slug reuse) → the verb returns route `orphan_ambiguous` with the candidate list in the payload and does **not** auto-select; the user pick stays in prose (kept affordance). Acceptance: a real-fs test with a stubbed `gh pr list` returning 2 PRs asserts `route == "orphan_ambiguous"`, a non-empty `candidates` array, and that no `pr.json` is written; a single-match test asserts `pr.json` is written then routed to a Branch-4 state. **Phase**: complete-route classifier

7. **CWD-based root resolution.** The verb resolves the lifecycle dir / `pr.json` from the physical CWD (the `_resolve_user_project_root_from_cwd` contract `lifecycle_event` uses), **not** `CORTEX_REPO_ROOT`, so an invocation from inside an `interactive/{slug}` worktree reads that worktree's artifacts. Acceptance: a worktree-CWD test (modeled on `cortex_command/tests/test_lifecycle_event.py`'s `_setup_worktree`) asserts the verb reads the CWD-rooted `events.log`/`pr.json` and ignores a divergent `CORTEX_REPO_ROOT`. **Phase**: complete-route classifier

7a. **Speculative-caller boundary enforced, not just asserted.** Because the verb does network `gh` I/O and a conditional `pr.json` write on every invocation, it must be invoked only from the Complete orchestration path — never speculatively from a latency-sensitive renderer. Acceptance: `grep -rEc "cortex-lifecycle-complete-route|complete_route" claude/statusline.sh cortex_command/dashboard/ hooks/` = 0 (no render/observability surface references the verb); the rewritten complete.md is the only consumer. **Phase**: complete-route classifier

8. **complete.md Step 7 prose collapses to a verb call.** Step 7's Branch 1-4/4a-4g narration is replaced by "run `cortex-lifecycle-complete-route <slug>`; if `terminal`, print `message` and exit; else proceed to Step 8," preserving the on-main short-circuit and the `feature_wontfix`/`feature_complete` precedence semantics. Acceptance: `grep -c "complete-route" skills/lifecycle/references/complete.md` ≥ 1, and the Branch 4a-4g / gh-command / evaluation-order tokens no longer appear in the Step 7 region of complete.md (`grep -c "gh pr view\|merge-base --is-ancestor" skills/lifecycle/references/complete.md` reflects only any intentional residual, with Step-7 routing narration removed). Verification of removal rides on the migrated Req-3 test (which now targets the verb), **not** the finalization-region guard in Req-15 (which covers only Step 11a). **Phase**: complete-route classifier

### Phase 2 — stage-artifacts staging verb

9. **Shared staging verb with per-phase path sets.** `cortex-lifecycle-stage-artifacts --phase {complete|refine}` stages the exact path set each phase stages today, over a single shared staging engine that owns the "enumerate explicit paths, never directory-glob" discipline. `--phase complete`: `cortex/lifecycle/{slug}/{research,spec,plan,review,index}.md` + `events.log` (those present) + the review-drift `**File**:` path iff `review.md` has a `## Suggested Requirements Update` section + the backlog write-back staging (see Req 11). `--phase refine`: `research.md, spec.md, index.md, events.log` (+ `cortex/backlog/{NNN}-{slug}.md` on Context A), with `spec.md` omitted on the cancel path. Acceptance: a real-git test (modeled on `tests/test_revert_merge_real_git.py`'s `_git` harness) asserts sorted `git diff --cached --name-only` equals the expected set for each phase/sub-mode; pass if `.venv/bin/pytest tests/test_*stage_artifacts*.py -q` exits 0. **Phase**: stage-artifacts staging verb

10. **refine approval/cancel sub-mode auto-detected.** Under `--phase refine`, the verb scans `events.log` bottom-up for the most recent transition (approval = `phase_transition specify→plan`; cancel = `lifecycle_cancelled`) and stages accordingly (cancel omits `spec.md`). Acceptance: two fixtures (one ending in each event) assert the staged set includes/excludes `spec.md` respectively. **Phase**: stage-artifacts staging verb

11. **Backlog write-back staging — narrowed to the files this feature touches (drops the `-u` sweep, fixes bug 2).** Under `--phase complete`, the backlog write-back stages exactly two **explicit** paths and **drops today's `git add -u cortex/backlog/` sweep** (which captures unrelated dirty tickets — the cross-session-contamination hazard): (a) the resolved ticket file `cortex/backlog/<resolved-filename>` from `cortex-resolve-backlog-item {slug}` (exit-0 stdout `filename` field — never a `{slug}.md` glob, since the lifecycle slug is a truncated prefix matching zero files; resolver exit 3 silently skips), and (b) `cortex/backlog/index.md` (the Step-10-regenerated index — `-u` previously caught it, so it must now be staged explicitly or the committed index goes stale). Both are explicit-path adds; neither sweeps. On `none`/external backends the resolver exits 3 and `index.md` is unmodified, so neither path stages (emergent; no backend branch needed). Acceptance: a real-git test asserts the resolved ticket file + `index.md` ARE staged, a `{slug}.md` glob is not used, and an exit-3 fixture stages neither and still succeeds. **Phase**: stage-artifacts staging verb

12. **Stage-first guard and halt-on-commit-failure preserved.** After staging, the verb (or its caller per the contract) runs `git diff --cached --quiet`: exit 0 (nothing staged) → caller skips `/cortex-core:commit` silently and continues (complete → Step 12; refine → lifecycle Step 3); exit 1 → commit. Commit-failure halts before the next step (complete: before Step 12; refine: the Halt-Before-Plan gate). Acceptance: a test asserts the verb reports a `nothing_staged`/`staged` signal matching `git diff --cached --quiet`'s exit on a no-op re-run vs a real stage. **Phase**: stage-artifacts staging verb

13. **Negative-token controls ported 1:1, plus the no-sweep control.** The negative-token guards in `tests/test_complete_md_finalization_commit.py:21-35` (forbidding `git add -u cortex/requirements/`, bare `git add cortex/backlog/`, slug-glob `cortex/backlog/*-`) are ported to the verb's staged-set test as `git diff --cached --name-only` assertions, with fixtures containing an unrelated dirty tracked file under `cortex/lifecycle/` and `cortex/requirements/` that must NOT be staged. **Plus a no-sweep control**: a fixture with an unrelated dirty tracked `cortex/backlog/OTHER.md` asserts it is **NOT** staged (the narrowed Req-11 behavior), behaviorally exercising the dropped `-u` sweep rather than string-checking it. **Phase**: stage-artifacts staging verb

14. **complete.md Step 11a and post-refine-commit.md staging prose collapse.** Both sections are replaced by a `cortex-lifecycle-stage-artifacts --phase {complete|refine}` invocation plus the residual prose (commit call, stage-first/halt framing as control flow). Acceptance: `grep -c "stage-artifacts" skills/lifecycle/references/complete.md skills/lifecycle/references/post-refine-commit.md` ≥ 2 (≥1 each). **Phase**: stage-artifacts staging verb

15. **Finalization-region guard updated.** `tests/test_complete_md_finalization_commit.py` (the `<!-- finalization-commit-step -->` region token guard, Step 11a only) is updated to assert the collapsed prose (positive token: the verb invocation) while its negative-token intent migrates into the verb's behavioral test (Req 13). Acceptance: `.venv/bin/pytest tests/test_complete_md_finalization_commit.py -q` exits 0. **Phase**: stage-artifacts staging verb

### Phase 3 — event-emission migration (gated on #330)

16. **pr_opened / feature_complete emission migrated to `cortex-lifecycle-event log --field`.** complete.md's Step 5 (`pr_opened`) and Step 11 (`feature_complete`) raw-JSON blocks become `cortex-lifecycle-event log --event … --field …` prose invocations through #330's extended verb. **Deliberate ticket reconciliation**: ticket #331 says the emission is "built INTO these verbs"; structurally no verb owns Steps 5/11 (`complete-route` is the Step-7 classifier and cannot emit `feature_complete` idempotently; `stage-artifacts` runs at Step 11a), so the spec instead has #331 migrate the emission **call sites** (from raw JSON to the verb) within its own PR — preserving the ticket's underlying intent ("move the sites in #331, not #330, to avoid double-touching `lifecycle_event.py` lines") while keeping both new verbs single-responsibility. **Emission-ordering fix (bug 1)**: `feature_complete` is emitted only **after** a successful Step 11a commit — today it fires at Step 11 *before* the commit, so a commit failure leaves it logged and Branch 2 then skips the retry forever (unrecoverable). Emitting post-commit preserves the exact field-set+values (the pin governs content, not ordering) and does not touch `complete-route`'s Branch-2 routing or its golden table. Acceptance: `grep -c '"event": "pr_opened"\|"event": "feature_complete"' skills/lifecycle/references/complete.md` = 0 (raw-JSON literals removed) AND `grep -c "cortex-lifecycle-event log" skills/lifecycle/references/complete.md` ≥ 2, AND a test asserting that on a simulated Step-11a commit failure no `feature_complete` row is written (so a re-invocation re-runs the finalization cleanly rather than short-circuiting via Branch 2). **Phase**: event-emission migration

17. **Field-set + values invariant pinned per event type (NOT literal bytes).** The migrated rows preserve field-set + field-values, not cosmetic byte format (→ Proposed ADR 0017). `pr_opened` keeps **exactly** `{schema_version=1, ts, event, feature, number, url, head_branch, repo}` with **no** `worktree_path`; `feature_complete` (Step 11 merge form) keeps **exactly** `{ts, event, feature, tasks_total (int), rework_cycles (int), merge_anchor="merge"}` with **no** `schema_version`/`worktree_path`. `feature` and `ts` are load-bearing — `metrics.py` hard-indexes both via unguarded `[]`, so their absence KeyErrors the whole corpus. Separator/timestamp format follows #330's canonical emitter. Acceptance: a round-trip test (frozen `ts`) asserts, per event, the emitted row's parsed object has the exact key-set above **and** the exact values/types (`merge_anchor == "merge"`, `tasks_total`/`rework_cycles` are ints, `schema_version` present for `pr_opened` and absent for `feature_complete`), AND classifies each event through a real consumer: `feature_complete` → `detect_lifecycle_phase` + `pipeline/metrics.py` (`merge_anchor`, `feature`, `ts` reads); `pr_opened` → `hooks/scan_lifecycle.py` (Python-reachable awaiting-merge sub-state) plus a separator-tolerance assertion standing in for the bash-only `statusline.sh` consumer (which a pytest round-trip cannot exercise). **Phase**: event-emission migration

18. **Events-registry producer rows updated.** `bin/.events-registry.md` rows for `pr_opened` (line 137) and `feature_complete` (line 12) are updated so their `producers`/`scan_coverage` reflect the migrated emission path; no new event types are introduced. Acceptance: `just check-events-registry && just check-events-registry-audit` exit 0. **Phase**: event-emission migration

## Non-Requirements

- **Does NOT make `stage-artifacts` backend-aware.** Per-backend staged-set variance is emergent (on `none`/external, Steps 9/10 wrote nothing so the backlog adds no-op); the verb does not branch on backend. Backend routing stays in the existing Step 9/10 prose (#326 territory).
- **Does NOT modify the 3-key `feature_complete` close form** in `backlog-writeback.md` (Step 2 "already complete" close) — that emission site is #326's scope. complete-route's Branch-2 detection is shape-agnostic (Req 2).
- **Does NOT offload the kept-affordance prose**: the `/cortex-core:commit` invocation, the PR-authoring step (Step 3), the merge-wait phase-exit pause (Step 6), the Step 8 worktree cleanup + "cd out before cleanup" hard guard, the Step 12 preserve-lifecycle-dir guard, and the Branch-3 multi-match user pick all stay prose.
- **Does NOT thread complete-route's dirty/ancestor snapshot into Step 8's destructive cleanup gate** — Step 8 re-verifies dirty + is-ancestor live (TOCTOU defense).
- **Does NOT change the byte-level serialization format of events.log rows** (separators, timestamp precision) — only field-set + values are pinned (ADR 0017); the cosmetic format follows the canonical emitter.

## Edge Cases

- **gh binary absent vs gh present-but-erroring vs PR genuinely not found**: absent/auth/network → Branch 4a (`pr_state: unknown`, exit 0); successful gh call resolving to "not found" → Branch 4b. The verb distinguishes by exit code + stderr pattern, not just returncode.
- **Invoked from inside an interactive worktree**: reads the worktree-rooted `events.log`/`pr.json` (Req 7); the orphan-recovery `pr.json` write lands in the worktree copy.
- **Resolver ambiguity (exit 2)**: surfaced per the canonical ambiguous-slug handling; resolver exit-3 (no match) silently skips the backlog-file add (Req 11).
- **Nothing staged on re-run** (common on the worktree path post-merge): stage-first guard exit 0 → skip commit silently, continue (Req 12).
- **`merge_anchor` reader back-compat**: legacy `feature_complete` rows lacking `merge_anchor` default to `"review"`; the migrated row must carry `"merge"` explicitly (Req 17).
- **`feature_complete` already present (Branch 2)**: no duplicate event, no re-cleanup, no second `pr.json` write — short-circuit to Step 12.
- **Step 11a commit fails** (bug-1 fix): `feature_complete` is not yet emitted (emission is post-commit), so a re-invocation re-runs the finalization cleanly rather than short-circuiting via Branch 2.
- **A concurrent session's dirty `cortex/backlog/` ticket is present** (bug-2 fix): only the resolved ticket file + `index.md` are staged, so the unrelated dirty ticket is not captured into this finalization commit.

## Technical Constraints

- **#330 dependency, isolated to Phase 3.** `cortex-lifecycle-event`/`log_event` emits a fixed 5-key compact row today and cannot produce the `pr_opened`/`feature_complete` shapes (extra fields + omitted `schema_version`/`worktree_path`). Phase 3 requires #330's `--field` extension (arbitrary fields + omittable `schema_version`/`worktree_path`). Phases 1-2 touch no emission and are independent of #330 (reviewer-verified: neither verb emits; Branch-2 detection is shape-agnostic; the registry/test/prose edits touch no `lifecycle_event.py` emission path). #330 is currently unstarted (mid-refine in a concurrent session); coordinate trunk commits with explicit `git commit -- <files>` pathspec.
- **No new `build_row`.** Emit through #330's `--field` rather than a #331-local serializer; a second serializer is the silent-drift the byte-identity discipline exists to prevent, and a shared builder would have to be introduced by #330 (epic mandates #330 first), making a #331-introduced `build_row` circularly ordered.
- **`complete-route` structure vs reputation.** Model the verb's **structure** on `_cli_detect_phase`/`detect_lifecycle_phase` (`cortex_command/common.py`) — a pure classifier function + thin `main(argv)->int` CLI, tolerant per-line `json.loads`, `json.dumps(separators=(",",":")) + "\n"`. But unlike `detect_lifecycle_phase` (whose documented contract is "runs on every invocation and must stay git-free and fast", and which the bash statusline mirrors precisely to stay under its <500ms budget), `complete-route` does network `gh` I/O and a conditional `pr.json` write — so it must NOT inherit detect-phase's safe-to-call-anywhere usage; Req 7a enforces the speculative-caller boundary structurally.
- **Read-only-except-recovery.** The Branch-3 single-match `pr.json` reconstruction is the verb's only write — a recovery side-effect, atomic via `os.replace`, idempotent across sequential re-invocations (once `pr.json` exists Branch 3 does not refire). Document the verb as a stateful recovery+routing verb, not a pure classifier.
- **Tests live in top-level `tests/`** (since `just test` runs only `pytest tests/`); modules in `cortex_command/lifecycle/`. Real-git via the `_git` harness with `core.hooksPath /dev/null`; gh via `tests/fixtures/gh-stub.sh`.
- **L1 ratchet does not apply** — `complete.md`/`post-refine-commit.md` are reference files, not `SKILL.md`.
- **Authoring**: rewritten prose is What/Why ("run it, act on `route`"), soft positive-routing phrasing; keep the stage-first guard and halt-on-failure as structural control flow, not prose-only.

## Open Decisions

None remaining. The two posture decisions surfaced by critical-review were resolved at spec time by operator decision to **fix both** (rather than preserve + follow-up):

- **OD-1 (resolved → FIX)** — the Step 11→11a emission-ordering hazard is fixed by emitting `feature_complete` only after a successful Step 11a commit (Req 16).
- **OD-2 (resolved → FIX)** — the `git add -u cortex/backlog/` cross-session sweep is narrowed to the two explicit paths this feature touches (Req 11), with a no-sweep test control (Req 13).

Both fixes are folded into the offload as a deliberate, operator-authorized exception to the epic's "offloads are not correctness fixes" carve-out, scoped to bugs the offload would otherwise entrench (see Scope note).

## Proposed ADR

### Proposed ADR: 0017-events-log-migration-preserves-field-set-not-bytes

**Context**: The offload epic (#336) pins "byte-identical events.log rows" when migrating hand-written emission sites to CLI verbs. Research (and the critical-review byte-invariant angle) found the `events.log` corpus is already heterogeneous — `lifecycle_event.py` emits compact separators + microsecond/offset timestamps, `pipeline/state.py` emits spaced separators + microsecond timestamps, and the hand-written `Z` second-precision form in skill prose is produced by no Python emitter (a copy-paste artifact). Every consumer parses tolerantly (`json.loads`/substring/regex). **Decision**: the migration invariant is **field-set + field-values identity** — the exact key set per event (presence/absence of `schema_version` and `worktree_path`), the values/types of `merge_anchor` (`"merge"`), `tasks_total`/`rework_cycles` (ints), and the presence of `feature` and `ts` (both unguarded-indexed by `metrics.py:223/292`, so their absence KeyErrors the whole corpus) — **not** literal bytes; the cosmetic separator/timestamp format follows the canonical emitter, and the load-bearing semantics are pinned by per-event round-trip tests that classify each row through a real consumer. **Trade-off**: gives up literal fidelity to the ticket's "byte-for-byte" wording and accepts a cosmetic format shift on migrated rows, in exchange for not adding a bespoke timestamp formatter that exists nowhere, not fighting the two existing serializers, and not duplicating #330's `--field` with a second row-builder. This ADR governs #330, #331, and #332's emission migrations uniformly.
