# Plan: unified-backlog-lifecycle-slug-resolver-extend

## Overview
Extract a pure `resolve()` library function from `cortex_command/backlog/resolve_item.py`, extend it from the current 3-step order to a 5-step order (UUID-prefix ‚Üí numeric ‚Üí exact stem with-or-without `NNN-` prefix ‚Üí exact `lifecycle_slug` frontmatter ‚Üí ranked title-substring), and retrofit `cortex_command/backlog/update_item.py:_find_item` as a thin consumer that surfaces ambiguity as exit-2 + stderr candidate list. A frozen 3-step baseline fixture (captured before any resolver change) gates order-drift regression. Six skill files gain an exit-2 handling sentence in the same PR so the contract change rolls atomically.

**Architectural Pattern**: layered ‚ÄĒ the change reifies a pure-library / CLI-translation split that already exists in spirit (resolution helpers in `resolve_item.py` are pure today; only `main()` produces exit codes). Both consumers (`cortex-resolve-backlog-item` and `cortex-update-item`) become thin CLI shims over the same library layer, and each retains its own argparse + exit-code translation.

## Outline

### Phase 1: Resolver extension (tasks: 1, 2, 3, 4)
**Goal**: Land the 5-step `resolve()` library function and prove it does not drift from the 3-step baseline on any existing `CURATED_INPUTS` case.
**Checkpoint**: `pytest tests/test_resolve_backlog_item.py` is green; `tests/fixtures/predicate_3step_baseline.json` exists; new UUID-prefix and lifecycle_slug-frontmatter cases resolve as designed; `cortex-resolve-backlog-item` exit codes (0/2/3/64/70) unchanged on the existing corpus.

### Phase 2: Consumer retrofit + skill-prose updates (tasks: 5, 6, 7, 8, 9)
**Goal**: Replace `_find_item`'s inline resolution with a library call, surface ambiguity as exit-2 in `cortex-update-item`, and update six skill files to recognize exit-2 in their failure-handling prose.
**Checkpoint**: `grep -c "slug_or_uuid in p.stem" cortex_command/backlog/update_item.py` = 0; `pytest tests/test_update_item_resolution.py` is green; all six skill files contain the exit-2 prose; `just test` is green overall.
