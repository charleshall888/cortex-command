# Specification: extract-backlog-management-skill-into-optional

> Epic context: P1 of #315 (Optional backlog plugin + configurable backend). Background design: `cortex/research/backlog-optional-plugin/research.md`. This ticket is the plugin extraction + the full operator-mandated slash-namespace rename; the configurable-backend resolver, backend-routing, external best-effort, and overnight refusal are later tickets.

## Problem Statement

The backlog management skill ships inside cortex-core, so installing the harness forces the local `cortex/backlog/` surface on every user — including teams who track work in GitHub Issues or Jira. This ticket makes the interactive backlog surface opt-in by extracting **only** `skills/backlog` into a new optional `cortex-backlog` plugin (mirroring the cortex-overnight optional-plugin pattern), while `skills/backlog-author` stays in cortex-core because discovery and morning-review compose ticket bodies through it on the external-tracker path even when the local plugin is absent. The backlog **engine** (`cortex_command/backlog/*` and its `cortex-*` console scripts) stays in the wheel, untouched — this is packaging-only at the engine layer. It is **not** a no-op at the consumer layer: moving the skill changes its slash command `/cortex-core:backlog` → `/cortex-backlog:backlog` (a real user-facing rename plus a new install requirement to retain the backlog UI), so all live references are renamed comprehensively in the same ticket (operator decision: rename everything, catch everything).

## Phases
- **Phase 1: Atomic extraction** — create the cortex-backlog plugin and move the `backlog` skill into it, co-landing every build/parity/marketplace/test/doc-amendment edit the gates force together, in one commit. The build-plugin recipe `case` branch and a build-derivation check are explicit requirements (the drift gate alone does not enforce them for a new plugin).
- **Phase 2: Comprehensive namespace rename** — rename every live `/cortex-core:backlog` reference to `/cortex-backlog:backlog` (skills, docs, init template, two test fixtures), handling the L1-ratchet side effect, and verify zero live references remain.

## Requirements

**Priority (MoSCoW)**: All requirements below are **must-have**. Phase 1 must co-land as one commit, and Phase 2's comprehensive rename is the operator's explicit mandate ("rename everything, catch everything"). There are no should-have/nice-to-have items; the deliberate scope cuts (the won't-do set) are enumerated under Non-Requirements.

## Edge Cases
- **plugin.json missing before `just build-plugin`** → build silently skips cortex-backlog and an incomplete commit can pass. Expected: hand-create plugin.json first (requirement 1); requirement 4's delete-then-rebuild check independently proves the mirror is recipe-built.
- **Missing `cortex-backlog)` recipe branch** → with the name in BUILD_OUTPUT_PLUGINS but no branch, the recipe falls to `*) continue` and never builds the mirror while gates pass green. Expected: requirement 2 mandates the branch; requirement 4 proves build-derivation.
- **Orphaned `plugins/cortex-core/skills/backlog/`** after `backlog` leaves cortex-core's array → `rsync --delete` is per-skill so it never touches the removed dir and the drift gate sees no diff. Expected: explicit `git rm -r` (requirement 5).
- **Incomplete rename inventory** → `skill_trigger_phrases.yaml:52` is inside the completeness-grep file-set; missing it both fails requirement 14's gate and breaks `test_skill_descriptions.py`. Expected: requirements 14 + 15 cover it.
- **L1 ratchet** — the description rename adds +3B over the 319B budget. Expected: trim, not re-cap (requirement 17).
- **init-artifacts hash** — there is NO checked-in hash to bump; the hash recomputes live, so editing the README template is safe (requirement 19).
- **Cross-plugin `/backlog-author` resolution** — kept bare; demonstrated working by morning-review's production bare cross-plugin call. Expected: post-install verification (requirement 20).
- **Renamed `docs/backlog.md` heading** — no anchor links target it (verified safe).

## Technical Constraints
- **Phase-1 co-land**: the pre-commit Phase-1 plugin classification gate and Phase-4 dual-source drift gate force most of the set together — but the drift gate is **blind to a build-output plugin the recipe skips** (a no-op build leaves worktree == index), so the `cortex-backlog)` recipe branch (req 2) and the build-derivation check (req 4) are explicit requirements rather than gate-implied. Canonical sources + generated mirrors commit together (CLAUDE.md "Drift hook + shared-checkout coupling").
- **build-plugin sequencing**: the plugin's `.claude-plugin/plugin.json` must exist before `just build-plugin` (justfile:593 skips unmaterialized plugins).
- **Orphan cleanup is manual**: the recipe rsyncs per-skill only; a removed skill's mirror dir must be `git rm`-ed (req 5).
- **parity scanner**: `PLUGIN_NAMES` membership excludes a token from both candidate (parity_check.py ~635) and wiring (~607) evaluation; `RESERVED_NON_BIN_NAMES` excludes from candidates only. `TOKEN_RE` is word-boundaried, so `cortex-backlog-*` scripts are not shadowed by the `cortex-backlog` plugin token — moving out of RESERVED is complete and safe (no self-test pins cortex-backlog in RESERVED).
- **plugin-list-matches-justfile self-test** asserts `PLUGIN_NAMES == BUILD_OUTPUT ∪ HAND_MAINTAINED`; marketplace.json and docs/setup.md have no completeness self-test (reqs 9 and 18 enforce those edits explicitly).
- **L1 ratchet** is keyed on canonical skill dir name; the rename's only ratchet impact is the `backlog` description trigger phrase (319B at budget, +3B). Resolution: trim to ≤319 (req 17), do not raise the budget row.
- **init-artifacts hash is computed live** (`scaffold.py:_compute_init_artifacts_hash` returns `f"v1:{h.hexdigest()}"`); there is no hash constant to bump and no consumer in-repo pins the value, so editing a `_HASH_INPUT_TEMPLATES` template is safe (req 19).
- **Slash namespace is plugin-derived**; a bare skill invocation resolves cross-plugin by name (demonstrated by morning-review), which is why req 12 keeps the bare form.
- **Dual-source discipline**: edit canonical top-level sources only; never hand-edit `plugins/**` mirrors.
