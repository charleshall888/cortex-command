# Plan: route-agentic-search-fanouts-to-sonnet-grunt-tier

## Overview
Add a criticality-independent `searcher → sonnet` constant to `cortex-resolve-model` (modeled on the existing `synthesizer` constant), then thread the resolved model into the **core wave** of the interactive research/discovery fan-out — leaving the always-last adversarial wave inheriting the parent — so interactive `/research` and `/discovery` run their breadth-first gather agents on Sonnet while the Opus orchestrator synthesizes. Routing rule is authored once in the shared `fanout.md` so research and discovery cannot drift; resolver failure degrades-loud-to-inherit (not halt).
**Architectural Pattern**: shared-state
<!-- The `cortex-resolve-model` verb is the single owned source of the model name; every dispatch site resolves from it rather than hardcoding "sonnet" (single-matrix-owner). This variant centralizes the routing rule in the shared fanout.md protocol consumed by both entry points. -->

## Outline

### Phase 1: The role (tasks: 1, 2)
**Goal**: Add `searcher → sonnet` as a criticality-independent constant in `cortex-resolve-model` and pin it in the golden-anchor test. Self-contained, no consumer yet.
**Checkpoint**: `.venv/bin/python -m pytest tests/test_resolve_model.py` exits 0 with the new `test_searcher_is_constant_sonnet` green; `cortex-resolve-model --role searcher` resolves `sonnet` with and without `--criticality`.

### Phase 2: The wiring (tasks: 3, 4, 5, 6, 7, 8)
**Goal**: Route the research/discovery core wave to the resolved `searcher` model via the shared `fanout.md` protocol, keep the adversarial wave inheriting, degrade-loud-to-inherit on resolver failure, document the rationale + judgment-inherit contract in `sdk.md`, record ADR-0023, and regenerate the plugin mirror in the same commits.

## Risks
- **Same-mirror-tree commit sequencing (Tasks 3 + 4) — a real hazard, not "either is fine".** Both edit the `plugins/cortex-core/skills/research/**` mirror tree, and the pre-commit drift hook diffs the *whole* plugin subtree (working-tree-vs-index) AFTER running `just build-plugin`, which regenerates the entire subtree via `rsync -a --delete`. So if both canonical files are edited into the working tree first and Task 3 is then committed alone via explicit pathspec, build-plugin regenerates BOTH mirror files and the unstaged Task-4 mirror change trips `drift_found` → the Task 3 commit hard-fails (exit 1). Mitigation (the implement phase must pick one): either (a) strict per-file edit→`just build-plugin`→stage canonical+mirror→commit before touching the next same-tree canonical, or (b) combine Tasks 3+4 into a single commit. Mirror regen is never deferred to Task 8 either way.
- **PATH binstub staleness.** `cortex-resolve-model` (and the `cortex-check-*`/`cortex-adr-citation-audit` verbs) on `$PATH` resolve to the installed wheel, which lacks the new `searcher` role until reinstall. All in-flight verification uses the source (`.venv/bin/python -m ...` or `CORTEX_COMMAND_FORCE_SOURCE=1`) — the released binstub catching up is expected, not a defect.
- **Discovery needs its own resolve+bind (Task 5), not a pointer.** Discovery dispatches from `research.md`'s own inlined two-wave steps, not by executing `fanout.md` (which it consumes only for sizing/angle-selection), so the resolve+bind must live in discovery's executed body for it to route to Sonnet. Task 5 plants that runnable resolve+bind (mirroring research Task 4) and cites fanout.md as the rule's source — the single-authoring R7 design is preserved (one rule, two consumers each binding it), and the parity tests keep both mirrors honest. The residual risk is duplicated-but-agreeing resolve lines in research and discovery; accepted because each entry point has an independent executed body and the alternative (a shared executed body) does not exist.
- **Tradeoffs angle runs on Sonnet.** The wave-not-angle routing sends the mildly-evaluative Tradeoffs angle to Sonnet (it joins the core wave). Accepted per spec R4 / ADR-0023: its output is folded into the Opus synthesis and re-challenged by the adversarial pass.
