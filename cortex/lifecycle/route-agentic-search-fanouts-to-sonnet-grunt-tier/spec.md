# Specification: route-agentic-search-fanouts-to-sonnet-grunt-tier

## Problem Statement

The harness's interactive parallel-research fan-out dispatches its core-wave agents with **no model parameter**, so they inherit the parent session model — Opus, in an interactive session. That runs breadth-first read-and-report "grunt" work on the most expensive model for no quality benefit (the orchestrator synthesizes the outputs; per-agent depth is not the bottleneck). This change adds a `searcher` role to `cortex-resolve-model` resolving to the Sonnet tier and routes the core wave to it, so the orchestrator stays strong (Opus) while the grunt fan-out runs on Sonnet — the "Opus orchestrator + Sonnet workers" pattern Anthropic's own multi-agent research system validated (+90% over single-agent Opus). It benefits anyone running `/cortex-core:research` (and `/cortex-core:discovery`'s research phase) interactively: lower cost, same or better artifact. This is the interactive research/discovery fan-out specifically — the one model-routing surface not already covered by the pipeline matrix (Path B, already Sonnet-heavy) or the lifecycle role matrix (builder/reviewer, already routed). Other model-less judgment dispatches (research-adversarial, critical-review reviewers, clarify-critic) are deliberately left inheriting (see Non-Requirements and the judgment-inherit contract in R8); this change does not claim to convert every model-less surface, only the gather fan-out.

## Phases

- **Phase 1: The role** — add a criticality-independent `searcher → sonnet` constant to `cortex-resolve-model` and pin it in the golden-anchor test. Self-contained and independently testable; no consumer yet.
- **Phase 2: The wiring** — thread the resolved `searcher` model into the research (and discovery) **core-wave** dispatches via the shared fan-out protocol, leave the always-last adversarial wave inheriting the parent, fall back to inherit-parent (with a surfaced warning) if the resolver fails, document the rationale and the judgment-inherit contract in `sdk.md`, and regenerate the plugin mirror.

## Requirements

1. **`searcher` role resolves to `sonnet`, criticality-independent.** Acceptance: `cortex-resolve-model --role searcher` prints `sonnet` and exits 0; `cortex-resolve-model --role searcher --criticality low` also prints `sonnet` and exits 0 (the constant resolves with or without `--criticality`, matching `synthesizer`). **Phase**: The role.
2. **Modeled as a `_CRITICALITY_INDEPENDENT` constant, not a tier-keyed row.** Acceptance: `cortex_command/lifecycle/resolve_model_cli.py` contains `"searcher": "sonnet"` inside the `_CRITICALITY_INDEPENDENT` dict (`grep -c '"searcher": "sonnet"' cortex_command/lifecycle/resolve_model_cli.py` ≥ 1); the frozen golden matrix `_expected_golden_matrix()` and its `{review, builder, orchestrator-fix, competing-plan}` set assertion in `tests/test_resolve_model.py:172-181` are unchanged. **Phase**: The role.
3. **Golden-anchor test extended for the new constant.** Acceptance: `tests/test_resolve_model.py` adds `test_searcher_is_constant_sonnet` (mirroring `test_synthesizer_is_constant_opus:80`, asserting `sonnet\n` with and without `--criticality`) and adds `"searcher"` to `_ALL_ROLES:33`; `.venv/bin/python -m pytest tests/test_resolve_model.py` exits 0. **Phase**: The role.
4. **The entire core wave routes to the resolved `searcher` model.** The routing unit is the **wave**, not the angle: the core wave (mandatory core — Codebase, Web, Requirements & Constraints — plus *all* orchestrator-chosen angles, including Tradeoffs) is dispatched on the `searcher` model. This matches the actual fan-out seam (`skills/research/SKILL.md:186` / `fanout.md:32` group every non-adversarial angle into one core-wave batch) and restores the operative rule from `research.md` OQ1 ("every chosen angle including Tradeoffs → the Sonnet role"). Acceptance: `skills/research/SKILL.md` Step 3 resolves `cortex-resolve-model --role searcher` in the orchestrator **body** and binds the captured value as the `model:` parameter of the core-wave `Agent` dispatch batch; observable: `grep -c 'cortex-resolve-model --role searcher' skills/research/SKILL.md` ≥ 1 AND the core-wave dispatch instruction (Step 3 §1) carries the `model:`-binding directive. **Phase**: The wiring.
5. **Only the always-last adversarial wave inherits the parent — and the ACs witness both sides.** The adversarial-wave dispatch (Step 3 §2) does **not** bind the `searcher` model; it omits `model` and inherits the parent (it is the error-correction layer and runs only at high/critical). Acceptance (both observable conditions must hold, so an all-waves or no-waves wiring fails): (a) the core-wave dispatch instruction binds the resolved `searcher` model; (b) the adversarial-wave instruction contains no `searcher`/`model:` binding and states it inherits the parent — verifiable by reading `skills/research/SKILL.md` Step 3 §1 (binds) vs §2 (omits). **Phase**: The wiring.
6. **Degrade-loud-to-inherit on resolver failure — do not halt.** On nonzero exit from `cortex-resolve-model --role searcher`, the orchestrator falls back to dispatching the core wave with **no `model:` parameter** (inherit-parent, exactly as today) and surfaces a one-line warning to the output noting the gather wave is running on the inherited model because role resolution failed. It does **not** halt the research run. Rationale: unlike the criticality-keyed sibling roles (where a bad substitution risks a quality *downgrade*, so halting protects correctness), a `searcher` resolver failure's only fallback is the inherited parent — a non-regressing inherit — and the realistic trigger is wheel-vs-mirror version skew (the skill ships in the plugin, the verb in the wheel), not a global fault; halting would break interactive `/research` for zero correctness benefit, against project.md's "keep working unless blocked." Acceptance: the Step-3 wiring includes a fallback-to-inherit-plus-surfaced-warning branch on resolver nonzero (not a halt); observable: a "fall back"/"inherit"/"warn" instruction adjacent to the resolve call, and no `halt`/`MUST`/`CRITICAL`/`REQUIRED` token introduced (MUST-escalation policy). The warning is surfaced to the run output; it does **not** depend on an `events.log` (research has a standalone mode with no lifecycle session). **Phase**: The wiring.
7. **Routing lives in the shared fan-out protocol so research and discovery cannot drift.** The core-wave model-routing instruction (and the adversarial-inherits carve-out) is authored in `skills/research/references/fanout.md` (the canonical shared source cited by research, refine, lifecycle, discovery), and `skills/discovery/references/research.md` inherits/references it rather than restating a divergent rule; observable: `grep -rl 'searcher' skills/research/references/fanout.md` matches, and discovery's research dispatch does not contradict it. **Phase**: The wiring.
8. **Rationale AND the judgment-inherit contract recorded in the owning doc.** `docs/internals/sdk.md` gains (a) a `searcher → sonnet` rationale bullet extending the existing "parallel agents → sonnet" line (`sdk.md:149`), and (b) a one-line statement of the cross-skill contract that judgment dispatches — research-adversarial, critical-review reviewers, clarify-critic — deliberately inherit the parent model rather than routing to `searcher`, so a future editor of any of those surfaces finds the omission documented at the model-selection owner doc. Acceptance: `grep -c 'searcher' docs/internals/sdk.md` ≥ 1 AND `sdk.md` contains the judgment-inherit note naming at least critical-review reviewers and clarify-critic. **Phase**: The wiring.
9. **Plugin mirror regenerated in the same commit.** Acceptance: after the Phase-2 commit, `just build-plugin` produces no diff and `.venv/bin/python -m pytest tests/test_dual_source_reference_parity.py tests/test_plugin_mirror_parity.py` exits 0 (the `plugins/cortex-core/skills/research/**` mirror is regenerated and committed alongside the canonical edit). **Phase**: The wiring.
10. **Research frontmatter untouched (L1 budget).** Acceptance: `skills/research/SKILL.md` frontmatter `description`/`when_to_use` are byte-unchanged (`git diff` shows no change in lines 1–15); `.venv/bin/python -m pytest tests/test_l1_surface_ratchet.py` exits 0 (research stays at its 379B baseline, which has zero headroom). **Phase**: The wiring.

## Non-Requirements

- Does **not** touch the overnight **pipeline** matrix (`cortex_command/pipeline/dispatch.py` / `_MODEL_MATRIX`). Path B is deliberately separate and already Sonnet-heavy; out of scope.
- Does **not** pin the currently-inheriting **judgment** dispatches to a Sonnet role: research-adversarial, critical-review's parallel reviewers (`critical-review/SKILL.md:58,66`), and the clarify-critic all stay inheriting the parent. This boundary is **file-locality**, not feature-size: this change edits `skills/research`/`fanout.md`/`discovery`, so it addresses research-adversarial in-scope but does not reach into the critical-review/clarify skill files. The judgment-inherit contract is anchored in ADR-0023 and `sdk.md` (R8) so the omission is documented; planting per-skill breadcrumb comments at `critical-review/SKILL.md:58` and the clarify-critic is a deferred, optional follow-up, not required here. (Routing critical-review's reviewers to Sonnet specifically would be a *downgrade* the ADR opposes; making their model criticality-explicit while preserving Opus is a different change entirely — neither is in scope.)
- Does **not** add a Haiku tier or route the read-only Codebase-search agent to Haiku. The user's framing was "Sonnet grunt"; a cheaper Haiku tier for pure file discovery is a possible future refinement, not this change.
- Does **not** add an opt-out / per-run model-override flag or env var. The single owned constant is the global override surface; criticality is the depth lever (it scales angle count and the adversarial pass, not the gather model); robustness on resolver failure is handled by R6's inherit-fallback. No interactive per-run override precedent exists, and adding one re-imports the cost this change removes.
- Does **not** route the `parallel-execution.md` worktree dispatch (it runs a whole `/cortex-core:lifecycle` — an orchestrator, not a gather agent) or devils-advocate (inline; no sub-agent dispatch exists).

## Edge Cases

- **Resolver nonzero (incl. wheel-vs-mirror version skew)**: fall back to inherit-parent and surface a warning (R6) — do not halt. The realistic trigger is version skew: the skill wiring ships in the plugin (`/plugin install`) while the verb ships in the wheel (`uv tool install`); if the plugin updates first, `cortex-resolve-model --role searcher` exits 2 (invalid choice) at that single site while `review`/`builder`/`synthesizer` resolve fine. This is **not** a global install fault, and the degrade-loud fallback keeps `/research` working (on the inherited model) until the wheel catches up, with the warning making the degraded state visible.
- **Standalone `/research` (no lifecycle session)**: `searcher` resolves with no `--criticality` and no lifecycle-state read (constant), so the core wave routes to Sonnet even when no lifecycle/criticality exists. No `cortex-lifecycle-state` call is added to the gather path; the R6 warning surfaces to output, not an events.log.
- **Parent session already on Sonnet**: pinning `searcher = sonnet` is cost-neutral and observationally near-identical to inheriting (same bare alias resolves to the same model; sub-agents have independent contexts, so no parent-cache delta of note).
- **Discovery research phase**: inherits the same core-wave routing via the shared `fanout.md` (R7), so it does not diverge from `/research`.
- **A future core-wave angle**: any newly-added orchestrator-chosen angle joins the core wave and routes to `searcher`; only the always-last adversarial wave inherits. The seam is the existing core-vs-adversarial wave boundary, which the fan-out already provides — so the routing rule needs no per-angle adjudication.

## Changes to Existing Behavior

- **MODIFIED**: research core-wave agents (mandatory core + all chosen angles, including Tradeoffs) previously inherited the parent (Opus, interactively) → now dispatch on Sonnet via the `searcher` role.
- **MODIFIED**: `/cortex-core:discovery`'s research-phase core wave likewise routes to Sonnet (via the shared `fanout.md`).
- **ADDED**: a new `searcher` role in `cortex-resolve-model` (additive — no existing role's resolution changes; `_ROLE_CHOICES` auto-includes it via `resolve_model_cli.py:75`).
- **UNCHANGED (explicit)**: the research adversarial wave, critical-review reviewers, and clarify-critic continue to inherit the parent model (the judgment-inherit contract, now documented in `sdk.md` per R8).

## Technical Constraints

- **Single-matrix-owner**: the tier must be a role inside `cortex-resolve-model`, never a hardcoded `"sonnet"` in a skill (`docs/internals/sdk.md:144`, `resolve_model_cli.py:5`, `criticality-matrix.md:24`). This is the durable form of the rejected archived approach (`cortex/research/archive/subagent-model-routing/` "Option D", which scattered literals and flagged its own drift risk).
- **Constant modeling** (`_CRITICALITY_INDEPENDENT`, not a 4-identical-cell row): minimizes golden-test churn and avoids the empty low/med/high-cell smell the module warns against (`resolve_model_cli.py:65-72`).
- **Body-only wiring** in `skills/research/SKILL.md`: research frontmatter has zero L1 headroom (379B == baseline), so no model-resolution wording may enter `description`/`when_to_use`.
- **ADR-0009 propagation**: resolve in the orchestrator body and propagate the resolved `$model` into the Agent dispatch; do not place raw `${CLAUDE_SKILL_DIR}` or bare relative paths into the fenced agent-prompt blocks (`cortex-check-skill-path` SP001/SP002).
- **Soft phrasing** for the resolver-failure branch (post-4.7 MUST-escalation policy) — no `MUST`/`CRITICAL`/`REQUIRED` token; the branch degrades, it does not command.
- **Mirror/drift**: the Python module is not plugin-mirrored (ships via wheel); the `skills/research` edit requires regenerating `plugins/cortex-core/skills/research/**` in the same commit (pre-commit `just build-plugin` + dual-source parity test). The wheel-vs-mirror cadence split is the reason R6 degrades rather than halts (see Edge Cases).
- **Bare alias**: `cortex-resolve-model` emits the bare alias `sonnet`, which the CLI (verified 2.1.197) resolves to `claude-sonnet-5`; there is no alias→pinned-ID layer in the repo, and none is added.

## Open Decisions

None. Both forks the critical-review surfaced are resolved in-spec: the routing unit is the **wave** (whole core → searcher, adversarial inherits — R4/R5), matching the real fan-out seam and `research.md` OQ1; and resolver failure **degrades-loud-to-inherit** (R6), not halts. The "how to be loud" sub-question is resolved: surface a one-line warning to the run output, with no `events.log` dependency.

## Proposed ADR

### Proposed ADR: 0023-route-core-research-fanout-to-sonnet-searcher-tier

**Context.** The interactive parallel-research fan-out dispatches its read-only core-wave agents with no model parameter, so they inherit the parent (Opus) session model — running Sonnet-appropriate breadth-first work on the most expensive model. **Decision.** Add a criticality-independent `searcher → sonnet` role to `cortex-resolve-model` (modeled on the existing `synthesizer` constant) and route the **entire core wave** (mandatory core + all orchestrator-chosen angles) to it; the **always-last adversarial wave** continues to inherit the parent model, and reviewers/critics in other skills are untouched. On resolver nonzero exit, **degrade-loud-to-inherit** (fall back to the inherited parent model and surface a warning) rather than halting. No opt-out flag. **Trade-offs.** (1) The routing unit is the **wave**, not the angle — chosen because the fan-out's only structural seam is core-vs-adversarial; an angle-level gather/judgment split has no structural anchor and produced contradictory routing for chosen angles like Tradeoffs. The cost: the Tradeoffs angle (mildly evaluative) runs on Sonnet, accepted because its output is folded into the Opus synthesis and re-challenged by the adversarial pass. (2) Constant role, not a tier-keyed row — chosen for minimal test churn and to avoid empty-cell drift; the cost is a small asymmetry vs the tier-keyed roles. (3) Degrade-loud-to-inherit, not halt — chosen because the `searcher` failure's only fallback is the inherited parent (a non-regressing inherit, unlike the quality-downgrade risk that justifies halting at criticality-keyed sibling sites), the realistic trigger is wheel-vs-mirror version skew (a single-site failure, not a global fault), and halting would break interactive `/research` against project.md's "keep working unless blocked." Hard-to-reverse as the established convention for interactive fan-out model routing; surprising without this context (why interactive research runs on a different model than the session); a real trade-off on each axis above.
