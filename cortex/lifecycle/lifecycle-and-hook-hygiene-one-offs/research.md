## Web Research

### Sub-item 1 — Resolver pass-through / redundant glob avoidance

- **Rspack-resolver cache system** ([deepwiki.com/web-infra-dev/rspack-resolver/2.3-cache-system](https://deepwiki.com/web-infra-dev/rspack-resolver/2.3-cache-system)): canonical pattern is "cache the resolved entity, not the lookup inputs."
- **DuckDB glob result cache** ([github.com/duckdb/duckdb/discussions/21678](https://github.com/duckdb/duckdb/discussions/21678)): session-level cache eliminates redundant I/O.
- **Claude Agent Skills overview** ([platform.claude.com/docs/en/agents-and-tools/agent-skills/overview](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/overview)) and the [SKILL.md pattern article](https://bibek-poudel.medium.com/the-skill-md-pattern-how-to-write-ai-agent-skills-that-actually-work-72a3169dd7ee) emphasize progressive disclosure — redundant tool surfaces consume catalog tokens.
- **Geeky-gadgets token-cost piece** ([geeky-gadgets.com/agent-skills-code-markdown](https://www.geeky-gadgets.com/agent-skills-code-markdown/)) cites a 100x token-cost gap (50k vs 200 tokens) for redundant skill surface — supports reframing as token waste.

### Sub-item 2 — SessionStart hook active-state gating

- **Claude Code hooks reference** ([code.claude.com/docs/en/hooks](https://code.claude.com/docs/en/hooks)) documents the `source` field on SessionStart (`startup`/`resume`/`clear`/`compact`) and warns: "SessionStart runs on every session, so keep these hooks fast."
- **Lefthook `glob:` filter** ([lefthook.dev/configuration/glob.html](https://lefthook.dev/configuration/glob.html)): skips the command entirely if no files match — direct analogue for "skip if no active pipeline."
- **Atlassian Git Hooks tutorial** ([atlassian.com/git/tutorials/git-hooks](https://www.atlassian.com/git/tutorials/git-hooks)): canonical `git diff --cached --name-only | grep -E ...` early-exit pattern; detect "is anything in scope" and `exit 0` immediately when not.

### Sub-item 3 — Delete vs. Chesterton's-fence for solo-dev tooling

- **Pro-delete**: [Understand Legacy Code: "Delete unused code"](https://understandlegacycode.com/blog/delete-unused-code/) — "If nobody's touched it in a year, and runtime telemetry shows zero usage, it's dead." Recommends git-tag-recovery (`git tag deprecated-<name>-<date>`) before delete. [Pensero "Guide to Dead Code Identification 2026"](https://pensero.ai/blog/dead-code): cruft cost accrues continuously.
- **Chesterton's-fence counter**: [George Stocker](https://georgestocker.com/2020/03/24/chestertons-fence/), [Symflower](https://symflower.com/en/company/blog/2022/programming-principle-chestertons-fence/) — understand *why* before deleting; load-bearing code can have no recent usage signal.
- **Solo-dev resolution**: literature converges on a practical middle ground — solo-dev tooling lowers Chesterton's bar because git history is fully accessible to the same author and blast radius is bounded. Mitigation: git-tag-recovery makes the deletion reversible and converts the epistemic concern into a 30-second `git show` lookup.

### Sub-item 4 — Scoped pre-edit hooks

- **pre-commit.com**: `--files` flag scopes hook invocation; `files:` regex in `.pre-commit-config.yaml` scopes which hooks fire. Doctrine: full test suites are too slow for pre-commit.
- **Lefthook glob docs** ([lefthook.dev/configuration/glob.html](https://lefthook.dev/configuration/glob.html)): `glob:` filters against `{staged_files}`; skips command entirely if no files match.
- **pytest-testmon** ([github.com/tarpas/pytest-testmon](https://github.com/tarpas/pytest-testmon)): tracks code→test deps via Coverage.py; runs only affected tests.
- **Jest `--findRelatedTests`** ([thesametech.com](https://thesametech.com/under-the-hood-jest-related-tests/)): analyzes file dep graph.
- **Token-cost framing for Cut 4**: pure latency unless hook stdout enters the model's context. `cortex-skill-edit-advisor.sh:43-45` does emit truncated stdout (`head -20`) into agent context via PostToolUse `additionalContext`, so framing as "context bloat per advisor fire" is defensible. Reframe required if shipping under epic #187 without splitting.

## Adversarial Review

### Failure modes and edge cases

- **Sub-item 1 — Step 1's `slugify($ARGUMENTS)` does not resolve against disk.** The 4 globs are first-reads, not duplicates of an upstream call. The ticket's framing of "the resolver already has the answer" conflated lifecycle SKILL.md Step 1 with `bin/cortex-resolve-backlog-item` (used by refine/research/discovery, NOT by lifecycle Step 1). Any spec that says "reuse the resolver's output across 4 sites" without first building or invoking a resolver is a no-op.
- **Sub-item 1 — Filename ambiguity is load-bearing.** Backlog file `170-fix-pre-existing-scan-lifecycle-test-failures-in-tests-test-hookssh.md` contains "scan-lifecycle" in its name. A user invoking `/cortex-core:lifecycle scan-lifecycle` would match this against the wrong file. The current 4 separate scans each have independent fail-silent semantics; collapsing them puts the disambiguation rule in one place and makes a single wrong decision propagate to all 4 surfaces.
- **Sub-item 1 — No-arg invocation path.** Step 1 already handles "no $ARGUMENTS" via fall-through to scanning incomplete lifecycle dirs (`SKILL.md:48`). Any single-scan rewrite must preserve this branch.
- **Sub-item 2 — L417/L446 are already gated.** The ticket's stated target does not exist as described. Any spec needs to either retarget the cut or drop it.
- **Sub-item 2 — Sequencing trap on L244-256.** `active_phase` (L310-343) ← `incomplete_phases` (L300-301) ← `batch_output` (L244-256). Gating L244-256 propagates to the entire downstream phase-aware context block — exceeds out-of-scope.
- **Sub-item 3 — Orphan-reference risk.** 6 sites must update coherently or the skill ships internally inconsistent (SKILL.md description trims trigger phrase but body still tells Claude to read the deleted reference file).
- **Sub-item 3 — `test_lifecycle_references_resolve` parallel check for skill-internal references.** If a similar test validates SKILL.md-referenced reference files exist, deleting `auto-scan.md` will fail it. Verify before merging.
- **Sub-item 4 — `test-skill-design` red on main.** Pre-existing `test_lifecycle_references_resolve.py` failure due to a broken reference in `lifecycle/audit-cortex-coreresearch-skill-output-shape-for-token-waste-in-researchmd-sections/review.md:39`. Scoping the advisor to `test-skill-design` ships permanent failure-message context bloat until that's fixed.
- **Sub-item 4 — Cap stdout regardless of pass/fail.** Current `head -20` truncation runs even on success and dumps ~60-200 tokens into context per fire. Spec should cap to ≤500 chars and prefer concise success summary over verbose output dump.

### Anti-patterns to avoid

- Adding "reuse the resolver's output" prose to SKILL.md without an actual resolver in Step 1 — that's honor-system without a contract.
- Forcing all four sub-items under epic #187 (token-waste) when Sub-item 2 is wall-clock-only and Sub-item 4 is mixed — invites a follow-up audit (#185-style) that finds the token savings unverifiable. Honest categorization: split or explicitly reframe in the spec/PR.
