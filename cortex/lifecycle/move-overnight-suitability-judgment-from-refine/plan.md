# Plan: move-overnight-suitability-judgment-from-refine

## Overview
Relocate overnight-suitability judgment from `/refine` to overnight's interactive curation gate, via three sequenced changes: (1) a curation→launch frozen-list handoff so `cortex overnight launch` executes the operator's approved set instead of re-selecting (closing the latent `[R]emove`-doesn't-stick gap), (2) a Step-6 three-pool curation surface that sets aside poor-fit specs with reasons and re-adds them, and (3) removal of the now-redundant `/refine` advisory. The architecture is a pipeline: select → curate/freeze → launch executes exactly the frozen set (Terraform `plan`→saved-plan→`apply` shape). Records the decision as **ADR-0021** (the spec's proposed `0018` is already taken by #329 — 0017–0020 are all consumed).
**Architectural Pattern**: pipeline

## Outline

### Phase 1: Curation→launch frozen-list handoff (tasks: 1, 2, 3, 4)
**Goal**: `cortex overnight launch` accepts and executes a dependency-closed curated slug set (`--only`); absent the input behavior is unchanged; a non-closed or ineligible set is refused fail-loud.
**Checkpoint**: new tests in `test_cli_launch_prepare.py` green for (a) curated subset → state == subset, (b) no input → unchanged, (c) empty → `nothing_ready`, plus dependency-closure refusal; `just test` exit 0.

### Phase 2: Overnight curation suitability set-aside (tasks: 5, 6)
**Goal**: `/overnight` Step 6 sets aside poor-fit specs (bias-to-exclude, per-item reasons, re-addable via `[I]`, judged once, re-displayed before every approval), and Step 7 passes the frozen active pool to `launch --only`.
**Checkpoint**: `new-session-flow.md` carries the set-aside rubric, `[I]` verb, judge-once/re-apply direction, re-display-before-approval direction, and the Step-7 `--only` handoff; mirror regenerated; `just test` exit 0.

### Phase 3: Remove the /refine advisory + ADR + governance (tasks: 7, 8, 9)
**Goal**: excise `/refine`'s overnight-candidate advisory and its run-mode detection (keeping the §5 skip-note), record ADR-0021, regenerate all mirrors, and confirm governance gates green with the affordance-preservation note captured.
**Checkpoint**: `grep -c 'Overnight-candidate advisory' skills/refine/SKILL.md` = 0; ADR-0021 present; `just test` + dual-source drift hook exit 0.

> **Phase ordering is a correctness constraint** (spec §Phases): the Phase 3 removal of the only suitability signal must not land before BOTH Phase 1's `--only` execution (Tasks 3–4, which make `launch` actually honor the frozen set) AND Phase 2's curation surface (Tasks 5–6) are live — else a window opens with no suitability signal anywhere. Encoded by **Task 7 depending on `[4, 5, 6]`** (full replacement live), not on `[5, 6]` alone: `[5, 6]`'s transitive closure excludes Task 3, so a valid schedule could remove the advisory while `launch` still ignores `--only` and re-selects everything. The enforcement is the dependency ordering (not single-PR atomicity): the implement phase runs all tasks before the Complete phase opens its single PR, so removal and replacement land on the same branch together.

Residual under-flagging risk (a poor-fit spec the prose judge misses runs unattended) is an accepted trade backstopped by the human gate + morning-report recoverability (spec §Edge Cases).
- **Dispatch mode**: this edits the `cortex_command` package; per project convention, prefer sequential trunk dispatch (the editable install means a worktree would verify stale code). Same-file tasks (2/4 on the test file; 5/6/7 on skill files) are sequenced via dependency edges to avoid races.
