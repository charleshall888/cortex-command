# Specification: explore-best-mechanism-for-lifecycle-worktree

## Problem Statement
`cortex init` splices a cortex-managed `EnterWorktree` authorization fence into a consumer repo's human-curated `CLAUDE.md` (ADR-0006), breaching ADR-0003's "the only write cortex makes outside its own tree is `~/.claude/settings.local.json`" invariant for consumer repos. The empirical gate test run during research resolved the load-bearing question **as far as a soft gate allows**: `EnterWorktree` is a soft model-adherence instruction (the harness does not enforce it — a fence-free picker-selection call was accepted, failing only on `path=` validity), and a user's selection of a worktree-labeled picker option authorizes the call. **Residual:** the selection-vs-mention seam is underdetermined in the rule's literal text — a stricter model could decline — so the picker-fired authorization is opportunistic, not guaranteed; the cd-shim fallback is the structural safety net for any decline. The standing fence is load-bearing for exactly one path — the suppressed-picker (`branch-mode: worktree-interactive`) path, which has never fired in this repo. **Option B**: delete the fence apparatus and keep `EnterWorktree` on the picker-fired path (live-selection-authorized), routing the suppressed-picker path explicitly to the already-shipped cd-shim.

**Recommendation note (post-critical-review).** Removing the fence deletes ADR-0006's entire decision body, so this is a **supersession** of ADR-0006 (new ADR), not an amendment — which means Option A (full removal / cd-shim everywhere) and Option B now carry the *same* ADR/ticket-reconciliation cost. B's remaining edge over A is narrow: it keeps the gate-test-validated picker-fired `EnterWorktree` entry (a safety-superset of A — it degrades to A's cd-shim behavior on any decline) and preserves #249's auto-enter capability, at the cost of modestly more skill complexity (the suppressed-picker routing split + retaining the `cortex-worktree-precondition` probe). A is simpler and fully deterministic. This spec recommends **B**, but A is a close, reasonable alternative — surfaced at the §4 approval for the operator's reserved decision.

## Phases
- **Phase 1: Remove the foreign-write fence apparatus** — stop `cortex init` writing to consumer `CLAUDE.md`; delete the fence functions (incl. the revoke-only liveness helpers), CLI verbs, template, and their tests; handle existing fences and the init-hash flip.
- **Phase 2: Rewire worktree entry and reconcile docs/ADRs** — keep `EnterWorktree` on the picker-fired path, drop *only* the verify probe (retain the already-in-worktree probe), route the suppressed-picker path explicitly to the cd-shim; regenerate the dual-source mirror; supersede ADR-0006 with a new ADR, fix the ADR-0004→0005 mis-citation, reconcile ADR-0003 and `project.md`, and reconcile #249/#250.

**Phase atomicity (constraint).** Phases 1 and 2 must land in a single PR/release. Phase 1 deletes the `--verify-worktree-auth` CLI verb while `implement.md` (edited in Phase 2) still invokes it; the canonical-vs-mirror drift gate cannot see this prose-vs-CLI skew (it compares bytes, and Phase 1 touches no `skills/` file). Shipping Phase 1 without Phase 2 would release a mirror that calls a removed subcommand.

## Non-Requirements
- Does NOT keep the fence behind an opt-in flag (Option D — dead-weight serving a never-fired path; leaves the ADR-0003↔0006 contradiction live).
- Does NOT relocate the clause to another consumer surface such as `.claude/rules/` (Option C — still a write into a repo cortex does not own).
- Does NOT force the picker on every implement invocation (the picker-fires-only path ADR-0006 rejected) — `branch-mode: worktree-interactive` may still suppress the picker; that path degrades to the cd-shim.
- Does NOT add a cortex-owned standing authorization surface for the suppressed-picker path now (the B+D blend — deferred to a future opt-in ticket only if a consumer reports needing suppressed-picker auto-enter; YAGNI).
- Does NOT change worktree creation (always `git worktree add`; ADR-0004 bypass is permanent).
- Does NOT alter the kept §1 branch-picker pause or the `Agent(isolation: "worktree")` sub-agent dispatch.
- (If Option A is chosen at §4 instead) full removal additionally deletes the `EnterWorktree` call site, both step-v probes, and marks #249/#250 superseded — out of scope for the B spec.

## Edge Cases
- **Soft-gate decline at runtime on the picker-fired path** (a stricter future model declines despite the selection): the `EnterWorktree skipped` branch fires → cd-shim. No correctness impact — only the orchestrator re-root is lost. This is the irreducible epistemics of a soft gate, accepted because the cd-shim floor is exercised today.
- **Consumer with the fence AND `branch-mode: worktree-interactive`**: this is the one configuration the fence was load-bearing for. Post-change it is a **deliberate, narrow behavior regression** — the suppressed-picker path structurally routes to the cd-shim (worktree still created and used; orchestrator session re-root absent), and the now-meaningless fence is handled per the §4 stranded-fence decision. Not framed as costless.
- **Existing consumer repo with a stranded fence** (if left-stranded chosen): no runtime effect once init stops regenerating it; one-line manual delete documented. (See Open Decisions — this conflicts with the ticket's "no cortex cruft in consumer CLAUDE.md" premise and is an operator call.)
- **Old plugin-mirror skill against the new CLI** calls the deleted `--verify-worktree-auth`: argparse exits 2 → `implement.md` routes any non-zero exit → cd-shim. Backward-compat safe across *released* version skew (distinct from the intra-change Phase atomicity constraint above).
- **init-artifacts-hash flip**: removing the template from `_HASH_INPUT_TEMPLATES` changes the hash → one-time `cortex init --ensure` drift refresh on marker-present consumers (the migration carrier; no version bump needed).

## Technical Constraints
- **Dual-source + Phase atomicity**: `skills/`/`SKILL.md` edits mirror into `plugins/cortex-core/` via `just build-plugin` (commit canonical + mirror together); `cortex_command/init/*`, `cli.py`, the template, and `tests/` ship in the wheel only. Phases 1 and 2 must land in one PR (see Phase atomicity) — the drift gate cannot catch the prose-vs-CLI skew.
- **Backward-compat**: `implement.md` already routes any non-zero `--verify-worktree-auth` exit to the cd-shim, so immediate subcommand deletion is safe for released skew.
- **Kept-pauses parity**: the §1 picker pause and the `cortex-lifecycle-branch-mode` structural marker (±35-line tolerance, `tests/test_lifecycle_kept_pauses_parity.py`) are unchanged — the picker stays.
- **ADR protocol** (`cortex/adr/README.md`): supersession = `status: superseded` + `superseded_by:` + a new ADR (the README's mechanism for "decision no longer applies"). Used here because Option B removes ADR-0006's entire decision body and adopts a different authorization model.
- **Destructive-ops** (`project.md`): any fence-cleanup sequence (if non-stranded cleanup is chosen at §4) skips on uncommitted state and lives in a named script.
- **Authoring policy**: What/Why-not-How; no new MUST without an evidence artifact (soft positive-routing for any new skill prose).
- **Gate model**: `EnterWorktree` is Permission Required: No (soft gate); the cd-shim is the structural safety net for any runtime decline, and the suppressed-picker→cd-shim routing is structural (not soft-gate-dependent) per R8.

## Open Decisions
- **Stranded-fence handling** (operator values call, surfaced at §4): existing consumer fences may be **left-stranded** (no further cortex write, but permanent cortex-authored cruft remains in the consumer's hand-curated `CLAUDE.md` — in tension with the ticket's premise) OR **cleaned once** via a clean-tree-gated, named destructive-ops script that removes only cortex's own fenced block on the next `cortex init --ensure` (one final write that removes cortex content rather than leaving it). Cannot be resolved purely from research — it trades "no further write" against "no lingering cruft," a values judgment for the operator. Recommended default: left-stranded (matches ADR-0006's accepted uninstall behavior), but the operator decides at §4.

## Proposed ADR
**New ADR `<NNNN>` superseding ADR-0006.** Context: the empirical gate test established that `EnterWorktree` is a soft gate and a user's picker selection authorizes it, so the standing consumer-`CLAUDE.md` fence is unnecessary for the picker-fired path and was the sole ADR-0003 breach for consumers. Decision: cortex writes no consumer-`CLAUDE.md` authorization fence; the lifecycle implement phase authorizes `EnterWorktree` on the picker-fired path via the user's live selection of the worktree-labeled option, and routes the suppressed-picker (`branch-mode: worktree-interactive`) path structurally to the cd-shim. Trade-off: the suppressed-picker path loses its orchestrator re-root (degrades to cd-shim) — accepted because that path is a never-fired power-user opt-in and the picker-fired authorization is gate-test-validated (soft, with a deterministic cd-shim fallback). This ADR supersedes ADR-0006 (`status: superseded` + `superseded_by: <NNNN>`), restores ADR-0003's "only write outside our tree" invariant for consumers, and is paired with the ADR-0004:49 citation fix. Meets the three-criteria gate (hard-to-reverse coordinated change; surprising-without-context; real trade-off vs. Options A/C/D). If the operator selects Option A at §4 instead, this ADR's Decision changes to "cd-shim everywhere; `EnterWorktree` call removed" and #249/#250 are both marked superseded.
