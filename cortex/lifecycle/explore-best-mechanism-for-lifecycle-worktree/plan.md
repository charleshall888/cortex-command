# Plan: explore-best-mechanism-for-lifecycle-worktree

## Overview
Implement Option B: delete the consumer-`CLAUDE.md` authorization-fence apparatus (so `cortex init` writes no foreign fence), keep `EnterWorktree` on the picker-fired path (authorized by the user's live worktree-option selection), and route the suppressed-picker path **structurally** to the already-shipped cd-shim. Reconcile the ADR/backlog/`project.md` surface that described the fence as a live decision. Phase 1 (wheel-side deletions) and Phase 2 (skill rewire + doc/ADR reconciliation) **must ship in one PR** — Phase 1 removes the `--verify-worktree-auth` CLI verb that the pre-edit `implement.md` still invokes, and no gate catches that skew: the pre-commit contract checker (`just check-contract`, codes E101/E103) does scan skill/doc prose against the live CLI surface, but its binary regex matches only hyphenated `cortex-*` console scripts, so the space-form dispatcher subcommand `cortex init --verify-worktree-auth` is invisible to it.
**Architectural Pattern**: layered
