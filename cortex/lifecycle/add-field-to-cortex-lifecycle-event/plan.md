# Plan: add-field-to-cortex-lifecycle-event

## Overview
Refactor `cortex-lifecycle-event log` into a uniform field-driven emitter (`{ts, event, feature, <ordered --set/--set-json fields>}`, spaced+`Z` serialization, flock+`O_APPEND` append), extend the events-registry scanner to recognize `--event <name>` **before** any `"event":"X"` literal is removed, then route the deterministic event emissions in 8 lifecycle/refine reference files through the verb — so the references keep only the event name + typed fields and the verb becomes the safe inherited emission contract for epic #336.
**Architectural Pattern**: layered
<!-- The verb is a serialization/emission layer that all deterministic event-producing sites route through; the static gate (scanner) and the format contract are the layer's interface. -->

## Outline

### Phase 1: Verb foundation & gate coverage (tasks: 1, 2, 3, 4)
**Goal**: The verb emits the canonical field-driven row, its lone existing consumer is migrated in the same commit, the events-registry scanner recognizes `--event <name>`, backfill detection becomes explicit-marker-driven (un-nulling real overnight rows the shape heuristic mis-flagged), and the ADR records the contract.
**Checkpoint**: `python3 -m pytest cortex_command/tests/test_lifecycle_event.py cortex_command/pipeline/tests/test_metrics.py tests/test_check_events_registry.py` exits 0; `bin/cortex-check-events-registry --staged --root .` exits 0; `just check-contract` exits 0; no reference file yet references `--set`/`--set-json` except `implement.md` §1a.

### Phase 2: Site migration (tasks: 5, 6, 7, 8, 9, 10, 11)
**Goal**: The deterministic events in the 8 in-scope reference files emit through the verb (string fields via `--set`, numeric/array via `--set-json`); call-site-pinned round-trip tests exercise the on-disk invocations; the registry producer columns point at the gate-scanned skill sites.
**Checkpoint**: every migrated event's raw `"event":"X"` literal is gone and its `--event X` replacement is present (per-event grep); `python3 -m pytest tests/test_lifecycle_event_roundtrip.py` exits 0; `bin/cortex-check-events-registry --staged --root .` exits 0; `just build-plugin && git diff --quiet plugins/` (drift hook clean).
**Hard dependency**: every Phase 2 task depends on Tasks 1 + 3 — no literal may be removed before the verb declares the flags (Task 1) and the scanner recognizes `--event` (Task 3).

### Phase 3: Hardening (task: 12)
**Goal**: The one spacing-sensitive non-`json.loads` consumer (`skills/refine/SKILL.md`'s grep) is made separator-tolerant.
**Checkpoint**: the should-have grep hardening is in place and its mirror regenerates.
