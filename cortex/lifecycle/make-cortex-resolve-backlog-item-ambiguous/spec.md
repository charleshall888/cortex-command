# Specification: make-cortex-resolve-backlog-item-ambiguous

## Problem Statement

`tests/test_cortex_resolve_backlog_item_parity.py` byte-compares the resolver's golden output against fixtures that snapshot **live backlog content**, so the parity cases drift on unrelated backlog changes. The headline case is `test_stderr_parity[title_phrase_ambiguous]`: it pins the exact ambiguous-match count (`ambiguous: N matches` + the first 5 `filename\ttitle` lines + `... (N-5 more)`), and because the query is `lifecycle`, every "lifecycle"-titled item added or removed drifts `N` and breaks `just test` (re-captured at `c582a84e`, `904bb80f`; a third drift is live now — #276 committed plus #277/#278 untracked push the live count to 34 vs. the pinned 33). Critical review surfaced that `test_stdout_parity[numeric_unambiguous]` shares the identical drift class: it byte-pins item 252's *live frontmatter title* and breaks if 252 is retitled, renumbered, or archived. `no_match` is genuinely stable (its stderr echoes only the input argument). The resolver's *matching* behavior (which items resolve, exit codes, slug-priority) is already covered by `tests/test_resolve_backlog_item.py`, but that suite only substring-checks `"ambiguous" in stderr` — it pins **none** of the output *format*, so these parity cases are the *sole* automated guard of the stderr/stdout format contract. The fix replaces the two drift-prone byte comparisons with structural assertions that pin the **format contract** (the surviving value) while deliberately not pinning incidental live data (the count, titles, ordering), keeping the resolver genuinely under test — not weakened to a no-op.

## Phases
- **Phase 1: Drift-immune structural assertions** — replace the byte-comparison of the ambiguous stderr and the numeric stdout with structural assertions that read the resolver's live output and pin format (not incidental live content); neutralize the volatile `.stderr`/`.stdout` snapshots as drift sources.
- **Phase 2: Fixture-doc hygiene** — update the test module docstring (quintuple invariant + documented structural exceptions) and the fixture README (case table, backlog snapshot, and the "How to recapture" recipe so it no longer regenerates the de-pinned snapshots).

## Requirements

1. **Ambiguous stderr structural assertion**: `test_stderr_parity` special-cases `title_phrase_ambiguous` and asserts the stderr *format* against the resolver's live output instead of byte-comparing a pinned count. The assertion must check: exit code == 2; the first stderr line matches `^ambiguous: (\d+) matches$` with captured `N > 1`; the next `min(N, 5)` lines each split on their **first** TAB into a `.md` filename and a non-empty title; and **when `N > 5`** a final line matches `^\.\.\. \((\d+) more\)$` with the captured number equal to `N - 5` (and no such line when `N <= 5`). Acceptance: `python3 -m pytest "tests/test_cortex_resolve_backlog_item_parity.py::test_stderr_parity[title_phrase_ambiguous]" -q` exits 0 (pass if exit code = 0). **Phase**: Drift-immune structural assertions

2. **Ambiguous formatter stays genuinely under test (no no-op)**: the assertion must fail on a real `_format_candidates` regression. Acceptance — Interactive/session-dependent (a no-op-freedom property is behavioral, not a grep-able property of the test source): verified by a mutation check — temporarily alter `_format_candidates` (e.g. change the header token `ambiguous:` → `ambig:`, change `matches` → `match`, replace the `\t` separator with a space, or reword the `... (N-5 more)` line) and confirm `python3 -m pytest "tests/test_cortex_resolve_backlog_item_parity.py::test_stderr_parity[title_phrase_ambiguous]" -q` exits non-zero in each case; then revert. An assertion checking only exit 2 + non-empty stderr does NOT satisfy this. **Phase**: Drift-immune structural assertions

3. **Ambiguous drift immunity across backlog growth**: the ambiguous case passes for any `N > 1` regardless of how many "lifecycle"-titled items exist. Acceptance: `just test` exits 0 on the current working tree (live count 34, #277/#278 untracked) AND after `git add` of #277/#278 (pass if exit code = 0 in both states). **Phase**: Drift-immune structural assertions

4. **Numeric stdout structural assertion**: `test_stdout_parity` special-cases `numeric_unambiguous` and asserts the resolved JSON *shape* instead of byte-comparing the live title. The assertion must: parse stdout as JSON; assert the four keys `filename`, `backlog_filename_slug`, `title`, `lifecycle_slug` are present; assert `filename` and `backlog_filename_slug` match `^252-`; assert `title` is a non-empty string; assert `lifecycle_slug` is present and non-empty — without byte-pinning the live title text. Acceptance: `python3 -m pytest "tests/test_cortex_resolve_backlog_item_parity.py::test_stdout_parity[numeric_unambiguous]" -q` exits 0 (pass if exit code = 0); drift check — editing item 252's `title:` frontmatter in `cortex/backlog/` does not change the test's pass/fail (Interactive: edit title in a scratch state, run the test, revert). **Phase**: Drift-immune structural assertions

5. **No committed fixture pins volatile live content**: neither `title_phrase_ambiguous.stderr` (count) nor `numeric_unambiguous.stdout` (live title) remains a byte-compared drift source. Recommended realization: delete both files; if either is kept, the structural branch must run before any `_read_expected_stderr`/`_read_expected_stdout` byte-read for that case. Acceptance: `test ! -f tests/fixtures/cortex-resolve-backlog-item/title_phrase_ambiguous.stderr` returns 0 AND `test ! -f tests/fixtures/cortex-resolve-backlog-item/numeric_unambiguous.stdout` returns 0 (if deleted), OR — if kept — neither case routes through the generic byte branch (verifiable by reading the special-case branches). **Phase**: Drift-immune structural assertions

6. **Case discovery preserved**: `_discover_cases()` (globs `*.argv`) still discovers all three cases. Acceptance: `python3 -m pytest tests/test_cortex_resolve_backlog_item_parity.py --collect-only -q | grep -c -E 'title_phrase_ambiguous|numeric_unambiguous|no_match'` ≥ 3 (pass if ≥ 3). **Phase**: Drift-immune structural assertions

7. **Documented structural exceptions in the test docstring**: the test module docstring states that the `title_phrase_ambiguous` (stderr) and `numeric_unambiguous` (stdout) cases use structural assertions — not byte-exact comparison — because their output embeds live backlog data, and corrects the now-false "error-formatter-shape is NOT opted in / reproduce byte-for-byte" framing as it applies to those cases. The "quintuple" (five-file-per-case) invariant note is updated to record that the structurally-asserted cases omit their de-pinned snapshot. Acceptance: `grep -c -i 'structural' tests/test_cortex_resolve_backlog_item_parity.py` ≥ 1 in the docstring region (pass if ≥ 1) and the docstring no longer claims those two cases are byte-compared. **Phase**: Fixture-doc hygiene

8. **Fixture README de-drifted, including the recapture recipe**: `tests/fixtures/cortex-resolve-backlog-item/README.md` no longer pins a stale ambiguous count, no longer claims the affected cases reproduce byte-for-byte, its backlog-snapshot line is corrected or de-pinned, AND the "How to recapture" section no longer regenerates the de-pinned snapshots (`title_phrase_ambiguous.stderr` at line ~113, and `numeric_unambiguous.stdout`) — otherwise a future recapture silently resurrects the drift source. Acceptance: `grep -c '32 matches' tests/fixtures/cortex-resolve-backlog-item/README.md` == 0 (pass if 0) AND `grep -c 'title_phrase_ambiguous.stderr' tests/fixtures/cortex-resolve-backlog-item/README.md` == 0 (pass if 0, i.e. the recapture command for that snapshot is removed). **Phase**: Fixture-doc hygiene

## Non-Requirements

- Does NOT change the `no_match` case — it is genuinely stable (its stderr `no match for '<input>'` echoes only the input argument, independent of backlog content) and remains byte-exact via `assert_byte_identical`.
- Does NOT modify `tests/test_parity_contract.py` or extend the closed `TOLERANCE_CATEGORIES` set. The per-case-tolerance-routing idiom (`test_cortex_lifecycle_state_parity.py`'s `_CASE_TOLERANCES`) reuses **existing** named categories; the shape-matching these two cases need (a regex over a count line; JSON key-presence) is not an existing category, so routing through the contract would require adding single-use categories to a closed set shared by ~21 sibling tests. The local structural assertions leave the closed set unchanged.
- Does NOT reuse the existing `error-formatter-shape` tolerance — it compares only `{empty/non-empty} × {zero/non-zero exit}` and never inspects bytes, so it would leave the format unguarded (the "weaken to a no-op" the ticket forbids).
- Does NOT migrate the parity suite to a frozen synthetic backlog corpus (option b′). b′ would restore byte-exactness by byte-pinning *incidental* live data (the count, titles, candidate ordering) against a synthetic corpus — more machinery than the format contract needs. The structural assertions pin the format contract directly while deliberately leaving incidental data unpinned.
- Does NOT change the resolver implementation (`cortex_command/backlog/resolve_item.py`) — the formats under test are unchanged; only the tests assert them differently.
- Does NOT retire or restructure the broader parity-fixture family — the migration-cruft cleanup (now that the bash sources are gone) is a separate concern noted for a future backlog item.

## Edge Cases

- **Ambiguous count 2–5**: the `... (N-5 more)` truncation line is absent; the assertion requires it present iff `N > 5`. Note: the truncation-line regression guard (R2) only exercises when `N > 5` — true now (N=34) and for any realistic "lifecycle" count; below 6 the header/candidate-line guards and the mutation check still apply.
- **Ambiguous count ≤ 1**: the query would no longer be ambiguous (exit 0 or 3, not 2) and the test fails. This is correct, not a defect: the case's premise — that `lifecycle` is an ambiguous query — would have genuinely broken. The real justification for tolerating this is that `lifecycle` is a core domain term that matches dozens of items and will not realistically fall to ≤1; it is not "self-documenting" cover for an unhandled state.
- **Numeric: item 252 archived or renumbered**: the query `252` would no longer resolve unambiguously (exit 3), and the numeric assertion (expecting exit 0 + JSON) would fail. This is the same premise-broke category as the ambiguous ≤1 case, and is much lower-frequency (it requires mutating one specific completed item); accepted as a bounded residual, not silently claimed away.
- **Title containing a literal TAB**: candidate lines are `filename\ttitle` and titles are free-text frontmatter; the assertion splits each candidate line on its **first** TAB (the filename has no TAB and ends in `.md`), tolerating additional TABs inside the title.
- **Deleted snapshot + the generic byte branch**: if `title_phrase_ambiguous.stderr` / `numeric_unambiguous.stdout` are deleted, the structural branch must execute before any `_read_expected_*` call for that case, or the parametrized test would raise `FileNotFoundError`.

## Changes to Existing Behavior

- MODIFIED: `test_stderr_parity[title_phrase_ambiguous]` byte-compared a count-pinned golden fixture → asserts the stderr format structurally against live output.
- MODIFIED: `test_stdout_parity[numeric_unambiguous]` byte-compared (under structural-equivalence tolerances) a stdout golden that pins item 252's live title → asserts the resolved JSON shape structurally.
- REMOVED: the count-pinned `title_phrase_ambiguous.stderr` and title-pinned `numeric_unambiguous.stdout` byte snapshots as drift sources (recommended: both deleted).
- MODIFIED: the test module docstring and fixture README claims that those cases reproduce byte-for-byte, and the README "How to recapture" recipe that regenerated the de-pinned snapshots.

## Technical Constraints

- Read the resolver's **live** output via the existing `_invoke_case` invocation, not from a fixture file.
- The structural branch for each affected case must precede any `_read_expected_*` byte-read for that case.
- `_format_candidates` reference format (cortex_command/backlog/resolve_item.py:282–296): header `ambiguous: {N} matches`; candidate `{path.name}\t{title}` for the first 5 matches; truncation `... ({N-5} more)` only when `N > 5`; lines joined by `\n` with a trailing newline at print time; exit 2.
- Numeric stdout JSON keys (from the resolver's exit-0 object): `filename`, `backlog_filename_slug`, `title`, `lifecycle_slug`. Slug-priority behavior (frontmatter `lifecycle_slug` winning over title-derivation) is covered by `tests/test_resolve_backlog_item.py`, so the numeric structural assertion focuses on JSON output shape rather than re-verifying slug-priority.
- `_discover_cases()` parametrizes on `*.argv` (test file lines 64–69); keep all three `*.argv` files. The structurally-asserted cases' `.exitcode` (`2` / `0`) and `.stdin` (empty) remain read and stay stable; `no_match`'s full quintuple stays byte-exact.
- Both structural assertions are intentional, documented exceptions to the parity contract's "byte-identical by default; every tolerance named and per-stream" discipline, justified because their output embeds live backlog data (Abseil Tip #135: test the contract, not the implementation). With `no_match` remaining byte-exact via the contract, this evolves the resolve-item parity test from byte-exact migration snapshots toward format-contract regression guards — a deliberate, documented shift, recorded at the test-file level so future authors see the rationale at the call site.
