# Plan: consolidate-tier-supersession-into-a-shared

## Overview
Extract the event-fold body of `reduce_lifecycle_state` into a pure `reduce_lifecycle_events(records) -> (state, rejected_positions)` core in `common.py`; have the Path reader delegate to it while keeping file I/O, line-number tracking, the `skipped_lines` merge, and the `LifecycleStateReduction`/`corrupted` wrapper as shell concerns; point `extract_feature_metrics`'s final-tier computation at the same core while keeping `initial_tier` a local first-seed latch that reuses the canonical `TIER_VOCABULARY`; and harden `parse_events`'s intake to `errors="replace"`. This is research's Approach B (functional core, imperative shell with two shells over one core). **Scope of the "by construction" claim** (matches spec §Scope precision): the supersession **rule** — the fold + vocabulary gate — is single-sourced, so it agrees by construction for every reader. The **intake** (file decode + JSONL parse) deliberately stays caller-local: `parse_events` (drops records lacking `event`/`ts`) and the Path reader (folds every parsed dict) remain two separate shells, so this is *not* a claim that the two readers agree on every byte-corrupt log. R8 narrows only the most severe intake divergence (a crash), not the full parse asymmetry.
**Architectural Pattern**: layered

## Outline

### Phase 1: Shared pure reducer (tasks: 1, 2)
**Goal**: Extract the pure `reduce_lifecycle_events` core and make `reduce_lifecycle_state` delegate to it with byte-for-byte-identical behavior for every existing consumer.
**Checkpoint**: `python3 -m pytest tests/test_reduce_lifecycle_state.py -q` exits 0, including the new pure-core, interleaving, and NamedTuple-field-set tests.

### Phase 2: Metrics delegation + intake hardening + parity coverage (tasks: 3, 4, 5, 6)
**Goal**: Make `extract_feature_metrics`'s final tier come only from the shared core, keep `initial_tier` local but vocab-gated by the shared predicate, harden the metrics intake against non-UTF-8, and add the metrics fold to systematic parity coverage against an independent oracle.
**Checkpoint**: `just test` exits 0; the metrics fold is exercised by an independent-oracle parity matrix that includes a non-UTF-8 intake case.

## Tasks

### Task 1: Extract pure `reduce_lifecycle_events`; delegate the Path reader
- **Files**: `cortex_command/common.py`
- **What**: Add a pure `reduce_lifecycle_events(records)` that performs the supersession fold + per-value vocabulary gating over already-parsed records, and rewrite `reduce_lifecycle_state(events_path)` to parse the file + track line numbers itself and delegate the fold to the new core — preserving its exact signature and tolerant-read contract.
- **Depends on**: none
- **Complexity**: complex
- **Context**:
  - Current single-pass implementation lives at `common.py:660-738`; `LifecycleStateReduction` NamedTuple at `:623-657`; `TIER_VOCABULARY`/`CRITICALITY_VOCABULARY` at `:619-620`.
  - **New core signature** (research §Contract, Approach B): `def reduce_lifecycle_events(records: Iterable[dict]) -> tuple[dict[str, str], list[int]]`. Returns `(state, rejected_positions)` where `state` has at most `{"criticality","tier"}` in criticality-then-tier insertion order, and `rejected_positions` are **0-based indices** into `records` of records carrying an out-of-vocab value on a recognized event. The core does **no** file I/O, no `json.loads`, no line-number tracking, and does **not** construct a `LifecycleStateReduction`. Move the event-dispatch logic currently at `:704-736` (the `lifecycle_start`/`criticality_override`/`complexity_override` branches and the per-value vocab checks) into this core. Non-dict records and unrecognized events are no-ops (not rejections), preserving the `:699-702` silent-skip rule. `Iterable` comes from `typing`/`collections.abc` — confirm the existing import list at the top of `common.py` before adding.
  - **Each rejected record's position appears at most once** in `rejected_positions`. A single `lifecycle_start` can carry *two* out-of-vocab values (both `tier` and `criticality`); the current code collapses both axes into one `skipped.append` via the per-line `line_rejected` boolean (`:705`/`:736`). The core must mirror that: use a **per-record rejected flag** (one append per record), NOT a per-value append, so `rejected_positions` is duplicate-free by construction and is a faithful image of today's per-line skip semantics — a per-value append would emit the position twice and silently diverge the core's published contract. (Locked by the Task 2 double-axis case.)
  - **Path shell** (`reduce_lifecycle_state`): keep the `errors="replace"` read (`:683`) and the missing-file early return (`:684-685`); keep the `splitlines()` loop, blank-line skip, and `json.loads`-in-try/except tracking **parse-failure 1-based linenos** (`:690-698`). Build a parallel list of `(lineno, record)` for every successfully-parsed JSON value (dicts and non-dicts alike) **in line order**, pass the records to `reduce_lifecycle_events` **in that same order** (non-dicts occupy their positions so the core's 0-based positions index the parallel list 1:1), then map each returned position back to its lineno. Produce `skipped_lines` as `tuple(sorted(set(parse_failure_linenos) | set(vocab_rejection_linenos)))`. Byte-for-byte fidelity rests on **two** invariants, not just set-disjointness: (a) **load-bearing** — within the vocab-rejection set each rejected record contributes its lineno *at most once* (guaranteed by the per-record flag above; today's code appends each line once); (b) the parse-failure and vocab-rejection sets are disjoint (a parse failure `continue`s before vocab checking). With both invariants the `set()` is belt-and-suspenders, not the thing that rescues fidelity. Wrap as `LifecycleStateReduction(state=..., skipped_lines=...)`.
  - **Interface preserved → callers untouched.** `reduce_lifecycle_state(Path) -> LifecycleStateReduction` keeps its exact signature/return type, so its callers require **no** changes — do NOT edit them: `_read_criticality_inner` (`:524`), `_read_tier_inner` (`:580`), `lifecycle_state_corrupted` (`:761`), `refine._reduce_current_state` (`refine.py:126`), `state_cli` (`state_cli.py:86,152`), `outcome_router._review_required` (`outcome_router.py:1001`). All use named-attribute access (research §Call graph). Preserve the `__wrapped__`/`lru_cache` hooks on the cached readers — they key on file stat, not on this function's internals.
  - **Do NOT grow `LifecycleStateReduction`** — it keeps exactly `(state, skipped_lines)`; no `initial_tier` field (R3 constraint; A-Option-2 rejected in research §Adversarial).
- **Verification**: `grep -c "def reduce_lifecycle_events" cortex_command/common.py` = 1 (pass if exactly 1); then `python3 -m pytest tests/test_reduce_lifecycle_state.py -q` — pass if exit 0 (every pre-existing test — torn-line lineno `:57`, mixed-line per-value `:84`, corruption suite `:133-199`, missing-file `:97`, reader delegation `:103` — still passes unchanged against the delegating reader).
- **Status**: [x] done

## Risks
- **Pure-core signature is locked by this plan, not the spec.** Spec §Technical Constraints left the exact signature to Plan/Implement. This plan commits to research's Approach B: `reduce_lifecycle_events(records: Iterable[dict]) -> (state, rejected_positions)` with 0-based positions, core returns bare `state` (not a `LifecycleStateReduction`). This is the lowest-touch shape on protected `common.py` and dissolves the synthetic-lineno footgun (research §Adversarial). Revisit only if a second pre-parsed consumer of the *full* reduction is named — none exists.
- **Permissive→strict on the metrics axis.** Vocab gating now drops out-of-vocab tiers in `extract_feature_metrics` (previously accepted any non-`None` string). The empirical audit found 0 out-of-vocab values across 340 real vocab-bearing rows, so this is a no-op on all real data; it is locked by R7. This is an intentional behavior change, documented in the spec's "Changes to Existing Behavior."
- **`initial_tier` is a value substitution, not just a drop, for one (non-real-data) sequence.** For an out-of-vocab first seed followed by an in-vocab seed (e.g. `["trivial","complex"]`), the new "first in-vocab" rule returns `"complex"` where the current first-non-`None` latch returns `"trivial"` — a substitution, not a drop-to-`None`. This is spec-sanctioned (line 53 "keeps the first in-vocab seed"), vocab-consistent, a no-op on all real data (audit: 0 out-of-vocab), and locked by the Task 4 discriminating fixture. Flagged because it changes a forensic field's value beyond the spec's "drop" wording.
- **`:1749` dispositioned by annotation, not strengthening** (Task 4) — chosen because the R9 independent-oracle matrix (Task 5) is the stronger, non-tautological replacement guard. If Task 5 were dropped, `:1749` would need strengthening in place instead.
- **Commit hygiene & dispatch**: runs on `main` with possible concurrent sessions → **sequential (not worktree) dispatch**, and commit with explicit `git commit -- <pathspec>`. Sequential dispatch matters because `just test` and the in-process test imports (Task 4/5 oracles call `extract_feature_metrics`/`reduce_lifecycle_events` directly) resolve against the editable install's `.pth` → the working tree, so they read current code only if no worktree intervenes; no worktree also means none of the editable-`.pth`-rewrite hazard. (The parity test's `BIN_STATE` *subprocess* leg pins separately via `_pinned_env()` — a distinct mechanism that covers only that leg, not the in-process oracle.) `cortex_command/` is not dual-source mirrored, so no canonical+mirror co-commit. No new event minted → no `bin/.events-registry.md` row.

## Acceptance
The supersession **rule** (lifecycle_start seeds → complexity_override `.to` supersedes → per-value `TIER_VOCABULARY` gating) is single-sourced in `reduce_lifecycle_events` and agrees by construction, with the Path reader and `extract_feature_metrics` both projecting their final tier from it — verified by a behavioral test pinning that delegation yields the correct superseded tier (hand-computed expected value) plus a grep that the inline fold is gone. The **intake** (`parse_events` vs the Path reader) deliberately stays two separate shells; R8 narrows only the crash divergence, and the parity matrix asserts reader agreement on clean tier-bearing lines (not on the byte-corrupt record itself). Every existing `reduce_lifecycle_state` consumer behaves byte-for-byte as before (`tests/test_reduce_lifecycle_state.py` green unchanged, including the new double-axis-rejection case), the metrics intake survives non-UTF-8 input without raising, the metrics fold is covered by an independent-oracle parity matrix, and `just test` exits 0.
