# Plan: harden-the-refine-research-considerations-hand (#337)

## Overview
Migrate the refine→research alignment-considerations hand-off from a model-parsed `research-considerations="<bullets>"` value-arg to a file channel: refine writes `cortex/lifecycle/{slug}/research-considerations.md` (overwriting, coupled to the arg, sequenced before the dispatch) and emits only the benign `research-considerations-file=<path>`; research's body reads the file and injects its content into the existing `{research_considerations_bullets}` placeholders. The escaping caveats are deleted and replaced by test-guarded producer disciplines. Phase 1 ships the two-endpoint migration with its discriminating prose-contract tests and the inject-content-not-path guard; Phase 2 locks the contract with schema/gitignore-template registration and ADR-0022.
**Architectural Pattern**: pipeline — a producer→consumer hand-off mediated by a transient file artifact replaces the inline value channel; the two endpoints couple through the file + path-arg pipeline boundary rather than an escaped inline string.

## Outline

### Phase 1: Migrate the hand-off channel (tasks: 1, 2, 3)
**Goal**: Move both endpoints onto the file+path-arg channel, delete the escaping caveats, gitignore the transient file, and guard the new behavior — including the genuinely-new file-reader contract (absent/empty/whitespace ⇒ no injection) — with discriminating prose-contract tests (incl. the inject-content-not-path guard).
**Checkpoint**: `just test` green with both new test files collected; refine emits `research-considerations-file=` coupled to a pre-dispatch overwrite write; research reads-and-injects file content; mirrors byte-identical; the transient file is gitignored.

### Phase 2: Structural test hardening + decision record (tasks: 4, 5, 6)
**Goal**: Register the hand-off field in the schema fixture, add the gitignore-template test case, and record ADR-0022 with both SKILL.md edits back-pointed to it.
**Checkpoint**: `tests/test_skill_handoff.py`, the gitignore-template test, and `tests/test_lifecycle_references_resolve.py` green; `cortex/adr/0022-*.md` exists, is cited as `ADR-0022` from both skills, and its previously-`unresolved` citations now resolve in the (report-only) audit.

## Risks
- **Dispatch mode**: this is a trunk feature with a concurrent live lifecycle session (`overnight-runner-honors-a-lifecycle-wait` at Specify) and mirror-coupling on every skill edit. Recommend **sequential trunk dispatch with explicit-pathspec commits** (`git commit -- <files>`), not worktree/parallel — Tasks 2, 3, and 6 each run `just build-plugin` (whole-tree rsync) and would race the mirror if parallelized, and the shared index risks leaking the other session's staged files.
- **Two mirror regens on the same files**: Tasks 2/3 edit the SKILL.md for the migration; Task 6 re-edits them to add the ADR back-point. This is two `just build-plugin` rounds on the same files. It is the cost of honoring the spec's Phase 1 (migration) / Phase 2 (decision record) split — the ADR is a deliberately lower-urgency Should-have, and keeping the back-point co-located with the ADR write avoids citing a file that does not yet exist (a human-navigation cleanliness choice — NOT an audit-enforcement one; the citation audit is report-only and never blocks). Accepted, not a stop-gap.
- **Behavioral half-migration window (benign)**: the `[2]`/`[2,3]` serialization is justified above by the mirror race, but the same Task 2→Task 3 commit boundary also half-migrates the *channel* (refine emits the new arg before research parses it). This degrades safely — research drops the unknown key, injecting nothing, exactly today's empty-value mode — and no committed test pins the channel until Task 4, which lands after both endpoints. Named here so it is argued design, not undocumented luck.
- **R3 placement-coherence (not anchor fragility)**: the R3 write-before-dispatch *anchors* are robust — `/cortex-core:research topic=` is unique and the ADR filename cannot spoof the write anchor (reviewer-confirmed). The residual risk is moved-block *coherence*: putting the block inside `### Research Execution` (not before its heading) and explicitly relocating the index-registration sentence so the propagation heading does not end up governing an unrelated line. The Task 2 placement guidance and the write-instruction-line R3 test address this.
- **Existing-repo gitignore propagation gap** (documented limitation, not fixed here): the template rule reaches new repos only; existing repos must re-sync `cortex/.gitignore` to pick it up. Until then the transient file is untracked there and could trip dirty-tree gates — this is the spec's accepted Technical Constraint, carried as-is.
