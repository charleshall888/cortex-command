# Specification: single-source-research-skill-fan-out (#350)

## Problem Statement

`skills/research/SKILL.md` restates grid, floor/corner, angle-roster, dispatch-rationale, and considerations-hand-off content that `skills/research/references/fanout.md` already owns canonically. The duplication costs weighted tokens — some of it (the considerations content) rides into every fanned-out searcher prompt, multiplying with agent count. This trim single-sources that content by **pointing SKILL.md at fanout.md and deleting the inline copies from SKILL.md** — fanout.md itself is not edited — and folds the terminal Step 5 routing into Step 1, executing the seven verified epic-#347 audit verdicts while preserving every phrasing-sensitive test pin. It benefits the maintainers (leaner shared skill, one canonical source per fact) and every fan-out run (cheaper searcher prompts).

## Phases
- **Phase 1: Considerations hand-off single-sourcing (s3 + s6)** — merge the two duplicate considerations-contract statements into one canonical occurrence; keep the L45 wording verbatim.
- **Phase 2: fanout.md-resident dedup (s4, s7, s13)** — cut the inline grid/roster/dispatch-rationale that fanout.md already owns; keep the pointers, the upper-bound rider, the template h4 tags, and every ADR-0023 mechanism line.
- **Phase 3: Output-structure strip + routing fold (s17, s18)** — strip the duplicate bracketed annotations and fold Step 5 into Step 1's mode-detection block.

**Per-phase gate (every phase checkpoint):** regenerate the cortex-core mirror (`just build-plugin`), stage canonical + mirror together, run `just test` green, and re-check the cross-cutting invariants R10 (fanout.md byte-identical to baseline) and R11 (mirror parity + fanout-pointer URL form + frontmatter untouched). R10/R11 are not one-phase gates — a stray fanout.md edit or dropped pointer prefix in *any* phase must be caught at that phase's checkpoint, not a later one.

## Edge Cases

- **Merge drops L45's exact tokens**: Step 3's L63 phrasing ("injects its content" / "inject nothing") does NOT match the `substitut\w+ its literal content` / `no…injection` regexes — if the implementer keeps L63's wording as the survivor or rewrites into fresh prose, two red-green tests flip to failing. Expected: keep L45 verbatim as the canonical statement; trim L63 to applicability + a "see Step 1" pointer.
- **Bare `research-considerations` token**: any surviving mention written with a space ("research-considerations file") or as "research-considerations bullets" trips `test_no_stale_bare_value_key`. Expected: every mention stays exactly `research-considerations-file`.
- **s18 slice swallows the considerations paragraph**: `### ` subheadings do NOT terminate the standalone slice (the regex needs a space in the 4th char). Everything from the capital-S anchor to `## Step 2` is asserted — the anchor block must sit after the L45 paragraph, and the whole span must stay free of the pinned tokens.
- **URL-side `${CLAUDE_SKILL_DIR}/` prefix drop**: dropping the prefix from a pointer's `](…)` URL while leaving the display-text token breaks off-repo resolution (ADR-0009) but a line-counting `grep -c` still counts the line — R11(d)'s fixed-string match on the `](…)` target form is what catches it.
- **Naive "add to fanout.md" reading**: adding the cut prose into fanout.md would re-duplicate and produce a spurious mirror diff. Expected: fanout.md stays byte-identical (R10).
- **Mirror drift at commit**: the pre-commit drift hook fails the commit if the mirror is stale and does not auto-stage. Expected: `just build-plugin` then `git add` both trees before committing.

## Changes to Existing Behavior

- **MODIFIED**: `skills/research/SKILL.md` Steps 1–3 prose is single-sourced against fanout.md (fewer inline restatements; pointers retained). No change to the emitted research.md schema or the fan-out dispatch behavior.
- **REMOVED**: The `## Step 5: Route Output` heading — its routing content folds into Step 1's mode-detection block.
- **MODIFIED (read-path)**: Step 2 no longer carries inline floor/corner grid values, so computing `agent_count` for any cell now requires consulting fanout.md's grid (previously a simple+low run could read the inline `3` and skip fanout.md). This is a strong structural incentive — the orchestrator cannot size the fan-out without the grid — but it is **not** control-flow-enforced: a model could still recall a value from prior context. The safety this buys for s7/s13 is therefore "strongly incentivized residency," not a hard guarantee (see Technical Constraints).

## Technical Constraints

- `test_research_handoff.py` regexes are file-wide (one surviving occurrence suffices) except `test_standalone_reads_nothing` (position-sensitive to the capital-S anchor) and `test_three_placeholders_retained` (exactly 3 placeholders in the core templates). Because several regexes are file-wide and non-discriminating, the spec's manual greps (R3, R6, R7b, R8c, R11d) are the real guards where the suite is blind.
- **Residency backstop for s7**: s7's cut is safe only because fanout.md's angle-selection rules are in context when the orchestrator dispatches. s4 removing the inline grid forces the Step-2 fanout.md consult, which pulls the whole file (including the no-keyword-router rule at fanout.md's angle-selection section) into context — this residency is the backstop for s7's cut. **s13's cut, by contrast, is pure rationale re-encoded in fanout.md with no runtime dependency** — s13's mechanism lines stay inline in SKILL.md, so s13 does not lean on fanout.md residency. (This corrects any framing that couples s7 and s13 as sharing one residency premise.)
- ADR-0023 requires each consuming orchestrator body to carry its own runnable resolve+bind — s13 may not fold the mechanism into a pure fanout.md pointer.
- ADR-0009 / `cortex-check-skill-path`: the fanout.md pointer links are body prose outside subagent-prompt fences, so the lint does NOT catch a dropped `${CLAUDE_SKILL_DIR}/` prefix — R11(d)'s fixed-string URL-target grep is the manual guard.
- Mirror discipline: canonical (`skills/research/`) and mirror (`plugins/cortex-core/skills/research/`) must be byte-identical and committed together; `just build-plugin` regenerates.
- `just test` is the project test command (`cortex/lifecycle.config.md`).
