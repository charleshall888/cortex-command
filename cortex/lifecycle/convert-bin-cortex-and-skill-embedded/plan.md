# Plan: convert-bin-cortex-and-skill-embedded

## Overview

Pure `[project.scripts]` console-script migration for the four `python3 -c` skill snippets, plus retirement of the two redundant bin/* wrappers whose console-script entries already exist. Five-phase decomposition: telemetry-bridge first (lets us delete wrappers without breaking the byte-equivalence test), then author four new helper modules + register entries, then rewrite skill prose with consumer-side parser migration, then extend the telemetry-gate and parity-test regex, then docs + full-suite verification.

**Architectural Pattern**: layered

## Outline

### Phase 1: Pre-flight + telemetry bridge + bin/* retirement (tasks: 0, 1, 2, 3)
**Goal**: lock R5/R13 strategies into a tracking file so parallel dispatch is race-free; enable wrapper deletion without breaking `test_telemetry_byte_equivalence`; delete the two redundant bash wrappers; migrate three subprocess-by-path tests.
**Checkpoint**: `test -f cortex/lifecycle/convert-bin-cortex-and-skill-embedded/.implementer-choices` exits 0 AND `pytest cortex_command/backlog/tests/test_telemetry_byte_equivalence.py -q` exits 0 AND `ls bin/cortex-backlog-ready bin/cortex-morning-review-complete-session 2>/dev/null` returns empty AND `cortex-check-parity --audit` exits 0.

### Phase 2: Author four console-script helpers (tasks: 4, 5, 6, 7)
**Goal**: create the four new entry-point modules with single-purpose `main()` functions; register as `[project.scripts]`; reinstall to activate.
**Checkpoint**: `cortex-lifecycle-branch-mode --help`, `cortex-lifecycle-picker-decision --help`, `cortex-critical-review-resolve-feature --help`, `cortex-critical-review-write-residue --help` each exit 0 after `uv tool install -e . --force`.

**Goal**: replace all four `python3 -c "import cortex_command..."` snippets with console-script invocations; migrate the bash JSON consumer in `implement.md`; update `implement.md:22` self-documenting prose.

**Goal**: restructure `--check-entry-points` from "name list under one base dir" to "(package, module) tuples"; extend kept-pauses parity-test marker regex to include the new CLI token.
**Checkpoint**: `bin/cortex-invocation-report --check-entry-points` exits 0 (all 10 modules pass) AND `pytest tests/test_lifecycle_kept_pauses_parity.py -q` exits 0.

### Phase 5: Documentation + full-suite verification (tasks: 12, 13)
