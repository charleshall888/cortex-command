# Specification: offload-lifecycle-step-2-backlog-write

> Epic #336 reference: last child of "Offload deterministic lifecycle mechanics to CLI verbs." Foundations (#330 event verb, #331 stage/complete verbs, #329 parse-args, #322 + ADR-0019 backend convention) have landed; #326 reuses them. Full investigation in `research.md`. **Scope narrowed after critical-review** — see the note under Non-Requirements on the close path.

## Problem Statement

Lifecycle Step 2 still hand-narrates deterministic procedure the agent re-executes every run: the lifecycle-start `cortex-update-item` write-backs (in_progress status + lifecycle-slug) each wrapped in a three-arm backend-routing branch (`cortex-backlog`/`none`/external), and the 7-field `index.md` frontmatter template with inline-array and wikilink-padding rules. This is the lifecycle analog of #322 and the highest-leverage finding of the 2026-06-25 lifecycle reference-file audit. Moving the deterministic *creation* template for `index.md` and the lifecycle-start write-backs into CLI verbs shrinks a high-traffic skill body, removes per-run re-derivation of the byte-fragile index.md creation template, and consolidates the lifecycle-start backend routing into one place. (The artifact-registration *append* path and the already-complete *close* path stay prose — see Non-Requirements.)

## Phases

- **Phase 1: index.md offload** — a dedicated `cortex-lifecycle-create-index` verb owns the index.md creation template; thin `discovery-bootstrap.md`'s creation block.
- **Phase 2: lifecycle-start write-back offload** — a `cortex-lifecycle-start-sync` verb owns the lifecycle-start `cortex-update-item` write-backs (in_progress + lifecycle-slug) with caller-passed backend routing; thin only the "Backlog Write-Back (Lifecycle Start)" section of `backlog-writeback.md`; preserve the cross-referenced canonical blocks.

## Requirements

1. **`cortex-lifecycle-create-index` creates index.md byte-faithfully.** New module `cortex_command/lifecycle/create_index.py` + `[project.scripts]` entry. Given `--feature {slug}` and `--backlog-file {path|""}` (the Step-1-resolved path), it writes `cortex/lifecycle/{slug}/index.md` with the 7 fixed-order frontmatter fields, inline `artifacts: []`, and the wikilink body on a backlog match (full filename stem on both sides, unquoted `title`), with heading/body omitted on `--backlog-file ""`. Acceptance: a golden-byte test (`tests/test_create_index.py`) asserts `main([...])` output equals an inline golden string for both the matched and no-match shapes; `python3 -m pytest tests/test_create_index.py` exits 0. **Phase**: index.md offload

2. **index.md field forms match the two real consumers and a pinned canonical style.** `parent_backlog_uuid`/`parent_backlog_id` emit **bare unquoted** `null` (never `"null"`) so `wontfix_cli._frontmatter_value` does not treat them as a real target; `tags` emit as an **unquoted inline array** `tags: [a, b]` (the dominant corpus form, ~95%) — parseable by `load_requirements_cli._extract_tags`; `created`/`updated` emit **date-only** `YYYY-MM-DD` (no time/`Z` suffix). Acceptance: golden test includes (a) a null-field case asserting the literal line `parent_backlog_uuid: null`, (b) a populated-`tags` case asserting the emitted line is the unquoted form AND that `_extract_tags` round-trips it, (c) a date assertion matching `^\d{4}-\d{2}-\d{2}$`. Pass if all three hold. **Phase**: index.md offload

3. **create-index is idempotent via a structural skip-if-exists guard.** If `cortex/lifecycle/{slug}/index.md` exists, the verb makes no write and returns a no-op signal. Acceptance: test writes a sentinel index.md, runs the verb, asserts the file bytes are unchanged and stdout reports the skip. Pass if the file is byte-identical after the call. **Phase**: index.md offload

4. **create-index uses an injectable date-only seam.** Today's date for `created`/`updated` comes from an injectable `_today()` helper that returns a **date-only** ISO string (`YYYY-MM-DD`) — defined locally in `create_index.py` (do NOT reuse `cortex_command/lifecycle_event.py:_now_iso`, which returns a full `…T..:..:..Z` timestamp), monkeypatched in tests. Acceptance: `grep -nE 'def _today' cortex_command/lifecycle/create_index.py` returns ≥1 line; the golden test patches it; and Req 2(c)'s date-only assertion holds. **Phase**: index.md offload

5. **create-index write-root matches the backlog write tree (no worktree split).** The verb resolves its write root with the `CORTEX_REPO_ROOT`-honoring resolver (`_resolve_user_project_root`, as `cortex-update-item` uses) — not the cwd-walking `_resolve_user_project_root_from_cwd` — so the index.md write and the backlog write-back land in the same tree under overnight (`CORTEX_REPO_ROOT`=main, CWD=worktree), per the #319 precedent. Acceptance: a test sets `CORTEX_REPO_ROOT` to a different tree than CWD and asserts index.md is written under the env-root tree. Pass if the write lands in the env tree. **Phase**: index.md offload

6. **`discovery-bootstrap.md` creation block replaced by a verb invocation.** The "Create index.md (New Lifecycle Only)" template/wikilink prose is replaced by a single full-flag `cortex-lifecycle-create-index` invocation; the skip-if-exists pointer, the "consume Step 1's result" invariant, and the epic-context What/Why guidance remain terse prose. Acceptance: `grep -c 'cortex-lifecycle-create-index' skills/lifecycle/references/discovery-bootstrap.md` = 1; the 7-field-enumeration prose is removed (file shrinks). **Phase**: index.md offload

7. **`cortex-lifecycle-start-sync` owns the lifecycle-start write-backs only.** New module `cortex_command/lifecycle/start_sync.py` + `[project.scripts]` entry. Interface `cortex-lifecycle-start-sync --backend {value} --backlog-file {path|""} --phase {none|…} --session-id {id} --lifecycle-slug {slug}`. It runs `cortex-update-item <path> --status in_progress --session-id <id> --lifecycle-phase research` and, when `phase==none`, additionally `--lifecycle-slug <slug>`. Acceptance: an argv-assertion test pins the **exact** `cortex-update-item` flag list per phase; `python3 -m pytest tests/test_start_sync.py` exits 0. **Phase**: lifecycle-start write-back offload

8. **Backend routing follows ADR-0019: caller-passed `--backend`, structural-guard skip, no external-adapter logic.** The skill resolves the backend once via `cortex-read-backlog-backend` and passes `--backend`; the verb routes on the passed value — `cortex-backlog` → run the write-backs; `none` → skip them, emit a one-line advisory; **non-local (external)** → skip the local write-back and emit the advisory, leaving the external `gh issue` to the skill's existing (unchanged) external-arm prose. The verb **never self-resolves** the backend and carries **no external-tracker adapter logic** (no `needs_agent` token, no `gh` composition). Acceptance: test asserts (a) `cortex-backlog` runs the writes, (b) `none` runs zero writes + advisory, (c) external runs zero local writes + advisory and emits no external-adapter payload; plus a negative control per arm. Pass if all three arms + controls hold. **Phase**: lifecycle-start write-back offload

9. **exit-2 ambiguous-slug passthrough preserved.** If a `cortex-update-item` call exits 2, `start-sync` re-emits the candidate stderr and returns exit 2 (the `wontfix_cli` pattern). Acceptance: test stubs an exit-2 `cortex-update-item` and asserts the verb returns 2 with the candidate list on stderr. Pass if exit code = 2. **Phase**: lifecycle-start write-back offload

10. **Only the "Backlog Write-Back (Lifecycle Start)" section is thinned; cross-referenced canonical blocks survive (discriminating check).** Replace just the in_progress + lifecycle-slug write-back prose (and its continue-path backend-routing) with a full-flag `cortex-lifecycle-start-sync` invocation. The `cortex-update-item` exit-2 canonical block (referenced by `complete.md`) and the artifact-registration recipe (referenced by `plan.md` + `review.md`) — both separate sections of `backlog-writeback.md` — must remain. Acceptance (discriminating, grepping the **target** file, not consumers): `grep -c 'Exit-2 Handling (canonical)' skills/lifecycle/references/backlog-writeback.md` = 1 AND `grep -c 'Registering an Artifact in index.md (canonical)' skills/lifecycle/references/backlog-writeback.md` = 1; and `grep -c 'cortex-lifecycle-start-sync' skills/lifecycle/references/backlog-writeback.md` = 1. **Phase**: lifecycle-start write-back offload

11. **New verb prose invocations satisfy `cortex-check-contract`.** Every `cortex-lifecycle-create-index` and `cortex-lifecycle-start-sync` invocation in skill prose is written in **full** with all `required=True` flags (placeholders allowed). Acceptance: `cortex-check-contract` (E101/E102/E103, run at pre-commit) exits 0 on the staged skill edits. **Phase**: lifecycle-start write-back offload

12. **Console-script registration and prose invocation land in the same commit (W003).** Adding a `[project.scripts]` entry makes a verb "deployed"; its only wiring signal is the prose invocation, so the pyproject registration (Req 1/7) and the corresponding prose invocation (Req 6/10) must be committed together to avoid a W003 orphan at the pre-commit parity gate. Acceptance: `cortex-check-parity` exits 0 on each commit (no W003 orphan in any intermediate commit). **Phase**: lifecycle-start write-back offload

13. **Mirrors regenerated and committed with canonical edits.** After thinning the reference prose, `just build-plugin` regenerates `plugins/cortex-core/skills/lifecycle/`; canonical + mirror are committed together (drift hook). Acceptance: the `.githooks/pre-commit` drift check passes; `git diff --quiet` between canonical and regenerated mirror. **Phase**: lifecycle-start write-back offload

## Non-Requirements

- **The already-complete *close* path stays entirely prose (scope narrowed after critical-review).** The status-check, the Close/Continue `AskUserQuestion`, the `feature_complete` event, and the close-path `cortex-update-item --status complete` write all remain prose in their **current event-first order**. Rationale: the close path runs only when the item is already `status: complete`, so the original "fixes a latent ordering bug" justification was false — the current order has no orphan window, and flipping to write-first would instead leave `feature_complete` absent on a mid-close crash, making `detect-phase` (`common.py:358`) report the user-Closed feature as active. Without that (false) benefit, offloading the rare close-path write does not earn the verb↔prose seam and the untested `phase==none` short-circuit it would introduce. The lifecycle-start path (the common case) is still offloaded.
- **No close-path write in the verb; no order flip.** Consequently the `feature_complete` event line and the Close/Continue kept-pause are **untouched**, so `test_lifecycle_event_roundtrip.py` and `test_lifecycle_kept_pauses_parity.py` stay green trivially (no line movement, no event relocation).
- **No new events; no change to `feature_complete` emission** (#330 owns it).
- **No `status-check` read verb.** The `status == complete` comparison stays prose (the skill holds `status` from Step 1; the prompt decision depends on `AskUserQuestion` availability, which only the skill knows).
- **No `init-ensure` fold.** Overridden: `init_ensure.main` is argless and feature-blind (`parser.parse_known_args` then a discarded result; builds a fixed `Namespace`), and feature-specific templating is a different responsibility from repo-level `cortex init --ensure` scaffolding. A dedicated verb is used.
- **No artifact-*registration* offload.** #326 offloads index.md *creation* only; the `--register`/append path (used by `plan.md`/`review.md`) stays prose. (Consolidating it is a possible future cleanup.)
- **No change to the overnight fail-closed backend guard** (must not be touched or DRY-merged with the interactive fail-open reader).
- **No epic-detection offload** (`discovery-bootstrap.md`'s "Epic Research Detection" stays prose).

## Edge Cases

- **Resolver exit-3 (no backlog match):** `create-index` writes the no-heading Shape-B index.md with bare-`null` uuid/id; `start-sync` silent-skips the `cortex-update-item` calls. Both produce valid output, no error.
- **Special-char title:** the #326 ticket's own `title` contains `+`, `#`, `(`, `)` and is single-quoted in frontmatter; `create-index` unquotes it and embeds it verbatim in the wikilink. The golden test uses a special-char title.
- **Special-char tag:** a tag containing a YAML-special char (`:`, `,`, `[`) is quoted only as needed; plain kebab tags (the norm) render unquoted. The golden covers a plain-kebab tags case; document the plain-kebab assumption.
- **Backend `none` / external:** `start-sync` performs zero local writes and emits the advisory; the skill's unchanged external-arm prose composes the `gh issue` independently.
- **Overnight worktree (`CORTEX_REPO_ROOT`≠CWD):** `create-index` writes index.md under the env-root tree (Req 5), matching where the backlog write-back lands — no split-brain.
- **Re-run on existing index.md:** skip-if-exists guard makes it a no-op; a hand-edited index.md is never overwritten.

## Changes to Existing Behavior

- **MODIFIED:** `discovery-bootstrap.md` index.md creation prose → a `cortex-lifecycle-create-index` invocation; templating moves to `create_index.py`.
- **MODIFIED:** `backlog-writeback.md` "Backlog Write-Back (Lifecycle Start)" section's `cortex-update-item` write-backs + continue-path three-arm routing → a `cortex-lifecycle-start-sync` invocation with caller-passed `--backend`.
- **ADDED:** two `cortex-lifecycle-*` console-scripts (`create-index`, `start-sync`) + modules under `cortex_command/lifecycle/`.
- **UNCHANGED:** the entire close path (status-check + Close/Continue `AskUserQuestion` + `feature_complete` event + `--status complete` write, in current order); the external-tracker arm prose; the exit-2 and artifact-registration canonical blocks; the epic-context guidance.

## Technical Constraints

- **Module shape:** clone `cortex_command/lifecycle/stage_artifacts.py` — pure helpers + thin `main(argv) -> int`, compact JSON `json.dumps(…, separators=(",",":")) + "\n"`, `try/except CortexProjectRootError → return 1`, `_telemetry.log_invocation(...)` first line. Console-script **only** — no `bin/` wrapper, no `bin/` mirror; `cortex_command/*.py` is not mirrored.
- **Backend per ADR-0019:** verb never calls `resolve_backlog_backend()` itself; acts on caller-passed `--backend` as a structural guard. Note the shape vs the ADR-0019 precedent (`emit-lifecycle-start`): there the primary action (events.log write) always runs and only a slug read is guarded; here the primary action *is* the local backlog write, so skipping it on `none`/external is the correct structural guard — still no external-adapter logic in the verb. This applies ADR-0019's principle, not a new decision.
- **Resolver re-parse:** the resolver stdout JSON does not expose `uuid`/`tags`; `create-index` re-parses the resolved backlog file's frontmatter (`resolve_item._parse_frontmatter`) and derives `parent_backlog_id`/`NNN` from the filename prefix.
- **index.md templating:** manual f-string assembly (PyYAML cannot reproduce ordered-block-doc + inline-unquoted-arrays + unquoted-dates + bare-`null`); atomic write (temp file + `os.replace`); pin the trailing newline; canonical form per Req 2.
- **Byte-identity scope:** a canonical *creation* form + golden regression guard, not matching the (non-canonical) legacy corpus; the genuine consumer couplings are the bare-`null` lines (`wontfix_cli`) and `_extract_tags`-parseable inline `tags` (`load_requirements_cli`) — verified exhaustive (no other code parses index.md content).
- **Gates that bind:** `cortex-check-contract` (full-flag prose invocations), `cortex-check-parity` W003 (Req 12 co-commit) + the discriminating cross-ref check (Req 10), the dual-source mirror drift hook, `tests/test_lifecycle_kept_pauses_parity.py` (trivially green — kept-pause untouched), `tests/test_lifecycle_event_roundtrip.py` (trivially green — event line untouched). `just test` runs the editable install → use sequential (trunk) dispatch for the `cortex_command` edits, not a worktree.

## Open Decisions

None — the research Open Questions and the critical-review objections were resolved at spec time: drop `status-check` (Non-Req); two verbs (Phases 1/2); **close-path write stays prose** (Non-Req, per the critical-review A-finding that the write-first "bugfix" was false); `feature_complete` event untouched; epic-detection out of scope; external arm = skip + advisory, no `needs_agent` (Req 8, ADR-0019).
