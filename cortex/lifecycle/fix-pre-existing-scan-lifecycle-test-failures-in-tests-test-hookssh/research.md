## Web Research

### Canonical Claude Code hooks schema

Per `code.claude.com/docs/en/hooks` (canonical post-redirect from `docs.anthropic.com/en/docs/claude-code/hooks`):

> **`additionalContext` is ALWAYS nested inside `hookSpecificOutput`** across all events:
> `{ "hookSpecificOutput": { "hookEventName": "EventName", "additionalContext": "..." } }`

SessionStart shape:
```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "..."
  }
}
```

Required: `hookSpecificOutput.hookEventName == "SessionStart"`, `hookSpecificOutput.additionalContext` (string). Plain stdout (non-JSON) is also accepted as raw context injection.

**There is no documented top-level `additionalContext` or `additional_context` shape.** A hook emitting `{"additional_context": "..."}` at top level is not consumed by Claude Code as context.

### Env vars Claude Code injects into hooks

Documented: `CLAUDE_PROJECT_DIR`, `CLAUDE_ENV_FILE`. **`AGENT` is NOT injected by Claude Code** — confirming the env-var gate in the hook is not satisfiable through any Claude-side mechanism.

### Sources

- Hooks reference: https://code.claude.com/docs/en/hooks
- Known surfacing bugs (don't affect schema correctness): issues #16538, #13650, #9591

## Tradeoffs & Alternatives

- **Alternative A — Fix the hook (remove the dead env gate).** Delete the `${AGENT:-}` conditional at lines 459–471 and emit `hookSpecificOutput.additionalContext` unconditionally. Pros: restores live SessionStart context injection (broken for every real Claude session since `ea51d0c`); aligns with every other hook in the repo; aligns with the official Claude Code spec; tests pass without modification; tiny diff. Cons: must let the pre-commit drift hook regenerate the plugin mirror (no real downside).

- **Alternative B — Fix the tests (set `AGENT=claude` in test invocations).** Prefix `AGENT=claude bash "$SCAN_HOOK" ...` in the two failing tests. Pros: tiny diff, no hook touch. Cons: entrenches a phantom env-var contract that no upstream caller sets; tests would pass while real Claude sessions continue to receive no lifecycle context; hides a live production regression; contradicts the issue author's open framing in backlog #170 ("Is the hook itself broken, or have the tests' expectations drifted?").

- **Alternative C — Delete tests and/or the hook.** Wrong: the hook is load-bearing live infrastructure (overnight runner depends on `LIFECYCLE_SESSION_ID` injection; SessionStart context continuity across `/clear`). Tests are the regression-detection surface that just caught the bug. Both must remain.

- **Recommended approach — Alternative A.** Concrete fix:

  At `hooks/cortex-scan-lifecycle.sh:459–471`, replace the `if/else` with the unconditional Claude shape:

  ```bash
  if [[ -n "$context" ]]; then
    jq -n --arg ctx "$context" '{
      hookSpecificOutput: {
        hookEventName: "SessionStart",
        additionalContext: $ctx
      }
    }'
  fi
  ```

  Let the pre-commit drift hook regenerate `plugins/cortex-overnight/hooks/cortex-scan-lifecycle.sh`. Optionally clean up `tests/test_lifecycle_phase_parity.py` line 511 (`env["AGENT"] = "claude"`) and line 545 (snake_case fallback parser) — both become dead but neither blocks acceptance.

## Adversarial Review

### Failure modes

- **A1's "fix the tests" prescription would entrench a phantom env-var contract.** Git archaeology shows `AGENT` was always a shell-local variable derived from stdin (`428e54e`), never an env var. `ea51d0c` deleted the assignment but left the consumer; `fde7dc6` papered over `set -u` rather than excising the dead branch. Adding `AGENT=claude` to test invocations would canonize a contract with no upstream definition, no documentation, and no other writer — pure dead scaffolding.

- **Live SessionStart impact is real, not just test-cosmetic.** The current default `else` branch emits `{"additional_context": "..."}`, a key Claude Code does not consume. Every real Claude SessionStart since 2026-04-01 has received no lifecycle context injection from this hook. The failing tests are catching a production regression, not a test drift.

- **A4's claim that Claude Code "silently discards unknown top-level keys"** is true per current spec but not future-proof. If a future Claude Code release tightens hook output validation, the current state could go from "no-op" to "fatal SessionStart hook failure." Reason to fix promptly.

### Assumptions verified

- The hook gate is truly dead: exhaustive `grep` for `AGENT=` across the repo finds exactly one caller-side assignment (`tests/test_lifecycle_phase_parity.py:511`), a defensive set by a test that worked around the bug. No settings file, plugin manifest, CI script, overnight runner, or status line sets `AGENT`.
- Plugin registration in `plugins/cortex-overnight/hooks/hooks.json` is a bare command invocation with no env wrapper.
- No hidden consumer of `additional_context` (snake_case) exists outside the hook and the phase-parity test's fallback parser.
- No other test depends on the non-Claude branch.

### Anti-patterns

- Current `${AGENT:-}` defensive default is "fix the symptom, not the cause." The deeper lesson: dead-branch detection in `set -euo pipefail` code requires inspecting where the variable is *set*, not just where it's read.

### Security

- No security implications either way. Hook reads SessionStart input, writes JSON, no privilege boundary.

### Verdict on the A1 vs A4 contradiction

**A4 is correct; A1 is wrong.** A1's path produces green tests but freezes a phantom contract, leaves production SessionStart broken, and contradicts the backlog author's open framing. A4's path produces green tests, fixes the live regression, and aligns with the documented Claude Code spec and the repo's other hooks.
