# Plan: skill-prose-to-cli-argparse-contract

## Overview

Build a stdlib-only pre-commit lint (`cortex_command/lint/contract.py`) that AST-extracts argparse surfaces from `[project.scripts]`-registered modules and validates `cortex-*` invocations in skill prose against those surfaces. Land the lint, its exception ledger, the four in-PR drift fixes, and the Phase 1.55 hook block in a single PR with hard-block enabled on merge. Mirrors `cortex_command/parity_check.py` in shape and reuses the fence state machine from `cortex_command/lint/prescriptive_prose.py:113-156`.

## Outline

### Phase 1: Lint core and exception governance (tasks: 1, 2, 3, 4, 5, 6, 7, 8)
**Goal**: Produce a working `bin/cortex-check-contract` binary, the AST-extractor + scanner + validator core, the exception-ledger surface, and a fixture-driven test suite that passes in isolation. The lint runs against any tree but is not yet wired into pre-commit.
**Checkpoint**: `uv run pytest tests/test_check_contract.py -v` exits 0 with ≥7 parametrized cases AND `bin/cortex-check-contract --help` exits 0 AND `uv run python3 -m cortex_command.lint.contract --self-test` exits 0.

### Phase 2: Pre-commit and corpus integration (tasks: 9, 10, 11)
**Goal**: Seed the initial exception ledger, fix the four named drift callsites in the same PR, then wire the lint into `.githooks/pre-commit` as Phase 1.55. After Phase 2 the lint hard-blocks on any subsequent drift in the scan corpus.
**Checkpoint**: `bin/cortex-check-contract --audit` exits 0 against the post-fix tree AND `grep -n "Phase 1.55" .githooks/pre-commit` returns ≥1 match AND the surrounding block invokes `just check-contract --staged`.

## Tasks

### Task 1: AST-based argparse surface extractor
- **Files**: `cortex_command/lint/contract.py` (new), `cortex_command/lint/__init__.py` (verify or create)
- **What**: Implement `extract_surface() -> dict[str, ParserSurface]` that reads `pyproject.toml` `[project.scripts]` via `tomllib`, resolves each `module:attr` target to a source path, parses the source with `ast`, walks for `argparse.ArgumentParser(...)` constructors and `.add_argument(...)` / `.add_subparsers(...)` / subparser `.add_parser(...)` calls, and builds a static `ParserSurface` dataclass exposing `required_flags`, `optional_flags`, `subcommands`, and per-subcommand surfaces. AST-unparseable modules emit `E201 cannot AST-parse module source for cortex-X at path Y` and are recorded as "extraction failed"; the function returns the partial surface plus an extraction-errors list. Handles the "multiple `ArgumentParser` instances" edge case by picking the parser with the most `.add_argument` calls; ambiguous (tied) parsers emit `E202 ambiguous main parser for cortex-X` and treat the module as exempt-equivalent. Implementation references `cortex_command/parity_check.py` for the tomllib loading pattern and `cortex_command/lint/prescriptive_prose.py` for module-scaffolding conventions.
- **Depends on**: none
- **Complexity**: complex
- **Context**: `pyproject.toml` lines 21-57 enumerate `[project.scripts]` (~33 entries). The `ParserSurface` dataclass has fields `binary: str`, `module_path: Path`, `required_flags: set[str]`, `optional_flags: set[str]`, `subcommands: dict[str, "ParserSurface"]`, `extraction_status: Literal["ok","ast_error","ambiguous","not_argparse"]`. AST walking uses `ast.walk()` on the parsed tree; pattern-match on `ast.Call` nodes where `func` resolves to `argparse.ArgumentParser` or has `.add_argument`/`.add_subparsers`/`.add_parser` as the method name. Use `ast.unparse()` only for diagnostic emission, never to round-trip code. The `[project.scripts]` value format is `cortex-foo = "cortex_command.module.path:main"`; resolve the path via `importlib.util.find_spec()` (do NOT import the module) and read `spec.origin` as a `Path`. Do not run `importlib.import_module` — that triggers side-effect imports (`_telemetry.log_invocation`, `claude_agent_sdk`, `yaml`) per Tradeoffs section M2 / Technical Constraint #1.
- **Verification**: `uv run python3 -c "from cortex_command.lint import contract; surface, _ = contract.extract_surface(); s = surface['cortex-create-backlog-item']; assert '--status' in s.required_flags and '--type' in s.required_flags, s"` exits 0

## Risks

- **Audit-corpus drift beyond the 4 named callsites**: research notes "8 distinct drift signatures across ≥9 skill files" — more than spec Requirement 12's 4 named callsites. After Task 10 the audit may still emit violations if the remaining 4+ drifts are in scope of v1 (not `cortex-update-item` key=value, not `python3 -m`). If `bin/cortex-check-contract --audit` exits non-zero after Task 10, expand Task 10's file list and/or Task 9's ledger to cover the residual drift before Task 11 lands.
