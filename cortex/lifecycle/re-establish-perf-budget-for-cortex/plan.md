# Plan: re-establish-perf-budget-for-cortex

## Overview

Refactor `cortex_command/log_invocation.py` to defer heavy imports and remove the `subprocess git rev-parse` call (replaced by a pure-Python walk over `[Path.cwd(), *Path.cwd().parents]`). Reorder `bin/cortex-log-invocation` branches so the working-tree `pyproject.toml` cortex-command predicate fires before the wheel-import probe. Re-establish the perf budget test against the new measured floor and bind the invariants the deleted delta test was implicitly defending via extended parity fixtures and a structural branch-order test. **Sequencing**: test scaffolding (extended parity fixtures, fixed `_p95`, durable import-time assertions) lands BEFORE the module refactor, so the refactor is verified against new fixtures that exercise the walk's edge cases.

## Risks

- **Measured floor on dev hardware may exceed the recommended 50ms p50 starting budget**, requiring iteration. If after Phase 2+3 the floor is e.g. 45ms p50 but with ±15ms variance, the budget must be 60ms+ to pass — and may not leave room to detect a 20ms regression. Task 8's contingency is to re-skip with a follow-up pointer rather than loosen iteratively, AND file a follow-up backlog item naming the calibration carry-over (user-confirmed resolution: re-skip + follow-up ticket counts as #264 complete because Phases 1-3 structural tests cover the regression classes that motivated #264). Task 8 MUST file the follow-up ticket before re-skipping — the file-and-link is part of the contingency, not deferred.
- **Task 4's session-id-first invariant is bound structurally by Task 3's `test_log_invocation_main_session_id_check_first` (AST-based first-statement check in `main()`).** A future edit that places any executable statement before the `LIFECYCLE_SESSION_ID` guard in `main()` will trip the assertion. The test is structural (ast.parse + first-non-docstring-body inspection), so it has no runtime cost and no platform variance. Remaining gap: the AST test does not catch a regression where deferred imports inside `main()` move ABOVE the session-id guard via expression order rather than statement order — e.g., if the session-id assignment's right-hand side is rewritten to embed an import expression. Mitigation for that gap: the un-skipped budget test (Task 8) catches the perf signature.
- **The structural branch-order test (Task 6) is brittle to predicate-text changes**. If a future contributor renames the cortex-command package or rewords the wheel-probe one-liner, the test's substring anchors need updating. Mitigation: the anchors are tied to literal stable text (`^name = "cortex-command"` is the package name; `cortex_command.log_invocation` is the module path).
- **Plugin mirror regeneration in Task 5 may surprise if the pre-commit hook is not installed**. The user is instructed to run `just setup-githooks` after clone (per CLAUDE.md); if not done, the mirror won't auto-regenerate. Task 5's contingency is direct `cp` if the hook is unavailable. Both files must land in the same commit.
- **macOS physical-path emission in breadcrumbs** is an accepted behavioral change (spec-documented).
- **`GIT_DIR` / `core.worktree` divergences from `git rev-parse`** are accepted scope-doc changes (spec-documented).
- **Task 4 is a complex multi-refactor task**. Bundling defer-imports + remove-subprocess + pure-Python-walk + reorder-main in one task is justified by the test scaffolding from Phase 1 — Task 1's extended fixtures and Task 3's import-discipline test bind enough invariants that the bundled task's correctness is durably verified. Splitting Phase 2 would be lower-risk but would mean more intermediate commits without proportional benefit since each sub-change touches the same file.
