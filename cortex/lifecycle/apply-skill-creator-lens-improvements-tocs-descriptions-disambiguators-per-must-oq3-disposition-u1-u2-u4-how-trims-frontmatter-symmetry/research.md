The closed evidence list collapses to one form (transcript URL/excerpt) in practice. Any retro-citation criterion that grep'd events.log for F-rows fails closed.
- **FM-2**: User's amendment ("soften both, no parser-cite") inverts the ticket's verification clause at backlog/178:94 ("MUSTs retained with parser-cite at `metrics.py:221` documented"). Implementation will hit contradictory criteria unless ticket-178 verification is updated **before** spec writing.
- **FM-3**: review_verdict observed corpus has only 3 events across 158 features. Cannot empirically distinguish "MUSTs prevent drift" from "review_verdict is so rare we haven't observed drift yet."
- **FM-4**: `clarify-critic.md:155` cross-field invariant ("any post-feature event with `origin: alignment` MUST have `parent_epic_loaded: true`") is a **schema invariant**, not a behavior directive. Softening eliminates the only enforcement (writer-side norm) before the planned validator lands. Same parser-protection class as review.md.
- **FM-5**: `clarify-critic.md:26` warning-template MUST is a **closed allowlist** ("MUST NOT echo raw filesystem error text or helper stderr"). Softening enables information-leak vectors (filesystem paths, error strings into user-facing warnings). Security-adjacent.
- **FM-6**: U2 corpus-wide trim using "events.log F-row" criterion **trims ALL rows mechanically** â€” zero F-rows exist, so the grep returns no matches for any row. The "â‰¤2 retro-cited rows" reduces to "0 rows" under literal application.
- **FM-7**: Pre-commit dual-source drift hook validates byte-identical mirror sync only â€” it does NOT parse YAML or check that softened review.md prose still produces parser-conforming verdict JSON downstream. First failure surface is silent observability degradation (`review_verdicts: None` in morning report).
- **FM-8**: Ticket 176 vs 178 overlap: the title-string ambiguity (`refine/clarify.md` vs `refine/clarify-critic.md`) is an edit-conflict risk. Verify before implementation.
- **FM-9**: Description char-count ceiling. Empirical: lifecycle 901, critical-review 895 chars. Anthropic 1024 cap leaves ~130 chars headroom. Adding 4-6 trigger phrases + disambiguator clauses (~80-120 chars) WILL exceed the cap unless additions replace existing tokens or move to `when_to_use`.
