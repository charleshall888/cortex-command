# Specification: cortex-lifecycle-enter-state-accept-a (#379)

## Problem Statement

A lifecycle's stored identity is currently whatever token the caller happened to type, so the same backlog item resolves to two different identities depending on which command ran first: `/cortex-core:refine 340` stores `core-skill-efficiency-survivors-of-the` while `/cortex-core:lifecycle 340` stores `340` (verified live). Worse, `cortex-lifecycle-enter` materializes a directory for **any** string it is handed — in the reported wild-light incident, `enter --feature 269` silently created a shadow `cortex/lifecycle/269/`, mis-bound `.session`, and stranded critical-review residue where the morning report could never find it. Recovery was manual. This fixes the producer so a ticket number is always a valid **input** but never a stored **identity**, and hardens the one verb that writes so it fails loud instead of inventing a lifecycle.

## Phases

- **Phase 1: Writer guards** — `enter` refuses to materialize a lifecycle dir it cannot account for. Independently shippable; closes the reported incident with no identity-semantics change.
- **Phase 2: Producer normalization** — `resolve_invocation`'s `new` branch emits the canonical `lifecycle_slug`. **Depends on Phase 1**: Phase 2 removes the `is_dir()` conjunct that currently holds a path-traversal vector inert, so the guard must land first.

## Requirements

1. **`enter` fails loud on a resume of a non-existent lifecycle**: with `--phase` ≠ `none` and no `cortex/lifecycle/{feature}/` dir, write a `cortex-lifecycle-enter: …` line to stderr and **exit 3**, creating nothing. Acceptance: `cortex-lifecycle-enter --feature no-such-thing --phase research --session-id X --backend cortex-backlog --backlog-file ""`; then `echo $?` == `3` (the command's OWN exit code, not a downstream check); stderr matches `no such lifecycle`; `test -d cortex/lifecycle/no-such-thing` = exit 1. Grounding: `cortex_command/lifecycle/enter.py:136-194` (guard precedes `create_index` at `:169`); exit-code idiom at `:261-269`. **Phase**: Writer guards

2. **`enter` still creates a legitimately-new lifecycle**: `--phase none` with no dir proceeds unchanged. Acceptance: `--feature brand-new-slug --phase none --backlog-file ""` → `state` == `ready`; `cortex/lifecycle/brand-new-slug/index.md` exists. Grounding: `enter.py:169`, `create_index.py:162-165`. **Phase**: Writer guards

3. **`enter` rejects an unsafe feature token**: adopt the house `_reject_unsafe_slug` pattern (rejects `/`, `\`, `..`) before any filesystem op, failing via the same stderr + **exit 3** channel as R1. Acceptance: `cortex-lifecycle-enter --feature '../../../tmp/evil' --phase none …`; `echo $?` == `3`; no file created outside `cortex/lifecycle/` (`test -e /tmp/evil` = exit 1). Grounding: pattern at `describe.py:55-63`, `plan_decision.py:71-79`, `review_verdict.py:101-109`, `next_verb.py:134-144`, `advance.py:171-179`, `implement_transition.py:88-96`, `spec_approve.py:116-124`; absent from `enter.py` (zero grep hits). Note `wontfix_cli.py:46` uses the stronger **whitelist** idiom (`_SLUG_RE = ^[a-z0-9]+(-[a-z0-9]+)*$`); implement may adopt either, but must match one existing idiom rather than invent a third. **Scope limit**: this closes the `next` → `enter` chain only — see Non-Requirements for the paths it does not cover. **Phase**: Writer guards

4. **Grandfathered numeric lifecycles keep working**: a feature whose dir exists passes every guard regardless of token shape. Acceptance: `cortex-lifecycle-enter --feature 374 --phase specify …` → `state` != `error` (dir exists, so `not feature_dir.is_dir()` is False). Grounding: `cortex/lifecycle/374/`, `cortex/backlog/374-*.md` (`lifecycle_slug: "374"`). **Phase**: Writer guards

5. **`enter` refuses to reuse a dir belonging to a different backlog item**: when the target dir exists AND its `index.md` frontmatter `parent_backlog_uuid` is non-null AND differs from the uuid of the item named by `--backlog-file`, fail via stderr + **exit 3** and leave the existing dir byte-identical. Compare the **uuid** (immutable, survives renames/renumbering), not the filename. Closes the silent-merge path Phase 2's derived-slug identity makes reachable. **Inert by design when `--backlog-file` is `""`** (Context B / no_match): `parent_backlog_uuid` is `null`, there is no item to collide on, and the check is skipped — this is a real coverage limit, not universal protection. Acceptance: create a lifecycle for item A; invoke `enter` with A's slug and B's `--backlog-file`; `echo $?` == `3`; A's `index.md` sha256 unchanged before/after. Grounding: `create_index.py:164-165` (`if target.exists(): return {"signal": "skipped"}`) is the silent-merge mechanism; `create_index.py:_render` emits `parent_backlog_uuid`/`parent_backlog_id` into index.md frontmatter (verified against the live `cortex/lifecycle/cortex-lifecycle-enter-state-accept-a/index.md`). **Phase**: Writer guards

6. **The guards fail loud without touching skill prose**: all three guards use a purpose-built exception caught in `main()`, a `cortex-lifecycle-enter: …` stderr line, and **exit 3** — a distinct code, matching the file's own contract-violation idiom. Exit 3 is chosen because exits 1 and 2 already carry distinct contracts (`enter.py:261-269`) that CLI callers branch on. **This costs zero skill prose**: `SKILL.md:56` enumerates only exit 2, while exit 1 has been live and unenumerated at `enter.py:267` since it shipped — an unenumerated exit code is the established norm, and a nonzero exit surfaces its stderr to the caller without a prose arm. Do **not** route these through `state: "error"`: that channel is the exit-0-always catch-all for *unexpected* exceptions (`enter.py:270-271`), regression-pinned by `test_cli_exits_0_with_error_state_on_unexpected_exception` (`test_enter.py:339-355`), and reusing it would make the guards invisible to every caller that branches on exit code rather than parsing stdout JSON — including `bin/cortex-lifecycle-enter`'s own `exec` passthrough wrapper. Acceptance: (a) `git diff --stat` for the whole change lists zero files under `skills/` or `plugins/*/skills/`; (b) `test_cli_exits_0_with_error_state_on_unexpected_exception` still passes unmodified. **Phase**: Writer guards

7. **The guard's message distinguishes its two causes**: "dir vanished mid-flight" (TOCTOU) vs "caller mis-threaded the identity" produce the same trip but different remediation. Acceptance: `Interactive/session-dependent: message wording reviewed at implement time`; the message names the feature and the served-slug remedy. **Phase**: Writer guards

8. **`resolve_invocation`'s `new` branch emits the canonical slug**: when a backlog item resolved and `slug != feature`, emit `feature = lifecycle_slug` (str-coerced) and `resolved_from = <raw token>`. Acceptance: `cortex-lifecycle-next 340` → `feature` == `core-skill-efficiency-survivors-of-the`, `resolved_from` == `340`. Grounding: `cortex_command/lifecycle/resolve.py:204-211` (backlog dict in scope at `:208`); coercion idiom at `:184-187`. **Phase**: Producer normalization

9. **`state: new` is preserved** — #370's edge ("a backlog ID with no lifecycle dir under its slug must still resolve `new`") is honored; only `feature`'s value and `resolved_from`'s presence change, never the state. Acceptance: same call as R8 → `state`/route still resolves as a new feature at research, not `resume`. Grounding: `resolve.py:204-211`; #370 Integration text. **Phase**: Producer normalization

10. **Context B is untouched**: with no backlog match (`backlog is None`), `feature` stays the caller token and no `resolved_from` is emitted. Acceptance: `cortex-lifecycle-next some-adhoc-slug-with-no-item` → `feature` == `some-adhoc-slug-with-no-item`; `resolved_from` absent. Grounding: `resolve.py:85-102`, `:180` (`isinstance(backlog, dict)` gate). **Phase**: Producer normalization

11. **The pinned-slug corpus is unaffected**: the change fires only when `slug != feature` AND the slug's dir does not exist — i.e. exclusively first entries on not-yet-pinned items. Acceptance: (a) `cortex-lifecycle-next 374` → `feature` == `374`, resume, `resolved_from` absent (byte-identical to pre-fix); (b) a corpus check emits zero drift rows — for every `d` in `cortex/lifecycle/*/`, `cortex-lifecycle-next "$(basename $d)"` returns `feature` == `basename $d` (count of rows where they differ = 0). Grounding: `resolve_item.py:135-142` (frontmatter is priority-1). **Phase**: Producer normalization

12. **The pinned test is updated deliberately, not incidentally**: `test_numeric_id_without_slug_dir_stays_new`'s `assert "resolved_from" not in r` flips to assert its presence and the normalized `feature`; `assert r["state"] == "new"` **must remain**. Acceptance: `just test` green; the test's `state` assertion unchanged in the diff. Grounding: `cortex_command/lifecycle/tests/test_resolve.py:194-202`. **Phase**: Producer normalization

- Does **not** close the broader vivifying-writer class — `cortex-lifecycle-event`, the 4 standalone B1 CLIs, `record_pr_opened`, `create_index` Shape B, `finalize`. Deferred to a follow-up ticket (research Open Question 4).

## Edge Cases

- **TOCTOU — dir deleted between `resolve` and `enter`**: guard trips, `enter` fails loud rather than silently resurrecting a shadow dir. This is the desired behavior; R7 requires the message distinguish it from a mis-thread.
- **Title edited between Step 1 and Step 2**: first-entry identity is provisional until `enter` pins it. Window is one skill turn — `enter` pins at Step 2 before refine re-resolves at Step 3, and frontmatter is priority-1 thereafter. Accepted and documented in code docstrings only.
- **Standalone refine (no `enter`, no pin)**: `refine.py:212`/`:289` mkdir the dir directly and never pin frontmatter, so it re-derives from the title on every call — **already true today**; this change adds no new exposure there.
- **Two items truncating to the same 6-word slug**: 372/372 unique today, but 204 of 225 unpinned titles are truncated. R5 catches the collision at `enter` and fails loud instead of merging.
- **`lifecycle_slug: ../../../../tmp/evil` in frontmatter** (bug, corrupted write, or autonomously-authored item): R3 rejects it **on the `next` → `enter` chain only**. The value reaches `feature` unsanitized because `next_verb.py`'s sanitizer is scoped to the `resume` arm (`:433`) and `new` returns verbatim via `_ROUTING_PASSTHROUGH` (`:88-96`, `:421-424`). **This scenario is NOT closed by this spec.** `/cortex-core:refine` resolves the identical field and feeds it to `refine.py:211-212`/`:288-289` (unguarded, CWD-relative mkdir) and `register_artifact.py` **without ever invoking `enter`** — reachable before Phase 1, after Phase 1, and regardless of phase ordering. See Non-Requirements: the class is live today and deferred to a security ticket.
- **Hand-typed `enter --feature 999 --phase none`**: creates `999/`. Permitted — #378 ratified numeric tokens as first-class input; the producer fix simply means the loop never produces this.
- **Two sessions entering the same ticket concurrently**: first-entry identity is provisional until `enter` completes; a second entrant should re-resolve rather than reuse a cached `feature`. Documented, not solved.
- **`--phase` override on a non-existent feature** (`myfeature plan`): `resolve.py:209` can carry `phase: "plan"` on a `new` envelope, but `SKILL.md:56` hardcodes `none` for `new`, masking it. Pre-existing; unchanged.

- **ADDED**: existence/phase, unsafe-slug, and same-item-uuid guards in `enter.py` — the fourth application of the shipped `_lifecycle_dir_exists` pattern (`critical_review/__init__.py:484-494`).

## Technical Constraints

- **No skill-prose edits** (operator constraint). Satisfiable **without weakening the failure signal**: `SKILL.md:53` threads `{feature}` opaquely, and an unenumerated nonzero exit needs no prose arm — `SKILL.md:56` names only exit 2 while exit 1 has been live and unenumerated at `enter.py:267` since it shipped. The apparent conflict between "no prose" and "fail loud" was false.
- **Guard the callers, never the primitive**: `critical_review/__init__.py:490-492` — "The guard lives in the callers, NOT in `log_event_at` (which must keep creating the dir for the legitimate fresh-lifecycle first-write at Site A, `refine.py`)." A blanket guard on the event-append path would break standalone refine.
- **No `die()` helper exists** in `cortex_command/` (zero grep hits); fail-loud is a purpose-built exception caught in `main()` + a `cortex-lifecycle-enter: …` stderr line + a distinct exit code. R6 follows that idiom with exit 3 rather than overloading exits 1/2 or the exit-0 `state: "error"` channel.
- **ADR posture**: ADR-0019 is untouched (it is scoped to `--backend`-shaped flags; arg+filesystem validation is normal verb work per `project.md:35`). The fix site is inside ADR-0024's sanctioned surface — `resolve.resolve_invocation` is `next`'s in-process identity engine (`next_verb.py:6`, `:72`, `:416`). **ADR-0019, ADR-0024, ADR-0027 are `status: proposed`, not accepted** (`cortex/adr/README.md:61-63`) — recorded intent, surfaced, not auto-binding.
- **`lifecycle_slug` is already on `STRING_INTENDED_KEYS`** (`frontmatter_quote.py:36-37`); any frontmatter write routes through the quoter (ADR-0027).
- **Protocol**: this changes a field's value semantics without changing envelope shape, so no bump per `protocol.py:24-27`. **Known gap**: that policy has no mechanism to flag semantic-value drift to out-of-repo consumers — the wild-light scenario that produced #379.
- **Test command**: `just test` (`cortex/lifecycle.config.md`).
- **Zero `next_verb.py` changes required** — `new` is in `_ROUTING_PASSTHROUGH` and flows through verbatim.

## Proposed ADR

### Proposed ADR: 0029-lifecycle-identity-is-the-canonical-slug

**Context.** Lifecycle dirs are keyed by whichever token the first caller typed, so identity is route- and order-dependent: `/cortex-core:refine 340` and `/cortex-core:lifecycle 340` produce different identities for one item, and `overnight/backlog.py` independently assumes a third convention. #370 established that "the slug is the canonical identity; other tokens are input normalization" but deliberately bounded the remap to the resume path (its acceptance criterion: "Only emit `state: new` when neither the numeric-token dir nor the backlog-slug dir exists"). #378 separately ratified numeric IDs as a first-class **input** and `374`/`378` as legitimate numeric-keyed dirs, explicitly dropping a proposal to reject bare-numeric tokens at parse time. Nothing in the repo governs lifecycle directory naming, and the record is silent on whether `state: new` may return a normalized `feature`.

**Decision.** A lifecycle's identity is the backlog item's canonical `lifecycle_slug` (`resolve_item.py:113`'s chain: frontmatter → spec/research dirname → capped `slugify(title)`). A ticket number, uuid prefix, filename stem, or title phrase is **input normalization** — accepted everywhere, stored nowhere *by the served loop*. `resolve_invocation` emits the canonical slug for **every** state including `new`, extending #370's own rule to the branch it bounded out.

**Scope of that claim, stated precisely.** This governs the `resolve_invocation`-mediated path (`/cortex-core:lifecycle` → `next` → `enter`) only. A **direct, hand-typed `cortex-lifecycle-enter --feature <number> --phase none` still creates a numeric-keyed dir, and remains permitted** — so numeric-keyed identity is **not** a closed set, and this ADR does not deprecate it. #378 preserved bare-numeric tokens as first-class input on functional grounds (rejecting them "would orphan the real item, miss the `resume/complete/wontfix <id>` forms, and break the `#378` literal-id sigil") and never held that new numeric identities would stop being producible; this decision does not extend #378 into a closure it did not make. **Consequence, stated so it is not mistaken later**: the #378 defensive coercions at `resolve_item.py:137-141` and `resolve.py:184-187` **MUST be retained** — they are not dead code guarding only `374`/`378`, because the hand-typed path keeps producing exactly the values they defend against.

**Trade-off.** Two objections are accepted rather than answered. **First**, the Kubernetes Name-vs-UID design argues the opposite — durable identity should be the system-assigned immutable key (the ticket number), with the human-friendly name as an alias, precisely because a title-derived slug drifts. First-entry identity becomes title-sensitive for one skill turn until `enter` pins it to frontmatter. (Note an immutable key *does* exist and is already recorded — `index.md`'s `parent_backlog_uuid` — and R5 uses it for collision detection; making it the *primary* identity was not attempted here and remains open.) **Second**, this buys identity determinism for the served loop at the cost of a **split rule**: the loop normalizes, a direct CLI call does not. That asymmetry is deliberate (ADR-0024 sanctions resolution only in the served-verb class, and ADR-0019 keeps `enter` a dumb arg-actor) but it means "identity is the canonical slug" is a property of the *path*, not an invariant of the *system* — a future contributor should not assume every lifecycle dir on disk is slug-keyed. It is taken because the alternative — identity determined by which command a user happened to run first — already produced a live data-corruption incident (#379), already disagrees with the overnight runner's readiness gate, and cannot be reconciled without picking one producer. Rejected alternatives: resolving inside `enter`/`state` (duplicates resolution across dumb verbs; a third ADR-0019 scope-extension precedent); and normalizing `refine` to store raw tokens instead (wider blast radius — `lifecycle-slug` is consumed by name across `refine/SKILL.md:86,94,98`, and would mean un-deriving a field two call sites already share).
