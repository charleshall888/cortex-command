# Plan: investigate-and-standardize-path-resolution-for

## Overview
Fix the `${CLAUDE_SKILL_DIR}` path-resolution defects that break plugin-distributed skills off-repo, then re-establish the deleted authoring guardrail structurally. Phase 1 closes the only failure that is both skipped-every-run and indistinguishable-from-noise (pr-review evidence-grounding) plus the two sibling raw-token prompt bugs. Phase 2 adds a context-scoped lint, sweeps the Class-2/3 bare-relative sites the lint is designed to catch, adds a positive-routing CLAUDE.md principle, and lands ADR-0009. Mechanism: explicit **body-propagation** — only a SKILL.md *body* resolves `${CLAUDE_SKILL_DIR}` (its own dir directly; a *sibling* skill via `${CLAUDE_SKILL_DIR}/../<sibling>`), so the owning body resolves the value and propagates the absolute path or inlined content to every reference file, shell line, and composed subagent prompt that needs it (a reference file cannot resolve the token itself). No new CLI subcommand for pr-review (the `2>/dev/null` swallow, not a missing CLI, is the bug).

## Outline

### Phase 1: Close the silent harm + sibling Class-1 prompt bugs (tasks: 1, 2, 3, 4)
**Goal**: pr-review evidence-grounding runs on faithful runs and fails loud+distinct (via a concrete, testable guard) when the script is absent; the Stage-4 subagent prompt receives inlined rubric/output-format content; the critical-review and diagnose raw-token prompt bugs are removed.
**Checkpoint**: `grep -c 'CLAUDE_SKILL_DIR:-$TMPDIR}/scripts/evidence-ground.sh' protocol.md` = 0; the absent-script guard emits a distinct non-empty stdout sentinel when run against a nonexistent path; the three Class-1 reference files carry no raw `${CLAUDE_SKILL_DIR}` token inside a dispatched prompt; piping a valid payload through `evidence-ground.sh` emits non-empty grounded JSON; cortex-core mirror clean.

### Phase 2: Anti-recurrence lint + Class-2/3 sweep + convention (tasks: 5a, 5b, 5c, 5d, 6a, 6b, 7, 8)
**Goal**: every bare-relative Read/execute site is converted to a body-propagated robust form, a two-detector structural lint (whose D2 exempts `${CLAUDE_SKILL_DIR}/`-prefixed paths) enforces the invariant going forward, CLAUDE.md carries the positive-routing principle, and ADR-0009 records the decision.
**Checkpoint**: `cortex-check-skill-path --audit` exits 0 on the real tree (including the pre-existing and newly-written `${CLAUDE_SKILL_DIR}/../` body forms, which D2 must NOT flag); every Class-2/3 per-site grep for its pre-fix broken form returns 0; `just test` exits 0; CLAUDE.md and ADR-0009 in place.

> **Scope note**: this plan is **full scope** (both phases, recommended — satisfies AC4 in this lifecycle). The §4 approval surface offers **minimum** (Phase 1 only; Phase 2 deferred to a new follow-up ticket) as the alternative. Under minimum scope, only Tasks 1–4 run and the Phase-2 tasks become a separate ticket.

> **Mirror-serialization note (do NOT parallelize)**: Tasks 3, 4, 5a, 5b, 5c each run `just build-plugin`, a **global** `rsync --delete` over the entire `plugins/cortex-core/` mirror. They are chained via `Depends on` into a strict serial order (3 → 4 → 5a → 5b → 5c) so the implement phase never dispatches two of them concurrently against the shared working tree (concurrent global regens race and corrupt each other's mirror slice). This is deliberate — see Risks. Tasks 1, 2 (pr-review, hand-maintained, no mirror), 5d (android, no mirror), 6a, and 8 remain parallel-eligible.

## Risks

- **Reference-file fixes need the owning body to propagate (Tasks 5b, 5c).** A reference file cannot resolve `${CLAUDE_SKILL_DIR}`, and a body only resolves its OWN dir — so a cross-sibling consult (discovery → lifecycle) requires the body to propagate the **sibling** absolute path `${CLAUDE_SKILL_DIR}/../lifecycle/…` (which the body CAN resolve) or inline the content; "resolve relative to the dir the body knows" is insufficient. The `cat …|bash` shell sites in `implement.md` are the hardest: a reference-file shell line gets no substitution pass, so the body must propagate the absolute helper path and `implement.md` must reference it. This is body-propagation per spec Req 5, not a new console script. Residual: the agent constructing the absolute path from the body-established value is still prose-guided — but, unlike the cross-sibling case, the information genuinely exists in context (the lifecycle body resolves the helper's own dir), so it is a robust-prose form, not a bare-CWD path.
- **Lint D2 must exempt the plan's own fix form (Tasks 5a/6).** The correct body form `${CLAUDE_SKILL_DIR}/../lifecycle/…` contains a `../` segment that a naive D2 would flag, which would make Task 6b's `--audit` gate un-passable on a tree that already contains such lines (`refine/SKILL.md:168,177`) and gains more from Task 5a. D2 is therefore specified to exempt a relative segment carried by a resolved `${CLAUDE_SKILL_DIR}/` prefix, and Task 6a carries a dedicated false-positive fixture proving it. The exemption is precise — broadening it to "any line containing `${CLAUDE_SKILL_DIR}`" would reintroduce token-less Class-2 false-negatives the spec forbids. If the real-tree `--audit` still trips on an unanticipated legitimate form, use the ignore-sentinel — do not broaden the detector.
- **The loud+distinct failure must be behaviorally testable, not grep-only (Task 2).** The headline value (a missing script fails loud and distinct, not silently into the synthesis-failure fallback) is encoded as a concrete shell guard emitting a distinct non-empty **stdout** sentinel (a channel the retained `2>/dev/null` does not suppress), with a behavioral verification that runs the guard against a nonexistent path. Residual: the main-agent's routing decision (treat the sentinel as distinct-failure vs fallback) remains partly prose-gated in protocol.md — but the detectable signal is now concrete, non-empty, and provably distinct from the empty-stdout condition that previously aliased to the fallback.
- **Parallel mirror-regen race — serialized, not deferred (Tasks 3/4/5a/5b/5c).** `just build-plugin` is a global `rsync --delete`; concurrent runs against the shared tree corrupt each other's mirror slice and the slice-scoped `git diff` checks become non-deterministic. The five mirror-touching tasks are chained via `Depends on` into a strict serial order (3→4→5a→5b→5c). Deferring mirror regen to a single final barrier task is rejected (memory `feedback_drift_hook_shared_checkout_coupling`: regen cannot be deferred to a final task; canonical+mirror commit together).
- **pr-review mechanism choice.** Body-propagation is chosen over the operator's earlier waived-CLI option: CLI adds an unprotected plugin↔wheel version-skew surface (pr-review has no CLI_PIN), a net-new wheel package, and a drift surface — for zero token saving and no reliability gain, and it does not fix the `2>/dev/null` swallow (the actual bug). Confirmed at the §4 approval surface.
- **Scope decision (full vs minimum).** This plan is full scope (recommended; satisfies AC4 in-lifecycle). Minimum scope runs Tasks 1–4 only and defers Phase 2 (Tasks 5a–8) to a new follow-up ticket — chosen at §4 approval.
