# Specification: investigate-and-standardize-path-resolution-for

> Background: `cortex/research/claude-skill-dir-fix/investigation.md` (prior bake-off) and this lifecycle's `research.md` (authoritative re-weigh). Revised after a critical-review pass that corrected the pr-review fix mechanism, decoupled acceptance from the to-be-built lint, and tightened the lint's invariant.

## Problem Statement

`${CLAUDE_SKILL_DIR}` is a Claude Code load-time substitution that resolves **only in a SKILL.md body** — not in reference files, the shell, YAML frontmatter, or composed subagent prompts. Plugin-distributed skills ship under a version-hashed cache path, so hardcoded and `~/.claude/skills/...` paths are also unavailable. The worst case is pr-review's Stage 3.5: `bash "${CLAUDE_SKILL_DIR:-$TMPDIR}/scripts/evidence-ground.sh" 2>/dev/null` — in a shell the var is unset, so the path resolves to `$TMPDIR/scripts/evidence-ground.sh` (nonexistent), the script never runs, and the empty/error result routes to the synthesis-failure fallback. The fallback **is announced** (Stage 5 prints a degraded-mode preamble), so the failure is not invisible at the output level — the defect is that **evidence-grounding is skipped on every faithful run, indistinguishable from a genuine Opus-synthesis failure**. Separately, raw `${CLAUDE_SKILL_DIR}` tokens and bare consult-references sit *inside* composed subagent prompts (a fresh subagent cannot resolve them), and bare relative skill-load paths resolve against **CWD, not the skill dir** (OQ3, verified live on Claude Code 2.1.168) so they break off-repo. The recurrence root cause: the authoring convention that prevented this (#010's `claude/reference/claude-skills.md` row) was deleted 2026-04-23 (`08d1102c`) and never relocated, with no lint enforcing it.

## Scope & Phasing

This is **one cohesive change**: a lint that enforces the path-resolution invariant can only land green after *every* offending site is fixed, so the lint and the source fixes are coupled and not independently deferrable. The work breaks into two phases, but **acceptance for each requirement is independently grep-checkable and does NOT route through the lint** (the lint is an additional cross-check, never the sole arbiter).

- **Phase 1: Close the silent harm + sibling Class-1 prompt bugs** — fix pr-review's evidence-grounding (correct the path so the script runs; preserve stderr→/dev/null JSON-parse hygiene; add an explicit script-resolvable loud check), inline the rubric/output-format content the Stage-4 subagent prompt currently only points at, and fix the two sibling raw-token bugs inside subagent prompts in critical-review and diagnose.
- **Phase 2: Anti-recurrence lint + Class-2/3 sweep + convention** — add the structural lint, fix every bare-relative skill-load site it is designed to catch, add a positive-routing CLAUDE.md principle, land ADR-0009.

**Priority (MoSCoW):** Phase 1 is **must-have** (closes the only failure that is both skipped-every-run and indistinguishable-from-noise). Phase 2 is **should-have** — it is AC4-mandated (the anti-recurrence convention), but if the operator elects the minimum at §4, Phase 2 ships as a **separate follow-up ticket**, NOT as a deferred-but-referenced phase of this lifecycle (so no Phase-1 acceptance depends on an unbuilt artifact). The §4 surface offers **full scope (both phases, recommended — satisfies AC4 here)** vs **minimum (Phase 1 only this lifecycle; AC4 deferred to a new ticket)**.

## Non-Requirements

- **No CLI subcommand for pr-review.** The operator waived pr-review's standalone contract, but the investigation found CLI adds a net-new `cortex_command/pr_review/` wheel package, an **unprotected** plugin↔wheel version-skew surface (pr-review has no MCP server / no CLI_PIN), and a new dual-source drift surface — for zero token saving and no reliability gain (the path, not a missing CLI, is the bug). Surfaced at §4 as the one mechanism divergence to confirm.
- **No load-time `!`-injection.** Abandoned; OQ4 (write-form + `$TMPDIR` parity) is rescoped out of the acceptance criteria (operator-approved at Research Exit Gate).
- **No deletion of the `2>/dev/null` stderr redirect** — it is required for main-agent JSON-parse hygiene; the fix targets the path, not the redirect.
- **No recreation of a standalone `claude/reference/` doc** (orphaning risk that lost the prior convention).
- **No change to `${CLAUDE_SKILL_DIR}` semantics or the harness; no change to main-agent-resolvable cache paths or dispatch prose; cortex-ui-extras and cortex-dev-extras are verified clean and out of scope.**

## Edge Cases

- **pr-review script path unresolved/missing** → the Req-1 pre-invocation check emits a distinct surfaced error, NOT the shared empty-stdout→synthesis-failure path (so a real failure is distinguishable from a normal zero-grounded-findings result, which returns non-empty JSON and is not a failure).
- **Lint false-positive on a legitimate body/main-agent token** → D1 is fence/`*-prompt.md`-scoped; a `${CLAUDE_SKILL_DIR}/references/foo.md` in a SKILL.md body or a `:-$TMPDIR` cache path in main-agent shell must pass.
- **Lint false-positive on a "do not load" citation** → D2 fires only in Read/execute contexts, not on markdown-citation links the prose marks as non-loadable.
- **Lint false-negative on token-less Class-2** → D2 catches bare `../`/`references/`/`skills/…|bash` in Read/execute contexts that carry no `${CLAUDE_SKILL_DIR}` token.
- **Prompt block without BEGIN/END fence** (Stage-3 critic prompts, whole-file `*-prompt.md`) → the fix adds fences / treats `*-prompt.md` as a whole-file prompt block so D1 has a boundary.
- **Fenced illustrative example of a bad form** → ignore-sentinel prevents self-tripping.
- **android-dev-extras hand-maintained** → fixed directly in `plugins/`; cortex-core-family in `skills/` (mirror regenerates).

## Proposed ADR

### Proposed ADR: 0009-skill-path-resolution-for-plugin-distributed-skills

Context: `${CLAUDE_SKILL_DIR}` resolves only in SKILL.md bodies; bare relative paths resolve against CWD (verified, 2.1.168); plugin skills ship under a version-hashed cache path. Decision: standardize the **invariant** — no reference file region that reaches a fresh subagent (a composed prompt) may carry a raw `${CLAUDE_SKILL_DIR}` token or a bare consult-reference, and no bare-relative path may appear in a Read/execute context; every such consumer receives the resolved value explicitly via body-propagation (resolve in the body; propagate the absolute path / inline reference content). The CLI mechanism is reserved for executable/state steps in skills already coupled to the cortex toolchain; it is **not** adopted for pr-review. Enforced structurally by a context-scoped lint, not prose alone. Rejected alternatives: pure-CLI for pr-review (new unprotected plugin↔wheel version-skew + wheel package + drift surface, zero gain, and does not by itself fix the path bug); load-time `!`-injection (unverified sandbox-write/`$TMPDIR`-parity, silent-failure); recreating a standalone `claude/reference/` doc (orphaning risk). Clears the three-criteria gate: hard-to-reverse (cross-skill convention + lint), surprising-without-context (the body-only substitution boundary and the CWD resolution of bare paths), real trade-off (CLI and injection considered and rejected with reasons).
