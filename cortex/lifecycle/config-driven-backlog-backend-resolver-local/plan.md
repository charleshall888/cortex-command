# Plan: config-driven-backlog-backend-resolver-local (#317)

## Overview
Introduce a config-authoritative backlog-backend seam: a `resolve_backlog_backend()` resolver (default `cortex-backlog`) + a graceful `cortex-read-backlog-backend` reader, a scaffolded `backlog:` config block, an in-process fail-closed overnight refusal guard, and inline per-consumer routing (local/none arms) across five interactive consumers. Every path is designed to *fail toward today's local behavior*, keeping a normal `cortex-backlog` repo byte-identical and off the token hot path.
**Architectural Pattern**: plug-in
<!-- Config string selects the active backlog backend; the LLM-plus-`instructions` is the adapter, with no per-tool typed client. The resolver is the selection seam; consumers route on the resolved key. -->

## Dual-source mirror discipline (applies to every task that edits `skills/` or `bin/`)
`just build-plugin` regenerates the `plugins/cortex-core/` and `plugins/cortex-overnight/` mirrors via a **per-skill** `rsync -a --delete` loop over the fixed SKILLS array (`rsync -a --delete skills/$s/ plugins/$p/skills/$s/`) plus one scoped `bin/` rsync (`justfile:616-627`) — `--delete` is confined to each per-skill (and the bin) subtree, not one whole tree. The pre-commit drift gate, however, diffs the **whole** `plugins/<plugin>/` working tree against the index (`.githooks/pre-commit:647`). Therefore, for every task below that edits a canonical `skills/<s>/…` or `bin/cortex-*` file: (1) run `just build-plugin`, (2) stage **both** the canonical and its regenerated mirror, and (3) co-land them in the SAME commit ([[feedback_drift_hook_shared_checkout_coupling]]). Because the rsync regenerates the shared mirror trees from current canonical, the Phase-3 consumer tasks are serialized by an explicit `Depends on` chain (Tasks 7→8→9→10) so this plan's own tasks cannot race each other's `build-plugin` runs — note this serializes only *this* plan's tasks, **not** the concurrent session (which is not bound by this graph). **Concurrent-session handling (corrected — the pathspec is NOT a foreign-mirror-dirt workaround):** an explicit `git commit -- <pathspec>` scopes only the *index*, so the concurrent session's foreign *staged non-mirror* files (e.g. `common.py`) don't leak into this commit ([[feedback_trunk_shared_index_concurrent_session]]) — that is its sole job here. It does **not** shield the commit from foreign *mirror* dirt, because the drift gate diffs the whole `plugins/<plugin>/` working tree vs the index regardless of the index pathspec (Risk #3). So before each Phase-3 commit, re-verify `git status plugins/<this-task's-plugin>/` shows only this task's mirror dirty; if the concurrent session has left foreign mirror dirt for the **same** plugin, there is no pathspec workaround — **halt and coordinate** (let the foreign session co-land its canonical+mirror, or wait) before committing. Plan-time reviewer check: the concurrent `harden-overnight-review` session is editing only `cortex_command/`+`tests/` (no swept skill tree), so `just build-plugin` currently produces zero foreign `plugins/` drift — the hazard is latent, not active, but re-verify per-commit since the foreign footprint can change.

## Outline

### Phase 1: Resolver + config seam + ADR (tasks: 1, 2, 3, 4, 5)
**Goal**: Stand up the resolver, register the `cortex-read-backlog-backend` console-script **with its first wiring reference in the same commit** (the name is "deployed" the moment the pyproject entry lands — see Task 2), deploy the graceful binstub + mirror, scaffold the `backlog:` block, and write ADR-0016.
**Checkpoint**: `resolve_backlog_backend` + `cortex-read-backlog-backend` resolve `cortex-backlog` for every degenerate input; scaffold reads back through the resolver as `cortex-backlog`; ADR-0016 present; `just test` green; the parity gate (no W003 orphan at the name-registration commit, no E002 drift) and the dual-source drift gate pass.

### Phase 2: Overnight safety guard (tasks: 6)
**Goal**: A structural, in-process, fail-closed first-check in `handle_prepare`/`handle_launch` that refuses any non-`cortex-backlog` backend via a JSON error envelope before any selection or bootstrap.
**Checkpoint**: a non-local backend makes both prep verbs return the `backend_not_supported` envelope + non-zero exit with `select_overnight_batch`/`bootstrap_session` never reached; an absent block proceeds; `just test` green.

### Phase 3: Consumer routing + degrade + criticality-feed decoupling + gate fail-safe (tasks: 7, 8, 9, 10, 11)
**Goal**: Inline local/none routing in the four remaining interactive consumers (refine, discovery, dev, morning-review), the refine criticality-feed decoupling (local byte-identical, with the seed→reconcile→gate invariant made explicit), the structural-consumer degrade, and the non-local §3b gate fail-safe that closes the resume-to-spec seed-tier hole. Tasks run **sequentially** (build-plugin serialization).
**Checkpoint**: each consumer carries a reachable local-default guard with a `none` arm; refine's local seed/reconcile stay Context-A byte-identical while the non-local branch omits `--backlog-slug` and passes Clarify's **computed** flags; the §3b gate refuses to skip-silent on a non-local seed-tier feature with `research.md` present; `just test` green; escalation lints pass.
