# Specification: wire-cortex-corelifecycle-wontfix-invocation-and (#329)

## Problem Statement

`/cortex-core:lifecycle` advertises reserved-first-word and phase-first invocation forms (`wontfix <slug>`, `resume <feature>`, `complete <slug>`, and bare `<phase>`) across multiple prose surfaces — the frontmatter `argument-hint`, the Invocation block (SKILL.md:24–26), the honor-phase line (SKILL.md:169), Step 1's parse, and `complete.md` — but Step 1 actually parses *first-word-as-feature* with no reserved-word handling. The advertised forms were never implemented, so **all four are broken**: `wontfix <slug>` → `feature="wontfix"`, `resume <feature>` → `feature="resume"`, `complete <slug>` → `feature="complete"`, and bare `<phase>` (e.g. `plan`) → `feature="plan"` (a phantom lifecycle). The grammar is prose-specified across drifted surfaces with no test, so the breakage went unnoticed. Separately, the wontfix terminal-state workflow (`references/wontfix.md`) enforces a load-bearing 3-step order (archive → `feature_wontfix` append → backlog update) by prose only. This feature **structuralizes the invocation grammar** (a tested parser CLI) and **the wontfix ordering** (an order-enforcing verb), and adds a **docs-derived drift-guard** so the grammar cannot silently drift from the docs again. The routing *dispatch* that consumes the parse result remains model-executed SKILL.md prose (as all skills are), but is reduced to a thin act-on-`mode` table and guarded by a mode-coverage test + the drift-guard — so this is a substantial reduction of the prose surface, not a full elimination.

## Phases

- **Phase 1: Structural invocation parser + reserved-sub-command routing** — replace Step 1's prose parse with a tested `cortex-lifecycle-parse-args` helper owning the full grammar, wire `resume`/`complete <slug>`/bare-phase-token behavior, reconcile every advertised surface to match the parser, and add a docs-derived drift-guard + a mode-coverage test.
- **Phase 2: Wontfix order-enforcing verb + reference thinning** — add `cortex-lifecycle-wontfix`, wire the `wontfix` route to it, thin `references/wontfix.md`, and update the events-registry/ADR/mirror surfaces.

## Priority (MoSCoW)

All requirements are **Must-have** unless tagged, and that is not laziness: each Must maps to a broken advertised form being fixed (Reqs 4/5/6/7/16), the structural parser + recurrence guard the scope was chosen for (Reqs 1/2/3/8/9/10), the order-enforcing verb and its correctness/safety contract (Reqs 11/12/13/14/15), or a pre-commit gate that would otherwise block the commit (Req 18). Genuine **Should-haves** (valuable, non-blocking) are tagged inline: the `feature_wontfix` producer-column `scan_coverage` flip to `manual` (Req 18 — hygiene, not gate-required). **Won't-do (this ticket)** items are the explicit boundaries in `## Non-Requirements` (chiefly bare-phase-token active-feature routing).

11. **Order-enforcing verb exists**: a new `cortex_command/lifecycle/wontfix_cli.py` module + console-script `cortex-lifecycle-wontfix = "cortex_command.lifecycle.wontfix_cli:main"`, argparse with a **positional `slug`** + `--reason` + optional `--backlog-slug` (no required flags — contract-lint-clean). It performs, as sequential gated statements in one function: (a) archive-move, (b) byte-faithful `feature_wontfix` append, (c) backlog terminalization. Acceptance: `cortex-lifecycle-wontfix --help` exits 0; `python3 -m pytest cortex_command/lifecycle/tests/test_wontfix_cli.py` exits 0. **Phase**: Wontfix order-enforcing verb + reference thinning.

12. **Archive-move uses `os.rename`, named main-root resolver, mandatory pre-flight guard**: step (a) is `os.rename` (fallback `shutil.move`) of `cortex/lifecycle/<slug>` → `cortex/lifecycle/archive/<slug>`, anchored to the **canonical main repo root via `cortex_command.common._resolve_user_project_root()`** (the env-aware, `.git`-precedence resolver canonicalized in #319/#311 — NOT the CWD-following `_resolve_user_project_root_from_cwd`). The 4-case existence guard runs first: dst-exists-not-src → skip step (a) (no-op-if-archived); src-not-dst → move; both-exist → error (non-zero, no nesting); neither → unknown-slug error. Acceptance: tests cover (i) an **untracked** lifecycle dir archives successfully (the case `git mv` fails on), (ii) re-running on an already-archived slug is a clean no-op, (iii) both-exist errors without nesting, and (iv) **invocation from inside a git worktree resolves to the main-resident dir, not a worktree-local copy**. `pytest … -k archive` exits 0. **Phase**: Wontfix order-enforcing verb + reference thinning.

13. **Byte-faithful `feature_wontfix` row via atomic append**: step (b) emits `{"ts": "<…Z>", "event": "feature_wontfix", "feature": "<slug>", "reason": "<reason>"}` using default `json.dumps` separators `(", ", ": ")`, insertion-ordered keys, and a `Z`-suffixed timestamp (`strftime("%Y-%m-%dT%H:%M:%SZ")`), written to the **archived** events.log via the reused atomic-append discipline (`flock` + tempfile + `os.replace`, parametrized with an archive-aware path) — NOT `lifecycle_event.py:log_event`, NOT a bare `printf >>`. Step (b) is idempotent (append only if no `feature_wontfix` row present). Acceptance: a test asserts **semantic** parse-compatibility — the emitted row feeds through `cortex_command.common.detect_lifecycle_phase` to `phase=complete` AND matches the `claude/statusline.sh` grep patterns (NOT byte-golden, which would fail on a legitimate field add); a companion test asserts the row equals the documented template for the current field set as a regression check. **Phase**: Wontfix order-enforcing verb + reference thinning.

14. **Backlog terminalization with a real producer, verified, slug-validated**: step (c) updates the backlog item to `--status wontfix --lifecycle-phase wontfix --session-id null`. The backlog target is derived **from `cortex/lifecycle/<slug>/index.md`'s `parent_backlog_id`/`parent_backlog_uuid` (read before the move)** — a deterministic producer `cortex-update-item` resolves reliably — with `--backlog-slug` as an operator override. If `index.md` is absent or its parent fields are null (an ad-hoc lifecycle with no backlog item), step (c) is a **documented no-op** (nothing to terminalize), not an error. The verb **verifies step (c) actually terminalized an item** (or was a deliberate no-op) and surfaces a **zero-match** as non-zero rather than silent success. The verb validates `slug` against `^[a-z0-9]+(-[a-z0-9]+)*$` before any filesystem op (path-traversal guard for direct callers). Exit codes: `0` success; `2` ambiguous backlog resolution (propagated; candidates to stderr) and argparse usage; `1` move/IO failure or zero-match-where-a-parent-was-expected; stderr prefixed `cortex-lifecycle-wontfix:`. Acceptance: tests cover (i) terminalization via index.md parent id, (ii) a `..`/`/`-bearing slug rejected pre-move, (iii) ambiguous-slug exit-2 propagation, (iv) a no-parent/ad-hoc lifecycle no-ops step (c) cleanly, (v) a zero-match surfaces non-zero. **Phase**: Wontfix order-enforcing verb + reference thinning.

15. **Fail-forward recovery re-asserts each postcondition idempotently**: the verb is fail-forward (never rolls back the move). On re-invocation it independently re-asserts **each** step's postcondition — move if not yet archived (Req 12 guard), append if no `feature_wontfix` row (Req 13 idempotency), terminalize if the backlog item is not yet terminal (Req 14) — rather than "retrying only step (c)." So an (a)-success/(b)-failure partial run is fully repaired on re-run. Acceptance: a test simulates a step-(b) failure after a successful move, then a re-invocation, and asserts the `feature_wontfix` row is present and the backlog item terminal afterward. **Phase**: Wontfix order-enforcing verb + reference thinning.

## Non-Requirements

- **Bare-phase-token active-feature routing** is NOT implemented. Routing `plan`/`review`/`research`/`specify`/`implement` (no explicit feature) to "the active feature" needs an active-feature concept that does not exist today; #329 only makes them non-broken (feature-required fallback) and recognizes them in the grammar. The full routing is deferred to a **cross-linked sibling ticket filed at completion**. Because of this, line 25 and line 169 are reworded to advertise the implemented behavior, not the deferred routing (Req 9).
- **End-to-end testing of the model's prose dispatch** is out of scope (and not feasible): SKILL.md's act-on-`mode` branches are executed by the model, not by Python. The structural guards are the parser unit tests (Reqs 1–3), the verb tests (Reqs 11–15), the mode-coverage proxy (Req 10), and the drift-guard (Req 8) — together they bound the failure surface; they do not eliminate the residual prose-dispatch risk.
- **No backlog status-vocabulary change**: `wontfix` is already a `TERMINAL_STATUS`; `feature_wontfix` is already registered. No new event names, no new statuses.
- **No new `AskUserQuestion` / operator pause** (so no `kept-pauses.md` / parity change); the exit-2 ambiguity remains a halt-and-surface.
- **No `bin/cortex-*` shell wrapper** for either new console-script (the 5 modern lifecycle console-scripts have none); no `bin/`-mirror (Python is wheel-only).
- **Not a redesign of the lifecycle state machine, event log, or phase detection** — confined to the invocation front door and the wontfix terminal workflow.

## Edge Cases

- **Untracked lifecycle dir** (premise rejected before any commit — the dominant wontfix case): `os.rename` archives it successfully (the empirically-confirmed case where `git mv` exits 128). Req 12.
- **Already-archived slug** (re-invocation / partial prior run): 4-case guard → clean no-op, no nesting, no duplicate row. Reqs 12/13/15.
- **`#`-sigil collision**: `#wontfix` is a **literal feature id/slug** (the `#` suppresses reserved-matching, consistent with the sigil's documented "this is an id" semantics) — the documented mechanism ruling out a feature literally named `wontfix`/`resume`/a phase token. Req 3.
- **Reserved word with no second word**: `wontfix`/`resume`/`complete` alone → error asking for the target; never creates a feature named after the reserved word.
- **Path-traversal slug** via direct console-script call: rejected by the verb's own slug validation before any filesystem op. Req 14.
- **Ad-hoc lifecycle with no backlog item** (`index.md` absent or null parent): step (c) is a clean documented no-op. Req 14.
- **Backlog resolution finds nothing where a parent was expected**: surfaced as non-zero, not silent success. Req 14.
- **Concurrent-session zombie resurrection** (low-probability): a concurrent session writing to the active events.log path after the move can recreate the dir (`lifecycle_event.py:88` `mkdir(exist_ok=True)`), resurfacing the lifecycle without a marker. Documented limitation; wontfix is operator-initiated and rare, so no `.session`-liveness gate is added in #329.
- **`wontfix` mis-routed to Step-2 fall-through** (prose-dispatch risk): partially mitigated — on a wrongful fall-through, Step 2 finds the dir already moved; and the verb's no-op-if-archived guard prevents a re-run from re-archiving or duplicating the row.

## Technical Constraints

- `feature_wontfix` row consumers are order/whitespace/extra-field tolerant (`common.py:_detect_lifecycle_phase_inner` json.loads + `.get`; `statusline.sh` grep). The test contract asserts **semantic** parse-compatibility, not bytes. Bash/Python phase-detection parity (`tests/test_lifecycle_phase_parity.py`) must stay green.
- **The routing dispatch is model-executed prose** (SKILL.md). This is a structural limit, not an omission: no Python test can assert the model correctly executes "on mode=wontfix, run the verb and halt." The spec mitigates with the parser tests, the mode-coverage proxy (Req 10), and the verb's own idempotent/no-op safety net — but the residual risk is acknowledged, not eliminated.
- Editing the editable-installed `cortex_command` package → **sequential/trunk dispatch, not a worktree** (`just test` runs the editable install).
- Console-scripts must be argparse-with-positional (no required flags) to stay contract-lint-clean (E101/E103), or carry a `bin/.contract-lint-exceptions.md` row if hand-rolled.
- Parity (W003): the new console-scripts must be referenced from an in-scope `skills/**/*.md` file; the wiring rides in the same commit as the `skills/*` edit (pyproject is not a parity trigger).
- SKILL.md size cap 500 (currently 183) and L1 ratchet (`lifecycle` cluster budget 890, frontmatter-only) are not binding — keep edits in the body.
- Atomic-append discipline (`flock` + tempfile + `os.replace`) per `pipeline.md:126`. Main-root resolution via `cortex_command.common._resolve_user_project_root()`.

## Proposed ADR

### Proposed ADR: 0018-structural-lifecycle-invocation-grammar

**Context**: `/cortex-core:lifecycle` advertised reserved-first-word and phase-first invocation forms across multiple prose surfaces that drifted out of sync with the actual Step-1 parse, leaving four advertised commands (`wontfix`/`resume`/`complete <slug>`/bare `<phase>`) silently broken with no test to catch it. **Decision**: make a small `cortex-lifecycle-parse-args` CLI the single, unit-tested source of truth for the invocation grammar (reserved verbs + phase tokens + feature/phase, with reserved/phase-first forms inverting the default `<feature> [phase]` order), and add a **docs-derived** drift-guard that scrapes advertised forms from the live doc bytes and asserts the parser classifies each correctly (with a bidirectional negative control). **Trade-off**: rejected the lower-cost prose reserved-word table (A1) because the already-broken `complete <slug>` is direct evidence prose-only first-word parsing here has already drifted — the failure CLAUDE.md's "prefer structural over prose-only for sequential gates" warns against. **Scope honesty**: the structural parser closes the **doc↔parser** drift class; the **parse↔dispatch** glue (SKILL.md acting on `mode`) remains model-executed prose, reduced to a thin act-on-`mode` table guarded by a mode-coverage proxy — not eliminated. The decision is hard to reverse once docs/tests depend on the parser, surprising without context (the feature/phase inversion + a separate parse CLI + a docs-scraping test), and a real trade-off (A1 vs A3) — clearing the ADR three-criteria gate.
