# Specification: reconcile-sessionstart-lifecycle-phase-summary-against

## Problem Statement

The SessionStart additionalContext lifecycle-summary block (emitted by `cortex_command/hooks/scan_lifecycle.py`) advertises each incomplete lifecycle's phase from artifact-presence inference and a single events.log rung, but does not cross-check against backlog frontmatter `status:`. Two recent operator-visible divergences (#075: shipped feature still advertised as "Review"; #209: paused-mid-implement feature still advertised as "Review") routed the operator onto wrong work paths. This spec makes the summary derive phase from on-disk truth — including the `feature_paused` signal currently invisible to the canonical detector — and surface terminal-vs-non-terminal disagreement between events-derived phase and backlog status with a structurally-distinct `[mismatch: backlog=<status>]` annotation, so the operator can decide work paths from accurate state.

## Phases

- **Phase 1: Detector extension + downstream ripple updates** — extend `_detect_lifecycle_phase_inner` to recognize `feature_paused` and emit `{phase}-paused` compound values; update `_encode_phase` so `implement-paused` carries `checked/total`; render via `_phase_label`; update the four closed-set consumers (statusline bash mirror, dashboard slow-flag classifier, lifecycle SKILL.md routing table, dashboard template); update parity tests.
- **Phase 2: SessionStart reconciliation** — load `cortex/backlog/index.json` once per hook invocation; cross-check events-derived phase against backlog `status:` using the terminal-vs-non-terminal asymmetry rule; render `[mismatch: backlog=...]` annotation on the matching entries (both active-feature header and others enumeration).
- **Phase 3: Cap mitigation + session-bound diagnostic** — sort mismatch-bearing entries first; truncate non-mismatch tail with `… +N more` under a 9,000-char soft budget; append `mismatches: N total` fragment to the section header; emit structured per-lifecycle diagnostics to a session-bound JSONL log so future divergences are debuggable.

## Requirements

1. **Detector recognizes `feature_paused`**: When the most recent significant event in `cortex/lifecycle/{feature}/events.log` (where significant = `phase_transition`, `feature_complete`, `feature_wontfix`, `feature_paused`) is `feature_paused`, `_detect_lifecycle_phase_inner` in `cortex_command/common.py` returns the phase that would otherwise be derived from the remaining ladder (typically `implement` or `review` from plan.md tally) with the suffix `-paused` appended (e.g., `implement-paused`, `review-paused`). When `feature_paused` is followed by a more recent `phase_transition`, the resume is the most recent significant event and the suffix is NOT applied. **Acceptance**: `python3 -m pytest tests/test_lifecycle_phase_parity.py -k paused` exits 0; pass if exit code = 0. **Phase**: Phase 1: Detector extension + downstream ripple updates

2. **`_encode_phase` carries progress through `-paused` suffix**: `_encode_phase` in `cortex_command/hooks/scan_lifecycle.py` (currently lines 60-74, using `if phase == "implement":` exact-equality) is widened so that `implement-paused` inputs also append the `:checked/total` payload, producing wire-format `implement-paused:3/5`. Equivalent handling for `review-paused` (no progress payload needed; wire-format stays bare). The acceptance is the wire-format round-trip: `_encode_phase("implement-paused", 3, 5, 0)` returns `"implement-paused:3/5"`. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k encode_paused` exits 0; pass if exit code = 0. **Phase**: Phase 1: Detector extension + downstream ripple updates

3. **`_phase_label` renders `-paused` suffix**: `_phase_label` in `cortex_command/hooks/scan_lifecycle.py` (currently lines 77-129) recognizes the `-paused` suffix on any phase value and renders as `{Base Phase Label} — paused` (em-dash + space + "paused"). Compound wire-format forms (e.g., `implement-paused:3/5` per R2) render as `Implement (3/5 tasks done) — paused`. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k label_paused` exits 0; pass if exit code = 0. **Phase**: Phase 1: Detector extension + downstream ripple updates

4. **Statusline bash mirror updated to match**: `claude/statusline.sh`'s phase-detection ladder (currently lines 395-454) gains a `feature_paused` rung consistent with R1 so that the bash mirror computes the same `{phase}-paused` value as the Python detector. The cross-language parity test at `tests/test_lifecycle_phase_parity.py:282` (`test_statusline_ladder_matches_canonical`) continues to pass for both pre-existing fixtures and the new paused fixtures added in R5 below. **Acceptance**: `python3 -m pytest tests/test_lifecycle_phase_parity.py::test_statusline_ladder_matches_canonical` exits 0; pass if exit code = 0. **Phase**: Phase 1: Detector extension + downstream ripple updates

5. **Parity test coverage for paused**: `tests/test_lifecycle_phase_parity.py` adds at least two fixtures under the fixture directory it iterates — (a) `events.log` ending with `feature_paused` while `plan.md` has unchecked tasks → expects `implement-paused` (wire `implement-paused:X/Y`); (b) `events.log` ending with `feature_paused` while `plan.md` has all tasks checked → expects `review-paused`. **Acceptance**: `grep -c "paused" tests/test_lifecycle_phase_parity.py` ≥ 2 AND the fixture directory contains at least one `paused` subdirectory. **Phase**: Phase 1: Detector extension + downstream ripple updates

6. **Dashboard slow-flag classifier widened to recognize `-paused`**: `cortex_command/dashboard/data.py:1191` and `:1197` currently use exact-string membership (`if current_phase in ("implement", "implement-rework"):` and `elif current_phase == "review":`). These are widened so `implement-paused`/`implement-rework-paused`/`review-paused` are classified into the same slow-flag bucket as their non-paused counterparts (`implement-paused` joins the `implement` bucket; `review-paused` joins the `review` bucket). The dashboard template `cortex_command/dashboard/templates/feature_cards.html` (raw `{{ current_phase }}` interpolation at lines 62 and 110) is updated to render the value via the same human-readable label rule as `_phase_label` — either by hoisting `_phase_label` into a shared helper consumed by Jinja, or by mirroring the `-paused` suffix rule in a template filter. The `phase_order` list at line 3 of the template is reviewed to confirm it does not need to enumerate `-paused` values (it is a sort key, not a render set — confirm no consumer breaks on unknown values). **Acceptance**: `python3 -m pytest tests/test_dashboard_data.py -k phase_paused` exits 0 (or, if no test file exists for dashboard slow-flag yet, a new one is added) AND `grep -n "paused" cortex_command/dashboard/data.py` returns ≥ 1 line under the slow-flag classifier. **Phase**: Phase 1: Detector extension + downstream ripple updates

7. **Lifecycle SKILL.md routing table updated to recognize `-paused`**: `skills/lifecycle/SKILL.md`'s phase routing table (currently lines 93-96, listing 8 phase values) is updated to include the `-paused` variants. The semantics: a `*-paused` phase routes to the same phase reference document as its non-paused base (e.g., `implement-paused` routes the same as `implement` — re-enter Implement to address the pause), but the user-facing announcement names the pause. Implementation choice between two equivalent approaches: (a) enumerate `implement-paused`, `implement-rework-paused`, `review-paused` as explicit routing-table rows; OR (b) document a prose rule "strip `-paused` suffix when looking up routing target, but display the full label". Either choice is acceptable; the parity test at `tests/test_lifecycle_kept_pauses_parity.py` continues to pass. **Acceptance**: `grep -c "paused" skills/lifecycle/SKILL.md` ≥ 1 AND `python3 -m pytest tests/test_lifecycle_kept_pauses_parity.py` exits 0. **Phase**: Phase 1: Detector extension + downstream ripple updates

8. **Index.json loaded once per hook invocation**: `cortex_command/hooks/scan_lifecycle.py:main()` reads `cortex/backlog/index.json` once before the per-candidate phase-detection loop and builds an in-memory `dict[lifecycle_slug → status]` map keyed by the `lifecycle_slug` field (skipping entries where `lifecycle_slug` is null/absent). When the file is absent, unreadable, or unparseable, the map is the empty dict, a structured diagnostic is emitted (per R13), and the hook continues normally — no mismatch annotations are emitted for any entry. When duplicate `lifecycle_slug` values appear in index.json, the first occurrence wins and a duplication diagnostic is emitted. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k index_json` exits 0; pass if exit code = 0. **Phase**: Phase 2: SessionStart reconciliation

9. **Terminal-vs-non-terminal mismatch rule**: For each candidate lifecycle whose slug is found in the index.json map, the generator computes `events_terminal = (phase in {"complete", "escalated"} OR phase == "complete:awaiting-merge")` and `backlog_terminal = (status in TERMINAL_STATUSES from cortex_command.common)`. When `events_terminal XOR backlog_terminal` is true, the entry carries a `[mismatch: backlog={status}]` annotation. Otherwise, no annotation is appended (the entry renders as today). When the slug is absent from the map, no annotation is appended. The `*-paused` phase values are non-terminal for this purpose (they compose with non-terminal base phases). **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k terminal_mismatch` exits 0; pass if exit code = 0. **Phase**: Phase 2: SessionStart reconciliation

10. **Active-feature header carries the annotation**: When the active feature (the one being resumed, rendered by the active-feature header at scan_lifecycle.py:405-413) is itself mismatched per R9, the `[mismatch: backlog={status}]` annotation appears on the active header line in addition to (or in place of, when the entry is suppressed from the "Other" list per current behavior) the others enumeration. The text returned by `_interrupted_hint()` continues to be driven by the events-derived phase (not the backlog status) — the hint is operator-actionable resume guidance grounded in lifecycle state, not backlog metadata. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k active_header_mismatch` exits 0; pass if exit code = 0. **Phase**: Phase 2: SessionStart reconciliation

11. **Mismatch-first sort**: The "Other incomplete lifecycles" enumeration is sorted with mismatch-bearing entries first, then non-mismatch entries. Within each group, the existing scan-order is preserved (stable sort). **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k mismatch_first_sort` exits 0; pass if exit code = 0. **Phase**: Phase 3: Cap mitigation + session-bound diagnostic

12. **Soft-budget truncation of non-mismatch tail**: When the fully-assembled additionalContext block (active header + pipeline-state prepend + others enumeration + diagnostic summary) would exceed **9,000** characters, the generator truncates non-mismatch entries from the bottom of the "Other" enumeration with a final line `  … +N more` where N is the count of dropped entries. The 9,000 threshold reserves ~1,000 chars over the measured ~700-char real overhead (pipeline_state.context_string ≤120 chars per `_pipeline_state.py:162,199-202`; active header + interrupted hint <500 chars per scan_lifecycle.py:406-428; metrics summary <100 chars), giving meaningful headroom for prepend growth without truncating early. Mismatch-bearing entries are never truncated; if the mismatch-only block alone exceeds the soft budget, no truncation is performed (losing a mismatch defeats the point) and Claude Code's 10K hard cap may fire as the failure mode of last resort. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k soft_budget_truncation` exits 0; pass if exit code = 0. **Phase**: Phase 3: Cap mitigation + session-bound diagnostic

13. **`mismatches: N total` header fragment**: The "Other incomplete lifecycles" section header line carries a trailing ` — mismatches: N total` fragment when N ≥ 1, where N is the total mismatch count BEFORE any truncation. When N = 0, the fragment is omitted (current header preserved). This makes the mismatch count visible even when individual mismatch annotations are dropped by Claude Code's 10K hard cap. **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k mismatches_header_fragment` exits 0; pass if exit code = 0. **Phase**: Phase 3: Cap mitigation + session-bound diagnostic

14. **Session-bound JSONL diagnostic**: For each candidate lifecycle considered in `main()`, scan_lifecycle.py appends one single-line JSON object to `cortex/lifecycle/sessions/${LIFECYCLE_SESSION_ID}/scan-lifecycle-diag.jsonl` (the sessions/ directory is the documented session-bound write surface). The JSON object contains at minimum: `ts` (ISO 8601), `feature` (slug), `decision` ("included" / "excluded"), `exclude_reason` (when excluded — e.g., "stale", "morning_review"), `latest_event_ts` (ISO 8601 or null), `threshold_days` (int), `last_event` (event name or null), `events_phase` (resolved phase or null), `backlog_status` (resolved status from index.json or null), `index_json_resolved` (bool), `mismatch` (bool). The file is created if absent (parent dir is created with `mkdir -p` semantics if absent — sessions/ may not yet exist). Writes are best-effort (try/except wrapped); IO failures (disk full, permission denied) do not interrupt the hook. The destination is operator-reachable — `tail -f cortex/lifecycle/sessions/<id>/scan-lifecycle-diag.jsonl` shows the diagnostic stream in real time, and the file persists for post-mortem inspection. When `$LIFECYCLE_SESSION_ID` is unset (legacy fallback or hook misconfiguration), the diagnostic is silently dropped (no fallback to stderr — stderr from hooks is `silent by design` per docs/overnight-operations.md:570). **Acceptance**: `python3 -m pytest tests/test_hooks_scan_lifecycle.py -k session_diagnostic` exits 0 AND a test fixture verifies the JSONL file is created and contains the expected fields. **Phase**: Phase 3: Cap mitigation + session-bound diagnostic

15. **Reconciliation regression fixtures**: New fixtures under `tests/fixtures/hooks/scan_lifecycle/` cover (a) the #075-shape case: lifecycle dir with `plan.md` and no `feature_complete` event, backlog `status: complete` → mismatch fires; (b) the #209-shape post-fix case: lifecycle dir with `events.log` ending in `feature_paused` while `plan.md` has unchecked tasks, backlog `status: in_progress` → phase renders as `Implement (X/Y tasks done) — paused`, NO mismatch (both non-terminal); (c) clean alignment case: lifecycle in `implement` with backlog `in_progress` → no annotation. **Acceptance**: `ls tests/fixtures/hooks/scan_lifecycle/ | grep -E '075|209|clean'` returns ≥ 3 lines (or equivalent fixture naming); fixtures are referenced from at least one passing test. **Phase**: Phase 3: Cap mitigation + session-bound diagnostic

## Non-Requirements

- Does NOT modify backlog `status:` field — observability surface remains read-only per `observability.md:93`.
- Does NOT introduce a 4th lifecycle-slug → backlog-file resolver. Consumes the pre-existing `cortex/backlog/index.json` (regenerated by the pre-commit hook).
- Does NOT change the contract of `_OVERNIGHT_TO_BACKLOG` in `outcome_router.py` or `normalize_status` in common.py.
- Does NOT add a `feature_wontfix` rung to the detector — it is already rung 1 of the existing ladder, and `wontfix` is in `TERMINAL_STATUSES`, so the terminal-vs-non-terminal rule already handles it.
- Does NOT fix arbitrary non-terminal status drifts (e.g., `events=plan` + `backlog=backlog` where status wasn't bumped on lifecycle start). The mismatch rule is terminal-vs-non-terminal asymmetry only; non-terminal drifts are tolerated. This scope was confirmed on critical-review re-check against the ticket's literal Role example.
- Does NOT address upstream invariant gaps (sibling work — `complete.md`/`wontfix.md`/review handlers should reliably sync backlog status). File as a separate backlog item if desired.
- Does NOT attempt to root-cause the #075 staleness-filter bypass — the original lifecycle dir was cleaned and the artifact is unrecoverable. The R14 session-bound diagnostic makes any future bypass debuggable instead.
- Does NOT change Claude Code's 10,000-character hard cap behavior (out of our control). R12 and R13 mitigate; the hard cap is a documented failure mode.
- Does NOT emit hook diagnostics to stderr. Per `docs/overnight-operations.md:570`, hook stderr is "silent by design" and has "no log mechanism" — emitting per-candidate JSON to stderr would either be silently dropped (defeating the debuggability purpose) or surface as user-visible noise (a UX regression). R14's session-bound JSONL file is the correct destination.

## Edge Cases

- **index.json absent or unparseable**: empty map, R14 diagnostic emitted, hook continues without any mismatch annotations. No false positives, no failure.
- **Duplicate `lifecycle_slug` in index.json**: first occurrence wins; R14 duplication diagnostic emitted.
- **Lifecycle dir exists but no backlog item** (ad-hoc lifecycle): index.json lookup misses; entry renders without annotation. Today's behavior preserved.
- **Backlog item exists but `lifecycle_slug` field is null**: not in the lookup map; entry renders without annotation. Reconciliation is a privilege that requires the link to be present.
- **`feature_paused` with no prior phase context**: detector falls through to artifact-presence rungs (no prior phase to compose with); `-paused` suffix is NOT applied (no `null-paused` value).
- **`feature_paused` followed by a `phase_transition` (resume)**: phase_transition is the most recent significant event; `-paused` is NOT applied.
- **`complete:awaiting-merge` (events-phase) + backlog `in_progress`** (PR open, not merged): `complete*` is events-terminal per R9; `in_progress` is non-terminal → mismatch fires. Correct — operator should know the PR is awaiting merge.
- **Mismatch-only block exceeds soft budget**: R12 explicitly allows the hard cap to be the failure mode of last resort. R13's header fragment still conveys the count.
- **R14 write fails (disk full, permission denied)**: emission is best-effort; silently dropped; hook continues. The diagnostic file may have gaps under disk pressure; this is acceptable degradation.
- **`$LIFECYCLE_SESSION_ID` unset**: R14 diagnostic is silently dropped (no stderr fallback per Non-Requirements). The hook still produces the additionalContext block normally.
- **`cortex/backlog/index.json` regenerated mid-hook** by another process: stale read is acceptable; the next invocation picks up the new state.
- **`-paused` phase value reaches a consumer that wasn't widened in Phase 1**: by R6/R7/R4 the four known consumers (dashboard, lifecycle skill, statusline, parity test) are widened. Any future consumer added must widen in lockstep — this is now a Changes-to-Existing-Behavior pattern, not a one-off.

## Technical Constraints

- **Latency**: The SessionStart hook blocks Claude Code session setup. `observability.md:91` gives the statusline a `<500ms` budget; SessionStart has no published budget but the same user-perceived-blocking discipline applies. The implementation PR must capture a microbenchmark: run scan_lifecycle.py end-to-end with a synthetic fixture set of N=90 candidate lifecycles (mix of mismatched, aligned, paused) and report wall-time p50/p99. There is no fixed numeric target; instead, the PR description must include the measurement and a Plan-phase decision rule (e.g., "if p99 > 800ms, batch the index.json read with the existing events.log reads"). This replaces the prior decorative "≤50ms additional vs. baseline" target which named no baseline and accounted for no per-candidate cost.
- **Sandbox**: scan_lifecycle.py runs under the standard hook sandbox; reads under `cortex/`; writes only to stdout (additionalContext JSON envelope) and to `cortex/lifecycle/sessions/${LIFECYCLE_SESSION_ID}/scan-lifecycle-diag.jsonl` (already an allowed write path per the existing session-write surface). No new sandbox grants required.
- **Fail-open**: Every new read must fail-open (try/except wrapped, never raises into the hook's stdout path). Every new write (the R14 JSONL append) must fail-open too. The hook must always exit 0 (per the existing pattern at scan_lifecycle.py:533-539, 663-664, 721-726, 754-771).
- **No hook stderr**: Per `docs/overnight-operations.md:570`, hooks "have no log mechanism" and stderr is "silent by design". The diagnostic destination is the session-bound JSONL file (R14), not stderr. Future work that wants operator-visible diagnostics must either land them in the session-bound log or build a separate user-surface (e.g., `cortex overnight diag scan-lifecycle`).
- **10K char cap**: Claude Code's `additionalContext` hard cap is 10,000 characters. Over-cap silently swaps the entire block for a file-pointer preview. R12 (soft-budget truncation at 9,000) and R13 (header-fragment count) mitigate; the hard cap is the failure mode of last resort.
- **Events-log schema legacy-tolerance**: `events.log` readers MUST tolerate the absence of `schema_version` per the historical-compat doctrine in `project.md:37-38`. The `feature_paused` rung addition follows the existing event-name substring detection pattern (no new schema requirement).
- **Shared infrastructure with closed-set consumers**: The detector's `phase` return value space is consumed by at least five closed-set consumers (lifecycle SKILL.md routing table at lines 93-96; statusline.sh phase ladder at lines 395-454; dashboard data.py:1191,1197 slow-flag classifier; dashboard templates phase rendering; `_encode_phase`/`_phase_label` in scan_lifecycle.py). Adding a phase value is NOT additive — it requires a lockstep update at each consumer. R2/R4/R6/R7 codify this; future detector changes must follow the same pattern.
- **Index.json freshness**: `cortex/backlog/index.json` is regenerated by the pre-commit hook (`cortex-generate-backlog-index`). A stale index is acceptable — it only affects the mismatch annotation for newly-renamed `lifecycle_slug` values until the next commit. The next pre-commit regenerates it.
