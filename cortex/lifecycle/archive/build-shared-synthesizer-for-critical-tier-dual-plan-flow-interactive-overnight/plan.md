# Plan: build-shared-synthesizer-for-critical-tier-dual-plan-flow-interactive-overnight

## Overview

Bottom-up shared-artifact-first build: prompt fragment + event constants + config gate first, then the gate-read helper, then the two surface wirings (§1b interactive, Step 3b overnight) which fold in v2 schema emission, then the `runner.py` circuit breaker, then four targeted test files. Edits to dual-source files go to canonical `skills/...` paths; after editing, run `just build-plugin` to regenerate the working-tree mirror under `plugins/cortex-interactive/...`, then stage both canonical and regenerated mirror in the same commit. The `.githooks/pre-commit` Phase 4 drift detector rejects commits where the staged mirror diverges from the build output (it does NOT auto-stage), so the staging step is mandatory. Phase 1.5's `bin/cortex-check-parity` is a separate SKILL.md-to-bin token-reference linter — distinct from Phase 3-4's build-plugin/drift mechanism.
