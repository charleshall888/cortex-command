# Specification: verify-claude-reference-md-conditional-loading-behavior-under-opus-47

> Epic reference: [research/opus-4-7-harness-adaptation/research.md](../../research/opus-4-7-harness-adaptation/research.md). This spike resolves OQ5 from that epic; the deliverable gates DR-1's firmness and feeds ticket #085's scope.

## Problem Statement

Epic #82 contracted its scope to a prompt-delta audit (DR-1) provisional on OQ5: whether Opus 4.7's stricter instruction-following changes the conditional-loading behavior of five reference files referenced by `~/.claude/CLAUDE.md`'s natural-language conditional table. Without resolving OQ5, ticket #085 cannot scope its remediation surface. This spike produces a report at `research/opus-4-7-harness-adaptation/reference-loading-verification.md` answering, per file: (Q1) does it load when its trigger fires under 4.7, (Q2) does 4.7's stricter following change its behavior, (Q3) does it need P3 (negation-only) remediation in #085. Report length follows from methodology rigor (~1.5–2 pages expected).

## Operational Definitions

These definitions are referenced by Requirements and Edge Cases below. They exist to bind the methodology to reproducible criteria so two implementers produce comparable verdicts.

- **Loading event in JSONL**: A line in the JSONL transcript where `message.content[].input.file_path` (for a `Read` tool_use entry) equals the absolute path of the reference file under test (e.g., `/Users/charlie.hall/.claude/reference/verification-mindset.md`) OR `~/.claude/reference/<name>.md`. String mentions in user/assistant text or attachments do NOT count — only Read tool_use invocations.
- **Canonical trigger probe**: A user message whose surface phrasing matches the conditional-loading table row's trigger text verbatim or near-verbatim (e.g., for `verification-mindset.md`: "I just ran the tests and they pass — write the PR").
- **Adjacent near-miss probe**: A user message that EITHER (a) omits one item from the trigger row's listed items (e.g., for context-file-authoring.md's trigger "Modifying SKILL.md files, Agents.md, CLAUDE.md, or reference docs", a near-miss is "I'm editing a regular markdown file" — omits all listed items), OR (b) substitutes a near-synonym not present in the trigger set (e.g., "writing some agent prompts" instead of "modifying SKILL.md files"). Near-misses must be at least 2 content words different from the canonical phrasing AND must NOT contain any of the trigger row's listed items as exact substrings.
- **Hook event**: An entry produced by the `InstructionsLoaded` hook (if available) recording that a specific reference-file path was loaded into a probe session's context. The event's payload must include a path-identifying field (e.g., `file_path` or `instruction_path`); a bare "instructions loaded" event with no path is treated as "unavailable for per-file Q1 verdict" and routed to fallback.
- **Ambiguous probe result**: For a paired probe (canonical + ≥2 near-misses), the result is **ambiguous** if the model's behavior on the canonical probe is INDISTINGUISHABLE from its behavior on at least one near-miss probe (i.e., no observable behavioral differential — same answer style, no reference-file content cited, no Read tool call to the file under test). Single-trial behaviors that are merely "interpretable two ways" do not qualify.
- **Confidence tier mapping**:
  - **HIGH**: Hook event fired for this file in the probe session AND JSONL-grep also shows ≥1 historical loading event AND the paired probe shows a clear behavioral differential (canonical fires the file, near-miss does not).
  - **MEDIUM**: Either hook OR JSONL-grep shows loading evidence (not both), AND/OR paired probe shows partial behavioral differential. Or: the source agreed with the methodology but n=1 prevents HIGH-tier confidence.
  - **LOW**: No loading evidence from either source, OR paired probe ambiguous, OR hook and JSONL-grep disagree (one shows loading, the other shows none).
- **Probe-isolation regime (the only valid one)**: A `claude -p '<probe text>'` invocation executed from a directory that contains no project-level `.claude/CLAUDE.md` (e.g., a fresh `mktemp -d` directory), so user-global `~/.claude/CLAUDE.md` auto-discovery applies (preserving the conditional-loading table) without project-level CLAUDE.md interference. **`claude --bare` is explicitly forbidden** — its documented behavior in 2.1.114 disables CLAUDE.md auto-discovery, which is the mechanism under test.

## Edge Cases

- **`InstructionsLoaded` hook unavailable in installed Claude Code 2.1.114** (after Req 11's positive-control verification): Fall back to JSONL grep for Q1; document the fallback in Methodology and downgrade Q1 confidence tier per the Confidence Tier Mapping. Apply per-file mixed tiering — files where the fallback was used get MEDIUM (only one source: grep), not HIGH.
- **Reference file has zero recent loading events in 210-JSONL corpus AND no `InstructionsLoaded` evidence**: Mark Q1 as "insufficient passive evidence" with confidence LOW. Run an active probe per Req 4 for that file's Q1 (canonical + 2 near-misses); if probe shows behavioral differential, upgrade to MEDIUM (active probe is one source). Absence of evidence does not become evidence of absence by default.
- **Active probe gives ambiguous result** (per Operational Definitions: canonical and at least one near-miss show no behavioral differential): Mark Q2 confidence LOW. Q3 defaults to "needs P3 remediation in #085" as the conservative answer.
- **Probe phrasing accidentally primes the behavior under test**: Paired probes (canonical + ≥2 near-misses per file; ≥4 for epic-flagged files) are required — the differential, not the absolute, is the Q2 signal. Near-miss construction follows Operational Definitions.
- **All five files load reliably AND all section-level patterns survive 4.7 reading**: Report concludes "DR-1 contraction stands; #085 scope unchanged." This is a valid outcome, not a failed spike.
- **A sixth conditional-loading entry is discovered mid-investigation** (e.g., a CLAUDE.md edit added one after this spec was written): Out of scope per Non-Requirements; flag in Limitations and notify #085 explicitly.
- **Probe session has wrong context** because cwd has a project-level `.claude/CLAUDE.md` or because the probe forgot to omit `--bare`: Spike-notes.md documents the failed invocation; re-run from a fresh `mktemp -d` directory without `--bare`. Acceptance criterion 10's `grep -c -- '--bare' = 0` catches this at review time.
- **Hook fires but doesn't disambiguate which file loaded** (payload omits a path-identifying field per Operational Definitions): Treated as "hook unavailable for per-file Q1 verdict"; route Q1 evidence to JSONL-grep fallback. Document the payload limitation in spike-notes.md.
- **Hook and JSONL-grep disagree** for a given file (one shows loading, the other shows none): Confidence tier is LOW per Confidence Tier Mapping. Surface the disagreement in the verdict cell rationale.
- **Hook fires for some files but not others** (partial firing across the 5): Per-file mixed tiering — files with hook events use HIGH/MEDIUM per the mapping, files without get MEDIUM (grep-only) or LOW (no evidence either way). All 5 cells use the same Confidence Tier Mapping; methodology may differ per file but verdict criteria do not.
