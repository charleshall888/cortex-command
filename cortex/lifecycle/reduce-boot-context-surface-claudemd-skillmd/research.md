- **Edit canonical only** — `plugins/cortex-core/skills/*/SKILL.md` are auto-regenerated by the pre-commit hook from `skills/<name>/SKILL.md`. A "compress 11 descriptions" PR is really a 22-file PR (11 canonical + 11 mirror).

- **Pre-existing MUST language is grandfathered**: relocation of OQ3 from CLAUDE.md to docs/policies.md does not require the evidence-artifact gate.
- **Lifecycle required**: editing files in `skills/`, `hooks/`, `claude/hooks/`, `bin/cortex-*`, or `cortex_command/common.py` requires a lifecycle (already at `lifecycle/reduce-boot-context-surface-claudemd-skillmd/`). CLAUDE.md and `docs/policies.md` are NOT under the editing-restricted paths.

## Web Research

### Anthropic's documented routing semantics

**Open Agent Skills standard** (agentskills.io):
- Required frontmatter: `name`, `description`. Optional: `license`, `compatibility`, `metadata`, `allowed-tools`.
- Only `description` is routed against. `metadata` is a client-storage bag, not used for routing.
- `when_to_use:` is **NOT** part of the open standard.

**Anthropic platform docs** (platform.claude.com/docs/en/agents-and-tools/agent-skills/):
- "Each Skill has exactly one description field. The description is critical for skill selection... Your description must provide enough detail for Claude to know when to select this Skill."
- Description hard cap: 1024 chars. Must be non-empty, no XML tags.

**Claude Code specifically** (code.claude.com/docs/en/skills) — KEY FINDING:
- Claude Code DOES support `when_to_use:` frontmatter. Semantics: "Appended to `description` in the skill listing and counts toward the 1,536-character cap."
- I.e., `description:` and `when_to_use:` are **concatenated** in the routing model's listing. Both are L1 cost.
- Total skill-listing budget is configurable via `skillListingBudgetFraction` (default ~1% of model context window).
- When the listing budget overflows, less-used skill descriptions are **truncated** first. Run `/doctor` to inspect overflow state.

### Level 1/2/3 loading contract (confirmed)

| Level | When loaded | Cost | Content |
|---|---|---|---|
| L1 (Metadata) | Every session start | ~100 tok/skill | `name` + `description` + `when_to_use` (in Claude Code) |
| L2 (Instructions) | On skill trigger | <5k tok recommended | SKILL.md body |
| L3+ (Resources) | Read on demand | Effectively unlimited | Bundled files |

### Verdict on non-`description` routing

- **Anthropic API / open standard**: UNSUPPORTED. Only `description` is routed against.
- **Claude Code**: SUPPORTED — but **only as description-extension**. `when_to_use:` is concatenated onto `description:` for the routing model's listing. It is NOT a separate routing channel. Both fields share the 1,536-char combined cap.

**Critical implication for this work:** moving content between `description` and `when_to_use` is NOT a token reduction in Claude Code. The only L1 reduction is shrinking the COMBINED char count.
