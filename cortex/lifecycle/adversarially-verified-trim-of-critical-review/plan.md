# Plan: adversarially-verified-trim-of-critical-review

## Overview

Ship a string-regression gate over the load-bearing contract text in `skills/critical-review/references/verification-gates.md` (plus the SKILL.md total-failure literal), then run one recorded adversarial trim pass over that file and apply only the verifier-certified-safe subset (possibly empty). The gate is the primary deliverable; the trim is bounded by it. **"Adversarially-verified" here means challenged by a fresh, independent adversarial pass — not certified by a behavioral A/B eval** (none is built this round); the static gate detects string-regression, not behavioral equivalence. The three other reference files (`a-to-b-downgrade-rubric.md`, `residue-write.md`, `angle-menu.md`) are left untouched and unpinned.

## Outline

### Phase 1: Anchor (tasks: 1)
**Goal**: Add a sibling string-pin test that fails loudly when the total-failure literal (pinned in both files AND asserted byte-equal across them), the three SKILL.md-cited Step designators, the per-section exit-0/3/4 route markers for either subcommand, or the exit-2 stop-dispatch reaction are deleted/renamed/paraphrased.
**Checkpoint**: `tests/test_critical_review_reference_pins.py` is green on the pre-trim tree AND demonstrably fails on a scratch deletion of any pinned string (then reverted). Independently shippable — committed before any Phase-2 edit.

### Phase 2: Verified trim (tasks: 2, 3, 4, 5, 6)
**Goal**: Produce a recorded adversarial trim-map for `verification-gates.md`, apply only the certified-safe/downgraded cuts under the Phase-1 gate and all existing gates, hand-diff condensed route prose (exit 0/2/3/4) against the Python exit constants, and land the whole changeset (canonical+mirror+artifacts) in ONE atomic commit with the full suite green.
**Checkpoint**: `verification-gates.md` (+ mirror) reflects exactly the certified subset (net byte change ≥ 0); the tempfile-guard passage and both exit-4 rationales remain present; out-of-scope files show 0 changed lines. `just test` is run only AFTER `just build-plugin` regenerates the mirror (the parity test reads working-tree bytes, so it is red in the uncommitted Task 3→5 interval by design); at the single Phase-2 commit, `just test`, E101, dual-source parity, and the L1 ratchet are all green.

## Risks

- **Zero-byte trim is the likely outcome.** The research adversarial pass already refuted the tempfile-guard cut and downgraded the exit-4 dedupe, and the remaining pool is small How-narration. The plan treats a zero-byte applied trim as a passing result (R5) — the gate + recorded trim-map are the deliverable. Do not manufacture cuts to show a byte reduction; honest reporting is required.
- **The gate detects string-regression, not behavioral equivalence.** A green pin proves the pinned literals/designators are present; it does NOT prove the orchestrator still reacts identically to exit 2/3/4. That residual risk is handled by the R8 one-time hand-diff (Task 4, now covering exit 0/2/3/4 with quoted both-sides evidence), not the static gate — an accepted, stated limitation. Verifier independence (Task 2) is likewise a process property: it is carried by two distinct agent dispatches and the distinct `auditor_reason`/`verifier_reason` attributions, not by a string check.
- **Pin-set minimality is a judgment call.** Pinning too much freezes prose the trim wants to touch; too little misses a renumber. The chosen set (3 designators + per-section exit-0/3/4 markers + the exit-2 stop-dispatch reaction + 1 total-failure literal pinned in both files AND asserted byte-equal across them) is the minimal *localizing* set that catches a renumber/paraphrase of the SKILL.md-cited structure. The exit markers MUST be asserted per-section (sliced at the two Step headings), not by a whole-file count — each occurs exactly twice, so a count check cannot localize a single-section deletion. If a future round wants to trim a route reaction's prose, it must update the citing pointer in the same change.
- **No intermediate commit in Phase 2.** Tasks 3 (canonical edit) and 4 (hand-diff note) leave changes uncommitted; Task 5 regenerates the mirror and lands all five paths in one atomic commit. This is load-bearing: a commit between the canonical edit and the mirror regen would (a) land a parity-broken commit the drift hook can't catch (the staged lifecycle path doesn't match the source-trigger regex, so `build-plugin` never runs) and (b) make `just test` red in the interim (the parity test reads working-tree bytes). Corroborated by the drift-hook/shared-checkout memory note: mirror regen can't be deferred to a final task.
- **Sequential dispatch on main, explicit pathspecs.** Tasks 2–6 are near-sequential by dependency and touch dual-source files under a shared index with a concurrent live session. Implement should dispatch sequentially (not worktree) and commit with explicit `git commit -- <paths>` pathspecs — never `git add -A`/`git checkout -- .`, which would sweep in or discard the concurrent session's 298–302 backlog edits.
