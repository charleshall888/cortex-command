# Specification: restore-worktree-root-env-prefix

## Problem Statement

The same-repo daytime/lifecycle dispatch path (`/cortex-core:lifecycle <feature>` → "Implement in autonomous worktree", and the `Agent(isolation:"worktree")` parallel-execution dispatch path via `claude/hooks/cortex-worktree-create.sh`) fails its readiness probe under Claude Code's macOS Seatbelt sandbox with `hardcoded_deny`. `git worktree add` cannot check out `.mcp.json` under `<repo>/.claude/worktrees/` because the Seatbelt mandatory deny on filename `.mcp.json` (enforced by Anthropic's `sandbox-runtime`) operates below user-level `sandbox.filesystem.allowWrite`. The `harden-autonomous-dispatch-path-for-interactive` lifecycle (completed 2026-05-13) shipped R7 — a `cortex init` Step 8 that registers the worktree root in `allowWrite` — on the premise that the registration would relieve the deny. Empirical A/B test (research.md, Adversarial review) confirms the premise is structurally unachievable. Result today: autonomous worktree dispatch is broken under interactive Claude Code sessions, parallel-feature dispatch has the same failure mode, and `cortex init` Step 8 is dead code. This lifecycle restores end-to-end autonomous-dispatch correctness by moving the default worktree root out of the deny scope (single resolver chokepoint), retires R7 with cited evidence covering both dispatch paths, and corrects four collateral defects the Adversarial review surfaced.

## Phases

- **Phase 1: Resolver + cleanup correctness (atomic landing)** — change branch (c) default, canonicalize via `Path.resolve()`, tighten branch (b) marker, route `cleanup_worktree()` fallback through the resolver, update existing tests, add new structural regression tests. All Phase-1 source + test edits land in a single atomic commit so `just test` is never red mid-phase.
- **Phase 2: Init handler + hook consumer parity (atomic landing)** — remove `cortex init` Step 8, delete `tests/test_init_worktree_registration.py`, update `claude/hooks/cortex-worktree-create.sh` to shell out to the Python resolver (single-chokepoint), update `tests/test_hooks.sh`. All Phase-2 edits land in a single atomic commit.
- **Phase 3: Sandbox-aware verification + paused-feature guard** — add a Seatbelt-active integration test covering BOTH the Python resolver and the bash hook dispatch paths, run it in an active Claude Code Bash session as part of implementation verification (recording the result as an F-row analogue in `events.log`); add an atime-touch guard on lifecycle resume with a test pattern that distinguishes guard-set from creation-fresh atimes.
- **Phase 4: Documentation reversal + R7 supersession (atomic landing)** — sweep the active references to `.claude/worktrees/` across requirements/internals/skill prose (including `cortex/lifecycle/` and `cortex/research/`, excluding `*/archive/*`); update multi-agent.md's same-repo contract with the `.mcp.json` mandatory-deny rationale; record the R7 supersession with cited evidence in both this spec and the harden spec, byte-parity-verified.

**Cross-phase dependency**: Phase 4 R14 cites R10 (Phase 3); R14 must land after R10. Phase 1 must land before Phase 2 (R8 hook depends on R1 resolver default being live). Phase 3 may run in parallel with Phase 2.

## Edge Cases

- **Cross-repo dispatch**: branch (d) of the resolver is unchanged — overnight runner same-session dispatches continue to use `$TMPDIR/overnight-worktrees/{session_id}/{feature}/`. No behavioral change for overnight.
- **macOS `/var/folders` vs `/private/var/folders` symlink**: `Path.resolve()` (R2) canonicalizes the result. Downstream Seatbelt rules that match canonical form succeed.
- **`$TMPDIR` unset**: branch (c) falls back to `/tmp/cortex-worktrees/<feature>` (mirroring branch (d)'s pattern). `/tmp` is not on the Seatbelt sandbox allowOnly list — this branch is a defensive fallback for non-macOS contexts (Linux/WSL2 where Claude Code injects `TMPDIR=/tmp/claude`).
- **3-day `dirhelper` purge against paused features**: R11's atime-touch guard refreshes access times on lifecycle resume. The R11 test pattern distinguishes guard-set atime from creation-fresh atime (set atime back to a known-old value first, then invoke idempotent path, then assert atime moved forward).
- **Existing user with legacy `<repo>/.claude/worktrees/` in settings.local.json**: branch (b) returns the legacy path before falling through to (c). The single existing user (project operator) removes the entry manually BEFORE Phase 1 lands (recorded as a verification prerequisite for R1, not a separate requirement). After R3 lands, the substring-match defect that allowed the legacy entry to win is also closed — a recurrence of the manual-cleanup hazard for future users is structurally prevented by R3's tightened marker scheme.
- **Multi-user shared dev box**: each user's per-user `$TMPDIR` produces a per-user worktree path; repo-shared `.git/worktrees/` admin tree may collide on `git worktree add` for the same feature slug across users. Acknowledged as out-of-scope (cortex-command targets single-user laptop workflows).
- **Parallel-execution `Agent(isolation:"worktree")` from inside a worktree**: the hook (R8) now shells out to `cortex-worktree-resolve` regardless of CWD. The hook's path computation becomes CWD-independent for the result itself; the `git worktree add` cwd remains the calling worktree, which is correct (the new worktree branches from the calling worktree's HEAD).
- **`CORTEX_WORKTREE_ROOT` operator override (branch (a))**: unchanged — operators who explicitly set this env var continue to be honored at top precedence. Useful for testing or specialized setups.
- **`cortex-worktree-resolve` unavailable on hook PATH**: the hook exits non-zero with a diagnostic — it does NOT silently fall back to a duplicate path computation. This preserves single-chokepoint guarantees at the cost of a hook failure mode the implementer must verify is observable (R8 acceptance covers this via the mock-replacement test).

## Technical Constraints

- **Seatbelt mandatory deny on `.mcp.json` is binary-level**: `~/.claude/settings.local.json::sandbox.filesystem.allowWrite` does not relieve it (Anthropic settings array-merge precedence model). The only escape is to place the worktree outside any path matched by the deny glob — `$TMPDIR/...` is in the allowed set per macOS Seatbelt's per-user `$TMPDIR` carve-out.
- **macOS `$TMPDIR` is a symlink** to `/private/var/folders/...`; Seatbelt operates on literal path strings, so `Path.resolve()` canonicalization is required for cross-comparison correctness (R2).
- **Single resolver chokepoint, enforced structurally**: `resolve_worktree_root()` is the single resolver consumed by ALL dispatch paths: (1) `create_worktree`, (2) `daytime_pipeline._worktree_path`, (3) `readiness.verify_dispatch_readiness`, (4) `cleanup_worktree`'s new routed fallback, and (5) `claude/hooks/cortex-worktree-create.sh` via the `cortex-worktree-resolve` shell-out (R8). The hook's structural agreement with the Python resolver is verified by R8's byte-identity test, NOT by prose discipline. Changes at the resolver propagate to all five callers automatically.
- **`cortex-check-parity` SANDBOX_WATCHED_FILES**: this lifecycle's source-code edits do NOT touch `cortex_command/pipeline/dispatch.py`, `cortex_command/overnight/runner.py`, `cortex_command/overnight/sandbox_settings.py`, or `pyproject.toml`. (Adding the `cortex-worktree-resolve` `[project.scripts]` entry DOES touch `pyproject.toml`. Verify whether `pyproject.toml`'s `claude-agent-sdk` watch pattern triggers — if yes, a `preflight.md` is required; if no, proceed without.) The sandbox preflight gate's status is checked at Plan time and surfaced in the plan-phase tasks.
- **Plugin mirror regeneration**: `just build-plugin` regenerates `plugins/cortex-core/{skills,hooks,bin}/` from canonical sources. R13's doc sweep verification includes a `just build-plugin` step.
- **Hook environment portability**: `claude/hooks/cortex-worktree-create.sh` is invoked by Claude Code itself; its environment must include `cortex-worktree-resolve` on PATH. The hook exits non-zero with a clear diagnostic when the helper is unreachable — verified by R8's mock-replacement test.
