# Specification: Adversarially-verified trim of research/SKILL.md

## Problem Statement

`skills/research/SKILL.md` (253 lines / 15,018 bytes) loads on every research delegation from lifecycle, refine, and discovery — a hot path — but was deliberately excluded from the `harness-token-efficiency-trim` feature's 12-file trim because no adversarially-verified trim map existed for it. This feature builds that map: classify each section by content class and consumption mode, verify each proposed cut with an *independent* adversarial pass plus a behavioral-equivalence oracle, and apply only the verified-safe subset. A rider applies three lifecycle/refine micro-canonicalizations the precedent's review left as R2 PARTIAL. Benefit: lower per-session token cost on a hot path, plus consistency in the lifecycle/refine reference family — without degrading research-dispatch behavior.

## Phases

- **Phase 1: Verified-safe trim of research/SKILL.md** — classify, independently verify (per-cut discharge + a behavioral-equivalence oracle), and apply the verified-safe cut subset to the canonical file + mirror.
- **Phase 2: Rider micro-canonicalizations** — convert three inline-duplicated rules to pointers at their canonical homes, with all dependent back-references retargeted in the same change.

## Requirements

1. **Per-section trim map**: Produce a `trims_verified`-shaped classification of `skills/research/SKILL.md` covering every region, tagging each by content class (`load-bearing-gate` / `maintainer-rationale` / `how-narration` / `duplication`) AND consumption mode (`orchestrator-read` / `verbatim-shipped`). Recorded at `cortex/lifecycle/adversarially-verified-trim-of-research-skillmd/evidence.json` as a single `trims_verified` entry for `skills/research/SKILL.md` (precedent schema; not the 12-file panel apparatus). Acceptance: `test -f cortex/lifecycle/adversarially-verified-trim-of-research-skillmd/evidence.json` succeeds; the entry has one classified region per `## Step`/region with a content-class tag, a consumption-mode tag, and a keep/trim/relocate verdict each; the region containing the five fenced per-angle prompt blocks is tagged `verbatim-shipped` with its fenced bodies verdict=keep. **Phase**: Verified-safe trim of research/SKILL.md

2. **Independent per-cut adversarial discharge**: Every applied cut is verified by an agent **distinct from the agent that proposed/authored the cut** (role separation — the proposer does not certify its own cut). For each cut the verifier states (a) the behavior the original prose induced, (b) the *citable* location that still enforces it after the cut, and (c) rejects the cut if (b) is "nowhere — it relied on this prose." The verifier independently re-reads each cited anchor in the post-trim file to confirm it exists and actually enforces the named behavior (a plausible-but-unread anchor does not discharge). **Emphasis/weighting/scope-bearing prose** (prose that tunes a decision the model still makes, rather than encoding a discrete rule — e.g. "not a quota — do not pad with redundant agents") may be `condense` only, never `remove`, and its discharge must show the *surviving* text still carries the weighting/scope explicitly (not merely that some discrete rule survives elsewhere). Acceptance: in `evidence.json`, every applied proposal (`safe_proposals`/`downgraded_proposals`) carries a `verifier_reason` whose `anchor` field names a file+line that the entry's `verifier_anchor_recheck` array confirms was re-read post-trim; every proposal whose `content_class` is `how-narration` or that the verifier judged emphasis/scope-bearing has `action` ∈ {`condense`} (not `remove`) OR a recorded refutation; proposals lacking a confirmed anchor appear only in `refuted_proposals`. **Phase**: Verified-safe trim of research/SKILL.md

3. **Behavioral-equivalence oracle (independent content floor)**: Before trimming, record baseline answers to a fixed behavioral-question battery (below) derived from the pre-trim file; after trimming, a **fresh agent given ONLY the post-trim `skills/research/SKILL.md`** (no access to the pre-trim file, the diff, or `evidence.json`) answers the same battery. Every answer must match the baseline. Any mismatch is a behavior regression: revert or revise the offending cut. This is the independent oracle that gates behavior-neutrality and bounds the trim from below (an over-trim that drops load-bearing behavior changes an answer). The battery (recorded with baseline + post-trim answers in `evidence.json` under `behavioral_equivalence`): (i) for tier=complex criticality=high, how many agents dispatch? (ii) which three angles are the mandatory core? (iii) is the fan-out count matrix inlined here or referenced by pointer? (iv) which angle is dispatched LAST for high/critical, and is it given a summary of the other angles' findings? (v) which angles receive the considerations-injection and which are excluded? (vi) what is the one fixed-contract output heading? (vii) what distinguishes lifecycle-mode from standalone-mode output routing? Acceptance: `evidence.json` `behavioral_equivalence` records baseline and post-trim answers for all seven items and `match: true` for each; on any `match: false` the cut(s) implicated are reverted before the phase completes. **Phase**: Verified-safe trim of research/SKILL.md

The byte total is an outcome, not a target. Acceptance: `wc -c < skills/research/SKILL.md` < 15018 (a sanity bound only — it does NOT by itself certify the trim; R2 and R3 are the safety gates); no applied cut lacks a discharge record in `evidence.json`. **Phase**: Verified-safe trim of research/SKILL.md

- Does NOT run the precedent's full 3-role auditor+verifier+lock-in panel. It DOES require role separation (a verifier distinct from the cut-author, R2) plus an independent behavioral oracle (R3) — proportional to a single-file trim, but not self-certification.

## Edge Cases

- **Verification self-sealing**: The cut-author certifying its own cuts would make "verified-safe" hollow. Expected: R2 role separation (distinct verifier) + R3 fresh-agent behavioral oracle that never sees the pre-trim file or evidence.json provide the independent check.
- **Emphasis prose passes the anchor test but shifts behavior**: A cut of weighting/scope prose can cite a surviving discrete-rule line yet change how the orchestrator acts (Opus 4.8 does not re-infer deleted scope). This applies to ALL scope/emphasis-bearing prose, not one site. Expected: R2 forces such prose to `condense` (not `remove`) with scope kept explicit; R3 battery catches a residual behavior shift.
- **Accidental edit inside a fenced body**: The highest-consequence failure — silently degrades every research dispatch. Expected: R5 byte-identity check on the five fenced bodies (diff pre/post) catches it; the boundary is anchored to fence-pairs, not drifting line numbers.
- **Fence-parity off-by-one**: Trimming a `Tools:`/`Mode:` scaffolding line adjacent to a ```-marker can consume the fence. Expected: after any cut near a fence, re-count ```-fence parity (must stay even / matched); R5 fenced-body diff fails if a fence was consumed.
- **Over-trim with no lower bound**: `wc -c < 15018` passes for any reduction including a destructive one. Expected: R3's behavioral oracle is the content floor, not the byte count.
- **Rider line-number drift**: The R2 PARTIAL line numbers are as of 2026-06-10 and have drifted (the index.md rule has THREE sites incl. the back-reference; the exit-2 rule has a second back-reference at refine "Handle failures as in Step 3"). Expected: re-locate every site before editing; do not edit by stale line number.
- **Callgraph gate**: `tests/test_skill_callgraph.py::test_real_tree_clean` reads every line of the live tree. Expected: run `just test`; do not assume the file is test-invisible.
