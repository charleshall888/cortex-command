# Specification: Extract the critical-only competing-plans block to a lazy reference (#341)

> Citation convention: `<path>:<line>` tokens here are backticked per the repo convention. Precise mechanism: `tests/test_lifecycle_references_resolve.py` *suppresses* `stale` file-line citations originating under `cortex/lifecycle/` (`tests/test_lifecycle_references_resolve.py:478`–481), so this spec's own citations cannot fail it; the live staleness surface is a full-path `skills/lifecycle/references/plan.md:NNN` citation under `cortex/research/**` whose line exceeds EOF (empirically none exist today). plan.md is currently **314 lines** and drops to ≈210 after the §1b body is cut. Backticking is consistent hygiene — apply it in plan.md too — but it is belt-and-suspenders here, not load-bearing for this lifecycle dir.

## Problem Statement

The Plan-phase reference `skills/lifecycle/references/plan.md` is read top-to-bottom on every plan, but its §1b "Competing Plans (Critical Only)" block (≈10.9 KB, ≈43% of the file) runs only when criticality is `critical` — ≈2% of plans (5 of 257 measured features). The block is dead weight resident on ≈98% of plan reads. Physically extracting it into a sibling reference Read only on the critical arm removes those bytes from the hottest interactive path with no behavioral change — the single largest resident-byte reduction available at no architectural risk (discovery R3; epic #340).

## Phases

- **Phase 1: Extract & wire** — move the §1b body to a new sibling, leave a load-bearing heading stub, and wire the critical arm to read it through the SKILL.md propagation manifest.
- **Phase 2: Reconcile pins, mirror & verify** — re-anchor the shifted kept-pauses entry, repoint documentation citations for accuracy, regenerate the plugin mirror, and prove every gate green.

## Non-Requirements

- **No behavioral change** to the competing-plans flow itself — this is a pure physical relocation behind the existing `§1a` criticality branch. The critical arm runs byte-identical logic, just Read from a new file.
- **Do not rename or alter** the heading text `### 1b. Competing Plans (Critical Only)` — it is pinned by `test_skill_section_citations.py` and cited as an overnight documentation anchor (#332 guardrail).
- **No new `bin/cortex-*` verb**, no `bin/.parity-exceptions.md` edit — `cortex-check-parity` requires zero action (the only `cortex-*` tokens in the block, `cortex-resolve-model` and `cortex-lifecycle-state`, stay wired across other `skills/**` files).
- **Preserve the `§1a` routing outcomes** — critical still routes to the competing-plans block, low/medium/high still route straight to §2. Req 2 has §1a's `critical` branch carry the read directive (so the gate is structural), but the routing *targets* are unchanged — no new branch, no changed condition.
- **Do not** gate the read in the SKILL.md body (rejected Alternative B — fractures the §1a→§1b flow and pollutes the generic "Read only the current phase reference" contract), reorder within plan.md, use `offset/limit` partial Reads, or attempt a build-time include (all save zero bytes or are impossible because criticality is per-feature runtime state).

## Edge Cases

- **Cold-read under-triggering (dominant runtime risk):** after extraction the critical arm must perform a *separate* Read of competing-plans.md; if the model skips the pointer, the entire critical-tier flow silently no-ops with no test catching the runtime miss. Mitigation: the read directive lives in §1a's `critical` branch (Req 2) — the routing control flow reached only on the critical path, a structural gate rather than a prose mention — and is one level deep from plan.md (not nested behind another reference). The over-fetch dual (a non-critical reader pulling the file anyway) is closed by the same placement: a non-critical reader never reaches the directive. Req 9's wiring test guards the static surface; the runtime miss stays untestable and is accepted.
- **Stale citation (precise surface):** `test_lifecycle_references_resolve.py` *suppresses* `stale` file-line citations under `cortex/lifecycle/` (`tests/test_lifecycle_references_resolve.py:478`–481), so this lifecycle's own artifacts cannot fail it. The live failure surface is a full-path `skills/lifecycle/references/plan.md:NNN` citation under `cortex/research/**` whose line exceeds the post-cut EOF (empirically none exist today — all are backticked). The check is line>EOF only, so it does *not* catch a citation that now points into the moved region (silent content drift). Mitigation: backtick `plan.md:NNN` citations as hygiene, and when repointing any citation that targeted the moved §1b body, point it at competing-plans.md.
- **Mirror omitted from the commit:** dual-source parity fails and the drift hook blocks the commit. Mitigation: `just build-plugin` and commit canonical + mirror together (Req 7).
- **Internal `§1b.x` references orphaned from their heading:** the `### 1b.` heading stays in plan.md, so the moved body's `§1b.x` self-refs would anchor nothing. Mitigation: competing-plans.md restates its own top heading (Req 1).
- **Subagent handed a self-contradictory pointer:** `plan.md:78`'s "§3 … below" sits inside the verbatim plan-agent prompt template dispatched to a fresh subagent that sees neither plan.md nor competing-plans.md. Mitigation: de-reference it (Req 4).

## Technical Constraints

- **ADR-0009 / SP001-SP002 path resolution**: a reference file cannot resolve `${CLAUDE_SKILL_DIR}`; only the SKILL.md body can. The sibling read is wired solely through the manifest bullet + named-target idiom (Reqs 2–3). A bare `references/competing-plans.md` in a Read context would trip the `cortex-check-skill-path` D2 lint; D1 is inactive (no SUBAGENT-PROMPT fence in these files).
- **Dual-source mirror** (CLAUDE.md; `.githooks/pre-commit` → `just build-plugin`): edit canonical only; regenerate and commit the mirror in the same commit.
- **Size cap is SKILL.md-only** (`test_skill_size_budget.py`); the 500-line cap does not gate reference files, and the L1 surface ratchet is untouched (no frontmatter change).
- **Prior art**: `critical-review-gate.md` is the exact lazy-conditional sibling-read precedent (manifest bullet + branch-gated read); #334's `fanout.md` relocation (commit `df02ac4e`) is the precedent for doing extraction + citer/test repoint + mirror regen in one commit, with a follow-up that scrubbed a bare-noun token — so scrub all token forms, not only path-form links.
- **Wiring-test precedent**: `tests/test_post_refine_commit_wired.py` is the established shape for statically guarding an extract-into-sibling + manifest-bullet change (file+mirror existence, consumer-references-target, SKILL.md mention, content tokens); Req 9 mirrors it for competing-plans.md.
