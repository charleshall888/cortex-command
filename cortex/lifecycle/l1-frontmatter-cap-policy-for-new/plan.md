# Plan: l1-frontmatter-cap-policy-for-new

## Overview
Reframe the existing `tests/test_l1_surface_ratchet.py` snapshot into a deliberate per-skill byte-budget policy with two structural add-time gates — a row-completeness assertion (every canonical skill needs a budget row) and a non-cluster ≤400B SUM ceiling — then trim the four uncapped skills below that ceiling so the gate can turn on. These two gates structurally catch the two add-time holes that let the four skills land: a new skill with no budget row, and a non-cluster skill shipping >400B. The two adjacent governance surfaces — promoting a skill into the cluster exemption, and raising an existing budget row — are **deliberately prose-governed** in `project.md` (spec Non-Requirement: no new authoring-time lint), with the routing test providing incidental friction against a bare tuple-append. Enforcement stays entirely in the existing ratchet test (no new lint); the routing-pressure cluster is the single documented exemption surface.

## Outline

### Phase 1: Policy + enforced default + fixes + routing floors (tasks: 1, 2, 3, 4)
**Goal**: Reframe the ratchet to deliberate budgets with a row-completeness gate, document the policy, point authors at it, and land the regression-guard fixtures — no skill-text trims yet.
**Checkpoint**: `just test` green; ratchet expresses deliberate budgets + the completeness assertion; `project.md`/`CLAUDE.md` document the ≤400 default + cluster exemption + lifecycle-id'd re-cap rule; trigger-phrase fixtures present for the four uncapped skills.

### Phase 2: Trim the four uncapped skills + turn on the ≤400 default assertion (tasks: 5, 6)
**Goal**: Trim the four uncapped skills to ≤400B SUM, lower their budgets, regenerate mirrors, then enable the non-cluster-≤400 assertion that structurally enforces the default on all future non-cluster skills.
**Checkpoint**: the four skills measure ≤400B; ratchet passes with lowered budgets AND the non-cluster-≤400 assertion present; routing + mirror-parity tests green; ~768B recovered.
