# Specification: 378 — 374 follow-ups (lifecycle_slug coercion, spec-approve routing, CLI_PIN dedup)

## Problem Statement

Three residues from #374 have hardened into real defects. A tool-managed writer emits `lifecycle_slug`/`feature` as unquoted YAML integers, which `yaml.safe_load` reads as `int` — crashing the served next/advance resolve path (`resolve.py:181`, `Path / int` → `TypeError`) and reddening `test_lifecycle_references_resolve.py`; separately, a write-omission leaves 60 completed items frozen at `lifecycle_phase: research`. `specify.md` and `refine/SKILL.md` still command the kept-callable legacy verb `cortex-lifecycle-spec-approve`, an incomplete R17 (ADR-0024). And `CLI_PIN` is live-diverged (`v2.35.0` overnight vs `v2.34.6` core), making session-start `uv tool install --reinstall` ping-pong the CLI between two tags. A critical review reshaped the scope: numeric backlog IDs are a legitimate first-class input and the quote-on-write fix alone makes numeric slugs safe (so no separate "reject numeric" layer is needed), and the auto-release pipeline dormancy is operator/secrets work split to its own ticket. What remains is verb-owned determinism, correct type-safe writes, and self-guarding pin convergence — cutting prose/token surface where the served loop needs determinism.

## Phases

- **Phase 1: Numeric slug fix** — key-scoped quote-on-write helper, backfill 4 files, reader coercion; greens the test and stops the crash. Numeric IDs stay a valid input; the fix makes them type-safe, it does not reject them.
- **Phase 2: Stale lifecycle_phase** — close the write-omission on completion and backfill the 60 frozen items.
- **Phase 3: spec-approve reroute** — extend advance.py's `spec-approve` arm with the missing spec/areas projection, then reroute both prose files off the legacy verb.
- **Phase 4: CLI_PIN de-drift + guards + CI** — converge the pins, extend the release/parity machinery to cover cortex-core with a target-set invariant, and wire the item-1 regression test into CI.

## Edge Cases

- **`null` / `None` frontmatter values**: the quoter must leave the None sentinel bare (`lifecycle_slug: null`) **even on an allowlisted key**, or it breaks `test_lifecycle_references_resolve.py`'s None sentinel and the resolver's null-fallback. Covered by req-1's explicit None-sentinel exclusion.
- **`update_item` rewrites `updated` on every call**: `updated` is not on the string-intended allowlist, so it is emitted bare and not churn-quoted. Covered by req-1.
- **YAML metacharacters in a value** (`"`, `\`, `:` + space, `\n`, control chars): the quoter must escape all of them, not merely wrap. Covered by req-1(c).
- **Release race (req-10 vs req-11)**: bumping cortex-core alone re-strands on the next overnight-only release. A point-in-time equality check does NOT catch this (both pins equal now, re-drift later) — req-11(c) therefore binds the co-landing structurally via the rewriter-target-set invariant, so a partial landing fails the guard rather than passing green.
- **Concurrent skill-editing session**: Phase 3 touches the `skills/` dual-source tree; the pre-commit drift loop gates *all* commits, so sequence Phase 3 away from any concurrent skill-editing session (the "dual-source drift blocks all commits" hazard).
- **advance.py `spec-approve` status ownership**: `_project_status` already projects `status: refined` (lattice- and archive-shadow-guarded) for the `to_state==plan` arm — so req-7 adds ONLY `spec:`/`areas:` and MUST NOT re-write status, or it would demote an item already past `refined` and defeat the hazard-4 guard. Adding the new flags must not change behavior for existing callers that omit them.

## Proposed ADR

### Proposed ADR: 0027-frontmatter-scalar-write-contract

**Context**: Tool-managed backlog/lifecycle frontmatter is written by hand-rolled per-key line editors (deliberately, to preserve ordered-block + inline-array + bare-null shape that PyYAML can't reproduce), but readers split between string-only custom parsers and `yaml.safe_load`. A bare-numeric string scalar (`lifecycle_slug: 378`) therefore type-leaks to `int` and crashes the served resolve path. **Decision**: establish a single **key-scoped** YAML-safe scalar-quoting helper as the mandated write path for frontmatter scalar emission. The rule is deliberately NOT "quote anything that mis-resolves under YAML 1.1" — dates and the `null` sentinel *do* mis-resolve yet must stay bare — but "for keys on the **string-intended allowlist**, quote when the value would mis-resolve (except the None sentinel); all other keys emit bare." This is forced by the writer boundary erasing Python type (every value arrives as `str`), so intent must be carried by the key, not inferred from the value. **Trade-off**: a key allowlist must be maintained as new string-intended fields are added (a new numeric-looking field left off the list re-exposes the bug), but it is the only rule that reconciles "quote ambiguous slugs" with "leave dates/null/ints bare," and it fixes the class at the single producer boundary ("parse, don't validate") rather than scattering defensive coercion across every `yaml.safe_load` reader. Generalizes the ad-hoc, title-only `_yaml_safe_title_value` precedent into a key-scoped form.
