# Specification: gate-policy-taxonomy-and-critical-review

## Problem Statement

The critical-review module's gate inventory is opaque (no source-discoverable classification), one path-validation gate is mechanically broken on macOS (the ancestor-symlink check rejects sandbox-sanctioned `/tmp/claude` paths), the ad-hoc-input flow promised in skill prose has no implementation (the CLI rejects paths outside `cortex/lifecycle/` and `cortex/research/` despite `skills/critical-review/SKILL.md:29` promising "use conversation context if no lifecycle"), and two verifier subcommands are named for a property they cannot structurally enforce (reviewer/synth engagement). Operators rely on these gates during agent dispatch — fixing the four defects and adding a source-discoverable taxonomy lets future audits read gate policy from source instead of re-classifying from first principles, eliminates the macOS false-positive, restores the documented ad-hoc input contract, and aligns subcommand names with what they honestly verify. Beneficiaries: anyone running `/critical-review` during a lifecycle or ad-hoc (the documented promise becomes real), and any future operator auditing the gate inventory (the classification is grep-discoverable).

## Phases

- **Phase 1: Under-root scoping and verifier rename** — Fix 2 (replace ancestor-symlink check at `:82-89` and remove the redundant root-symlink gate at `:103-111` atomically, with a structural invariant test enforcing the atomicity, using `Path.resolve().is_relative_to()` scoping) plus Fix 4 (rename verifier subcommands and Python functions, drop the engagement-checking claim in docstrings). Fix 2 and Fix 4 are independent within the phase; the phase groups them only because both can land before Phase 2's new infrastructure.
- **Phase 2: Auto-resolve helper and gate-class taxonomy** — Fix 3 (auto-resolve ad-hoc input into `cortex/_adhoc/<sha>/` with `source_path:` and `snapshot_sha:` events.log fields, gitignore, and `cortex clean --adhoc` retention recipe) and Fix 1 (annotate every gate with `# gate-class:` and ship a parity test enforcing the annotations + docstring substrings). Phase 2 cannot land before Phase 1 because (a) `_adhoc/` snapshot paths resolve through `/private/var/folders/...` on macOS — without Phase 1's under-root scoping fix the current ancestor-symlink check rejects them — and (b) Fix 1's annotation grep count in Requirement 1 depends on G3 having been removed in Phase 1.

## Non-Requirements

- The four fixes do NOT add a programmatic gate-class accessor (a registry dict or Enum). The annotation is grep-discoverable only; an audit CLI is a follow-up ticket if and when the use case is concrete.
- The verifier rename does NOT close the orchestrator-fabricated-input bypass. `check-artifact-stable` and `check-synth-stable` detect SHA drift; structurally enforcing reviewer/synth engagement would require a different architecture (reviewers writing to disk out-of-band from the orchestrator) and is explicitly deferred per DR2. The rename's primary value is honesty (gate name, docstring caveat, and `advisory` gate-class annotation now match the gate's actual capability) rather than additional structural enforcement of the bypass — that distinction is documented openly in Requirement 11's docstring and in the gate's `advisory` annotation.
- The rename does NOT rename the sentinel strings `READ_OK:` / `SYNTH_READ_OK:` in the wire protocol. Out of scope; documented in the gate docstring.
- `cortex clean --adhoc` does NOT use LRU, size-quota, or generation-tagged retention. Time-based + `snapshot_sha:` pinning is the only retention strategy.
- The auto-resolve helper does NOT support symlinking the ad-hoc input into `_adhoc/` (only copying). Symlinking collides with Fix 2's symlink rejection and reintroduces the staleness class Fix 4 narrows.
- TOCTOU between `validate_artifact_path` and `sha256_of_path` is NOT addressed in this ticket. The validation-then-read race is a real concern but out-of-scope; filed as a sibling for separate triage.
- `cortex clean --adhoc` does NOT emit tombstone events into events.log when it deletes a snapshot. Events.log entries for completed/archived lifecycles that reference an already-GC'd snapshot become silently dangling references after cleanup — this is accepted as audit-trail decay rather than a referential-integrity guarantee. Re-snapshot of identical content produces an identical `snapshot_sha` and is idempotent.
- The cleanup scan's cost is O(N_lifecycles × events.log_size) per invocation with no incremental-scan optimization or recently-active-lifecycle scoping. This is acceptable at the current scale (≤100 lifecycles, ≤MB-class events.log files); a follow-up ticket would add incremental scanning if the scan latency exceeds 1 second in production.
- Bandit/Ruff-style stale-tag detection (RUF100 equivalent for `# gate-class:`) is NOT in scope. Requirement 2's parity test enforces tag presence + closed-set value; stale-tag detection is a follow-up.

## Edge Cases

- **Ad-hoc input with NUL byte in the path**: rejected at the `validate_artifact_path` boundary with `ValueError`. NUL is the one byte POSIX paths cannot legally contain.
- **Ad-hoc input with surrogate code points** (from `surrogateescape`-decoded argv carrying invalid UTF-8 bytes): rejected at the `validate_artifact_path` boundary with `ValueError` raised from `candidate.encode('utf-8', errors='strict')` → `UnicodeEncodeError`. The check fires before any snapshot or event emission.
- **Ad-hoc input with control characters or ANSI escapes in the path (excluding NUL)**: accepted at validation. Preserved verbatim in `source_path`. `json.dumps` JSON-escapes control characters on events.log write; downstream consumers must sanitize before any terminal `print` (acknowledged behavioral contract on consumers, not a validation concern at write time).
- **Newlines in path**: legal POSIX, accepted. Stored as escape-sequenced bytes via JSON serialization.
- **SHA collision in the fanout**: SHA-256 with 2-char fanout has astronomically low collision probability for the scale of ad-hoc usage. If two different files produce the same 256-bit hash, the second write overwrites the first — semantically a no-op (same bytes by definition). No special handling needed.
- **`_adhoc/` snapshot referenced by paused lifecycle's events.log**: Requirement 9 covers — the pin set is built from every `events.log` regardless of lifecycle status, so paused features keep their snapshots until the events.log itself is archived/removed.
- **Two concurrent `cortex clean --adhoc` invocations**: tombstone-rename makes deletion atomic; the second invocation sees `.tombstone-<sha[2:]>/` (already in the deletion queue) and skips. The first invocation owns the `rm -rf`. (Required by Requirement 9.)
- **`cortex clean --adhoc` racing with a snapshot being written**: snapshot writes use atomic temp-rename to `.staging-<sha[2:]>.<basename>` then `os.rename` after the parent dir exists. Cleanup ignores `.staging-*` paths. (Required by Requirement 6 and 9.)
- **Malformed JSONL row in an events.log during cleanup scan**: skipped with `WARN: skipped malformed row at cortex/lifecycle/<feature>/events.log:<lineno>: <reason>` on stderr; cleanup continues; exit code is 2 (warning). A CI consumer that wants to fail on warnings checks for exit code 2 explicitly. (Required by Requirement 9.)
- **Ancestor symlink that resolves outside the root**: e.g., `/tmp/claude/cortex/lifecycle/foo/spec.md` where `/tmp/claude` is a symlink to `/Users/charlie/scratch/`. After Phase 1, the realpath endpoint `/Users/charlie/scratch/cortex/lifecycle/foo/spec.md` is checked against the realpath of the root. They don't match; validation rejects. (Covered by Requirement 3's `test_module_api_rejects_realpath_escaping_root`.)
- **Direct symlink at the artifact path pointing into a sibling lifecycle**: e.g., `cortex/lifecycle/foo/spec.md` is a symlink to `cortex/lifecycle/bar/spec.md`. After Phase 1, both realpath endpoints live under `cortex/lifecycle/`, so under-root scoping accepts the path. The feature-narrowing gate at `:122-129` then rejects it if `--feature foo` was specified. Containment is preserved.
- **Phase 1 lands partially (only `:82-89` replaced)**: Requirement 4's parity-test invariant fires on the next pytest run with the message `Phase 1 atomicity invariant violated — root pre-resolution gate present`. The half-landing is detectable by name, not just by downstream test breakage.

## Technical Constraints

- **In-house under-root scoping pattern**: `Path.resolve().is_relative_to()` is already used at `cortex_command/critical_review.py:115` and `cortex_command/init/scaffold.py:167`. Fix 2 reuses this primitive. The `realpath().startswith()` shape is explicitly rejected in-tree by a comment at `scaffold.py:165-166` and by web research (TinaCMS GraphQL CVE, Python tarfile CVE-2025-4138).
- **macOS APFS case-insensitivity**: `scaffold.py:113-172` is the canonical in-house pattern for handling case-normcase under macOS APFS; Fix 2 reuses the same approach.
- **`cortex/` umbrella is sandbox-writeable** (ADR-0003): `cortex/_adhoc/` lives under the umbrella, so no additional sandbox registration is needed.
- **Events.log atomicity**: `append_event` at `critical_review.py:278-326` uses tempfile + `os.replace` and `json.dumps`. The new `source_path` and `snapshot_sha` fields are written through this helper, inheriting atomicity and JSON-escaping. Control-character safety on the events.log write side is delivered by `json.dumps`; consumers reading events.log are responsible for sanitization before terminal print.
- **Dual-source parity** (CLAUDE.md): canonical skill edits go on the `skills/` side; the pre-commit hook regenerates `plugins/cortex-core/skills/`. The rename surface in Requirement 12 must be edited as a single commit so the parity hook stays green.
- **Skill-helper module design** (`cortex/requirements/project.md:34-35`): the three-subcommand structure (`prepare-dispatch`, `verify-synth-output` (post-rename: `check-synth-stable`), `record-exclusion`) is load-bearing. The rename preserves the three-subcommand shape; no split or merge.
- **Events registry discipline** (`cortex/requirements/project.md:41-42`): the `source_path:` and `snapshot_sha:` field extensions to `sentinel_absence` must update `bin/.events-registry.md` in the same commit that adds the fields to the emitter.
- **Structural separation over prose-only enforcement** (CLAUDE.md): Phase 1's atomicity (Requirement 4) and the renamed verifiers' docstring contract (Requirement 11) are structurally enforced by Requirement 2's parity test — not by prose convention. The principle binds the spec to back its invariants with executable checks. Fix 4's gate-name rename is primarily an honesty change (the `advisory` annotation telegraphs the structural weakness); the bypass surface is unchanged and the rename is not defended on structural-separation grounds — only on the honesty grounds documented in Non-Requirement 2.

## Proposed ADR

### Proposed ADR: 0005-canonical-adhoc-scratch-directory

The auto-resolve helper introduces `cortex/_adhoc/<sha[:2]>/<sha[2:]>/<basename>` as a peer-of-`lifecycle/` scratch directory for ad-hoc input snapshots. Choosing peer-location over the DR3-as-written nested-location (`cortex/lifecycle/_adhoc/`) is hard-to-reverse (consumers across critical_review, the cleanup recipe, every lifecycle-slug iterator have or will encode the path), surprising-without-context (DR3 specified the nested form; the reason for divergence is the lifecycle-slug-iteration collision surfaced in adversarial review), and a real trade-off (peer location requires extending `_default_artifact_roots()` and diverges from DR3 prose; nested location requires hardcoded `^_` exclusions across every iterator with ongoing audit cost). The decision: peer location, with `_default_artifact_roots()` extended once and a single canonical home for ad-hoc snapshots that no lifecycle iterator will misinterpret as a feature. Snapshots are content-addressed via the full SHA-256 with 2-char fanout (`<sha[:2]>/<sha[2:]>`) to keep any single fanout dir below ~10k entries and to enable deterministic SHA-derivation from the snapshot's path for `cortex clean --adhoc`'s pin-set construction.
