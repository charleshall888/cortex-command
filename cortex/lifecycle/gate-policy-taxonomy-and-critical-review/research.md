## Web Research

**Inline policy-tag taxonomies.** Bandit `# nosec`, Semgrep `nosemgrep:`, and Ruff `# noqa` are the dominant references. Bandit issue #888 prohibits bare `# nosec` without specific rule IDs; Ruff's RUF100 rule flags stale suppressions. Both bake validators alongside the tag — `# gate-class:` without a paired validator becomes the prose-only-enforcement anti-pattern the rest of this work is trying to escape (see Adversarial item 6).

**Under-root containment.** `realpath(child).startswith(realpath(root))` has multiple CVEs — TinaCMS GraphQL (`startsWith` after `path.resolve`), Python tarfile CVE-2025-4138 / GHSA-hgqp-3mmf-7h8f (realpath gap), CWE-22 (Path Traversal). Canonical Python idiom: `os.path.commonpath([base, target]) == base`, or `Path.resolve().is_relative_to(base.resolve())`. Edge cases: trailing-slash normalization (Python tracker #1707768, #21039), macOS case-insensitive APFS (CERT VU#439395, git CVE-2014-9390), TOCTOU between resolve and open (filelock GHSA-w853-jp5j-5j7f, util-linux GHSA-qq4x-vfq4-9h9g). The codebase's in-house pattern at `scaffold.py:167` already incorporates case-normcase handling — Fix 2 reuses it.

**Content-addressed scratch with retention.** SHA-prefix layouts are universal: Bazel `~/.cache/bazel/.../content_addressable/sha256/<full-hash>/`, Nix `/nix/store/<base32>-<name>/`, pnpm CAFS `<2-char-prefix>/<rest-of-hash>` with SHA-512. 7-day retention is the dominant default (GitHub Actions cache, Cloudflare Workers, Gradle released distributions). Prefix-collision risk at 8 hex chars (32 bits) reaches ~50% by 65k entries — see Adversarial item 5 and the prefix-length Open Question.

**Bypassable engagement check → SHA-stability.** Industry consensus: narrow the claim, don't drop the check entirely. Latent Space "How to Kill the Code Review", arxiv 2506.08837 (Design Patterns for Securing LLM Agents), NCSC Security Architecture Anti-patterns ("management bypass"), OWASP LLM Prompt Injection Prevention all converge on: the verifier and the produced artifact must be structurally separated; the verifier must not depend on the agent it verifies. The rename to `check-artifact-stable` honors this principle for the SHA-drift subset — but only if the SHA is recorded out-of-band from the artifact-producing agent. See Adversarial item 3 for the residual bypass.

## Tradeoffs & Alternatives

**Fix 1 — gate-class annotation taxonomy**

- **A1 (inline `# gate-class: ...` comments)** — recommended. Matches the in-file `# Req 9X` precedent; S effort; pairs naturally with the parity test recommended in Adversarial item 6.
- A2 (module-level registry dict) — defer until an audit subcommand is concrete; a second source of truth for 8 entries doesn't earn its complexity yet.
- A3 (decorator-based marker) — requires a refactor the ticket did not scope; the `last_err` loop semantics in `validate_artifact_path` are subtle.
- A4 (Enum/Literal without comments) — not grep-friendly; only pays off if paired with A3.

**Fix 2 — under-root symlink scoping**

- **B2 (`Path.resolve().is_relative_to(Path(root).resolve())`)** — recommended. Already the in-house pattern at `scaffold.py:167` and `critical_review.py:115`. Reuse the existing case-normcase pattern for macOS APFS. Pairs with Adversarial item 1: drop the redundant root-symlink gate at `:103-111` atomically.
- B1 (`realpath().startswith()`) — explicitly rejected by an in-tree comment at `scaffold.py:165-166`.
- B3 (manual parent-walk) — custom loop where a stdlib primitive exists.
- B4 (reject only when artifact-file itself is a symlink) — too permissive; bypasses feature-narrowing containment.

**Fix 3 — auto-resolve helper**

- **C1 (copy on every invocation + 7-day time-based retention)** — recommended, with the Adversarial items 2, 4, 5 amendments applied (location, lifecycle cross-reference, prefix length).
- C2 (symlink instead of copy) — collides directly with Fix 2.
- C3 (relax allowed-dirs) — DR3 explicitly rejected this; downstream consumers depend on canonical paths.
- C4 (XDG `~/.cache/cortex/_adhoc/`) — breaks the "all artifacts under one umbrella" invariant.
- Retention: time-based 7-day matches the discovery research; LRU/size-quota/generation-tagged require per-feature bookkeeping the per-feature-less `_adhoc/` cannot supply.

**Fix 4 — verifier rename + rescope**

- **D1 (rename to `check-artifact-stable` / `check-synth-stable`)** — recommended. Only alternative that honors DR2 and CLAUDE.md's structural-separation principle. The rename must be paired with explicit documentation of the residual bypass (Adversarial item 3).
- D2 (keep names, update docstrings) — preserves the misleading name.
- D3 (delete verifiers entirely) — defensible follow-up if production telemetry ever shows zero SHA-stability hits, but the data isn't there yet.
- D4 (migrate engagement to synth prompt) — CLAUDE.md explicitly forbids; DR2 records the rejection.

**Inter-fix coupling** (corrected from the ticket's "no inter-fix ordering" claim):

- **Fix 2 → Fix 3 is a HARD order on macOS.** `_adhoc/` paths resolve through `/private/var/folders/...` and would be rejected by the current `:82-89` ancestor-symlink check before any `_adhoc/` logic runs. The ticket's Integration assertion is wrong on this pair.
- Fix 2 → Fix 1 is a SOFT order. Annotating the broken `:82-89` gate with a clean class is awkward before Fix 2 lands. Provisional `# gate-class: hygiene (broken — see ticket 255)` is acceptable, but cleaner if Fix 2 lands first.
- Fix 4 is genuinely independent.

## Adversarial Review

1. **`:103-111` root-symlink check must drop atomically with `:84-89`.** Today both gates pre-resolve `/private` consistently, masking the bug. Fix 2 that replaces only `:84-89` will produce a half-fixed state where the root-symlink gate still rejects `/tmp/...`-rooted callers. The spec must specify atomic removal/rework of both gates together.

2. **`cortex/lifecycle/_adhoc/` collides with lifecycle-slug iteration.** Multiple downstream consumers walk `cortex/lifecycle/*/` expecting per-feature directories: `hooks/cortex-scan-lifecycle.sh`, the lifecycle counters/state CLIs, the pipeline scanner. Siting `_adhoc/` inside `cortex/lifecycle/` invites a "convention collapses under load" failure. **Mitigation candidates**: relocate to `cortex/_adhoc/` (peer of `lifecycle/`), or hardcode an `^_` exclusion in every iterator. The location decision flows into the Open Questions section.

3. **The rename does not close the original bypass.** `_cmd_verify_reviewer_output` validates that the supplied SHA appears in the supplied file — the orchestrator-LLM controls both inputs, so it can construct a stub `READ_OK: <path> <sha>` file and pass the check without dispatching a reviewer at all. DR2 acknowledges this honestly ("the system loses a check it previously claimed to have but never actually delivered"). The spec must surface this in the rename's own docstring: `check-artifact-stable` detects uncoordinated concurrent writes during the dispatch window, not orchestrator misbehavior or coordinated fabrication.

4. **Retention vs paused-lifecycle cross-reference is undefined.** A 7-day retention will delete an `_adhoc/<sha>/` snapshot that an open lifecycle's `events.log` still references via `source_path:`. Failure modes: forensic replay (`/cortex-core:diagnose`) breaks with `FileNotFoundError`; audit of `sentinel_absence` rationale loses evidence. **Mitigation candidates**: (a) `cortex clean --adhoc` walks all `events.log` files and pins referenced snapshots; (b) retention keys off lifecycle status (`complete | archived` allows deletion); (c) snapshots are copied into the per-feature lifecycle dir at events-emit time. The mitigation choice flows into Open Questions.

5. **SHA-prefix length is unspecified.** DR3 writes `<sha-prefix>` without a length. 8-char prefix (32 bits) hits ~50% birthday-collision by 65k entries. The web research recommends ≥12 hex; pnpm's pattern is full-hash with a 2-char fanout. The spec must fix this — the recommended default is full-hash with a 2-char fanout.

6. **The `# gate-class:` annotation has no validator.** Discoverability ≠ correctness. Without a parity test that walks rejection sites and asserts each carries an in-scope annotation, the tag rots within two PRs — exactly the prose-only-enforcement anti-pattern the rest of the work tries to escape. **Required mitigation**: ship a pytest alongside Fix 1 that enforces the annotation, file the audit-CLI follow-up separately.

7. **The rename surface is undercounted.** Beyond the canonical skills/ paths, the rename touches `skills/critical-review/references/verification-gates.md:33, 40, 70, 73, 84` (five sites), `tests/test_variant_a_writer_sites_baseline.py:225, 238, 245`, and the plugin mirrors. The codebase agent's three-line count missed these. The implement plan must use the corrected enumeration.

8. **Under-root scoping must check the realpath endpoint, not the literal path.** A symlink at `cortex/lifecycle/foo/evil.md` pointing to `/etc/hostname` must still reject — the literal path is under the root, but the realpath escapes. B2's `is_relative_to` is correct *if* applied to the resolved path; the spec must say "the realpath of the candidate must be under the realpath of the matched root," not just "the candidate path."

9. **TOCTOU between `validate_artifact_path` and `sha256_of_path`.** A file can be replaced with a symlink between the two calls; the rename + rescope work does not fix this. Out of ticket 255's scope, but worth filing as a sibling ticket.

10. **`source_path:` is an injection vector.** Caller-supplied paths can contain JSON-meta characters, newlines, NUL bytes, ANSI escape codes. The TUI / status pipe ingests events.log; any downstream `print(event["source_path"])` to a terminal allows ANSI injection. The spec must specify input validation (UTF-8, no control chars, length-bounded).

11. **`cortex clean --adhoc` concurrency is undefined.** Two concurrent runs (cron + manual, or overnight + scheduled) racing on the same `_adhoc/<sha>/` will half-delete, error with "directory not empty," or delete a snapshot being actively written. **Required mitigation**: file-lock or tombstone-rename pattern (`_adhoc/.tombstone-<sha>/`, delete in second pass).

12. **Sentinel strings `READ_OK:` and `SYNTH_READ_OK:` are unchanged by the rename.** A future maintainer reading the docstring for `check-artifact-stable` and grep-ing for `check_artifact_stable` will not find the sentinel-matching regex. The spec must either rename the sentinel strings (high-risk, breaks fixtures and saved transcripts) or document the deliberate name divergence in the gate's docstring.

## Open Questions

- **`_adhoc/` location**: Site at `cortex/lifecycle/_adhoc/<sha>/` (as DR3 states) with an explicit `^_` exclusion added to every lifecycle-slug iterator, **or** relocate to `cortex/_adhoc/<sha>/` (peer of `lifecycle/`) to avoid the convention collision. The peer-location is structurally safer but diverges from DR3. **For Spec**: pick one and document the iterator-audit list if the nested path is kept.
- **SHA-prefix length**: 8 vs 12 vs full-hash. The recommended default from the web research and adversarial review is full-hash with a 2-char fanout. **For Spec**: fix the length; default recommendation is full-hash with two-char fanout for filesystem-friendliness.
- **Retention vs paused-lifecycle cross-reference**: Three mitigation candidates above (events.log pin, lifecycle-status key, copy-into-lifecycle). **For Spec**: pick the mitigation; the lightest-weight option is events.log pin — `cortex clean --adhoc` scans every `cortex/lifecycle/*/events.log` for `source_path:` references before deleting.
- **Sentinel string rename**: Rename `READ_OK:` / `SYNTH_READ_OK:` together with the subcommand rename, **or** document the deliberate name divergence. **For Spec**: pick one — fixture-breakage probably tips toward "document the divergence" but the spec should make the choice explicit.
- **Cleanup CLI surface**: `cortex clean --adhoc` as a `cortex_command/clean.py` subcommand, or as a separate `cortex-clean-adhoc` console script. **For Spec**: pick one and add the `[project.scripts]` wiring.
- **`is_file` gate-class assignment**: The discovery inventory called it "Security-adjacent hygiene." The taxonomy has only three classes (`security|hygiene|advisory`). **For Spec**: pick one — the closer-to-security framing argues for `security`, the file-only check is closer to a typing gate which argues for `hygiene`. Recommended default: `security` (rejecting non-regular files is the closest thing to a security check in the inventory).
- **Parity-test scope for `# gate-class:` annotations**: The mitigation in Adversarial item 6 says "walks rejection sites and asserts each carries an in-scope annotation." The exact walking strategy (AST-based vs. comment-window regex) is a spec decision. **For Spec**: pick the simplest reliable approach.
