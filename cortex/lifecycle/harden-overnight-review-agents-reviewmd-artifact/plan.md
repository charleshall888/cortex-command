# Plan: harden-overnight-review-agents-reviewmd-artifact

## Overview
Anchor the worktree-mode overnight review's writer and reader to one explicit absolute main-repo
`review.md` path. Three coordinated edits: (1) canonicalize the lifecycle-base resolver in
`common.py` so `outcome_router` can reach it without the `feature_executor`↔`outcome_router` import
cycle; (2) thread an absolute `{review_md_path}` through both prompt render seams (writer side); and
(3) pass an absolute `lifecycle_base` at all three review-gate call sites (reader side). Then a
regression guard pins writer/reader path agreement and absolute-path-ness across both cycle seams so
the divergence cannot silently return.

## Outline

### Phase 1: Path-contract fix (tasks: 1, 2, 3)
**Goal**: The review agent writes, and the gate reads, one absolute main-repo `review.md` path across all three review-gate call sites and both prompt render seams.
**Checkpoint**: All three `dispatch_review` calls pass an absolute `lifecycle_base`; both render seams emit an absolute `{review_md_path}`; `prompts/review.md` carries no bare-relative path literal; package import smoke is clean (no import cycle).

### Phase 2: Regression guard (tasks: 4, 5, 6)
**Goal**: Pin the bug-distinguishing property — the path the agent is told to write equals the absolute path the gate reads, with no bare-relative literal — for both cycle seams and all three call sites, with the full suite green.
**Checkpoint**: New contract tests pass (render seams absolute + no bare-relative; resolution absolute + equals `base/feature/review.md`; all three call sites resolve absolute); `just test` exits 0.

## Risks
- **Resolver relocation vs lazy import.** Task 1 moves `_resolve_lifecycle_base` to `common.py` (a small refactor touching `feature_executor`) rather than doing a function-local lazy import of `feature_executor` inside `outcome_router`. Chosen to keep a single canonical resolver (spec Technical Constraint: "Reuse `_resolve_lifecycle_base()` rather than introducing a new path-resolution path") and avoid a lazy-import smell; the re-import in `feature_executor` preserves backward compatibility. The lower-churn alternative (lazy import) is viable if the move surfaces an unexpected dependency, but leaves the resolver split.
- **R3 test setup is the fiddliest unit of work.** Constructing call contexts for `_recovery_review_gate` and `_repair_completed_review_gate` is heavier than the already-fixtured `apply_feature_result` path. If full gate setup proves disproportionate, drive the primary path via existing fixtures and construct minimal contexts for the other two — but do NOT downgrade R3 to a source grep (spec forbids; grep cannot distinguish an absolute base from a relative one).
- **Negative-lookbehind in the regression test.** The absolute path ends with the same `cortex/lifecycle/.../review.md` suffix as the old relative literal, so the "no bare-relative" assertion must use a lookbehind (or equivalent) rather than a plain substring check, or it will false-positive and the guard becomes vacuous.
- **Downstream main-anchored readers are served transitively, not separately tested.** The write-to-main also feeds the morning-report drift scan (`report.py:904/920/1129`) and lifecycle-phase detection (`common.py:406`), which glob/read `review.md` under the main lifecycle base. The fix serves them transitively; they are not separately unit-tested because exercising them requires a real sandboxed-agent write (out of `just test` scope). The `could_not_run` gate remains the safety net for a denied/absent write, so this is an accepted scope boundary, not a coverage hole to close in this lifecycle.
- **Writer-side sandbox permission is a load-bearing precondition, not an in-repo change.** The absolute-path write depends on Write-tool sandbox-bypass *and* cortex-init's user-scope `allowWrite` union (Task 2 Context). No task modifies the sandbox; if a future Claude Code change made `--settings` *replace* rather than union the sandbox config (and Write-tool writes started being sandboxed), the fix would need an orchestrator-side write instead. Recorded so the dependency is visible to a builder reading only the plan.

## Acceptance
A worktree-mode overnight review of a clean-merging feature has its `review.md` written to and read from the same absolute `<main>/cortex/lifecycle/<feature>/review.md` path, so a valid `APPROVED` verdict is no longer turned into a synthetic `ERROR`/`could_not_run` by the worktree-vs-main path split. In-suite proxy: the new contract tests pass for both cycle seams (rendered prompt absolute, no bare-relative literal, writer target == gate `review_md_path`), the resolution is absolute and equals `base/feature/review.md`, all three call sites resolve absolute, and `just test` is green — pinning writer/reader path agreement so the divergence cannot silently regress.
