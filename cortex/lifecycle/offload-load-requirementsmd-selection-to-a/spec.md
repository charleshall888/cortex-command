# Specification: offload-load-requirementsmd-selection-to-a

> Epic reference: parent #336 ("Offload deterministic lifecycle mechanics to CLI verbs"). #333's byte-identical-output discipline is the **selection set + the emitted fallback string**, not events.log rows.

## Problem Statement

`skills/lifecycle/references/load-requirements.md` narrates a deterministic file-selection algorithm (~55–65% of the file) that the model hand-executes in lifecycle Review, refine Clarify/Specify, and discovery Clarify/Research. Hand-execution is unreliable and non-deterministic — four divergent fallback-note forms already appear in real artifacts. Offloading the *selection* into a `cortex-load-requirements` verb makes the algorithm enforced (not advisory), deterministic, and pinnable, and collapses the shared reference to "run it, read what it lists, inject the list." The model keeps the one irreducible part — reading file contents into context and the glossary-surfacing judgment. Benefits every lifecycle, discovery, and refine phase; worth doing now as a tractable, high-leverage offload under epic #336.

## Phases

- **Phase 1: Verb + foundations** — build `cortex-load-requirements` (module + console-script + bin wrapper), wire it in-scope (so parity stays green with no consumer touched), pin its behavior with prose-derived unit/golden tests, fix the `project.md` data bug; land green.
- **Phase 2: Reference collapse + migration** — collapse `load-requirements.md` (with its broken protocol-shape tests fixed in the same commit), migrate the consumer references to the verb while retaining their protocol citation, reconcile the fallback-note phrasing, regenerate mirrors, and fix the inherited #328 stale-path test debt.

## Requirements

> **Priority (MoSCoW):** all requirements are **Must** — the offload is atomic. Building the verb (Phase 1) without collapsing the prose and migrating consumers (Phase 2) would leave *two* live sources of selection truth (the verb and the hand-executed algorithm), the drift anti-pattern the offload exists to remove. There are no Should/Could items; the **Won't** items are in Non-Requirements. R15 (the broader #328 sweep) is Must by the user's explicit bundle decision.

> **Fidelity framing:** the verb reproduces the prose's *intended* selection set, with **two deliberate, documented corrections** to prose defects (not silent drift): (i) empty/whitespace tags are stripped — the prose's bare substring rule treats `""` as a substring of every trigger and would load *all* area docs (R4); (ii) Global Context is loaded in the no-match fallback — reconciling the prose's step-1 ("always loaded regardless of tag matches") vs step-5 ("load project.md only") self-contradiction in favor of step 1 (R5). Both are stated here so "set-fidelity" is not read as byte-for-byte prose mimicry.

3. **Input contract — `--feature <slug>` with graceful fallback, byte-equal to no-feature.** `--feature <slug>` reads `cortex/lifecycle/{slug}/index.md`, extracts `tags:`, runs the full algorithm. Absent/tag-less index **or** omitted `--feature` ⇒ fallback (`cortex/requirements/project.md` + Global Context only + fallback note), never an error. The `--feature`-on-absent-index output MUST be **byte-identical** to the no-`--feature` output. Acceptance: unit tests for (i) `--feature` + matching tags → area docs loaded [positive control], (ii) `--feature` + absent index → fallback, (iii) no `--feature` → fallback; **plus an explicit equality assertion that (ii) stdout == (iii) stdout byte-for-byte**. Test file exits 0. **Phase**: Verb + foundations

4. **Algorithm fidelity with discriminating tests.** Match against the **trigger** text only (left of ` → `); ASCII case-insensitive **substring** match; **whole-tag** (do not split `harness-adaptation`); dedup; **strip empty/whitespace tags** before matching (documented correction i); `[]` ≡ absent ≡ missing-field ⇒ fallback. Acceptance: each negative control is paired with a positive control **in the same test**, on a fixture engineered to discriminate — (a) `tags: ["", "pipeline"]` loads ONLY the pipeline doc (not all; not none), (b) `tags: ["requirements"]` matches a trigger phrase containing "requirements" if one exists but NOT the area-doc *paths* (fixture includes a path with "requirements" in it and a trigger without), (c) a fixture with trigger `harness` present and tag `harness-adaptation` → the area doc is NOT loaded (whole-tag), while tag `harness` → it IS loaded. Test file exits 0. **Phase**: Verb + foundations

5. **Output shape — full repo-relative paths only, deterministic order, defined dedup, note on stderr.** Emit to **stdout**, newline-delimited (one path per line, single trailing newline), paths only (never file contents): `cortex/requirements/project.md` first (full repo-relative path, NOT bare `project.md`); then `## Global Context` entries in file order (each a full repo-relative path resolved literally against repo root; absent ⇒ ` (skipped: file absent)` suffix); then matched area docs in `## Conditional Loading` order. **Dedup is by resolved path**: a path appearing both as a Global Context entry and as a tag-matched area doc is emitted **once**, in its Global Context position, carrying its skip-status (Global Context placement wins). The **fallback note goes to stderr** (keeping stdout a pure path list). Acceptance: a unit test asserts (a) the first stdout line is exactly `cortex/requirements/project.md`, (b) stdout contains no requirements-file body text (every non-blank stdout line resolves to a path or a path + ` (skipped: file absent)`), (c) a duplicate Global-Context/area-doc path appears once. Test file exits 0. **Phase**: Verb + foundations

## Non-Requirements

- **Reading/concatenating requirements file contents.** The verb prints paths only; the model still reads file bodies into context.
- **De-duplicating the glossary-surfacing line's 4 copies.** Separate concern; the RULE_CARRIER ratchet already designates `load-requirements.md` canonical.
- **Building `--tags` or `--backlog-slug` arms.** Shelved as future escape hatches; no current consumer justifies them. (The standalone-refine-Context-A path is project.md-only today under faithful protocol execution — `refine/clarify.md` §1 reads only title/description/status, not tags, and no `index.md` exists at Clarify — so `--feature`-only introduces no regression; the `--backlog-slug` arm is the future path if deterministic backlog-tag loading there is ever wanted.)
- **Unifying every historical fallback-string rendering.** The invariant is forward-deterministic verb output + correct selection set, not reproducing past LLM renderings.
- **Touching the 3 SKILL.md path-propagation manifest entries** (the lifecycle, refine, and discovery SKILL.md files). They stay valid because `load-requirements.md` remains a thin reference.
- **Giving `specify.md` a live verb call.** Its §1 load is skipped by all callers; it remains a citation-only carrier.

## Edge Cases

- **No tags / no area docs / `tags: []` / absent index** → `cortex/requirements/project.md` + Global Context only (to stdout) + generic fallback note (to stderr). Not an error.
- **Empty-string or whitespace tag** (`["", "pipeline"]`, trailing comma) → stripped before matching (loads only real matches, not all).
- **Tag that substring-matches an area-doc path but not a trigger** (e.g. `requirements`) → no spurious load (trigger-only matching).
- **Hyphenated tag** (`harness-adaptation`) → matched as a whole token, not split.
- **Global Context file absent on disk** (e.g. `glossary.md` today) → emitted as `cortex/requirements/glossary.md (skipped: file absent)`, not silently dropped.
- **A path is both a Global Context entry and a tag-matched area doc** → emitted once, in Global Context position, carrying its skip-status.
- **Multiple tags matching one phrase / one tag matching multiple phrases** → each matched path loaded once (dedup by resolved path).
- **Discovery slug** → discovery omits `--feature` (it has no `cortex/lifecycle/{slug}/index.md`); fallback applies.

## Technical Constraints

- Edit canonical sources only (`skills/`, `bin/cortex-*`); `plugins/cortex-core/**` mirrors regenerate via `just build-plugin` — commit canonical + mirror together (drift hook). The `cortex_command` package is wheel-only (not mirrored).
- **Sequential dispatch, not worktree** — `just test` runs the editable install, so a worktree would verify stale `cortex_command` code (project memory). Reinstall the editable wheel before invoking the console script / running `just test`.
- **W003 wiring:** a deployed `cortex-*` name with no in-scope reference fails `cortex-check-parity` (run `--staged`, non-lenient, at pre-commit). The Phase-1 verb deploy MUST co-locate (same commit) with an in-scope, path-qualified `bin/cortex-load-requirements` reference (the R9 parity test). A `python3 -m cortex_command…` module path is NOT a wiring signal.
- **Collapse↔test coupling:** R10 (collapse) and the R14 protocol-shape test fixes it forces MUST land in the same commit; otherwise `just test` is red in the interval (no pre-commit pytest gate catches it, but it violates "land green").
- **Concurrent-session discipline (shared main):** commit with an explicit `git commit -- <pathspec>` so a concurrent session's staged files don't leak in; `just build-plugin` regenerates ALL mirrors, so verify only this work's mirror diff is staged (project memory: trunk shared-index + drift-hook coupling).
- Gates that apply: parity (E002/W003), skill-path (SP001/SP002 — bare PATH command, net-simplifies), bare-Python L201 (console-script name only), events-registry (no event), L1 surface ratchet (unaffected — frontmatter only).
- Read-only verb is a degenerate case of "Skill-helper modules": console-script entry applies; events-registry does not.
- Keep set-fidelity (selection set == prose's *intended* algorithm, with the two documented corrections) and string-determinism (pinned stdout) as **two separate invariants**; verify set-fidelity with a prose-derived oracle + a live-`project.md` selection test, not self-captured goldens.
