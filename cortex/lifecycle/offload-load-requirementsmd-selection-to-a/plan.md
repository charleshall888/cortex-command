### Task 1: Create the `cortex-load-requirements` verb module (pure selection fn + CLI shim)
- **Files**: `cortex_command/lifecycle/load_requirements_cli.py`
- **What**: Implement the deterministic requirements-file selection algorithm as a pure resolve function plus a thin `main()` argparse shim that prints a newline-delimited repo-relative path list to stdout and the fallback note to stderr. (R1, plus the R5 emission shape, R4 matching rules, and the R7 no-events property.)
- **Depends on**: none
- **Complexity**: complex
- **Context**:
  - Model the pure-fn + shim split on `cortex_command/backlog/resolve_item.py` (`resolve(...) -> result` + thin `main()`); model the argless-shim + project-root resolution on `cortex_command/lifecycle/backlog_backend_cli.py` (uses `cortex_command.common._resolve_user_project_root()`); model YAML-frontmatter parsing of `index.md` `tags:` on `cortex_command/backlog/load_parent_epic.py` (the closest analog — a "load" verb that takes a slug and parses frontmatter itself).
  - CLI surface: `cortex-load-requirements [--feature <slug>]`. `--feature <slug>` reads `cortex/lifecycle/{slug}/index.md`, extracts the `tags:` list (inline YAML form `tags: [a, b, c]`); absent/tag-less index **or** omitted `--feature` ⇒ fallback path. Never errors on a missing index/feature (R3).
  - Selection algorithm (parse `cortex/requirements/project.md`): `## Conditional Loading` bullets are `- <trigger phrase> → <repo-relative path>` (separator ` → `, U+2192) — split on the **first** ` → ` and match against the **trigger text only** (left of it); a `## Conditional Loading` bullet with **no** ` → ` is skipped (no match, no crash — never `IndexError`). `## Global Context` bullets are `- <repo-relative path>`. Match rule: **pure ASCII-casefold substring** — `tag.lower() in trigger.lower()` over ASCII, so a short tag that is a substring of a longer trigger token DOES match (e.g. tag `pipe` matches trigger word `pipeline`); this is intentional substring, NOT word-boundary matching. "**Whole-tag**" means the tag string is used whole (do not split `harness-adaptation` into `harness`+`adaptation`), not word-boundary. **Strip empty/whitespace tags before matching** (documented correction i — prose's bare-substring rule treats `""` as a substring of every trigger); `[]` ≡ absent ≡ missing-field ⇒ fallback (R4).
  - Output order + dedup (R5): `cortex/requirements/project.md` first (full repo-relative path, NOT bare `project.md`; if `project.md` is itself absent on disk, emit it with the ` (skipped: file absent)` suffix too — the verb never directs reading a non-existent file); then every `## Global Context` entry in file order (each resolved literally against repo root — **no bare-filename heuristic**; absent on disk ⇒ ` (skipped: file absent)` suffix); then matched area docs in `## Conditional Loading` order. Dedup is by resolved path — a path appearing both as a Global Context entry and a tag-matched area doc is emitted **once**, in its Global Context position, carrying its skip-status (Global Context placement wins). Two further collision classes (named so they are not left undefined): a `## Global Context` entry equal to `cortex/requirements/project.md` is emitted **once**, as the unconditional first line (the first-line slot wins; it is not re-emitted in the Global Context block); a path listed **twice within** `## Global Context` is emitted **once**, at its first occurrence. Paths only — never file contents.
  - Fallback (documented correction ii): the no-match path still emits `project.md` + Global Context entries to stdout (reconciling the prose step-1/step-5 self-contradiction toward step 1, which always loads Global Context). The **fallback note goes to stderr**, keeping stdout a pure path list. The note string is a module-level constant: `no area docs matched for tags: {tags}; loaded project.md only`, with `{tags}` rendered `[t1, t2, ...]` (unquoted, comma-space; empty ⇒ `[]`).
  - No events: the module imports/writes nothing to `events.log` and registers no event token (R7).
  - Relocate the prose's `## Why this protocol` rationale into the module docstring as a one-line intent pointer ("the verb selects the minimal tag-relevant requirements set, avoiding under/over-loading"); the `## Matching Semantics` content becomes the verb's behavior and `--help` text.
- **Verification**: run `python3 -m cortex_command.lifecycle.load_requirements_cli` from repo root — pass if exit 0 AND the first stdout line is exactly `cortex/requirements/project.md`; AND `grep -c '"event"' cortex_command/lifecycle/load_requirements_cli.py` = 0 (pass if 0). Fail otherwise.
