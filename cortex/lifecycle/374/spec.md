## Problem Statement

Lifecycle phase sequencing is routed today by prose conventions the model follows voluntarily, with phase derived from artifact presence and events written by multiple independent authorities. This build ships the served loop: a `next` verb that serves state, guard evaluation, fragment reference, and advance contract from a single wheel-owned transition table; an `advance` verb that executes transitions under a locked claim/commit protocol; and a protocol handshake making the prose and wheel layers skew-detectable, with a session-start healer. **Honest delivery claim:** this build establishes one authoritative *read* path (events-first with legacy artifact fallback) and routes every *in-repo* transition-writing site through the table's discipline; full single-oracle status — retiring the externally-callable legacy verbs — lands at the later protocol-floor bump, and the enumerated permanent exceptions (statusline derivation, pipeline-run FSM) remain by design. The operator benefits: predictable sequencing with an explainable derivation trace, structurally-tested writer discipline, and a documented roll-forward exit.

## Phases

Each phase is independently shippable and leaves the repo consistent — the descope lever is the phase boundary, not per-requirement triage.

- **Phase 1: Foundation hardening** — close the writer census structurally, ship the claim/commit locking primitive, pin one log resolver.
- **Phase 2: Protocol substrate (step 0)** — stamp the protocol into existing verb payloads, ship the wheel-side compat evaluator, the wrappers, and the session-start healer. (Loop-side halt behavior lands with the loop, Phase 5.)
- **Phase 3: Transition table and verbs** — `transition_table.py`, `next`, `advance`, `describe`, events, ADRs.
- **Phase 4: Phase-authority cutover** — events-first single resolver; the two transition-decision writers folded; reverse-direction golden; harness arms.
- **Phase 5: Loop integration and exit apparatus** — SKILL.md loop body, in-repo caller rewiring off the legacy verbs, kept-pauses parity transfer, roll-forward procedure, docs.

## Edge Cases

- Two advance invocations race one transition → one claims; the other is refused at claim ("in-flight transition") or at commit ("state moved since claim") — exactly one `advance_committed` (R3).
- A legacy typed-subcommand write lands between advance's claim and commit → commit-time re-reduce detects it, refuses completion, and names the interleaved row; side effects already run are idempotent and the orphaned claim is recoverable by `invocation_id` (R3, R12). In-repo, this class shrinks to near-zero because shipped prose no longer commands the typed transition verbs (R17).
- Crash between claim and side effects → retry detects the orphaned `advance_started` by `invocation_id`, resumes via existence probes; never a global halt (R12).
- Worktree session → machine verbs and lock agree on the main-root log; a CWD-resolved legacy append to the worktree copy is surfaced by the (permanent) mismatch detector (R4, R15).
- Legacy `feature_paused` row without kind/slug → reducer defaults to most-restrictive (R5).
- Stale plugin + fresh wheel → legacy invocation keeps working (coexistence); fresh plugin + stale wheel → per-verb expectation check (plugin-side expectation vs served payload) halts with remediation; SessionStart background install initiates healing, loop stays halted until the wheel matches (R6–R8).
- Operator hand-appends a recovery row → dedup tolerates it; `cortex-lifecycle-event log` is the documented sanctioned path (R12).
- Archive-shadowed feature dir → append path refuses; projector never writes status for it (R12).
- `plan.md` checkboxes hand-edited without advance on a machine-rows feature → events win; permanent mismatch detector reports the divergence (R15).
- Config sets a dormant/unknown workflow key → existing loud-warning behavior (A4) unchanged.

## Technical Constraints

- events.log stays the only durable state (ADR-0001/0004); machine is a pure transition function; all emission additive-only under ADR-0020 via the shared primitive.
- Network calls (gh) never execute under the log flock (R3's claim/commit split exists for this).
- ADR-0019 discipline for composed bodies; `next`/`advance` sanctioned by ADR-0024, not silently under 0019.
- ADR-0016: status projection cortex-backlog-only.
- House verb style: never-crash exit-0 `{"state": ...}` envelopes; exit 2 reserved; `KNOWN_STATES` closed tuples; `_reject_unsafe_slug` first (describe reads arbitrary feature dirs — traversal is the real injection surface).
- Pre-commit gates: dual-source mirror regen staged with sources; parity (W003/E002 ordering); events-registry `--staged`; SP001/SP002; L201.
- Protocol integer per-layer declaration (wheel serves / plugin expects), compat-range never exact-equality, per-verb-payload checking (wrappers resolve per-module — mixed generations possible on one machine).
- Skew paths must be exercised by scripted tests (branch-c working-tree fallback means this repo never hits them naturally).
- Dual-emission window contracts only at a protocol-floor bump decided by the operator (telemetry-informed, not calendar).

## Proposed ADR

### Proposed ADR: 0024-served-lifecycle-verb-class-and-coexistence
Bounds ADR-0019 for the served-verb class: `next`/`advance` read config, resolve identity, evaluate guards, and serve instructions — categorically beyond the dumb-arg-actor rule; sanctioning them silently under 0019 is the scope creep 0019 warns against. Also bounds the new skill-invocation directive class (the envelope may name only skills whose invocation condition is machine-readable state) and concedes the release-cadence coupling cost of the wheel-owned gate matrix. Contains the coexistence policy: absorbed verbs stay callable until a protocol-floor bump (events-registry deprecation template, differing trigger stated); in-repo prose invocations removed at Phase 5; advance-emitted legacy-vocabulary rows carry an optional `invocation_id` distinguishing them from independent legacy emissions during the window.

### Proposed ADR: 0025-events-as-phase-authority-with-legacy-fallback
Records the exit-gate decision: events.log is the authoritative phase source wherever machine rows exist; artifact derivation demotes to legacy fallback inside one shared resolver whose callers are enumerated and migrated; the two transition-decision writers are folded, not sanctioned; dual authority is refused. Permanent exceptions stated by name (statusline derivation, pipeline-run FSM). Trade-off stated honestly: forfeits the cheap prose-side rollback (the roll-forward procedure is the standing exit) in exchange for one predictable oracle; the mismatch detector runs permanently as the divergence tripwire.

### Proposed ADR: 0026-cortex-core-background-install
Reverses the #235/#263 cortex-overnight-only scope for the SessionStart background-install pattern on changed-premise grounds: cortex-core prose now depends on specific wheel verbs, and under events-authority a skewed consumer cannot advance. Healing is eventual (async initiate at SessionStart, documented skip predicates); the loop's per-verb detect-and-halt remains the correctness boundary; mid-session wheel replacement semantics stated (cross-protocol → next-call halt; same-protocol → compatible by contract).

(ADR-0020 receives an in-file amendment for the claim/commit advance schema — an evolution of the same decision, not a new ADR.)
