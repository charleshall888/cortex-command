## Web Research

- **Handshake precedents:** Terraform's plugin protocol (multi-major-version serving via distinct namespaces, negotiate-down, additive-minor/breaking-major split) is the strongest direct precedent; Docker's `min(client_max, server_reported)` negotiation is the algorithm shape. Kubernetes' skew-policy table is the *documentation* artifact to emulate — but it does not document violation-detection mechanics; that part must be designed, not copied. No surveyed system best-effort-interprets an out-of-range version: negotiate down or refuse. With no daemon and no remote flag service, the wheel-side protocol-version gate **is** the only kill switch available.
- **Two-row transitions:** golang-migrate's binary `dirty` flag (one interrupted migration halts everything until manual surgery) is the anti-pattern the `advance_started`/`advance_committed` design avoids — an orphaned `started` row is diagnosable and resumable via its key. Idempotency keys must be **deterministic, business-derived, persisted in the event itself** (never regenerated per retry; AWS durable-execution: generate once, reuse across every attempt). Idempotency must hold at every consuming stage, not just advance's front door.
- **Schema evolution:** protobuf discipline transfers — treat state IDs / event types / gate IDs as append-only, reserve-on-deprecate, never recycled.
- **Dual-emission windows:** expand→migrate→contract; size the window by **consumption telemetry, not release count** (LinkedIn: 30-day zero-read bake before contracting).
- **Rollback for unpinned floating installs:** npm's real lever is dist-tag repointing + deprecation warnings — it changes what *new* installs get, never touches installs that already pulled. For this repo, "rollback" of the wheel realistically means re-pointing/pinning tags for future installs; auto-revert dead-man's-switch patterns (Junos commit-confirmed) don't transplant (no daemon, no timer authority). Reverse-direction compatibility goldens do the job a revert-timer would do in a daemon architecture — validating why arm h is load-bearing.

## Tradeoffs & Alternatives

**Fragment flavor — selector recommended.** Selector (`next` returns a fragment path; model Reads it): single-copy corpus, no new sync surface, no ADR-0009 pre-resolution obligation, preserves authoring locality, fits the loop-body token budget, live in-tree precedent (ADR-0018 parse-args), cheapest to abandon. Conceded weakness: "convention, not guarantee" — nothing enforces the Read. Server (wheel-served content): closest to a structural guarantee but 1–2.3K tokens/call, ADR-0009 pre-resolution obligation, a **third sync surface with no pre-commit trigger**, moves the highest-churn edit surface behind wheel releases, hardest to roll back — rejected as blanket policy. Hybrid: earns its dual-path complexity only for a small named set of stable machine-ownable sequencing fragments — scoped exception at most, not grown speculatively. PostToolUse-injection: greenfield hook engineering, brittle matching ("never Bash-text regex" lesson), looks structural without being reliably so — deferred until selector's convention gap is shown to bite.

**Handshake — detect-and-halt now; auto-remediation deferred.** Detect-and-halt: zero new architecture, strict improvement over B1's accepted zero-skew-protection posture, delivers operator req 8, preserves the twice-reaffirmed #235/#263 scope boundary. Auto-remediation (SessionStart background-install ported to cortex-core): real working pattern exists in-tree but requires reversing a twice-made decision, has an unsolved mid-loop-reinstall race for the interactive path, and pays for a failure class with zero recorded occurrences. Hold it to the same evidence bar the epic applied to C itself. (See Open Questions for the narrower version-sync-hook variant the adversarial pass raised.)

## Rollback & Release Engineering

**Release mechanics:** every merge to `main` auto-releases (`auto-release.yml` bumps version + `CLI_PIN[0]` for cortex-overnight, tags; `release.yml` builds the wheel; no un-publish path, no hold lever short of disabling the workflow). `CLI_PIN` exists **only** for cortex-overnight; **cortex-core has zero skew protection** — with marketplace auto-update on, new SKILL.md prose can reach consumers before the matching wheel is installed. All 9 existing lifecycle bin wrappers honor `CORTEX_COMMAND_FORCE_SOURCE`; `cortex-lifecycle-resolve` has no wrapper at all.

**What rollback means per layer:** prose/plugin (SKILL.md loop body, references) — hot-revertable, one git revert + mirror regen, reaches consumers next marketplace refresh. Wheel (verbs, table) — roll-forward only: an operator can manually pin an older tag (documented generically in docs/setup.md), but there is no un-release. Emitted machine events — permanent; old readers must parse mixed legacy+machine logs forever (partially structural already: `reduce_lifecycle_state` no-ops unknown events, `detect_lifecycle_phase` only greps `feature_complete`; the genuinely untested surface is prose/jq consumers and the dashboard — arm h is narrower than "no protection exists" but still has no code). Net: "roll it back" is true for *the choice to keep invoking the loop*, false for *undoing what happened*.

**Roll-forward exit procedure (currently unwritten — a spec deliverable):** (0) preconditions shipped with C: force-source wrappers incl. `cortex-lifecycle-resolve`; (1) trigger + owner + decision-record home named up front, not ambient; (2) stop routing: revert SKILL.md prose to phase-table routing (one commit); (3) leave wheel verbs in place, callable, forever-interpretable; (4) verify tolerant reading via the reverse-direction golden (arm h); (5) quarantine event vocabulary as `deprecated-pending-removal` registry rows, 30-day window, **with a named owner** — #377's 12 already-lapsed rows prove windows accumulate debt unowned; note `--audit` is off the critical path and demonstrably unactioned, so name who runs it; (6) dual-emission stays live through the grace period.

## Adversarial Review

Findings folded into the sections above where they corrected fact (module naming, SKILL.md cap, census severity, arm-h scope, version-sync existence). The load-bearing structural findings:

1. **CAS is unsatisfiable with the current primitive — hard blocker.** `_append_event_atomic` holds the flock for one append only; no hold-lock-across-read-validate-append API exists; every existing "idempotent presence check" is check-then-append TOCTOU. `advance` needs a **new locked read-validate-append primitive** in `lifecycle_event.py`. Uncosted by other angles; now a named deliverable.
2. **Destructive writers void the permanence assumption.** The `os.replace` writers (critical-review, discovery) can silently erase an `advance_started`/`advance_committed` row mid-race — breaking the two-row crash reasoning and the "emitted events are permanent" premise of the rollback story. Critical-review auto-triggers inside the loop C serves. Writer triage (convert both to `log_event_at`; route refine/escalator bare appends through `log_event`; wrap or delete `orchestrator-round.md:387`) is small, behavior-preserving, and **must precede advance**.
3. **Phase authority is artifact-derived today — the table creates a competing oracle.** `detect_lifecycle_phase` derives phase from artifact presence (review.md verdict regex, plan.md checkboxes, spec/research existence); only `feature_complete` comes from events. `implement-rework`/`escalated` are states the table cannot enter by event append; no lock covers plan.md edits, so CAS over artifact-derived state is impossible. With `advance_lifecycle.py` this is a three-authority system. **The spec must decide phase authority openly** — see Open Questions; do not ship dual authority.
4. **Structural-guarantee strength and rollback cheapness are the same dial.** The cheap prose-side exit is honest only while phase authority stays artifact-based; migrating authority into events (the only route to a real guarantee) makes resolver/dashboard/overnight depend on event rows and the cheap exit is gone. The operator cannot have both; the spec picks openly.
5. **Handshake read-side correction.** Passing `--protocol N` to old-wheel verbs hits argparse's unknown-argument exit 2 — colliding with both the contract-violation channel and the wrapper wheel-not-found channel. Design read-side instead: stamp a protocol field into payloads; the loop checks field presence per verb call (absence = legacy); skew on new verbs is a `{"state": "protocol-skew"}` exit-0 payload per house style. Wrappers resolve per-module, so one machine can serve mixed code generations — the check must be per-verb-payload, not once-per-loop. Branch (c) working-tree fallback means this repo never exercises the skew path — it ships untested where it matters unless tests simulate it.
6. **C-as-scoped is largely convention + telemetry.** Selector flavor + detect-and-halt + deferred pause tooth + advisory table = the only new structural bind is advance refusing out-of-order event-recorded transitions, and prose routing through advance is itself prose-enforced. By ADR-0008's own doctrine, advance's refusal is guard output. This is acceptable — but ADR-0024 must say it honestly rather than claim a guarantee.
7. **The waived gate is undecidable post-hoc without minimal instrumentation.** With criteria and capture both waived, "the loop prevented misrouting" vs "there was nothing to prevent" cannot be distinguished; keep-by-inertia is the default outcome. Mitigations recorded (operator's call, per the fiat): record the fiat in ticket/ADR; optionally trial on disposable feature slugs to keep new vocabulary out of real logs until the keep decision; the roll-forward procedure names its trigger and owner regardless.
8. **Latent verifier flakes go live:** `complexity_escalator._verify_last_event` and refine's read-after-write last-line assertions produce false failures under any concurrent append — the loop multiplies write frequency. Fix in the same triage pass.
9. **Naming collision:** the resolver's envelope already carries a `next` directive field with routing semantics; a `next` verb gives loop prose two different `next`s to obey. Rename or namespace one in the spec.
10. **Pause consumption without C+1 has a real floor and a real trap.** Event-backed enforcement is available now (`spec_approved`, `plan_approved`, `feature_paused` exist as typed events — advance can structurally require them). The trap: the most-restrictive-default obligation colliding with judgment-suppressed/config-conditional kinds would make advance refuse transitions for which no evidence can ever exist, forcing an habitual override flag — reconstituting convention plus a footgun. Scope advance's pause enforcement to event-backed pauses only; judgment/config kinds are describe-only metadata in C.

## Open Questions

- **Writer triage scope** — resolved: hard precondition inside 374's plan (first implementation tasks): convert critical-review/discovery `append_event` to `log_event_at`, route refine/escalator bare appends through `log_event`, wrap-or-delete `orchestrator-round.md:387`, fix the read-after-write verifiers.
