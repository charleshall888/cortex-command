# Specification: cortex init scaffolds cortex/.gitignore for umbrella transient artifacts

## Problem Statement
Today `cortex init` gives a consumer repo a binary choice via a commented `# cortex/` toggle it appends to the *root* `.gitignore`: commit the entire `cortex/` umbrella (the default — sweeping every transient scratch file into history alongside real backlog/lifecycle/requirements artifacts) or uncomment `# cortex/` and ignore the whole umbrella including project state. There is no selective middle setting. Shipping a tracked nested `cortex/.gitignore` with the umbrella scaffold gives every consumer repo the same selective ignoring this repo hand-maintains — scoped to transient artifacts only (residue, session/lock files, runtime logs, indexes), independent of the host root `.gitignore`. The operator has chosen a **copy-if-absent** model: the scaffolder lays the file down once on first init and never auto-overwrites it; updates flow through explicit `cortex init --force`, and divergence is surfaced (non-destructively) by the `cortex init --update` drift report.

## Phases
- **Phase 1: Correct & ship the template** — fix the seeded content bugs and ship `cortex/.gitignore` via the existing copy-if-absent scaffold path, with tests.
- **Phase 2: Repo cleanup & doc accuracy** — de-dup the repo root `.gitignore` and reconcile now-imprecise residue wording (with plugin-mirror regen).

## Requirements

1. **Corrected, depth-complete seed content**: The shipped `cortex/.gitignore` ignores transient artifacts at *all* depths, fixing the bugs research found in the currently-seeded file (`cortex/.gitignore`). At minimum: residue/session/lock/activity rules must match archived paths (`cortex/lifecycle/archive/<slug>/...`), not only active-feature paths — the seeded single-level `lifecycle/*/critical-review-residue.json` misses archives; and `overnight-events-*.log` must widen to also catch `overnight-events.log`. **Acceptance** (pattern-only test via `--no-index`, which — unlike bare `git check-ignore` — ignores tracked-file precedence and tests the rule itself): from a repo containing the shipped file, `git check-ignore --no-index cortex/lifecycle/archive/x/critical-review-residue.json` exits 0; `git check-ignore --no-index cortex/lifecycle/feat/critical-review-residue.json` exits 0; `git check-ignore --no-index cortex/lifecycle/archive/x/.session` exits 0; `git check-ignore --no-index cortex/lifecycle/overnight-events.log` exits 0. Pass if all exit 0. **Phase**: Correct & ship the template

2. **Must-track project state never ignored**: The template ignores ONLY transient artifacts and never tracked work-products; the `learnings/` rule stays narrow to `recovery-log.md` even at archive depth. **Acceptance** (pattern-only via `--no-index`): `git check-ignore --no-index` exits 1 (no match) for each of `cortex/backlog/x.md`, `cortex/lifecycle/feat/spec.md`, `cortex/lifecycle/feat/learnings/outline.md`, `cortex/lifecycle/archive/x/learnings/outline.md`, `cortex/requirements/project.md`, `cortex/adr/x.md`. Pass if all exit 1. (The archive-depth `learnings/outline.md` case proves the narrow `recovery-log.md` rule does not over-match sibling work-products at archive depth.) **Phase**: Correct & ship the template

3. **Copy-if-absent via the scaffold path**: Ship the template at `cortex_command/init/templates/cortex/.gitignore` so the existing `scaffold()` writer lays it into the consumer repo only when absent (`scaffold(overwrite=False)`: `if dest.exists() and not overwrite: continue`) — no clobber, no churn — with no new writer code. **Acceptance**: a `cortex_command/init/tests/` test asserts a fresh repo gets `cortex/.gitignore` (`exists()` and `st_size > 0`); a second test writes a sentinel `cortex/.gitignore`, re-runs `cortex init` (default and `--update`), and asserts the file is byte-identical (sentinel survives). `just test-init` exits 0. **Phase**: Correct & ship the template

4. **Hash participation with copy-if-absent semantics (staleness visible, never auto-overwritten)**: The template participates in the init-artifacts hash (added to `_HASH_INPUT_TEMPLATES`) so the coverage test stays green and a *diverged* on-disk file is surfaced by the `cortex init --update` drift report — but an existing file is NEVER auto-overwritten on the default or `--ensure` paths (only `--force` applies an update). This is deliberate: under copy-if-absent, `--ensure` advancing the stored hash while skipping an existing file is the intended no-refresh contract, not a desync, and the drift report (not silent staleness) is the upgrade signal. **Acceptance**: a `cortex_command/init/tests/` test runs `cortex init --ensure` (and default) against a repo with a pre-existing hand-edited `cortex/.gitignore` and asserts the file is left byte-identical; a second assertion confirms a content-diverged `cortex/.gitignore` appears in the `--update` drift report (the existing `drift_files` surface). `just test-init` exits 0. **Phase**: Correct & ship the template

5. **`--force` updates with backup**: `cortex init --force` backs up an existing `cortex/.gitignore` and overwrites it with the shipped content, via the existing `--force` backup path. **Acceptance**: a `cortex_command/init/tests/` test asserts that after `--force` on a repo with a pre-existing `cortex/.gitignore`, the prior file is present under `cortex/.cortex-init-backup/<ts>/cortex/.gitignore` and the live file equals the shipped content. `just test-init` exits 0. **Phase**: Correct & ship the template

## Non-Requirements
- Does NOT auto-overwrite an existing consumer `cortex/.gitignore` on default init or `--ensure` — deliberate per the chosen copy-if-absent model; this departs from the ticket's literal "versioned, replace-on-version-bump" wording. Divergence is surfaced by the `--update` drift report; the apply path is `cortex init --force`.
- Does NOT add any versioning machinery — no in-file version sigil, no per-file hash state surface, no dedicated writer, no `_CORTEX_GITIGNORE_VERSION` constant. (The file rides the existing template/scaffold/hash plumbing.)
- Does NOT untrack the ~78 already-committed residue files in this repo (`git rm --cached`) — git only ignores untracked paths; this is a separate, ticket-deferred decision. Consequently the residue wording in this repo's docs is reconciled (Req 8), not deleted on a false "now-ignored" premise.
- Does NOT mirror the init package into any plugin — `cortex_command/init/` ships only in the CLI wheel; only the `skills/lifecycle/references/*.md` edits are plugin-mirrored.
- Does NOT modify or invoke the relocation migration (`cortex_command/init/_relocation_migration.py`) — dead code, not wired into init, moot for this work.
- Does NOT change the root `# cortex/` toggle behavior in `ensure_gitignore`.

## Edge Cases
- **Consumer already has a hand-authored `cortex/.gitignore`** → left byte-identical on default/`--ensure`/`--update` (copy-if-absent); never clobbered except by explicit `--force`, which backs it up first. A diverged file is listed (non-destructively) in the `--update` drift report — including the benign case of a consumer who added their own ignore lines (accepted informational cost of hash participation).
- **Consumer on a hash-matched install deletes `cortex/.gitignore`** → `cortex init --ensure` Case (i) (`marker_hash == installed_hash`) returns before the scaffold pass, so `--ensure` alone does NOT restore it; `cortex init` / `cortex init --update` (which always run the scaffold pass) lay it back down. This is parity with the existing `ensure_claude_md_authorization` fence behavior, not a new gap.
- **Consumer uncomments root `# cortex/`** (ignores the whole umbrella) → the nested file is inert/moot (git does not descend into an ignored directory); no conflict.
- **Archived artifacts** (`cortex/lifecycle/archive/<slug>/...`) → ignored, via the depth-complete globs (Requirement 1). Note: in *this* repo such artifacts that are already tracked stay tracked (git does not retroactively untrack); the rule only prevents *future* untracked leaks.
- **Negation (`!path`) rules** → inert once the umbrella toggle is on (git can't re-include under an ignored parent); the shipped file uses no negations, but the limitation is noted in the file header.

## Changes to Existing Behavior
- ADDED: `cortex init` writes `cortex/.gitignore` into consumer repos (copy-if-absent), changing the default tracked surface of a freshly-initialized umbrella.
- ADDED: `cortex init --update`'s drift report now lists `cortex/.gitignore` when the on-disk file diverges from the shipped template (informational, non-destructive).
- MODIFIED: residue-example wording in `skills/lifecycle/references/complete.md` (and reviewed in `post-refine-commit.md`) reconciled — the named residue is gitignored in *fresh consumer repos*, while in this repo it stays tracked-and-swept until a separate `git rm --cached`; the enumerated-staging guard is preserved.
- MODIFIED: this repo's root `.gitignore` no longer carries the cortex-scoped transient rules (now owned by the nested `cortex/.gitignore`).
- REMOVED: the stale `.gitignore:41` line reference in `cortex_command/overnight/runner.py`.

## Technical Constraints
- **`git check-ignore` semantics**: bare `git check-ignore <path>` reports a *tracked* file as not-ignored (exit 1) regardless of a matching rule; pattern-correctness acceptance therefore uses `git check-ignore --no-index <path>` (a pure pattern test). All `git check-ignore` acceptance criteria above use `--no-index`.
- **Widened globs span tracked paths, made safe by git, not by pattern scoping alone**: widening residue/`.session` rules to archive depth (Req 1) necessarily makes them *pattern-match* real tracked paths (e.g. tracked `archive/<slug>/.session`). This is safe because git never retroactively untracks a matched path — the rule only suppresses *untracked* future scratch. Must-track non-overlap (Req 2) is guaranteed by keeping the rules basename-scoped to transient artifacts (notably `learnings/recovery-log.md`, never `learnings/`), verified by the `--no-index` checks including at archive depth.
- **Copy-if-absent rides the existing scaffold/hash plumbing**: the template lives at `templates/cortex/.gitignore` and is added to `_HASH_INPUT_TEMPLATES`; `scaffold(overwrite=False)` provides copy-if-absent, `--force` provides backup+overwrite, and `drift_files()` provides the divergence signal. No dedicated writer, no `_iter_template_files` exclusion, no separate version state. The init code is wheel-only; this introduces no plugin-mirror coupling.
- **Seed correctness is first-ship-final for existing installs**: with no auto-overwrite, existing consumers only get content fixes via `--force` (surfaced by the drift report). All research-identified content bugs (archive-depth globs, `overnight-events*.log`) must be fixed in this work; the drift report is the non-destructive staleness signal that points a stale consumer at `--force`.
- **Template bytes sourced from a template under the init package**, resolved via `importlib.resources`, not hardcoded (per ticket). Verified: dotfile templates are included by the wheel/sdist build (hatchling) and surfaced by `importlib.resources.files(...).iterdir()` / `os.walk`.
- **Dual-source**: the `skills/lifecycle/references/*.md` edits require `just build-plugin` and committing the regenerated `plugins/cortex-core/skills/lifecycle/references/` mirror in the same commit (the `.githooks/pre-commit` drift hook enforces this). The init/template/hash code triggers no mirror gate.
- **Content decisions resolved from research**: ship the `backlog/index.json` + `backlog/index.md` ignore rules (derived build artifacts; matches this repo's resolved position in commit `9875226c`); keep `lifecycle/*/.dispatching` for root/nested parity (no writer exists, but harmless forward-coverage that the root file also carries).
- **Closest prior-art in-repo**: `scaffold()`'s `if dest.exists() and not overwrite: continue` (scaffold.py:360) is the copy-if-absent primitive; `_HASH_INPUT_TEMPLATES` + `_compute_init_artifacts_hash` is the hash participation; `--force` backup via `backup_existing`. (Note: `ensure_claude_md_authorization` is NOT the model here — it is a versioned fence writer and is itself a `_HASH_INPUT_TEMPLATES` member; this feature wants the plain template path, not a fence.)

## Open Decisions
None. (The integration form is resolved: a plain scaffold template at `templates/cortex/.gitignore` added to `_HASH_INPUT_TEMPLATES`, served copy-if-absent by the existing `scaffold()`/`--force`/`drift_files` plumbing.)

## Proposed ADR
None considered. The copy-if-absent choice departs from ADR-0006's versioned-fence discipline (and the ticket's "replace-on-version-bump" wording), but it does not meet the three-criteria ADR gate: it is **reversible** (a later switch to a versioned writer is additive), **unsurprising** for an additive ignore file laid down once via the existing template path, and the **trade-off** (refresh vs. simplicity) was decided with the operator. The departure is recorded in Non-Requirements rather than promoted to an ADR.
