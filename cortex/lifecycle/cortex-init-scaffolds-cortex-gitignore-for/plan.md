# Plan: cortex-init-scaffolds-cortex-gitignore-for

## Overview
Ship a corrected `cortex/.gitignore` as a plain scaffold template under `cortex_command/init/templates/cortex/`, riding the existing `scaffold(overwrite=False)` copy-if-absent path, `--force` backup/overwrite, and `drift_files()` divergence signal — no new writer, no version sigil, no handler wiring. The only init-code change is one tuple entry in `_HASH_INPUT_TEMPLATES`. Then clean up the now-redundant root `.gitignore` rules, reconcile stale residue wording in two plugin-mirrored references, and fix a stale code comment.
**Architectural Pattern**: layered

## Risks
- **Copy-if-absent means existing consumers never auto-receive the bug-fixed globs** — only `cortex init --force` applies the corrected content. Be precise about the staleness signal: adding the template to `_HASH_INPUT_TEMPLATES` makes every already-initialized consumer hit `--ensure` Case (ii) once on the release that ships it, which advances the stored `init_artifacts_hash` while `scaffold(overwrite=False)` skips the existing file — and the `--ensure` drift report is emitted to (buried) stderr. So the divergence is only surfaced where a human sees it when someone runs `cortex init --update` in a terminal; the auto-`--ensure` path reaches a stable, silently-stale state. This is the spec's deliberate Non-Requirement (departs from the ticket's literal "replace-on-version-bump"); the operator chose it. Revisit only if silent upgrade of the archive-depth fix to existing installs is considered required.
- **This repo's own umbrella file gains globs that pattern-match already-tracked files** — 43 archived `critical-review-residue.json` + 3 archived `.session` (archive-depth widen), PLUS `cortex/lifecycle/overnight-events.log` (the `overnight-events-*.log` → `overnight-events*.log` widen). Benign — git never retroactively untracks, so all stay tracked-and-swept; the rules only suppress *future* untracked scratch. Note `overnight-events.log` is a deliberately-committed dashboard **seed fixture** (regenerated by `seed.py`, guarded by an `overnight-seed` sentinel), NOT transient residue — keep it out of any later "untrack the residue" (`git rm --cached`) action, which the spec already scopes out. A reviewer may double-take; the spec's Technical Constraints document why the pattern-match is safe.
- **Phase 2 touches plugin-mirrored references on `main`** — Task 5 must build + commit the mirror atomically with the canonical edit or the drift pre-commit hook rejects the commit. Flagged in Task 5 context; no standalone risk if followed.
