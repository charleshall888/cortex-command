# Specification: Artifact template cleanups — D1 keep-optional-in-§3 with explicit-conditional+enum, D2 deferred to #182, D3 index.md compression preserving backlog↔index.md graph edge, D4 ## Open Decisions optional gated on #183

## Problem Statement

Per the epic-#172 audit, three per-feature lifecycle artifact sections cost tokens both in artifact storage and in reviewer/dispatch prompts that read full specs/plans, with weak or no consumers behind them: `## Open Decisions` is empty in 65.7% of specs (71/108 verified), the default plan.md `**Architectural Pattern**` field is not currently rendered (cited as 1.4% populated — verified 2/138 plans across the corpus, only via §1b critical-tier dispatch where P8 evaluates it), and `lifecycle/{feature}/index.md` body content (H1 wikilink + intro line + per-artifact wikilink list) is redundant with frontmatter for machine consumers. This ticket trims these surfaces in a way that (a) preserves the load-bearing observability primitives flagged by the C2 critical-review correction (full frontmatter), (b) preserves the "evaluated, not decorative" property of `**Architectural Pattern**` across **both** critical and non-critical paths (C3 correction — the field is added to the default §3 template with an explicit-conditional+enum annotation so non-critical authors who commit to a category emit a closed-set value, and authors with no architectural commitment omit cleanly), (c) preserves the load-bearing Obsidian backlog↔index.md graph edge (adversarial-discovered constraint: `.obsidian/` has graph/backlink/outgoing-link core plugins enabled with no community plugins) while accepting the loss of the four intra-lifecycle index.md→artifact graph edges (mitigation: the artifacts are siblings in the same folder; Obsidian's file-tree pane and the `artifacts:` frontmatter array still surface them; the most-traversed edge in practice — backlog↔index — is preserved), and (d) sequences D4 strictly after #183's Gate 2 removal via `blocked-by:[183]`.

Net effect: ~5 lines/file saved across new lifecycles (~690 lines/year at current pace), `## Open Decisions` becomes omittable in spec.md when empty (eliminates ~71 latent ceremonial sections going forward), and template prose carries no normative-judgment soft-MUST drift. D2 (Scope Boundaries deletion) is a no-op in this ticket — #182 owns that work via the Outline replacement.

## Edge Cases

- **Obsidian graph-view regression — partial mitigation only**: `.obsidian/core-plugins.json` shows graph, backlink, outgoing-link, and properties core plugins enabled; no community plugins installed (no `plugins/` subdirectory). Obsidian's native graph view and backlink panes do NOT index frontmatter wikilinks — only body-content `[[…]]`. The chosen B1+ shape preserves the H1 wikilink `# [[{NNN}-{backlog-slug}|{backlog title}]]` and the `Feature lifecycle for [[{NNN}-{backlog-slug}]].` intro line in index.md body — **both of which target the backlog filename stem, so the only graph edge preserved is backlog↔index.md (the load-bearing one when navigating from backlog to lifecycle)**. The four intra-lifecycle edges `index.md→research.md/spec.md/plan.md/review.md` previously provided by the per-artifact wikinks (`- Research: [[…]]`, `- Spec: [[…]]`, `- Plan: [[…]]`, `- Review: [[…]]`) are ELIMINATED by this change — those navigation paths now require the Obsidian file-tree pane or the `artifacts:` frontmatter array. B1+ is chosen over B2 (max compression) because preserving the backlog↔index edge has higher load-bearing value than the marginal ~2 lines saved by also dropping the H1 + intro lines.
- **Mixed-format index.md coexistence**: Old (body-rich) archive index.md files and new (body-trimmed) post-change index.md files coexist in the same `lifecycle/` tree. `bin/cortex-archive-sample-select` (L251-266) — the only program touching index.md — checks only directory-contents membership (`contents.issubset({"events.log", "index.md"})`), not body content. Dashboard (`cortex_command/dashboard/`), statusline, hooks, and tests do not walk index.md body. No runtime breakage from mixed formats.
- **Gate 2 transient-window before #183**: D4 sequencing strictly waits on #183 via `blocked-by:[183]`. If #180's D4 landed before #183: Gate 2 (`skills/lifecycle/SKILL.md` L270-274) would still scan spec.md for `## Open Decisions` bullet count, would find 0 (because section omitted), and would skip silently — no escalation regression in practice. But the gate prose in SKILL.md would reference a now-optional section, creating a doc-inconsistency. The strict sequencing eliminates this inconsistency.
- **Refine of a backlog item that produces no Open Decisions**: Under the post-D4 template, the spec.md author may omit `## Open Decisions` entirely. The §2b Pre-Write Check "Open Decision Resolution" continues to require attempted resolution before deferral. Specs with all-decisions-resolved (like this spec.md for #180 itself) omit the section without violating any post-Specify Checklist item (S1-S6 do not require `## Open Decisions`).
- **Empirical thinness of "Gate 2: 0 fires" claim**: #183's claim is structurally verified (13/16 archive `complexity_override` events fall cleanly in Gate 1 window; 3 are post-plan manual overrides; 0 fall in Gate 2 window with auto-firing semantics) but methodologically partial (events.log payload has no `gate` field — attribution is heuristic). D4's strict `blocked-by:[183]` sequencing is the structural mitigation — once Gate 2 is deleted entirely, the empirical-vs-heuristic distinction becomes moot.
- **Soft-MUST drift in template prose**: The annotation added for D4 ("Optional — omit when empty; include with items when uncertain at spec time") uses explicit conditional + concrete-instruction form, avoiding normative judgment phrases like "when load-bearing" that could be misread under post-Opus-4.7 tone semantics as "always assess → always emit."
- **Pre-commit hook failure on mirror regeneration**: `.githooks/pre-commit` Phase 3 (L168-174) re-runs `just build-plugin` with stderr surfaced AND exits 1 on failure. If a canonical-source edit causes the mirror regeneration to fail (e.g., syntax error in YAML), the commit is blocked and the error is visible — no silent drift.
- **Self-referential spec writing for #180**: This spec.md is written under the pre-D4 canonical specify.md template (since D4 hasn't shipped). To preserve canonical template compliance without inflating the artifact, this spec emits `## Open Decisions` with a `None — all resolved at clarify and spec interview time per §2b Pre-Write Check` attestation (see `## Open Decisions` below). This shape is structurally valid under both the pre-D4 canonical template (heading present, item placeholder filled) AND the post-D4 template (omission would also be valid, but the explicit "None" attestation is preferred to remove dependency on D4 actually shipping).
