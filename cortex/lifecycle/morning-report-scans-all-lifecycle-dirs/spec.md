# Specification: morning-report-scans-all-lifecycle-dirs

## Problem Statement
Two defects in `cortex_command/overnight/report.py` surfaced in wild-light run `overnight-2026-06-09-0222`. (A) The morning report's "Critical Review Residue" and "Requirements Drift Flags" sections glob the entire `cortex/lifecycle` tree instead of the session feature set, so the report fills with noise from unrelated historical lifecycles (13 residue entries in the run, none from the 4 session features) and degrades as repo history grows. (B) Session-finalization auto-filed tickets write an unquoted `title:` into YAML frontmatter; the embedded `: ` is invalid YAML and aborts the whole-backlog scan in `cortex-update-item` / `cortex-resolve-backlog-item` / `cortex-load-parent-epic`, blocking all backlog status updates until the bad file is removed. The operator (who reads the morning report and relies on backlog tooling) benefits: a session-relevant report and unbreakable backlog tooling. The report's tracked copy (`cortex/lifecycle/morning-report.md`) is git-committed to local `main` every session by the runner, so the scoping fix also cleans what lands in git history.

## Phases
- **Phase 1: YAML-safe generated-ticket titles (Bug B)** — generated retry/follow-up tickets, and the canonical backlog creator, serialize titles so the strict backlog parser round-trips them exactly and never aborts, and the tolerant index parser is never corrupted.
- **Phase 2: Session-scope the report sections (Bug A)** — the residue and drift sections render only this session's features, with correct counts, correct empty-vs-absent state handling, and CWD-independent path resolution throughout the drift path.

## Requirements

1. **Generated ticket titles round-trip exactly through the strict parser and never abort it.** `create_followup_backlog_items` (`cortex_command/overnight/report.py`, title write at ~line 307) serializes the `title` value so frontmatter containing `Follow up: {name}` / `Retry deferred: {name}` (embedded `: `) parses via `cortex_command/backlog/resolve_item.py::_parse_frontmatter` (`yaml.safe_load`) without raising, with `title` round-tripping to the in-memory value. Acceptance: a test in `cortex_command/overnight/tests/test_report.py` generates tickets for a deferred and a failed feature whose `name` contains a colon, reads each written file, and asserts `resolve_item._parse_frontmatter` (the real strict parser) returns the exact title. Pass: `.venv/bin/pytest cortex_command/overnight/tests/test_report.py -q` exits 0. **Phase**: YAML-safe generated-ticket titles (Bug B)

2. **The canonical backlog creator title is hardened.** `cortex_command/backlog/create_item.py:112` (`f'title: "{title}"\n'`, currently quoted but not escaped) serializes titles so embedded special characters do not break the strict parser. Acceptance: a test creates a backlog item with a title containing an embedded double-quote and a colon, then asserts the written frontmatter round-trips exactly through `resolve_item._parse_frontmatter` without raising. Pass: targeted `.venv/bin/pytest` over the create_item test exits 0. **Phase**: YAML-safe generated-ticket titles (Bug B)

3. **The serialization does not corrupt the tolerant index parser.** The fix must not break `cortex_command/backlog/generate_index.py::_parse_frontmatter` (best-effort: first-colon split + `.strip("\"'")`). Contract: for the realistic generated titles (`Follow up:`/`Retry deferred:` + a kebab-case slug, no embedded quotes), the title round-trips **exactly** through the tolerant parser. The fix MUST NOT use `json.dumps` (its backslash escapes leak literally into the backlog index) and MUST NOT emit a YAML document-end marker (`...`) or a multi-line/folded scalar (which the line-oriented tolerant parser truncates and which can break the strict parser). Acceptance: a test asserts the realistic generated titles round-trip exactly through `generate_index._parse_frontmatter`, and that the serialized title occupies exactly one physical line with no `...` marker. Pass: assertions hold in the Phase-1 pytest run. **Phase**: YAML-safe generated-ticket titles (Bug B)

4. **Titles are serialized as a sanitized single-line scalar.** The title is serialized on one physical line; any embedded newline/control character in `name` is collapsed/stripped before serialization so block-folding cannot occur. (Generated names are normally kebab slugs, but `lifecycle_slug`/feature names are read raw, so sanitize defensively rather than assume.) Acceptance: a test with a `name` containing an embedded newline asserts the written frontmatter is single-line for the title field and round-trips through the strict parser. Pass: assertion holds in pytest. **Phase**: YAML-safe generated-ticket titles (Bug B)

5. **Existing inline frontmatter layout is preserved.** The generated ticket frontmatter keeps its mandatory inline `tags: [...]` array and existing field order — the fix serializes the title scalar only, not the whole block (a whole-block `yaml.safe_dump` would emit block-style lists and reorder fields). Acceptance: the Phase-1 test parses the generated frontmatter and asserts `tags` is a list rendered inline (`grep -c 'tags: \[' <generated ticket>` = 1) AND the field order matches the pre-fix order; this is a layout check only — title validity is gated by R1. Pass: assertions hold in pytest. **Phase**: YAML-safe generated-ticket titles (Bug B)

6. **Residue section is session-scoped.** `render_critical_review_residue` (`report.py`, glob at ~1031) renders only residue whose lifecycle directory name (`path.parent.name`) is a member of `data.state.features`. Acceptance: a test builds session features `{feat-a}`, writes `critical-review-residue.json` into both `feat-a/` and an unrelated `old-unrelated/` lifecycle dir, renders, and asserts `feat-a` present and `old-unrelated` absent. Pass: `.venv/bin/pytest tests/test_report.py cortex_command/overnight/tests/test_report.py -q` exits 0. **Phase**: Session-scope the report sections (Bug A)

7. **Drift section is session-scoped as a narrowing gate, composed with existing exclusions.** `render_pending_drift` (`report.py`, globs at ~655/671) renders only review.md whose lifecycle directory name is in `data.state.features`. The keyset membership test is an ADDITIONAL narrowing gate, AND-composed with the function's pre-existing `merged` and `reimplementing` exclusions (`report.py:647–673`) — it removes non-session features and never re-includes an already-excluded one. This function has no existing coverage, so the change adds it. Acceptance: (a) a test writes `review.md` into an in-session and an unrelated dir and asserts the unrelated feature is absent; (b) a test with a feature that is in the keyset but `merged` asserts it is still excluded from drift. Pass: both new drift tests exit 0 under pytest. **Phase**: Session-scope the report sections (Bug A)

8. **The residue `(N)` header count equals the number of entries actually rendered.** The `## Critical Review Residue (N)` count (`report.py:1032`) is recomputed from the rendered set on whichever path is taken — never the raw glob count when a filter was applied. Acceptance: a negative-control test with session features `{feat-a, feat-b}` (both on disk with residue) renders header `## Critical Review Residue (2)`; the `{feat-a}`-only case renders `(1)`. Pass: both assertions hold in pytest. **Phase**: Session-scope the report sections (Bug A)

9. **Join on directory name; filter on the full keyset; verify the join key.** The membership test uses `path.parent.name` / `review_path.parent.name` (the lifecycle directory name) — not the writer-controlled `payload["feature"]` field (which the residue header displays) — and intersects against the **full** `data.state.features` keyset (paused/deferred/failed in-session features still surface), not a merged-only subset. Acceptance: (a) a deferred (not merged) in-session feature's residue still renders; (b) a divergence test where `payload["feature"]` differs from the dir name asserts the dir name decides inclusion — a residue whose dir name is in the session set renders even if its `payload["feature"]` is not, and one whose dir name is absent is excluded even if its `payload["feature"]` is present. Pass: both assertions hold in pytest. **Phase**: Session-scope the report sections (Bug A)

10. **Present-but-empty state renders empty; only absent state renders unfiltered.** When `data.state` is loaded (present) — including when `data.state.features` is the empty dict `{}` — both sections filter by the keyset, so a session with zero features renders an empty section (NOT the whole tree). Only when `data.state is None` (overnight-state.json absent or unloadable) do both sections render unfiltered as a degraded fallback (cannot scope; bias to over-show so a broken-state session never silently hides findings). The `(N)` count reflects what is actually rendered on the taken path. Acceptance: (a) the existing residue tests that pass `state=None` (`tests/test_report.py`, `Test_critical_review_residue`) still render their on-disk residue unchanged; (b) a new test with `data.state` present and `features == {}` (plus an on-disk residue dir) asserts the residue section renders empty and header `(0)`. Pass: `.venv/bin/pytest tests/test_report.py -q` exits 0 with both behaviors covered. **Phase**: Session-scope the report sections (Bug A)

11. **The entire drift path is CWD-independent.** Re-anchor ALL `cortex/lifecycle`-relative path constructions reachable from `render_pending_drift` to `_resolve_user_project_root()` — the two globs (655/671), the inline re-implementing read (`events_path` at ~657), and the per-feature helpers it calls (`_read_requirements_drift` ~876, `_read_drift_protocol_breaches` ~932) — so the enumerator and the per-feature content reads resolve against the same root. Acceptance: (a) `grep -c 'Path("cortex/lifecycle")' cortex_command/overnight/report.py` = 0 AND `grep -c 'Path(f"cortex/lifecycle/' cortex_command/overnight/report.py` = 0; (b) a test renders the drift section with the process CWD set to a directory other than the fixture root (while `CORTEX_REPO_ROOT` points at the fixture) and asserts the in-session drift still resolves correctly. Pass: grep assertions = 0 and the CWD-independence test exits 0. **Phase**: Session-scope the report sections (Bug A)

12. **Full suite passes.** Acceptance: `just test` exits 0. Pass: exit code = 0. **Phase**: Session-scope the report sections (Bug A)

## Non-Requirements
- **Cross-repo residue is not surfaced.** Features with `repo_path` pointing at another repo write residue CWD-relative inside that repo (gitignored), which never reaches the home-repo report. Glob-then-filter preserves the current home-repo-only behavior; surfacing cross-repo residue is out of scope.
- **`seed.py` is the third generated-title writer and is deliberately out of scope.** `cortex_command/dashboard/seed.py:1057` also hand-writes a `title:` line (using `title.replace('"','\\"')` then double-quoting). It is left untouched because its titles are static, colon-bearing-but-quote-free literals (e.g. `Seed: ...`) — adequate for those fixed inputs, not a general-purpose serializer. It is not "fixed" here; the chosen scope is the two writers with dynamic, potentially-colon-bearing titles (report.py, create_item.py).
- **No shared YAML serializer is extracted.** Per the chosen scope, the two dynamic sites are fixed in place; a shared `common.py` serializer is not built.
- **No residue session-stamping or bootstrap-clearing.** Bug A scopes by feature-set membership only; it does not stamp residue files with `session_id` or clear them at bootstrap.
- **Tickets 292 and 293** (review-gate cycle-0 crash; plan-parser metadata) from the same overnight run are tracked separately and not addressed here.

## Edge Cases
- **Title with embedded `"`, `\`, or multiple colons**: must round-trip exactly through the strict `resolve_item` parser (R1/R2). For the realistic kebab-slug titles the tolerant index parser also round-trips exactly; a title containing the wrapping quote character may render with the YAML-doubled quote in the backlog **index** only (cosmetic, non-corrupting) — accepted, since the strict parser (the one with whole-backlog blast radius) is exact.
- **Title with embedded newline / over-long title**: collapsed/sanitized to a single line before serialization (R4) so no block-folding occurs.
- **`data.state` present with `features == {}` (e.g. live planning-phase session)**: renders an empty residue/drift section with count `(0)` — NOT the whole tree (R10). This is the corrected behavior; the earlier "empty → unfiltered" reading would have reintroduced Bug A.
- **`data.state is None` / corrupt or missing `overnight-state.json`**: render unfiltered as a degraded fallback (R10) — biased toward over-showing so a broken-state session never silently hides B-class findings. This is the only path that can emit unscoped content; it commits unscoped content to `main` only when state is genuinely unloadable.
- **Malformed residue JSON**: the existing graceful per-file skip (`report.py:1046`) is preserved; filtering wraps it without changing skip behavior.
- **`lifecycle_slug` frontmatter diverges from the on-disk lifecycle dir name**: in the normal overnight flow the residue/review.md dir name equals the `data.state.features` key (both derive from `resolve_slug()` / the lifecycle dir). If a feature's raw `lifecycle_slug` diverges from its on-disk dir name, its residue could be filtered out. Accepted as a rare, pre-existing data-inconsistency edge — not enforced or worked around here.
- **Stale carryover residue (accepted)**: a feature re-entered in a later session that fails before critical review may show its prior-session residue (residue is gitignored and accumulates at a fixed per-feature path). Accepted: a re-entered feature that reaches critical review overwrites its residue with fresh findings; not solved by the keyset filter; not in scope.

## Technical Constraints
- **Title serialization contract**: produce a single-line YAML scalar for the title — no `...` document-end marker, no multi-line/folded output. Forbidden: `json.dumps` (backslash escapes leak through the tolerant `generate_index` parser, corrupting the index) and the bare `yaml.safe_dump(value)` reading (emits a `...` marker that breaks the strict parser when spliced). Acceptable: serialize the title scalar via a routine that yields one quoted line whose result the strict parser round-trips exactly (e.g. dump `{"title": value}` with folding disabled and take the title line, or an explicit single-line single-quote-with-doubling helper). The strict parser (`resolve_item`, `yaml.safe_load` — the one that aborts the whole backlog) is the load-bearing contract: exact round-trip, never raises. The tolerant parser (`generate_index`) must not be corrupted and round-trips exactly for the realistic kebab titles.
- **Preserve inline layout / field order**: serialize only the title scalar; do not whole-block `safe_dump` (it emits block-style lists, breaking the mandatory inline `tags: [...]` and `create_item.py`'s pinned field order).
- **Membership join**: `path.parent.name` / `review_path.parent.name` equals the `data.state.features` key in the normal overnight flow (`plan.py:418` `slug = item.resolve_slug()`, and the lifecycle dir is named from the same slug). This is a flow invariant, not a hard guarantee — see the `lifecycle_slug` divergence edge case. Filter on the full keyset; the keyset gate composes (ANDs) with `render_pending_drift`'s existing merged/reimplementing exclusions.
- **State presence vs emptiness**: `OvernightState.features` is `field(default_factory=dict)` (`state.py:233`) with no invariant forbidding `{}`, so a present-but-empty state is reachable mid-planning; treat present-empty as "scope to zero," and only `data.state is None` as the unfiltered fallback.
- **Blast-radius context**: `resolve_item.py:412–430` eagerly `yaml.safe_load`s every backlog file and raises on the first failure — the Bug B fix is what prevents one malformed ticket from blocking the whole backlog.
- **Test assertions must use the real `resolve_item._parse_frontmatter` (strict)**: the tolerant `generate_index._parse_frontmatter` does NOT raise on the bug, so asserting only against it would not catch a regression.
