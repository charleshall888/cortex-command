# Plan: morning-report-scans-all-lifecycle-dirs

## Overview
Two independent defect fixes in the overnight morning-report path. **Bug B**: serialize generated-ticket titles (`report.py`) and the canonical backlog-creator title (`create_item.py`) as sanitized single-line YAML scalars so colon/quote/newline-bearing titles round-trip through the strict backlog parser (`resolve_item._parse_frontmatter`, `yaml.safe_load`) and never abort the whole-backlog `resolve()` scan — the parser whose failure has whole-backlog blast radius. For the realistic generated titles (`Follow up:`/`Retry deferred:` + a kebab slug, no embedded quotes) the title also round-trips **exactly** through the tolerant index parser (`generate_index._parse_frontmatter`); for arbitrary user titles that combine a colon with an embedded quote, the strict parser stays exact while the backlog **index** may show a cosmetic doubled quote — accepted per the spec's Edge Cases, since the index parser is not the whole-backlog-blast-radius surface. **Bug A**: scope the residue and drift report sections to the session feature set (`data.state.features`), recompute the section `(N)` counts from the rendered set, handle present-but-empty (`{}`) vs absent (`None`) state correctly, and re-anchor the drift-render path's `cortex/lifecycle` reads to `_resolve_user_project_root()`.

## Outline

### Phase 1: YAML-safe generated-ticket titles (Bug B) (tasks: 1, 2)
**Goal**: Both dynamic title writers serialize the title as a sanitized single-line scalar that the strict parser round-trips exactly and never raises on; the whole-backlog `resolve()` scan survives a colon-bearing generated ticket.
**Checkpoint**: A test populates a backlog directory with a colon-titled generated ticket plus a normal item and asserts `resolve_item.resolve(...)` does not raise; the per-site round-trip tests pass.

### Phase 2: Session-scope the report sections (Bug A) (tasks: 3, 4, 5)
**Goal**: Residue and drift sections render only this session's features with accurate counts and correct present-empty vs absent handling; the drift render path resolves CWD-independently via `_resolve_user_project_root()`.
**Checkpoint**: Residue/drift tests assert in-session present, unrelated absent, `(N)` correct, present-empty → `(0)` with an accurate empty body, `state=None` → unfiltered, and that both the drift render and the `reimplementing` exclusion resolve off-CWD; `just test` exits 0.

## Tasks

### Task 1: Serialize generated-ticket titles + prove the whole-backlog scan survives (Bug B, report.py)
- **Files**: `cortex_command/overnight/report.py`, `cortex_command/overnight/tests/test_report.py`
- **What**: Replace the unquoted `f"title: {title}\n"` write in `create_followup_backlog_items` (the frontmatter f-string at `report.py:~307`) with a sanitized single-line YAML-scalar serialization, add per-site round-trip tests (R1/R3/R4/R5), and add an integration test that the whole-backlog `resolve()` scan survives a colon-titled generated ticket.
- **Depends on**: none
- **Complexity**: simple
- **Context**: The frontmatter block at `report.py:~304-320` builds an inline-`tags: [...]` mapping in a fixed field order — preserve that layout and order; serialize **only** the `title` scalar (do not whole-block `yaml.safe_dump`, which emits block-style lists and reorders fields, breaking R5). **Serialization contract (ordering is load-bearing):** (1) FIRST sanitize — collapse any `\r`/`\n`/control characters in `name` to a single space (R4); (2) THEN serialize the sanitized value to one physical line. Use `yaml.safe_dump({"title": sanitized}, default_flow_style=False, width=float("inf"), allow_unicode=True)` and extract the title value by splitting the emitted mapping on the first `": "` (not by grabbing "the first physical line" — that is unsafe if a stray control char survived and produces an unterminated quoted scalar that raises `ScannerError` in the strict parser, reproducing the whole-backlog abort). Strip the single trailing `\n`; assert the result is exactly one physical line before splicing. **Forbidden**: `json.dumps` (backslash escapes leak through the tolerant `generate_index` parser, R3) and the bare `yaml.safe_dump(value)` scalar form (emits a `...` document-end marker, R3). For realistic `Follow up:`/`Retry deferred:` + kebab-slug titles the result round-trips exactly through both `resolve_item._parse_frontmatter` (strict) and `generate_index._parse_frontmatter` (tolerant). Tests in `cortex_command/overnight/tests/test_report.py`: import the real `cortex_command.backlog.resolve_item` and `cortex_command.backlog.generate_index`; generate tickets for a deferred and a failed feature whose `name` contains a colon, read each written file, assert the strict parser returns the exact title (R1); assert exact round-trip through the tolerant parser and that the title is one physical line with no `...` (R3); a name with an embedded newline yields a single-line title field that the strict parser accepts (R4); `grep`/parse the generated ticket to assert `tags` is rendered inline and field order matches pre-fix (R5). **Integration test (closes the real Bug B symptom):** write the generated colon-titled ticket(s) into a tmp backlog dir that also contains one normal valid item, then call `resolve_item.resolve(<query>, backlog_dir=<tmp>)` (the eager whole-backlog scan at `resolve_item.py:423-431`) and assert it does NOT raise `ResolutionError` — the single-file round-trip alone does not exercise the eager loop that aborts the whole backlog.
- **Verification**: run `.venv/bin/pytest cortex_command/overnight/tests/test_report.py -q` — pass if exit 0 and all tests pass (including the `resolve()`-does-not-raise integration assertion).
- **Status**: [x] done

## Risks
- **R11 is narrowed from its literal acceptance, by decision at this plan's approval.** R11's acceptance is written as a **file-global** `grep -c 'Path(f"cortex/lifecycle/' = 0`, but its requirement prose scopes re-anchoring to literals "reachable from `render_pending_drift`." The two readings diverge: the global grep additionally catches seven non-drift sites — plan-acceptance reads (`_read_verification_strategy:787`, `_read_acceptance:804`, `_read_last_phase_checkpoint:829`), `_read_recovery_log_last_entry:946`, `_read_learnings_summary:963`, and the **session sidecars** `collect_tool_failures:1277` / `collect_sandbox_denials:1423`. Re-anchoring the two session sidecars is **unsafe**: the PostToolUse hook writes those files **CWD-relative inside the worktree** (`claude/hooks/cortex-tool-failure-tracker.sh:42`; agent tools run `cwd=worktree_path`), while `_resolve_user_project_root()` returns `CORTEX_REPO_ROOT` = the **home** repo (`runner.py:2245`) — so re-anchoring them would make the reader look in the home repo while the writer wrote in the worktree, **silently dropping all tool-failure and sandbox-denial data** from worktree-based overnight reports, and `just test` cannot catch it (every fixture bakes in a resolvable root). This plan therefore re-anchors **only the five drift-reachable sites** (Task 4), honoring R11's prose intent, and leaves the seven non-drift sites untouched. **Operator decision at approval:** accept this narrowing (recommended), or send back to revise R11's acceptance text. If you instead want full global CWD-independence, the session-sidecar *writers* must be re-anchored in the same change — a larger scope than Bug A.
- **Bug B's user-facing symptom is now tested at integration level.** Task 1 adds a `resolve_item.resolve()`-does-not-raise assertion over a populated backlog dir, not only a single-file round-trip — the single-file proxy could pass while the eager whole-backlog loop (`resolve_item.py:423-431`) still aborted.
- **No shared title serializer (spec Non-Requirements).** The sanitize-then-serialize logic is duplicated at `report.py:~307` and `create_item.py:112`. Accepted per the chosen scope; `seed.py:1057` (a third, static-title writer) is deliberately untouched.

## Acceptance
A wild-light overnight run's morning report shows residue and drift entries only for that run's session features with accurate `(N)` counts (present-empty state renders `(0)` with an accurate empty body, not the whole tree; only genuinely absent state renders unfiltered), and the drift render plus its `reimplementing` exclusion resolve correctly regardless of process CWD; auto-filed retry/follow-up tickets whose titles contain a `: ` round-trip through the strict backlog parser so the eager whole-backlog `resolve()` scan (and thus `cortex-update-item` / `cortex-resolve-backlog-item`) never aborts; and `just test` exits 0.
