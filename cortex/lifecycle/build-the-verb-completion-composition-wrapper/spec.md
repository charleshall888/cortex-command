# Specification: build-the-verb-completion-composition-wrapper

> Epic reference: this is Phase B of epic 371 (CLI-served lifecycle state machine). Funding rationale, phase boundary, and the Phase-C gate live in `cortex/research/cli-served-lifecycle-state-machine/research.md`; ticket-scoped research in `cortex/lifecycle/build-the-verb-completion-composition-wrapper/research.md`.

## Problem Statement

The lifecycle's multi-event transitions are hand-written in prose-ordered sequences whose ordering invariants are held only by sentences, and its recorded failure classes — emit-before-commit stranding, invented event schemas, drifting pause-inventory anchors, an overnight prompt that re-implements (buggily: its override read consults a field the event never carries) the events.log criticality reduction — are all symptoms of sequencing knowledge split between prose and code. This ticket moves the four multi-emission decision clusters into wrapper verbs whose function bodies are the ordering invariant, converts the kept-pauses inventory to generated output with an exact-equality parity net, and points the overnight orchestrator at the shared reducer — fixing every recorded failure class without building the gated Phase-C loop. Operators get durable consent events for the two highest-stakes pauses by construction.

## Phases

- **Phase 1: Test-harness foundation** — close the `just test` coverage blind spot and add the environment-fixture guard tests, so later pin retirements trade into enforced tests.
- **Phase 2: Marker-generated kept-pauses** — markers, data file, generator, parity collapse to marker-set equality. Runs **before** the prose rewrites so the line-anchored parity scheme retires before Phase 3 shifts its anchors.
- **Phase 3: Wrapper verbs and prose routing** — the four B1 verbs, write-side order tests, cluster prose rewrites, FILE_EVENTS retirement, registry updates.
- **Phase 4: Shared overnight reducer** — orchestrator-round.md consumes the shared reducer; heading-citation pin retires.

## Non-Requirements

- No `next`/`advance` served loop, no protocol handshake, no transition table, no bounding ADR for router verbs — all Phase-C, gated by ticket 374 on post-composition evidence.
- No runtime pause tooth (refusing transitions without recorded pause events) — Phase C+1.
- No overnight execution unification beyond the criticality read (the RQ3 transition table and pipeline-writer census are Phase C+2 prerequisites); the two `plan_comparison` direct appends in orchestrator-round.md stay as-is.
- Judgment content never migrates: plan-authoring rules, review criteria, prompt templates, the drift-apply loop, approval-surface wording, the overnight plan-format contract all stay prose.
- The residual four FILE_EVENTS rows keep their current exact counts — no verb owns them this wave.
- No tier read added to the overnight prompt (criticality only; the shared reducer already serves tier when Phase C wants it).
- No new pre-commit gate (kept-pauses freshness enforces via pytest locally plus the CI wiring in Requirement 5).
- No byte-identity reproduction of historical event rows — parsed-field semantics per ADR-0020's canonical shape; the legacy corpus's field-order drift is tolerated by readers, not imitated by writers.
- No automatic resume-time repair of a mid-verb partial emission sequence — the mismatch detector surfaces it and the idempotent verbs are the manual recovery tool; an automatic repair hook is Phase-C territory (the advance verb's CAS/idempotency design).

## Edge Cases

- **Crash between ordered emissions**: re-running the same verb repairs — each emission independently presence-checks (parsed `event`-field match, never substring) before appending. Resume flows do NOT re-invoke the verbs automatically (they key on artifact existence); the scan_lifecycle mismatch detector is the surfacing mechanism and manual verb re-run is the documented recovery.
- **Halt-before-Plan commit-gate failure after the spec-approve verb completed**: all rows and the write-back already exist; resume proceeds to Plan and the hand-edit commit rule in refine-delegation.md applies unchanged — nothing needs repair.
- **`--feature` containing path separators or `..`**: rejected with an error envelope before any filesystem access.
- **Corrupted events.log (interactive)**: verbs surface `"corrupted": true` semantics from the shared reducer; interactive prose routes to the review-requiring arm (criticality-matrix rule).
- **Corrupted events.log (overnight)**: criticality-present-despite-corruption uses the value; criticality-unknowable routes single-agent with a morning-report warning — the feature still executes (outcome_router precedent), it is not deferred.
- **Consumer repo with new plugin prose + old wheel (verb missing)**: the call site's halt arm fires — halt, instruct install, never hand-emit. The per-file zero sweeps (typed AND `log` form) make a hand-emission reintroduced in this repo fail the roundtrip test.
- **`--areas` omitted by the calling agent**: existing areas preserved (omission is the common case, not a clearing instruction).
- **Two markers with the same ID, or a marker inside the generated region**: exact set-equality parity fails loud; the generated file's header warns against hand-edits.
- **Re-run of the wait arm after `feature_paused`**: presence-checks make the double invocation emit nothing new; the discriminant still reports `wait-approved`.
- **Concurrent invocation of the same verb for the same feature** (two sessions): each append is individually atomic (flock + `O_APPEND`, inherited posture); presence-checks bound the damage to at-most-once per emission per completed run — the same posture `finalize`/`wontfix` ship with today. Cross-session gate-level CAS is a Phase-C hazard-register item, explicitly not built here.

## Changes to Existing Behavior

- MODIFIED: overnight criticality resolution — the shared reducer honors `criticality_override` events (the inline block read a field overrides never carry, so they were never applied); **latent under default config** — the read executes only when `synthesizer_overnight_enabled` (default false, fail-closed) is enabled, so no observable change until an operator flips the gate.
- MODIFIED: `phase_transition review→implement-rework` — single-owner (review-verdict verb); implement.md §3 no longer re-emits it.
- MODIFIED: specify→plan transition emission — the spec-approve verb is the sole emitter; lifecycle-wrapped refine passes `--emit-transition`, standalone refine passes `--no-emit-transition` (recording explicitly: standalone-refined features continue to carry **no** specify→plan row, today's operative behavior under specify.md §5's carve-out); refine-delegation.md's boundary-logging instruction for that boundary re-targets the verb.
- MODIFIED: `kept-pauses.md` — hand-authored inventory → generated output; the parity test's line-tolerance scheme → exact marker-set equality + freshness diff + marker-anchored semantic sub-checks (suppression wiring, step headings).
- MODIFIED: FILE_EVENTS rows for the four cluster files — exact counts → per-file zero sweeps covering typed and `log`-form invocations.
- MODIFIED: events-registry rows for cluster events — producers re-point to verbs, `scan_coverage` flips to `manual` with replacement-enforcement rationale.
- ADDED: typed `lifecycle-cancelled` subcommand; optional enum consent field on `spec_approved`; four verb console scripts; pause data file + generator; parity/freshness test wired into CI.
- REMOVED: `test_plan_md_has_1a_heading`; inline `_read_criticality` block; the four clusters' hand-written `cortex-lifecycle-event` prose invocations.

## Technical Constraints

- ADR-0019 dumb-arg-actor: caller-passed `--backend`/`--feature`; the spec verb's gated primary action is a recorded "taken knowingly" crossing. ADR-0016: 3-arm backend routing stays skill-side; verbs gain no backend awareness beyond the structural guard.
- ADR-0020: all emissions through the uniform `{ts, event, feature, …}` path; the hand-written exception class stays closed at three; additive-only against events.log (new optional fields and new event names allowed; no shape mutation, no rollback — ADR-0004).
- Events-registry discipline: renames/deletions take 30-day `deprecated-pending-removal` rows; `scan_coverage` labels must match the actual scan surface (Requirement 14).
- Implementation-phase invocation: after editing `cortex_command/`, verify via `python3 -m` or `CORTEX_COMMAND_FORCE_SOURCE=1` — binstubs run the installed wheel.
- Dual-source: `skills/` edits regenerate `plugins/cortex-core/` at pre-commit; the pause data file and regenerated kept-pauses.md ride the existing rsync mirror.
- Prose authoring: SP001/SP002 path propagation, L201 console-script-only invocation (which is why the overnight prompt's in-process import in Requirement 15 applies only to the prompt's Python blocks, not skill prose), What/Why-not-How, MUST-escalation evidence rule.
- `just test` is the enforcement surface for all new tests (Requirement 1 precedes every pin retirement); the kept-pauses parity/freshness test additionally rides CI per Requirement 5.
- Phase ordering is load-bearing: Phase 2 (markers) precedes Phase 3 (prose rewrites) so the line-anchored parity test retires before the rewrites would shift its anchors past tolerance.
