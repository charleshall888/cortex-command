## Event Schema & Registry

**Consent recording is field-additive, not new event types.** `plan_approved` already carries the required `dispatch_choice` enum (trunk/worktree-interactive/feature-branch/wait) — it *is* the plan-decision consent record. `spec_approved` carries zero fields; consent enrichment is optional field(s) on it. The registry's "Field-additive schema extensions" section (`bin/.events-registry.md:147-178`) is the established precedent (five events grew optional fields without new names); `detect_lifecycle_phase` (`common.py:294-297`) checks event-name presence only, never field values, so additive fields are fully back-compatible. Adversarial constraint: keep consent fields **enum/identifier-only** — events.log is append-only forever with no redaction path, so operator free text (revision rationale, "Other" cancel text) must not become event fields.

**ADR-0020 exception class unaffected.** The three hand-written exceptions (`plan_comparison`, `clarify_critic`, `pr_opened` — schema_version-before-feature shapes) don't intersect the four clusters; all cluster events already flow through `_EVENT_SUBCOMMANDS`/`log_event`. Routing emissions through verb function bodies keeps the class closed at three structurally.

**Gap: `lifecycle_cancelled` has no typed subcommand** — plan.md and specify.md cancel arms emit it via the generic `log` escape hatch today. The verbs owning cancel arms should add a typed subcommand (additive, allowed); `stage_artifacts.py:148` already discriminates approval-vs-cancel by scanning for it.

**Real-log shapes to reproduce** (parsed-field semantics, not byte-identity — ADR-0020 defines a new canonical, and the historical corpus has field-order drift the verbs must not imitate): approve arms emit event + `phase_transition` back-to-back (same-second `ts`); the wait arm emits `plan_approved{wait}` + `feature_paused` with **no** transition.

**Idempotency precedents.** `finalize._feature_complete_exists` (parsed `event` field match, never substring; a commit-retry double invocation never duplicates) and `wontfix_cli`'s fail-forward per-step postcondition re-assertion. Each wrapper-verb emission re-derives its precondition from current events.log state so a partial run (crash between (a) and (b)) is repaired by re-running the whole verb.

**Spec-verb write-back contract.** Caller-supplied args per the dumb-arg-actor rule: `--feature`, `--backlog-file` (basename, `""` when absent — enter/finalize convention), `--backend` (caller-resolved via `cortex-read-backlog-backend`, never re-derived), `--spec-path`, `--areas`. In-process `update_item` import per the finalize precedent (self-regenerating index) rather than subprocess. `--areas` must **preserve-on-omit** and clear only on an explicit empty flag — an LLM omitting a flag is the common case, not the edge.

**Emission path is mandatory.** All four verbs emit via `lifecycle_event`'s existing append helpers (flock + `O_APPEND`, spaced `json.dumps`) — `scan_lifecycle.py` substring-matches exact spacing, so a hand-rolled `json.dumps(separators=...)` in a new module silently breaks read-side consumers and drops locking.

## Overnight Reducer Consumption

**The inline `_read_criticality` (orchestrator-round.md:238-260) is buggy today**: it reads `entry.get("criticality")` for both `lifecycle_start` and `criticality_override` events, but overrides carry `from`/`to` only (`common.py:738-744`; registry row confirms) — so overnight has **never applied criticality overrides**. The B3 swap to `cortex-lifecycle-state --feature <slug> --field criticality` is a behavior fix, not a pure refactor: record it as an intentional change in the plan and registry (both upgrades and downgrades start applying).

**Swap design requirements**: (a) the subprocess call introduces a failure surface the pure-Python block lacks (command-not-found exit 127) — the prompt needs an explicit fallback arm; per the adversarial review and `state_cli.py`'s own contract ("fail safe toward review"), corrupted/unavailable must route to the **cautious** branch (dual-plan scrutiny or defer), not silently default `medium`; (b) a `"corrupted": true` arm per criticality-matrix.md:22-26 ("treat the feature as requiring review"); (c) apply the documented defaults yourself (the CLI prints `{}` or omits absent keys).

**Boundary**: the inline block is the only per-feature events.log *read* in the prompt; the two `plan_comparison` direct appends (~line 334, 376) are writes, out of B3 scope. PATH is confirmed available under launchd (plist snapshots user PATH; console-scripts ship in the same wheel). Prompt and verb are wheel-packaged together (`importlib.resources`) — they cannot skew. Registry rows citing `orchestrator-round.md:256` as consumer become stale prose to update. Tier is already served by the verb; B3 is criticality-only, no verb-contract change.

## Skill-Prose Integration & Marker Design

**Routing templates**: complete.md Step 7 ("Act on the verdict; do not re-derive it" — route arms on JSON discriminant) and backlog-writeback.md's needs-decision handling + canonical 3-arm backend routing.

**Per-cluster verb-boundary facts** (what "one function body" actually owns):

- **Plan** — branch-dependent, self-contained per arm: discriminants ≈ `{branch-mode-approved, wait-approved, cancelled, revise}`; only subsets of the three events fire per path. The AskUserQuestion surface and option-assembly judgment stay prose. Keep plan.md:123's warning that overnight eligibility doesn't honor `feature_paused` — the consent-to-wait event is durable but the pause tooth is explicitly Phase-C.
- **Review** — does **not** fit one straight-line body: the §4a drift auto-apply judgment loop (LLM-applied suggestion, reviewer re-dispatch, 2-retry cap, conditional `drift_protocol_breach`) sits between `review_verdict` and the transition. The prose calls the verb **after** the drift loop with post-hoc breach/retry args — the function body owns the truncated verdict→(breach)→transition tail, an explicit design decision for spec.
- **Spec** — three contexts, not two: standalone refine (transition suppressed), lifecycle-wrapped refine (refine-delegation.md owns boundary logging — two emission owners to reconcile or carve down), and the halt-before-Plan commit gate sits mid-sequence (refine-delegation.md item 5) — so the verb needs per-emission presence-check idempotency for resume-after-commit-failure, and a context flag under-models the ownership matrix unless refine-delegation's specify→plan logging is explicitly reassigned to the verb.
- **Implement** — fits: `batch_dispatch` is unconditional-mechanical; the §4 transition verb reads criticality/tier via the shared reducer. New drift pair to manage: moving the routing rule ("Review when criticality ∈ {high,critical} OR tier = complex") into wheel code while criticality-matrix.md keeps the prose statement — pin the code rule to the doc or rewrite the doc to defer to the verb.
- **Duplicate-ownership hazard (adversarial, verified)**: `phase_transition review→implement-rework` is emitted by review.md §5 **and** implement.md §3 — mechanizing both bakes in a same-session duplicate that subsequence-order tests pass by construction. Spec must assign single ownership + presence-check idempotency.

**Marker design**: HTML-comment markers (e.g. `<!-- pause: <id> <kind> -->`) are safe — the existing sentinels (`skill-path-lint:ignore-next`, `bare-python-lint:ignore-next`, `size-budget-exception:`) are prefix-scoped consume-next-line patterns with no generic comment scanning, and the events-registry checker matches only `"event": "..."` / `--event` literals. Reference files are not size-budget-counted (the 500-line cap enumerates `SKILL.md` files only); the L1 ratchet is frontmatter-only — markers cost zero budget. Web prior art (terraform-docs, embedme `--verify`, doctoc): regenerate-and-diff checks plus self-documenting "don't hand-edit, re-run the generator" text inside the generated region; no off-the-shelf orphan-marker detection exists — exact set-equality parity is the guard.

**Placement & distribution**: the pause data file must live inside the `skills/` tree (most naturally `skills/lifecycle/references/`) — `cortex/` is not rsync'd by `just build-plugin`, while `skills/lifecycle/` mirrors wholesale into `plugins/cortex-core/`, so the data file and regenerated kept-pauses.md ride the existing mirror with no new wiring. Generation happens in this repo at author time; the committed output ships via the plugin.

**Freshness enforcement**: prefer regenerate-and-diff **inside the pytest parity test** (it lives in `tests/`, which `just test` runs) over a new pre-commit phase — the dual-source gate already blocks every commit on plugin-tree cleanliness, and adding more pre-commit drift surfaces compounds the known cross-session blocking problem. (The `sync-install-guard --check` Phase-1.95 shape remains the fallback if commit-time enforcement proves necessary.)

## Web Research

- **Marker/generator prior art**: terraform-docs (BEGIN/END HTML-comment injection), embedme (`--verify` fails CI on drift), markdown-magic, doctoc (warning text baked into the marker). Convergent pattern: regenerate → diff → fail loud; self-documenting markers; stable IDs must be explicit, never derived from heading slugs. No tool ships orphan-marker detection — the parity test fills that gap.
- **Consent-event field templates**: OPA decision logs (`decision_id`, the *input* persisted alongside the *result*, policy revision — falsifiability comes from persisting input with result); GitHub deployment reviews (actor, state enum, comment, scope; self-approval prevention checked before recording); Step Functions human-approval (always time-bound pending states); LangGraph HITL (approve/edit/reject/respond enum + persisted resumable state).
- **Order-testing idioms**: Google Testing Blog "Change-Detector Tests Considered Harmful" + "Prefer Narrow Assertions" — assert the ordering relation between the specific events under test (subsequence/precedence), not exact counts or full-log equality; log-file readback (real append path) catches serialization bugs in-memory mocks hide; a 2025 mock-assertion study found ~half of call-order-caught faults invisible to state assertions.
- **CLI state-discriminant envelopes**: `gh api --batch` (uniform status/path/body wrapper), Terraform `-json` (one JSON object per line with a `type` discriminant), Nx (detects agentic callers, emits structured JSON), OPA (CLI return value and durable log row share the same decision id — the pattern for verb-envelope/event correlation). No exact prior art for single-verb gate+emit+return consumed by an LLM prose loop — novel composition, consistent with the epic research's framing.

## Adversarial Review

Findings folded into the sections above; the standing list, priority-ordered (each is a spec obligation):

1. **`just test` blind spot** — extend the recipe to `cortex_command/lifecycle/tests/` (and sibling dirs) as a blocker for pin retirement (§Test Lattice).
2. **Zero-count rows, not deletions**, for retired FILE_EVENTS rows; no-duplicate assertions in write-side tests (§Test Lattice).
3. **review→implement-rework double-emission** — single ownership + presence-check idempotency before either verb is written (§Skill-Prose).
4. **Spec verb: three-context ownership matrix + mid-sequence commit gate** — per-emission resume semantics; `--areas` preserve-on-omit (§Skill-Prose, §Event Schema).
5. **Overnight fallback fails toward caution** (review/defer, not `medium`); record the override fix as a behavior change (§Overnight).
6. **Emission-path discipline** — all verbs through `lifecycle_event` helpers; cheap slug-shape guard (reject path separators — input hygiene, not resolution, so dumb-arg-actor-compatible); consent fields enum-only (§Event Schema).
7. **Kept-pauses freshness via the pytest parity test**, not a new pre-commit gate; update project.md:27 + docs/policies.md in the same commit as the data file (§Skill-Prose, §Requirements).
8. **Skew posture, cheap hardening**: on command-not-found in a consumer repo (new plugin prose, old wheel), the agent knows the legacy hand-emission pattern and will plausibly improvise it — each rewritten call site carries an explicit "command not found → halt and instruct install; do NOT emit events by hand" arm, inside the ticket's accepted no-handshake posture.
