# Specification: Trim verbose lifecycle skill content (implement.md §1a + plan.md §1b + SKILL.md gate compression)

## Problem Statement

Three lifecycle skill files carry redundant prose that costs token-context on every feature execution: `implement.md` §1a inlines an atomic-write Python recipe duplicated in `cortex_command/overnight/daytime_pipeline.py`; `plan.md` §1b inlines synthesizer-internal protocol (scoring rubric, swap probe, worked downgrade examples) duplicated in `cortex_command/overnight/prompts/plan-synthesizer.md`; and `SKILL.md` describes complexity-escalation gates twice (inline protocol step plus a standalone Complexity Override section) with three inlined `complexity_override` JSON examples — and Gate 2 (Specify→Plan ≥3 Open Decisions) had zero observed fires across 153 lifecycles per epic-172-audit DR-2. Trimming these three files reduces per-feature context cost while preserving load-bearing dispatcher contracts and the security mitigation at `plan.md:120`.

## Edge Cases

- **`importlib.resources.files()` load failure at plan-time**: If `plan-synthesizer.md` cannot be loaded (file missing, package import error), the synthesizer Task sub-agent dispatch will fail. Expected behavior: existing failure handling in `cortex_command/pipeline/dispatch.py` raises and surfaces to the operator; this trim does not change that path. No new error handling is introduced.
- **Future plan.md author reads §1b after the trim**: They see the dispatcher contract (envelope extraction, schema validation, rationale-hiding rule, verdict routing) inline. To understand the synthesizer-internal protocol (scoring rubric, swap-and-require-agreement protocol body, worked downgrade examples), they follow the `importlib.resources.files()` pointer to `plan-synthesizer.md`. The pointer is one-level-deep per Anthropic's "one-level-deep references" guidance. Expected behavior: comprehension preserved.
- **Simple+low spec with ≥3 Open Decisions after Gate 2 removal (FM-3)**: Auto-escalation does not fire (Gate 2 removed); orchestrator-review is skipped per `orchestrator-review.md:9` (simple+low skip rule); manual escalation via `criticality_override` remains available. Expected behavior: documented in SKILL.md per Requirement 11; the user is informed at spec-time that this lifecycle's tier may be under-classified relative to its Open Decisions count.
- **`tests/test_skill_size_budget.py` enforcement pass after trim**: The trim is purely reductive — SKILL.md drops from 396 to ≤ 350 lines, well under the 500-line cap. No exception marker is needed.
- **Plugin mirror regeneration drift**: `just build-plugin` uses `rsync -a --delete`. After the canonical edit, the mirror is regenerated atomically. If the dual-source drift hook fails post-commit, expected remediation is re-running `just build-plugin` and re-committing — same as today.
- **Resumed lifecycle running on the trimmed prose**: An overnight runner that pre-loaded `implement.md` or `plan.md` before the edit will still operate on the cached pre-trim content for the active session; new sessions started after the merge load the trimmed content. Expected behavior: no migration step required because skill content is read each session.
- **Item 3 Gate 1 surviving prose now stands alone**: After deleting the standalone "Complexity Override" section, only the inline protocol-step subsection (Step 5 in SKILL.md) describes Gate 1. The state-detection paragraph at line 82 references `complexity_override` events generically and does not need editing. Expected behavior: Gate 1 remains fully described in one place; resumed lifecycles still detect tier from events.log per the unchanged Step 2.

## Changes to Existing Behavior

- **MODIFIED**: `implement.md` §1a no longer contains an inline atomic-write Python recipe. Executors of the daytime-dispatch path now invoke a helper module entry-point (existing or to-be-added per implementation; verify in plan phase) instead of inline `tempfile.mkstemp` + `os.replace` calls. The atomic-write semantics are preserved (single-source, in `cortex_command/overnight/`), and `dispatch_id` recovery-after-compaction is unaffected.
- **MODIFIED**: `plan.md` §1b is shorter; the dispatcher contract is preserved inline (envelope extraction, schema validation, rationale-hiding mitigation, verdict routing) while the synthesizer-internal protocol body is reachable only via the `importlib.resources.files()` pointer to `plan-synthesizer.md`.
- **REMOVED**: `SKILL.md` Gate 2 (Specify → Plan ≥3 Open Decisions auto-escalation). Specs with ≥3 Open Decisions no longer auto-escalate to Complex tier at the spec→plan transition. Compensating controls: (a) the `## Open Decisions` Pre-Write Check at `specify.md` §2b already pressures resolution upstream; (b) orchestrator-review S2/S5/S6 (`orchestrator-review.md:139–150`) evaluates spec quality at plan-approval stage for medium+ criticality OR complex tier; (c) `simple+low` lifecycles fall outside (b) per the orchestrator-review skip rule and rely on manual `criticality_override` escalation, documented per Requirement 11.
- **REMOVED**: `SKILL.md` standalone "## Complexity Override" section. Gate 1 (Research→Specify ≥2 Open Questions) is now described only in the inline protocol-step subsection (Step 5).
- **REMOVED**: Two of the three inlined `complexity_override` JSON examples in `SKILL.md`, replaced with a pointer to the canonical schema in `cortex_command/overnight/events.py`.
- **ADDED**: One new assertion in `tests/test_daytime_preflight.py::test_skill_contracts` pinning the helper-module pointer in `implement.md` §1a (closes Adversarial A4).

- **Dual-source drift hook**: Edits to `skills/lifecycle/{SKILL.md,references/*.md}` must be followed by `just build-plugin` to regenerate `plugins/cortex-core/skills/lifecycle/{SKILL.md,references/*.md}`. The pre-commit hook fails on drift.
- **`requirements/project.md` SKILL.md size cap (500 lines)**: Post-trim ~350 lines passes; no exception marker required.
- **`requirements/project.md` Workflow trimming clause**: Per line 23, hard-deletion is preferred over deprecation. Gate 2 removal follows this convention; the standalone "## Complexity Override" section deletion follows it; no tombstones, env-var soft-deletes, or deprecation notices are introduced.
- **`requirements/project.md` SKILL.md-to-bin parity**: Gate 2 prose has no `bin/cortex-*` references; parity is preserved automatically.
- **Anthropic skill-authoring "one-level-deep references"**: The `plan-synthesizer.md` pointer in `plan.md §1b` is one level deep (the file is read directly via `importlib.resources.files()`, not chained through another reference).
- **`cortex_command/overnight/events.py` schema for `complexity_override`**: Unchanged. `EVENT_TYPES` already lists `complexity_override`; no event-schema migration is required by Gate 2 removal.
- **`plan.md:120` rationale-hiding sentence is verbatim-pinned**: Requirement 7's acceptance criterion is a literal `grep` for the sentence; the implementation must not paraphrase or relocate it without updating that test.
- **`tests/test_daytime_preflight.py:326,379` event-flow contract**: Unchanged by the trim. Lines reference `(implementation_dispatch + dispatch_complete)` event flow which Items 1–3 do not touch.
