## Overview
Fold the inside-repo containment check that today lives in implement.md's 33-line `python3` heredoc into `cortex-worktree-create` (a pure helper + a guard placed *before* the idempotent/create branch split so it gates every return path), then delete the heredoc and trim the redundant §1/§1a branch/worktree narration to route on the CLI verbs' outputs — fixing the step-v dash/slash resolve bug along the way. ## Risks

- **Risk 1 — Dispatch mode AND release coordination (critical-review A1).** *Dispatch:* this edits the `cortex_command` package; `just test` runs the editable install (pytest imports the working tree), so a worktree would verify stale code — pick **"Implement on current branch"** (trunk) at the §4 approval surface, and no wheel reinstall is needed *for the tests*. *Runtime (the corrected claim):* the `cortex-worktree-create` **binstub resolves to the installed wheel**, not the editable working tree (project.md "Wheel-binstub vs working-tree invocation"), and the runtime `implement.md` is read from the **installed plugin**, not the repo working tree. So source-level phase ordering (Phase 1 before Phase 2) guarantees no *commit* lacks the check, but it does **not** guarantee runtime continuity: the containment check (in `worktree.py` → wheel) and the heredoc removal (in `implement.md` → plugin) only reach runtime on reinstall. **Mitigation:** ship both phases in a single coordinated release (the wheel carrying the check and the plugin lacking the heredoc must release together; never publish a plugin with the heredoc gone against a wheel without the check). Both phases land on the same branch, so one release after both phases satisfies this. **Dogfooding caveat:** if testing locally before release, reinstall the plugin and the wheel together (or neither) — a partial reinstall (new plugin, old wheel) opens a transient no-check window. `CORTEX_COMMAND_FORCE_SOURCE=1` does NOT help here — that escape exists only in the `bin/cortex-check-*` dual-channel wrappers; `cortex-worktree-create`/`cortex-worktree-resolve` are plain console scripts with no such branch.
