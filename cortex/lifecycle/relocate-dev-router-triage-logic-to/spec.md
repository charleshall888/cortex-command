# Specification: relocate-dev-router-triage-logic-to

> Epic context: [`cortex/research/skill-efficiency-remaining-work/research.md`](../../research/skill-efficiency-remaining-work/research.md) (#340, R5). Scope is the dev-router relocation only.

## Problem Statement

`skills/dev/SKILL.md` loads on every `/cortex-core:dev` invocation before Step 1 classifies the request into one of five first-match branches. Two presentation blocks — the Step 3c triage rendering (~4.8 KB, used only by Branch 1) and the Step 2 criticality heuristic table (~1.4 KB, used only by Branch 5 and the Step-4 decline path) — are always resident even on the branches that never execute them. Relocating each block to a reference read only when its consuming branch fires removes that dilution from the branches that don't use it (the epic's "hot-path resident-tokens and clarity-harm" thesis; Web-corroborated by Anthropic's Agent Skills progressive-disclosure guidance and context-rot research). The recommendation logic stays prose, not a CLI verb: it is presentation/routing judgment over the deterministic child-map `cortex-build-epic-map` already emits.

**Honest accounting of the trade (not a symmetric free win).** Each block's relocation benefits only the branches that do *not* consume it, and shifts the consuming branch's cost from resident to a Read round-trip:
- **Criticality table** (Branch 5 + Step-4-decline): relocation de-dilutes the other branches, including the common Branch-1 triage path, at zero added cost to triage.
- **Step 3c rendering** (Branch 1 triage — a common, arguably the busiest path): relocation de-dilutes only the colder Branches 2–5; Branch 1 itself trades resident tokens for a Read round-trip plus a conceded, untestable "read-but-not-applied" risk on its decision tree. This is a **user-accepted** trade, chosen knowingly at the research exit gate over the alternative of leaving Step 3c inline (see `## Open Decisions`). The byte win is against a metric no test enforces (the L1 ratchet is frontmatter-only); the value case rests on attention-dilution theory, not a failing gate.

## Phases

- **Phase 1: Relocate & wire the two references** — Extract both blocks to consumer-keyed references under a new `skills/dev/references/`, leave pinned-heading stubs + imperative Read pointers (each using the body-resolved `${CLAUDE_SKILL_DIR}/references/<file>.md` form), place the criticality pointer after the Step-2 resume-guard's terminal route-out, and regenerate the mirror.
- **Phase 2: Standing guard & full verification** — Add a wiring-guard regression test (including a body-absence assertion as negative control) and confirm the whole gate suite stays green against a captured baseline.

## Requirements

**Priority (MoSCoW)**: R1–R9, R11, R12 are **Must-have** — without them the relocation is either not actually performed (R1/R2 move-not-copy), mis-wired (R3–R6), unmirrored/unshipped (R9), unverified (R11), or constraint-violating (R12). R10 (standing wiring guard) is **Should-have** — it protects against a future re-inlining regression and carries the move's negative-control, but the shipped feature functions without the *test* itself (the sibling #341 pattern includes it, so it is planned, not deferred). Scope note: the Step-3c half is retained on the user's informed exit-gate decision, not because "the relocation is incomplete without it" — the criticality-only scope would be a complete, smaller change; Full Option B was chosen deliberately (see `## Open Decisions`). Won't-have items are in `## Non-Requirements`.

## Non-Requirements

- Does NOT convert the recommendation tree into a CLI verb — it is presentation/routing judgment over `cortex-build-epic-map`'s deterministic output.
- Does NOT modify `cortex_command/backlog/build_epic_map.py` — its child-map contract (`{id, spec, status, title}`, `schema_version "1"`) is unchanged.
- Does NOT move Step 3b mechanics (backend gate, index regen, epic-map invocation, exit-code routing) or the Step 2 `### Resumed Lifecycle` guard into a reference — deterministic routing/safety and control-flow stay resident.
- Does NOT combine both blocks into a single reference (Option A regresses both consuming branches above baseline).
- Does NOT introduce a separate "Reference-path propagation manifest" section — dev has only two in-body Read pointers and no downstream shell lines or composed subagent prompts to propagate a resolved path to (D1/SP001 is inert for plain prose references), so two well-formed `${CLAUDE_SKILL_DIR}/references/<file>.md` pointers are the entire wiring. A manifest would be redundant ceremony and would create a second occurrence of each path token that breaks single-occurrence acceptance checks.
- Does NOT fix the pre-existing Step 4 "trivial (Branch 5)" wording mislabel — out of scope; noted so the wiring uses the correct branches (Branch 5 is the non-trivial default; the trivial branch is Branch 4; the decline path is the second caller of the criticality block).
- Does NOT touch the L1 frontmatter or raise the ratchet budget.

## Changes to Existing Behavior

- MODIFIED: `/cortex-core:dev` Branch 1 (backlog triage) now Reads `triage-rendering.md` at `### 3c` before producing triage output — shifts Step 3c's resident cost to a Read round-trip on this common path (user-accepted; mitigated by imperative pointer phrasing). Dilution is removed from the colder Branches 2–5, not from Branch 1 itself.
- MODIFIED: `/cortex-core:dev` Branch 5 and the Step-4 decline path now Read `criticality-heuristics.md` before applying the heuristic table — except on the confirm-resume sub-branch, which routes out before the Read. This de-dilutes all other branches (including Branch 1) at no added cost to them.
- ADDED: `skills/dev/references/` (dev's first reference directory) with two files, plus their auto-generated mirror.

## Technical Constraints

- **SP001/SP002 (ADR-0009)**: the body's two Read pointers MUST use `${CLAUDE_SKILL_DIR}/references/<file>.md` (own-dir form; no sibling `../`). D2 fires on a bare-relative path; D1 is inert for plain prose references (no subagent-prompt fence), which is precisely why no propagation manifest is needed.
- **Mirror is tracked, not just present**: `git diff --quiet` and the pre-commit Phase-4 drift hook (`.githooks/pre-commit:647`, same construct) are both blind to untracked files. For dev's first `references/` dir the mirror files are brand-new untracked, so the drift hook does NOT block a commit that omits them — the tracked-state assertion in R9 (`git ls-files --error-unmatch`) and the fresh-checkout CI parity run are the real protection. `git add` all new canonical + mirror files explicitly (trunk `git commit -- <pathspec>` / `--only` drops untracked files).
- **Parity R6 carve-out**: keep a narrative mention of any `cortex-*` token that lands in a reference — a token reduced to a bare flat-table cell can be dropped from wiring detection. (No binstub token actually moves — `cortex-build-epic-map` stays in Step 3b.)
- **L201**: no bare-Python `cortex_command` imports in the new references (none present in the moving content).
- **Contract lint E101/E103 + callgraph**: preserve invocation lines verbatim during the move.
- **Mirror discipline**: `just build-plugin` runs with the canonical-edit commit; edit canonical sources only.
- **Reference size**: if either reference exceeds ~100 lines, add a top-of-file table of contents (Anthropic best-practice, to survive a partial `head` read). Both blocks are well under this today.

## Edge Cases

- **Missed read at `### 3c`** (model reaches 3c but skips the Read): mitigated by an imperative "Read `…/triage-rendering.md` before producing any triage output" gate line, not a bare bracketed link. Residual runtime risk is untestable.
- **Read-but-not-applied on the Step 3c decision tree** (all-refined / any-unrefined / blocked-note-prepend logic): the correctness-bearing branching moves one hop from the child-map it operates on; moved verbatim to preserve fidelity, imperative pointer to force the read. Untestable at runtime — documented, not eliminated; R10's guard covers static wiring only.
- **Resume path is two sub-branches**: the `### Resumed Lifecycle` guard routes out only on **confirm-resume** (resume the existing lifecycle) — that path never Reads `criticality-heuristics.md` because R5 places the route-out before the pointer. The **start-fresh** sub-branch (user chooses to discard and start over) correctly proceeds to the assessment and DOES Read the reference (a fresh start needs the table). "Dir exists" alone does not skip the Read — only confirm-resume does.
- **Second caller of the criticality block**: the Step-4 decline path reaches Step 2's criticality logic in addition to Branch 5; the single pointer must be reachable from both.
- **First `references/` dir for dev**: `rsync -a` in `build-plugin` sweeps the new subdir automatically; but the pre-commit drift hook does NOT block on a missing (untracked) mirror file — R9's `git ls-files --error-unmatch` tracked-state check and the fresh-checkout CI parity run are what actually catch an omitted mirror.
- **Stub token hygiene**: R1/R2/R10 now assert token *absence* from the body; a stub pointer must therefore name the target file, not restate the extracted-content tokens (else the absence-grep fails on the stub's own text). This is the inverse of #341's presence-leak concern.

## Open Decisions

None open. Recorded decisions:
- **Scope = Full Option B** (relocate both blocks) — resolved with the user at the research exit gate, with the tradeoff explicit: option (a) full Option B "accepts a Read round-trip on triage"; option (b) "criticality only, leave Step 3c inline, lowest correctness risk." The user chose (a) knowingly. The critical-review re-raised whether the Step-3c half is worth it (it de-dilutes only colder branches while adding a triage round-trip); this remains reversible to the criticality-only scope if the user prefers, but is not re-asked absent a signal.
