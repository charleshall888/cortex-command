# Specification: reconcile-discovery-skillmd-architecture-vocabulary-with

> Epic/background reference: follow-up to #268 (`auto-consolidation-pass-in-discovery-decompose`); see `cortex/lifecycle/reconcile-discovery-skillmd-architecture-vocabulary-with/research.md` for the full analysis and the #268 deferral rationale.

## Problem Statement

`skills/discovery/SKILL.md`'s two operator-facing surfaces — the GATE-2 brief-generation fallback (line 82) and the `revise` re-walk (line 85) — name Architecture sub-sections (`### Integration shape`, `### Seam-level edges`, and a `### Why N pieces` falsification gate) that the discovery research template no longer emits. The template (`research.md` §6) emits only `### Pieces` + `### How they connect`, expressing the piece-count concern as a soft inline hint. #268 already reconciled `decompose.md`'s §1/§2/§4 to the emitted vocabulary and ADR 0007 names those two headings as the authoritative coupling-signal input, leaving `SKILL.md` as the lone straggler. When brief generation fails or a user picks `revise`, the agent is instructed to display or re-emit headings the template does not produce — orphaned guidance that misleads (and, per the Wix eval evidence in research, degrades downstream agent reasoning). Removing the gate from SKILL.md additionally orphans one residual cross-reference in `decompose.md` (§3, line 50: "the falsification gate (research-phase R3) has already run") that #268 left intact because #268 did not touch the gate. This reconciles `SKILL.md` and that residual `decompose.md` reference so all three discovery surfaces describe one artifact vocabulary, and pins the result so it cannot silently re-drift.

## Phases

- **Phase 1: Reconcile discovery instruction vocabulary** — edit the SKILL.md GATE-2 fallback (line 82) and `revise` re-walk (line 85) to the emitted vocabulary; reconcile the dangling gate reference at `decompose.md:50`; regenerate and stage both plugin mirrors.
- **Phase 2: Regression test** — pin assertions so the drifted vocabulary cannot return and the `revise` re-walk's replacement pointer stays present.

## Requirements

1. **GATE-2 fallback sub-section list (`SKILL.md:82`) names only the emitted Architecture sub-sections.** Replace the parenthetical `(sub-sections ### Pieces, ### Integration shape, ### Seam-level edges, and optionally ### Why N pieces)` with one naming only `### Pieces` and `### How they connect`. Acceptance: observable state — `grep -c "Integration shape" skills/discovery/SKILL.md` = 0 AND `grep -c "Seam-level edges" skills/discovery/SKILL.md` = 0, while the line still names `### Pieces` and `### How they connect`. **Phase**: Reconcile discovery instruction vocabulary

2. **`revise` re-walk (`SKILL.md:85`) conforms to the emitted template, drops the superseded spec pointer and the orphaned gate, and points at the live template.** Replace "re-walks the Architecture write protocol per spec R4 GATE-2 (iii) (re-emit `### Pieces`, re-run `### Integration shape` and `### Seam-level edges`, re-run the `### Why N pieces` falsification gate if piece_count > 5)" with language that (a) points the agent at the live template in `references/research.md` §6, (b) names the emitted headings (re-emit `### Pieces` per the role-naming convention, then `### How they connect`), and (c) carries the piece-count concern as the template's soft "consider merging" guidance — no `### Why N pieces` gate, no `spec R4 GATE-2` pointer. Acceptance: observable state — `grep -c "Why N pieces" skills/discovery/SKILL.md` = 0 AND `grep -c "spec R4 GATE-2" skills/discovery/SKILL.md` = 0; AND the positive content of the replacement (the `references/research.md` §6 pointer plus the two emitted headings, present in the `revise` bullet specifically) is enforced by the pinned marker-phrase assertion in Req 5 — not by a bare file-wide token check. **Phase**: Reconcile discovery instruction vocabulary

3. **Reconcile the dangling gate reference at `decompose.md:50`.** The sentence "The Architecture section's falsification gate (research-phase R3) has already run the structural-coherence merge test. Do not re-run it here. The analytical piece-set at decompose entry is the merged set." references a gate this ticket removes from SKILL.md. Reword it to drop the removed-gate naming ("falsification gate (research-phase R3)") while **preserving the behavioral instruction** — the analytical piece-set delivered by research is final; do not re-derive or re-merge it at decompose. Do NOT re-open `decompose.md` §1/§2/§4 (already reconciled by #268). Acceptance: observable state — `grep -c "research-phase R3" skills/discovery/references/decompose.md` = 0, while §3 still instructs the agent not to re-derive/re-merge the piece-set (the "Do not re-run it here" / "piece-set … is the merged set" intent survives). **Phase**: Reconcile discovery instruction vocabulary

4. **Both plugin mirrors regenerated and staged in the same commit.** After editing canonical `skills/discovery/SKILL.md` and `skills/discovery/references/decompose.md`, run `just build-plugin` and stage `plugins/cortex-core/skills/discovery/SKILL.md` and `plugins/cortex-core/skills/discovery/references/decompose.md` so the pre-commit dual-source drift hook passes. Acceptance: command + expected output — `diff -q skills/discovery/SKILL.md plugins/cortex-core/skills/discovery/SKILL.md` exits 0 AND `diff -q skills/discovery/references/decompose.md plugins/cortex-core/skills/discovery/references/decompose.md` exits 0 (both pairs identical). **Phase**: Reconcile discovery instruction vocabulary

5. **Regression tests pin the reconciliation so it cannot silently re-drift.** (a) In `tests/test_discovery_gate_presentation.py` (raw-text, reading the existing `DISCOVERY_SKILL` constant via `read_text`, mirroring that file's module-level marker-phrase idiom): assert all four drifted tokens are absent (`"Integration shape"`, `"Seam-level edges"`, `"Why N pieces"`, `"spec R4 GATE-2"` not in text) AND a **positive marker phrase pinning the `revise` bullet's replacement** is present — a stable substring of the rewritten `revise` bullet that includes its `references/research.md` §6 pointer — so a degenerate edit that deletes/stubs the `revise` bullet fails. (b) In the decompose test module (`tests/test_decompose_rules.py`, where decompose heading assertions already live): assert `"research-phase R3" not in` the decompose.md body, while leaving the existing intentional non-emitted-heading failure-message strings untouched. Acceptance: command + expected output — the new/extended assertions exist as described and `just test` exits 0 (pass if exit code = 0). **Phase**: Regression test

## Non-Requirements

- Does **NOT** edit `skills/discovery/references/research.md` (the emitted template). It is the source of truth and is read-only for this ticket. The `### Why N pieces` gate is **not** restored to the template.
- Does **NOT** re-open `decompose.md` §1/§2/§4 (already reconciled by #268) — the only `decompose.md` edit is the §3:50 dangling-reference reconciliation in Req 3.
- Does **NOT** touch the `has_why_n_justification` event subsystem: the field, the `emit_architecture_written` function or its `--has-why-n-justification` CLI flag (`cortex_command/discovery.py`); the `architecture_section_written` events-registry row (`bin/.events-registry.md:115`) **including its stale `producers` column** (which lists `skills/discovery/SKILL.md` as a producer though no active skill emits the event — already false today, more misleading after this change); the registry's **dangling `tests/test_discovery_events.py` consumer reference** (a file that does not exist); and the **actual live test consumer `tests/test_discovery_module.py`** (4 functions — `test_emit_architecture_written_writes_jsonl`, `test_emit_architecture_written_validation_rejects_negative_piece_count`, the path-routing test, `test_cli_emit_architecture_written_appends_event` — that exercise the field directly and would break `just test` if the subsystem were retired without updating them). This pre-existing orphaned subsystem (no active skill emits the event — verified) is a separate, schema-touching concern of a different risk class. **Resolution: file a follow-up ticket** to retire (or deliberately re-establish) the Why-N concept across these four surfaces. Dropping the SKILL.md vocabulary does not degrade this subsystem.
- Does **NOT** update the fixtures under `tests/fixtures/discovery-brief/{diagnostic,simple,complex}-topic/research.md` (which still use the old headings). No current test negatively asserts their heading content, and conforming them risks the `generate-brief` fixture-driven tests. **Include in the same follow-up ticket.**
- Does **NOT** touch `cortex/adr/0007` (historical rejected-alternatives prose) or the existing `test_decompose_rules.py` negative-assertion failure-message strings that intentionally name the non-emitted headings.
- Does **NOT** add the discovery `SKILL.md`/`decompose.md` mirrors to `tests/test_plugin_mirror_parity.py` — they rely on the pre-commit drift hook, as today.

## Edge Cases

- **Historical research.md artifacts** (`cortex/research/*/research.md`) still using the old four-heading vocabulary: after conforming, the GATE-2 fallback displays their actual content correctly (no hallucination), but the fallback's described headings won't match the file's. Acceptable residual — the fix targets newly-emitted artifacts; historical files are not rewritten.
- **`revise` re-walk under-specification**: dropping the spec pointer without a replacement reference would leave "re-walk the Architecture write protocol" pointing at prose that does not exist in `research.md`. The replacement (Req 2) must point at the live `research.md` §6 template, and Req 5(a)'s pinned marker phrase enforces that the pointer is present — preventing both a low-effort agent reading `revise` as free-form text editing and future silent re-drift.
- **`decompose.md:50` reword must preserve behavior**: the reconciliation (Req 3) must keep the "do not re-derive/re-merge the piece-set at decompose" instruction. A degenerate fix that merely deletes the sentence would remove a real behavioral guard. Req 3's acceptance requires the don't-re-run intent to survive.

## Changes to Existing Behavior

- **MODIFIED**: `SKILL.md` GATE-2 fallback display instruction now names `### Pieces` + `### How they connect` (was: also `### Integration shape`, `### Seam-level edges`, optional `### Why N pieces`).
- **MODIFIED**: `SKILL.md` `revise` re-walk now points at the live `research.md` §6 template and names the emitted headings (was: superseded "spec R4 GATE-2 (iii)" + the orphaned `### Why N pieces` gate). This is an **instruction-level behavioral change to the `revise` path**: an agent running `revise` is no longer told to "re-run the `### Why N pieces` falsification gate if piece_count > 5." The piece-count concern survives as the template's soft "consider merging pieces" hint, which the agent reads when re-walking from `research.md` §6. No Python/code consumer is affected — no active skill emits the related `architecture_section_written` event.
- **MODIFIED**: `decompose.md:50` — the "(research-phase R3) falsification gate has already run" cross-reference is reworded to drop the removed-gate naming while preserving the "piece-set is final; do not re-derive/re-merge" instruction.

## Technical Constraints

- **Dual-source mirror**: edit canonical `skills/discovery/SKILL.md` and `skills/discovery/references/decompose.md`; regenerate via `just build-plugin`; stage all four files (canonical + mirror, both) in the same commit (pre-commit drift hook enforces; `just setup-githooks` must have been run).
- **SKILL.md size cap** 500 lines (`tests/test_skill_size_budget.py`); file is 114 lines — ample headroom.
- **MUST-escalation policy** (CLAUDE.md): the replacement prose uses soft positive-routing phrasing and describes the output shape/intent (the What and Why), not step-by-step method. No new MUST/CRITICAL/REQUIRED tokens introduced.
- **LEX-1 prescriptive-prose scanner** and **L201 bare-Python prohibition** run on `skills/**/*.md` but neither triggers on this vocabulary-only edit (no backlog section headers; no Python imports).
- **Source-of-truth posture**: the emitted research template is authoritative over prose describing the artifact's structure (research / industry consensus). ADR 0007 (accepted) instantiates this by naming `### Pieces` + `### How they connect` as decompose's authoritative coupling-signal input; the conform-down direction for SKILL.md follows from the same posture. (ADR 0007 governs decompose's input, not SKILL.md's `revise` prose directly — the authority for the SKILL.md edit is the source-of-truth posture, which ADR 0007 evidences.)
- Commit via `/cortex-core:commit`; test-command `just test` (from `cortex/lifecycle.config.md`).

## Open Decisions

None. The scope decision — keep #269 to **prose-vocabulary reconciliation across the discovery instruction surfaces** (SKILL.md lines 82/85 + the residual `decompose.md:50` gate reference + their mirrors + regression tests) and **file a follow-up** for the code/schema `has_why_n_justification` subsystem and the stale fixtures — is resolved at the spec approval surface below, not deferred to implementation. The boundary is principled: prose vocabulary (this ticket) vs. event-schema/CLI/fixtures retirement (follow-up, a different risk class).

## Proposed ADR

None considered. The conform-down decision is not ADR-shaped: it is easily reversed (prose), unsurprising given the #268 precedent, and already governed by ADR 0007's "emitted artifact is source of truth" posture.
