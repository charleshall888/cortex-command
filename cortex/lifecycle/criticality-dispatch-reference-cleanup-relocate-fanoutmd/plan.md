# Plan: criticality-dispatch-reference-cleanup-relocate-fanoutmd (#334)

## Overview
Introduce a pure, stdlib-only `cortex-resolve-model` verb that owns the deterministic *(role, criticality) → model* Lifecycle Matrix, prove it reproduces today's dispatched models with an independent golden-anchor test, migrate the six value-bearing dispatch sites to resolve their model at runtime (deleting the inline literals), then clear the adjacent misfiled/stale references (`model-selection.md`, the `criticality-matrix.md` model column + narration, and `fanout.md`'s location). No net change to dispatched models — the four tiered roles verified cell-for-cell against the parsed matrix, the synthesizer constant pinned by the verb's own contract test.
**Architectural Pattern**: layered — a deterministic resolver layer (mechanism) is pulled out from beneath the skill prose, which keeps only the role/criticality judgment (policy) and defers the lookup to the verb.

**Open Decision resolved**: module placement is `cortex_command/lifecycle/resolve_model_cli.py` (the resolver-verb family home — siblings `state_cli.py`, `backlog_backend_cli.py`, `dispatch_choice_cli.py`), with the matrix as an inline module-level constant. This keeps imports stdlib-only and avoids any reach into `pipeline/dispatch.py`.

## Tasks

### Task 1: Create the pure resolver module
- **Files**: `cortex_command/lifecycle/resolve_model_cli.py`
- **What**: New stdlib-only module encoding the Lifecycle Matrix as *(role, criticality) → model* plus a thin `main(argv)`/`_build_parser()` CLI. Fail-loud on unknown input; never defaults.
- **Depends on**: none
- **Complexity**: complex
- **Context**: Follow the shape of `cortex_command/lifecycle/state_cli.py` (module docstring, `_build_parser()`, `def main(argv: Optional[List[str]]) -> int`). Inline the matrix as a module-level constant `_LIFECYCLE_MATRIX: dict[str, dict[str, str]]` keyed role→{criticality: model}: `review` {low:sonnet, medium:sonnet, high:opus, critical:opus}; `builder` {identical to review}; `orchestrator-fix` {low:sonnet, medium:sonnet, high:sonnet, critical:opus}; `competing-plan` {critical:sonnet} (low/medium/high absent → undefined). Model `synthesizer` is **criticality-independent** → represent it as a separate constant set (`_CRITICALITY_INDEPENDENT = {"synthesizer": "opus"}`), NOT a matrix row — a constant can't drift and a row would invite a future editor to "fill in" empty low/medium/high cells (research adversarial finding). argparse: `--role` `required=True` (choices = the five role names); `--criticality` optional (choices `low|medium|high|critical`). Logic: criticality-independent role → print its model regardless of `--criticality`; tier-keyed role with `--criticality` omitted → exit 2 + stderr; unknown role OR unknown/undefined (role,criticality) cell → exit 2 + stderr naming valid values; success → print bare model name + newline, exit 0. **Do NOT** `import` from `cortex_command/pipeline/dispatch.py` and do not modify it. Add a load-bearing comment noting the one discriminator that makes `--role` mandatory: `review@high=opus` vs `orchestrator-fix@high=sonnet` (the full design rationale lives in `docs/internals/sdk.md`, added in Task 7).
- **Verification**: run `CORTEX_COMMAND_FORCE_SOURCE=1 PYTHONPATH=. python3 -m cortex_command.lifecycle.resolve_model_cli --role review --criticality high` prints `opus` (exit 0); `--role orchestrator-fix --criticality high` prints `sonnet`; `--role synthesizer` prints `opus`; `--role review` (no criticality) exits 2; `--role bogus --criticality high` exits 2 — pass if all hold. AND `grep -nE "import .*dispatch|from .*dispatch" cortex_command/lifecycle/resolve_model_cli.py` returns nothing (exit 1).
