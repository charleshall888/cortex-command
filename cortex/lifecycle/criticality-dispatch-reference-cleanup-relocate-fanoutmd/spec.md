# Specification: criticality-dispatch-reference-cleanup-relocate-fanoutmd (#334)

## Problem Statement

The lifecycle's model-selection policy — *which model a given dispatch role uses at a given criticality* — is deterministic, yet it lives as prose duplicated across four places (`model-selection.md`'s Lifecycle Matrix, `criticality-matrix.md`'s model column, and ~6 inline directive sites). The governing principle for this project is that **deterministic behavior belongs in the CLI and only genuine judgment stays with the LLM**; a duplicated prose policy table violates that. Today the copies happen to agree (there is *no* live drift — the `review` vs `orchestrator-fix` difference is a *deliberate* role threshold, both correct), so this is **preventive single-sourcing**: collapse the deterministic lattice into one executable source (`cortex-resolve-model`) that the value-bearing dispatch sites resolve at runtime, with an independent golden anchor proving the verb reproduces today's dispatched models exactly, then clear the adjacent misfiled/stale references (`fanout.md` location, `model-selection.md` rationale prose, a self-narration paragraph).

## Priority (MoSCoW)

- **Must** (core single-sourcing + the safety net that makes it non-regressing): Reqs 1–11, 14, 15, 16, 17. Includes the golden-anchor test (Req 6) — without it the "models unchanged" claim is unverifiable.
- **Should** (quality, deferrable as fast-follow): Req 12 (relocate the *durable* rationale to sdk.md — the stale-number deletion within it is Must; the destination of the qualitative prose is Should) and Req 13 (sdk.md pipeline-matrix back-pointer).
- **Won't** (this iteration): see `## Non-Requirements`.

## Phases

- **Phase 1: Resolver verb foundation + golden anchor** — a pure stdlib-only role×criticality module + `cortex-resolve-model` CLI + binstub + tests, including a test that **parses the still-present `model-selection.md` Lifecycle Matrix** and asserts the module reproduces it (the independent ground-truth check). Parity wiring is established within this phase.
- **Phase 2: Call-site migration** — the value-bearing dispatch sites resolve their model via the verb; no model literal survives in the directive; criticality-independent sites resolve criticality-free.
- **Phase 3: Asset dedup & rationale relocation** — freeze the (now-verified) golden values into the test, then delete `model-selection.md`, move durable rationale to `docs/internals/sdk.md`, point sdk.md at the verb/`resolve_model()` as authoritative, remove `criticality-matrix.md`'s model column and its line-24 narration.
- **Phase 4: fanout.md relocation** — move `fanout.md` to `research/references/`, repoint all citers (all path forms) and the hardcoded test path.

## Non-Requirements

- **Does NOT wire the orphan roles** (`exploration`, `parallel-research`): `research/SKILL.md` dispatches those agents with no model pin today; adding pins would be a silent behavior change, out of scope. Their historical policy note moves to sdk.md docs; pinning them is a candidate follow-up ticket.
- **Does NOT change the SDK-side pipeline model selection**: `resolve_model(complexity,criticality)` in `dispatch.py` and its in-process callers are untouched.
- **Does NOT change `verification-gates.md:44`** (`--model-tier`) or `synthesizer-prompt.md` (a prompt title): neither is a model *chooser*.
- **Does NOT fix the un-pinned critical-review reviewer dispatch** (a latent telemetry-vs-dispatch gap) — noted as a follow-up.
- **Does NOT introduce any MUST/CRITICAL escalation** in the migrated prose (soft positive routing per the MUST-escalation policy).

## Edge Cases

- **Orchestrator skips the RUN**: mitigated by the hard-fail handler (halt, never guess) and by the verb call being a concrete bash step. Note honestly: skill prose is **not** a control-flow gate — true structural gating (blocking dispatch until the verb resolves) is a per-site implementation option to apply where the skill flow allows, not a guarantee the prose provides. Where a site cannot reliably run the verb, flag it during implementation.
- **Standalone critical-review (no lifecycle session)**: the synthesizer site runs `cortex-resolve-model --role synthesizer` with NO `cortex-lifecycle-state` read, so a missing/corrupt lifecycle state cannot halt it (this is the fix for the regression the original spec would have introduced). The verb itself ships in the same wheel as the `cortex-critical-review` binstubs critical-review already depends on, so it adds no new availability class.
- **`cortex-resolve-model` not on PATH mid-implementation** (editable wheel not reinstalled): tests invoke `main([...])` in-process AND a subprocess smoke test (Req 6) provides the parity wiring; same-session phases use `python3 -m` or `CORTEX_COMMAND_FORCE_SOURCE=1`.
- **Unknown/typo'd role or tier-keyed role missing criticality**: verb exits nonzero → site halts and escalates, surfacing the miswiring instead of dispatching a wrong model.
- **Transcription error in the module matrix**: caught by the Req 5 golden-anchor test, which parses today's `model-selection.md` (Phase 1) rather than a same-change restatement.

## Changes to Existing Behavior

- **MODIFIED**: the value-bearing dispatch sites stop reading an inline model literal and resolve the model at runtime via `cortex-resolve-model`. Net dispatched models are unchanged — and this is now *verified* by the Req 5 golden anchor, not merely asserted.
- **ADDED**: a new global verb `cortex-resolve-model` (new command surface).
- **REMOVED**: `skills/lifecycle/assets/model-selection.md`; `criticality-matrix.md`'s model-selection column + line-24 narration; the version-pinned benchmark numbers.
- **MOVED**: `fanout.md` from `lifecycle/references/` to `research/references/`.

## Technical Constraints

- Pure, stdlib-only resolver module; bare model name on stdout; exit 2 on unknown input / tier-keyed-role-missing-criticality (matches the repo fail-loud posture). `--role` `required=True`; `--criticality` optional (role-conditional in verb logic).
- Parity (`cortex-check-parity`): a `[project.scripts]` entry counts as deployed — wired via the Phase-1 test's console-name reference (a matrix table cell does NOT count). Independent of the Phase-2 migrated sites, so the Phase-1 acceptance is satisfiable in-phase.
- L201 / SP001-SP002 / E101-E103 lints pass on migrated prose (`--criticality` optional → synthesizer mentions are contract-clean).
- Dual-source: edit canonical only; `just build-plugin` + commit canonical + mirror together (pre-commit drift hook). Explicit-pathspec commits if a concurrent session is live.
- Repoints honor ADR-0009 (`${CLAUDE_SKILL_DIR}` body-resolution; own-dir `${CLAUDE_SKILL_DIR}/references/fanout.md` for research after the move). #328 is the concrete template.
- Editing `cortex_command/` uses sequential dispatch, not worktree (`just test` runs the editable install).

## Open Decisions

- **Exact module placement** (`cortex_command/lifecycle/` vs a neutral `cortex_command/model_resolver.py`): both satisfy the constraints; the choice depends on import-graph cleanliness once the module is drafted. *Deferred: distinguishable only when writing the module (which sibling imports stay stdlib-only).*

## Proposed ADR

None considered. The refactor is mechanically reversible (re-inline the literals), so it fails the ADR three-criteria gate (criterion 1, hard-to-reverse). The durable rationale lives in `docs/internals/sdk.md` + a verb-module comment instead.
