# Specification: restructure-commit-skill-lazy-load-release

## Problem Statement

`skills/commit/SKILL.md` (99 lines) loads into context on every commit the lifecycle makes — ~9 builder loads per feature run — yet the release-type version-bump marker mechanics (regex, precedence, `BREAKING:` fallback, `--dry-run` procedure) and two worked examples are only relevant on the rare release-worthy commit, and three narration sections restate procedure a capable model performs unprompted or that the commit hook already enforces. The 2026-07 skill-value audit verified five sections trim-safe (s3, s4, s5, s7, s8) with named keep-lists, plus one provisional (s6). This feature applies those verdicts to cut resident hot-path tokens **without** dropping any load-bearing directive. Two sharp constraints govern: (1) a malformed or missing release-type marker fails **silently** (auto-bump defaults to `patch`; the regex runs only in post-merge CI), so the restructure must keep the marker *decision trigger* resident even as it moves the mechanics behind a lazy pointer; (2) the ticket's Integration precondition requires the column-0 `BREAKING:` clause to stay **visible** in the body, so a one-line resident `BREAKING:` backstop is kept (its fuller mechanics move to the ref).

## Phases

Group by blast radius. **This feature is intended to land as a single commit** (canonical `SKILL.md` + new ref + regenerated mirror). If the implementer splits the two phases into separate commits, **each commit that touches `skills/commit/` must run `just build-plugin` and stage the regenerated mirror** — the drift pre-commit hook is fail-closed on any `skills/`-touching commit, and Phase 1 creates files under `skills/commit/`, so mirror-regen cannot be deferred to a Phase-2-only step.

- **Phase 1: Extract release-type mechanics (s7, s8)** — move the marker regex, precedence, `BREAKING:` mechanics, `--dry-run` procedure, and both worked examples into a new lazily-loaded `skills/commit/references/release-type.md`; keep the marker *decision trigger*, the two marker tokens, the own-line MUST, and a one-line `BREAKING:` backstop resident behind a `${CLAUDE_SKILL_DIR}` pointer. (Regenerate the mirror in this commit if phased separately.)
- **Phase 2: Compress narration and fold Validation (s3, s4, s5, s6)** — compress Workflow, Commit Message Format, and Commit Command in place; fold Validation while preserving "do not bypass"; regenerate the cortex-core mirror.

## Requirements

**Priority (MoSCoW)**: Requirements 1–10 are all **Must-have** — the restructure is incorrect, unverifiable, or unshippable without any one of them. There is no Should-have tier: the previously-optional one-line resident `BREAKING:` backstop is now a definite keep (Req 3) to honor the ticket precondition and remove ambiguity. **Won't-do** items are enumerated under Non-Requirements. Nothing here is must-have-by-default padding. **Anti-vacuity note**: because every "keep" token already exists in the unchanged file, keep-greps alone cannot prove the restructure happened — each requirement below pairs keep-checks with **removal** checks and/or the Req-10 body-size gate so a no-op or a delete-instead-of-move implementation fails.

## Non-Requirements
- Does **not** change `skills/commit/SKILL.md` frontmatter (`name`, `description`) — this is a body-only edit; the L1 surface ratchet is not engaged and must not be perturbed.
- Does **not** change any Python (`cortex_command/auto_bump_version.py` already owns the regex contract) — prose + new ref file + mirror only.
- Does **not** attempt to single-source the marker regex, and adds no byte-identity test between the ref prose and `auto_bump_version.py`. The regex will exist in two places (the moved ref + `auto_bump_version.py`); `docs/release-process.md:124` mentions the `[release-type: minor]` token in prose but holds **no** regex copy, so it is unaffected. "Reduces duplication" is explicitly not a claimed benefit.
- Does **not** re-verify or re-open the s3/s4/s5/s7/s8 verdicts — they are `verified_survives`; this feature applies them.
- Does **not** touch the other epic #347 children (implement.md, research, project.md, lifecycle.config).

## Edge Cases
- **Agent composes a release-worthy commit but never opens the ref**: mitigated by Req 2 — the decision conditional + marker tokens stay resident folded into the message guidance, so the common case (drop a `minor` marker) needs no ref open; the ref is only for mechanics detail.
- **Marker written mid-line or indented**: silently ignored by the CI regex (falls through to `patch`). Mitigated by keeping the own-line rule resident (Req 2/5) and folding both worked examples into the ref (Req 1) as defense-in-depth.
- **Rejected commit "bypassed" via `git commit -F`/editor**: the hook is command-shape-scoped and blind to these forms; mitigated by preserving "do not bypass" as a standalone imperative (Req 9).
- **New ref file left untracked at commit**: the drift hook's working-tree-vs-index phase fails closed; mitigated by explicit `git add` of all four paths + running `build-plugin` before commit (Req 10).
- **Phase 1 committed separately from Phase 2**: Phase 1 touches `skills/commit/`, so its commit trips the fail-closed drift gate unless the mirror is regenerated in that same commit — mitigated by the Phases note and Req 10 applying per skills-touching commit.
- **A bare-relative read path leaks into the new ref file**: would trip SP002 (the ref is in the `skills/**/*.md` lint corpus). Mitigated by keeping the ref's `--dry-run`/prohibition prose free of bare `read`/`bash <path>` constructions; fenced ` ```bash ` blocks are safe only because the actual content carries no bare-relative path, not because fences are categorically exempt.

## Technical Constraints
- **Silent-misfire hazard is the governing constraint**: release-type regexes run only in post-merge CI; a missing/malformed marker defaults to `patch` with no commit-time signal (`cortex_command/auto_bump_version.py` `get_bump_scope`). Resident text must prompt the decision, not merely document the syntax.
- **Anti-vacuity acceptance discipline**: every "keep" token already exists in the unchanged 99-line file, so keep-greps alone are non-discriminating. Acceptance criteria pair keeps with removal-greps (Reqs 3, 7, 8, 9) and a body-size gate (Req 10) so a no-op or delete-instead-of-move implementation fails.
- **Skill-path lint (SP001/SP002)**: body pointer must carry the `${CLAUDE_SKILL_DIR}/` prefix (D2-exempt); the new ref file is in the lint corpus. SP001/D1 is N/A (no subagent-prompt fence in the commit skill). ADR-0009.
- **Mirror parity is a same-commit, fail-closed gate**: `just build-plugin` (`justfile` `rsync -a --delete` sweeps the new `references/` subdir automatically, no manifest change; `commit` is in the cortex-core `SKILLS` list); `.githooks/pre-commit` rejects any `skills/`-touching commit whose mirror was not regenerated + staged. Stage canonical + mirror together; new files must be explicitly `git add`ed.
- **L1 ratchet not engaged** (frontmatter untouched; `commit` not in the routing-pressure cluster); **SKILL size cap** (500 lines, `tests/test_skill_size_budget.py`) not the forcing function (file is 99 lines) — the forcing function is weighted hot-path token reduction per epic #347.
- **Locate sections by heading and pinned tokens**, not `master_candidates.json` line ranges (they predate the trim commit; here they happen to still match).

## Open Decisions
None. The three decisions surfaced in research are resolved above: (1) the `BREAKING:` clause keeps a **one-line resident backstop** (Req 3, now a definite Must-have) while its fuller mechanics move to the ref (Req 1) — honoring the ticket Integration precondition ("keep the column-0 clause visible") and the adversarial reframe together; (2) the s8 examples fold into the ref rather than being dropped (Req 1) — zero hot-path cost, defense-in-depth for a silent failure mode; (3) the decision trigger is folded into the message-composition guidance (Req 2).
