## Web Research

The lazy-ref move is **Anthropic's canonical progressive-disclosure pattern**, not a novel risk. Anthropic's "Agent Skills" model: L1 frontmatter (always resident) → L2 SKILL.md body (loaded on trigger) → L3 reference files (loaded only when navigated to). Direct guidance for this exact move: "If certain contexts are mutually exclusive or rarely used together, keeping the paths separate will reduce token usage." The `skill-creator` guidance sets the ≤500-line SKILL budget and the placement rule: reference files must be "referenced clearly from SKILL.md with guidance on **when** to read them," and all triggering/"when to use" information must stay in the body, never deferred to a reference.

**Classification heuristic (from the sources):** keep resident anything needed for (1) *triggering / deciding to act*, (2) core process/control-flow, (3) anything *wrong-by-default if omitted* (a capable model would guess incorrectly rather than degrade gracefully). Move anything *correct-by-omission until a rare branch fires*. Anthropic's "right altitude" context-engineering guidance (avoid both over- and under-specification) is close to a direct citation for the project's "prescribe What and Why, not How" principle; arXiv:2510.22251 gives narrow empirical support that over-procedural prompting can *degrade* frontier-model correctness, not just cost tokens.

**Named risk:** if the pointer isn't explicit about *when* to follow it, the model may never navigate to the reference and will lack the info at the moment of need — mitigation is structural: keep anything needed to *decide whether to act* in the always-loaded portion.

## Tradeoffs & Alternatives

**Decision A — separate `references/` file vs. compress-in-place → EXTRACT.** This is a *frequency*-mismatch problem (rare-need content resident on ~9 loads/run), and only lazy-loading fixes frequency; compress-in-place merely shrinks a tax paid every load without removing it, so it does not solve the stated problem. The added surface is minimal here: mirror sweep is automatic, and the pointer is the simplest ADR-0009 case (no subagent propagation). Matches s7's `verified_survives` verdict. This reasoning is **section-specific** — s3/s4/s5 (Workflow/Format/Command) apply to *every* commit and are correctly compressed in place, not extracted.

**Decision B — s6 (Validation) → INCLUDE (fold), with condition.** Folding loses no text if the directive is preserved. But s6 is UNVERIFIED (votes:0) and — per the adversarial pass — "do not bypass" is load-bearing: the hook is command-shape-scoped, so a rejected commit can be forced through via `-F`/editor, which the hook cannot structurally catch. **Preserve "do not bypass" as its own standalone imperative sentence**, not a paraphrased subclause.

**Decision C — s8 examples → fold-into-ref (mild lean) vs. drop.** Once the ref exists, hot-path cost is zero either way, so this is purely "does a worked example earn its place in a rarely-opened file." The failure mode (marker not on its own line → silently ignored → wrong semver) favors keeping two short examples as low-cost defense-in-depth. Dropping entirely is defensible (audit value:2, DELETE-leaning). Closest call — deferred to Spec.

## Adversarial Review

**The keep/move boundary must be reframed — this is the load-bearing finding.** The resident release-type text's value is **decision-prompting, not lookup.** A reference is consulted when you already know you need it; the release-type text is different — the always-visible prose is *what makes the author consider "is this minor/major?" at all.* Because the default is `patch` and a missing marker fails **silently** (verified: `get_bump_scope` returns `patch` on empty `explicit`; regexes run only post-merge in CI), the failure chain for a naive extraction is: agent composes a feature commit → nothing resident prompts the marker decision → agent never recognizes release-worthiness → never opens the ref → CI silently cuts a **patch** for what should have been **minor/major**. No automated gate catches it. **Worst in the overnight/autonomous path**, where speculative ref-reading is weakest and the under-bump is least likely to be caught before shipping.

**Corrected residue — keep the *whether*, move the *how*.** KEEP RESIDENT:
- The **decision trigger as an authoring conditional**, placed *at the message-composition step* (not only a trailing pointer): default is patch; a new backward-compatible feature → `[release-type: minor]`; a breaking change → `[release-type: major]`.
- The **two marker tokens** (so the common case needs no ref open).
- The **own-line MUST rule** (a mid-line marker is silently ignored — highest-consequence syntax rule).

MOVE TO REF (safe): the **precedence rule** (`major>minor>patch` is a CI-parse-time range resolution — an author only ever marks their own commit's scope and never needs precedence to do so correctly); the **exact regex** (source of truth is `auto_bump_version.py`); the **`BREAKING:` fallback mechanics** (the prose itself calls it a backstop — "prefer the explicit marker; rely on `BREAKING:` only as a backstop"); the **`--dry-run` pre-merge procedure**; the **examples**.

**s4 keep-list is larger than "keep imperative-mood."** Mapping every s4 rule against the hook: capital / no-period / blank-line are hook-enforced (safe to drop from prose); but **imperative-mood** (past-tense blacklist misses "Adds"), **"summarize the why, not the what"** (zero enforcement anywhere, exists ONLY here), and the **72-char wrap** (hook only checks ≥10) are NOT reliably enforced — **keep all three** (compressed; the Add/Fix/Remove exemplars earn their place). Drop only the derivable format-template fence and the hook-duplicated bullets.

**Pre-commit footguns:**
- **New files are untracked** → `git commit --only`/pathspec drops them, and the drift hook's working-tree-vs-index phase then fails closed. Stage all four paths explicitly (`git add` the 2 canonical + 2 mirror), run `build-plugin` in the same commit.
- Body pointer must use the `${CLAUDE_SKILL_DIR}/` prefixed form; keep bare-relative read/bash paths out of the ref (SP002).
- Byte-identity of the marker prose with `auto_bump_version.py` is **untested** — relocating to a less-trafficked ref slightly *raises* silent-drift risk from the CI source of truth. Combined with `docs/release-process.md:124` as a third copy: **"reduces duplication" is not a real benefit of this change** — don't frame it that way.
