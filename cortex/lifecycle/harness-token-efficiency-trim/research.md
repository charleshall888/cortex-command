# Research: harness-token-efficiency-trim

## Provenance

This research was produced in-conversation on 2026-06-10 via two multi-agent workflow passes, not via the standard `/cortex-core:refine` delegation:

1. **Deep-research pass** (run `wf_2bfda90f-131`, 30 agents, ~2.3M tokens): 12 per-file trim auditors, each followed by an adversarial verifier instructed to refute every proposed cut; 5 structural researchers (constraint inventory, progressive-disclosure design, cross-file duplication map, metrics-fix scoping, execution-process rules); a completeness critic.
2. **Lock-in adversarial panel** (run `wf_77f642ae-7bb`, 5 agents): steelman defense, dedicated attacks on mechanism bundling / execution mechanics / scope boundary, and a neutral comparator that scoped from the evidence before seeing the proposal.

The full machine-readable evidence (per-section proposals with verifier verdicts, duplication inventory, constraint anchors, panel arguments) is in `evidence.json` in this directory. User approvals for scope, mechanism, and packaging were given in-conversation; the corresponding events are logged in `events.log` with this file as the record of what was approved.

## Problem

A full complex-tier lifecycle loads ~270–330KB (~68–83k tokens) of instruction text into the orchestrator context and multiplies it through 30+ sub-agent dispatches. Much of the instruction mass is maintainer-facing rationale (ADR recaps, parity-anchor notes, repeated `${CLAUDE_SKILL_DIR}` explanations), defensive How-narration that post-Opus-4.7 models do not need, and cross-file duplication. Separately: the user's dominant usage pattern — `/refine` in one session, then a fresh session resuming `/lifecycle` at Plan — loads ~60.5KB before planning starts, of which ~10–12KB is dead on that path (refine-delegation steps, index.md creation, discovery bootstrap).

## Key findings

### F1 — Verified-safe trim inventory (evidence.json `trims_verified`)
Auditors proposed 48,934 bytes of cuts across 12 lifecycle/refine files; adversarial verifiers approved **36,539 bytes as safe**, downgraded 36 proposals to safer actions (typically condense-instead-of-remove), and refuted 2 outright. Verifier rejections cite concrete anchors: kept-pauses parity line positions, dispatched-verbatim prompt templates, test substring pins, recorded design decisions. Per-file safe savings: implement.md 6,835B; plan.md 4,205B; refine SKILL.md 4,010B; clarify-critic.md 4,145B; post-refine-commit.md 3,260B; complete.md 3,000B; SKILL.md 2,710B; specify.md 2,286B; clarify.md 1,965B; review.md 1,823B; orchestrator-review.md 1,490B; backlog-writeback.md 810B.

### F2 — Cross-file duplication (evidence.json `duplication`)
~9–10KB removable via four dedup moves, in order: (1) extract the specify/plan §3b critical-review gate (95%-identical 1.2KB pair) to a shared reference using the proven 2-line-pointer pattern already live for orchestrator-review.md; (2) trim critical-review SKILL.md's restatements of verification-gates.md (kills a live drift pair at SKILL.md:86 vs verification-gates.md:84-86); (3) canonicalize the `cortex-lifecycle-state` read protocol (8 sites, 6 files) in criticality-matrix.md, keeping bare commands inline; (4) micro-canonicalizations (glossary sentence → load-requirements.md, index.md update block, exit-2 handling, sub-task race rule). **Explicitly locked out**: the four dispatched-verbatim READ_OK copies (sub-agents cannot follow pointers), approval-surface skeletons (parity-test coupled), auto-advance reinforcements (plausible pause-suppression value), model-routing inline values (declared intentional).

### F3 — Progressive disclosure: option (b) surgical extraction
Resume-at-plan loads 60.5KB today; only ~10–12KB is path-dead because the rest (resolver, detect-phase routing, transition rules, completion gates, kept-pauses inventory, path-propagation manifest) is genuinely shared — the session auto-advances into Implement/Review so it needs the transition machinery. Option (b): extract SKILL.md's refine-delegation steps (~3.4KB, lines 140–163) to `references/refine-delegation.md`; make the discovery-bootstrap.md read conditional on `phase = none/research` (the file already self-scopes to that); split backlog-writeback.md's "Create index.md (New Lifecycle Only)" section out of the resume path. Net ~9KB off resume (~27% of pre-phase overhead); fresh path pays one read hop on content it reads anyway. The extraction carries 5 `${CLAUDE_SKILL_DIR}` occurrences — the body's Reference-path propagation manifest must pre-resolve the extracted file's targets per ADR-0009 (`cortex-check-skill-path` gates this), and `tests/test_post_refine_commit_wired.py` anchors plus post-refine-commit.md's three "lifecycle Step 3 §4" cross-references need same-commit updates.

### F4 — Option (c) dispatch-table consolidation: REJECTED (affordance analysis)
Option (c) proposed merging Step 2's routing table and Step 3's artifact-existence gate into one phase-keyed dispatch table. The lock-in panel refuted its premise: the two gates are **independent error-correctors, not redundancy**. `detect_lifecycle_phase` (common.py:360–368) checks only spec.md once it exists — it returns `plan` for the exact spec-without-research inconsistent state that Step 3's artifact gate catches (SKILL.md:138). Under a torn `spec_approved` events.log line (silently skipped at common.py:272–275) the detector and the artifact gate genuinely disagree; collapsing them forces choosing a winner in corruption states — a behavior change. (c) saves zero bytes (the "redundant" surface is ~1.8KB + 0.9KB that a merged table still needs), and the bundling rationale (shared parity churn) was measured as ~zero — the parity test's only SKILL.md anchor is line 60, above the edit region. Per CLAUDE.md's authoring guideline, this boundary's affordance is real: the Step 3 gate is the only check that catches detector-invisible inconsistent states. **Do not re-propose without new evidence.**

### F5 — Constraint surface (evidence.json `constraints`)
~25 test modules read these files. Keystones: `tests/test_lifecycle_kept_pauses_parity.py` (bidirectional inventory↔call-site parity, LINE_TOLERANCE=35, heading-regex section location); byte-equality snapshot of complete.md's `**Hard guard**:` paragraph against `tests/fixtures/complete_md_hard_guard.txt`; verbatim substring pins concentrated in implement.md (step-v ordered tokens, branch-mode wiring, EnterWorktree co-location) and complete.md (~40 substrings, `<!-- finalization-commit-step -->` marker region); `tests/test_lifecycle_references_resolve.py` hard-fails when cited lines pass EOF. Pre-commit phases: check-parity (E002/W003), check-contract (E101/E103 — inline `cortex-*` invocations need required flags), events-registry, prescriptive-prose, skill-path SP001/SP002, dual-source mirror regen (canonical+mirror must land in the same commit; regen works on feature branches). Unguarded hazard: `cortex_command/overnight/prompts/orchestrator-round.md:242,302,413` cite plan.md §1a/§1b normatively, `cortex_command/lifecycle_config.py:8-9,95-96` cite plan.md §5 / complete.md Step 2, `report.py:965` cites review.md §4a — no test covers these; a post-trim grep is mandatory.

### F6 — L1 frontmatter surface
The only always-loaded mass (every session, every project). Current total 8,339 bytes vs 5,777 at lifecycle #191's post-trim snapshot (+44%). Decomposition: four skills added since #191 contribute 2,368B uncapped (interview 758, requirements-write 685, requirements-gather 498, backlog-author 427); the original 13 grew only ~194B (+3.4%). #191's R6 caps had a documented MISS at close (research 502 now / 378 then vs 200 cap). A ~30-line ratchet test freezing today's bytes ships in this feature; the re-trim (really: cap policy for new skills + the research overage) is deferred. Baseline captured in evidence.json `l1_surface_baseline`.

### F7 — Execution context
Concurrent lifecycle #287 completed 2026-06-10T17:10Z; tree clean. Local main is 1 ahead (unpushed 020cbb5c, other session's) / 1 behind (origin v2.24.0) — this feature branches from `origin/main` (362bf765) to keep the PR diff clean of both. Worktree-mode is blocked for anything touching cortex_command verification (worktree `.venv` symlinks to the parent venv whose editable pth points at the main checkout — `just test` would validate stale code); this feature uses the main checkout on the feature branch with sequential dispatch. The discovery/lifecycle orchestrator-review near-duplicate (79 divergent lines) is pulled into scope: trimming one side while deferring the sync would entrench the drift.

### F8 — Out of scope, deferred with evidence
(a) Frontmatter cap policy + re-trim (F6) — ticket must carry the decomposition, not the misleading "+44% regrowth" framing. (b) research/SKILL.md (15,018B) and critical-review references (~45KB, loaded twice per complex feature) — no trim maps exist; building them mid-lifecycle would inflate this feature; tickets must inline the known constraint anchors (test_research_fanout_matrix.py, test_discovery_research_sizing.py; reviewer/fallback/synthesizer prompts are dispatched-verbatim and largely untrimmable). (c) Metrics fixes (tier attribution to final effective tier + initial_tier field; SessionStart float rounding) — approved, but ship as their own small PR for revert isolation and review-mode separation; they avoid common.py and are not lifecycle-gated.
