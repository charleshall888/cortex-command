# Specification: cortex-lifecycle-state-collapses-to-null

## Problem Statement

`cortex-lifecycle-state` (`cortex_command/lifecycle/state_cli.py:_reduce_events`) collapses its entire reduce to `None` on the first `events.log` line that fails JSON parse, then `main()` writes the filtered result and exits 0 with no diagnostic ‚ÄĒ so `--field tier` reads as absent and the spec-phase ¬ß3a/¬ß3b adversarial gates silently default to `simple` and skip, exactly on the complex/critical features that most need review. The sibling readers `common.py:read_tier`/`read_criticality` (consumed by the overnight runner's gating path) instead tolerantly skip malformed lines and return the last valid value ‚ÄĒ a split-brain in which two readers of the same log disagree. Research found this is the **third** hand-rolled reducer of these events (`state_cli`, `common.py`, `refine.py`), with `state_cli` the lone outlier; that the readers also diverge on **non-UTF-8 handling** ‚ÄĒ `common.py`'s strict decode raises `UnicodeDecodeError` into the overnight runner (a latent crash) while `state_cli` silently replaces ‚ÄĒ and on override field-precedence; and that the dominant real trigger is an LLM emitting a malformed `lifecycle_start` line. This feature unifies the three reducers behind one tolerant shared helper, **fails the gate safe toward review when corruption makes the tier unknowable**, validates accumulated values against the known vocabulary, and closes the latent crash. Beneficiaries: the lifecycle/refine spec-phase gates and the overnight review gate (they fire on un-classifiable corrupted state instead of silently skipping), the overnight runner (no crash), and operators (corruption is surfaced on the CLI/interactive paths).

## Phases

- **Phase 1: Unify reducers + reconcile torn-line / encoding / field-precedence / vocabulary** ‚ÄĒ one shared tolerant, vocabulary-validating reducer in `common.py`, called by all three reader sites, that also records whether any line was skipped; non-UTF-8 no longer crashes; readers agree by construction on identical content.
- **Phase 2: Gate fail-safe + observability + test re-founding** ‚ÄĒ the gate-deciding consumers fail safe toward review on un-classifiable corruption; a CLI-only human-readable warning on skipped lines; flip the bug-pinning fixture and re-found its README; harden and extend the cross-reader agreement test.

## Requirements

1. **Shared tolerant, vocabulary-validating reducer.** Add a single reducer to `cortex_command/common.py` (e.g. `reduce_lifecycle_state(events_path)`) that reads the log with `errors="replace"`, skips any line that fails `json.loads` (continue), accumulates tier/criticality per the canonical rule (`lifecycle_start` seeds both; `complexity_override`/`criticality_override` supersede), and **accepts a value only if it is in the known vocabulary** (`tier ‚ąą {simple, complex}`, `criticality ‚ąą {low, medium, high, critical}`) ‚ÄĒ an out-of-vocabulary value (e.g. a U+FFFD mojibake string that still parses as JSON) does NOT supersede a prior valid value and is treated as a skipped line. The reducer **records whether ‚Č•1 line was skipped** (parse failure or out-of-vocab) so corruption is detectable. It returns the `{tier?, criticality?}` accumulator plus the skip indicator ‚ÄĒ **never `None`**, **never raising on non-UTF-8**. Acceptance: `grep -c "def reduce_lifecycle_state" cortex_command/common.py` = 1; unit tests pass (`pytest -k reduce_lifecycle_state` exits 0) covering: non-UTF-8 byte returns without raising; an out-of-vocab override value does NOT overwrite a prior valid `tier`; a torn line sets the skip indicator true. **Phase**: Phase 1.
2. **`read_tier`/`read_criticality` rewired as thin wrappers** over the shared reducer, each projecting the accumulator to its existing defaulted bare-string return (`tier`‚Üí`simple`, `criticality`‚Üí`medium` when absent), and preserving their `@lru_cache(maxsize=128)` and `.__wrapped__` introspection contract (pinned by spec-R1 tests and `test_bin_lifecycle_state_parity.py`). Acceptance: `grep -c "read_tier.__wrapped__" cortex_command/common.py` ‚Č• 1 AND `grep -c "read_criticality.__wrapped__" cortex_command/common.py` ‚Č• 1; the spec-R1 introspection tests pass (`just test` exits 0). **Phase**: Phase 1.
3. **`state_cli` consumes the shared reducer; the `return None` torn-line branch is removed.** On a torn log `state_cli` emits the last-valid accumulator dict (never `null`). Acceptance: running `cortex-lifecycle-state` against the torn-line fixture with `--field tier` emits `{"tier":"complex"}` (not `{}` / `null`); `grep -c "return None" cortex_command/lifecycle/state_cli.py` = 0. **Phase**: Phase 1.

## Non-Requirements

- Does NOT add atomic-append, locking, fsync, or any write-side change to the `events.log` producers ‚ÄĒ write-side hardening would violate ADR-0001 / the no-locks forward-only constraint and is out of scope.
- Does NOT add a schema validator or daemon for `events.log`.
- The R7 corruption signal is consumed ONLY by the gate-deciding consumers (¬ß3a/¬ß3b, `plan.md`, overnight `requires_review`). Display-only consumers (resume reporting, morning-review walkthrough, `dev`/`implement` reads) are NOT changed and continue to default on absent ‚ÄĒ they do not read the corruption signal.
- Does NOT change the `errors="replace"` decision into a fail-loud hard error ‚ÄĒ the vocabulary-validation gate (R1), not strict decoding, is what closes the silent-acceptance path, so `errors="replace"` remains the encoding contract.
- Does NOT fix unrelated dual-implementation divergences (e.g. the `overnight/backlog.py` frozenset, already tracked separately).
- Does NOT address the `@lru_cache` stat-key cache-coherency edge (a same-size, mtime-preserving in-place rewrite could serve a stale cached value) ‚ÄĒ this is pre-existing behavior, orthogonal to torn-line agreement, and out of scope; see Technical Constraints for how "agreement" is scoped around it.

## Edge Cases

- **Torn final / mid-file line with a valid earlier `lifecycle_start`**: tier recovered (last-valid); corruption signal false; CLI emits a stderr warning naming the line.
- **Torn-`lifecycle_start`-only (the dominant trigger)**: tier unrecoverable ‚Üí corruption signal true ‚Üí the gate-deciding consumers fail safe and FIRE review (the headline failure is closed for the dominant trigger); CLI also emits the stderr warning.
- **Non-UTF-8 byte breaking JSON structure**: `errors="replace"` ‚Üí no crash; `json.loads` fails ‚Üí line skipped + warned, skip indicator set.
- **Non-UTF-8 byte inside a string value**: `errors="replace"` ‚Üí line parses, but the mojibake value fails vocabulary validation ‚Üí it does NOT supersede a prior valid value and is treated as skipped (skip indicator set). It is never silently accumulated.
- **Empty / no-state-event log**: returns the empty accumulator; corruption signal false (no lines skipped); each reader defaults (`simple`/`medium`).
- **Missing file**: returns the empty accumulator / default, exit 0; corruption signal false.
- **Multiple torn lines**: all skipped, last-valid returned; CLI warns once per skipped line.

- MODIFIED: `common.py` `read_tier`/`read_criticality` on a non-UTF-8 byte ‚ÄĒ was uncaught `UnicodeDecodeError` (latent runner crash) ‚Üí now tolerant (`errors="replace"`).
- MODIFIED: three independent hand-rolled reducers (`state_cli`, `common.py`, `refine.py`) ‚Üí one shared `reduce_lifecycle_state` helper.
- MODIFIED: gate-deciding consumers (¬ß3a/¬ß3b, `plan.md`, overnight `requires_review`) ‚ÄĒ on corruption-detected-and-tier-unknown, now FIRE review (fail-safe) instead of defaulting to `simple` and skipping.
- ADDED: vocabulary-validation gate ‚ÄĒ an out-of-vocabulary (e.g. mojibake) value never supersedes a valid value.

## Technical Constraints

- The shared helper lives in `cortex_command/common.py` (its stated consolidation home; "all callers should import from here").
- Preserve `read_tier`/`read_criticality`'s `@lru_cache(maxsize=128)` and `.__wrapped__` attributes ‚ÄĒ pinned by spec-R1 introspection tests and `test_bin_lifecycle_state_parity.py`.
- `errors="replace"` applied uniformly via the shared helper; the **vocabulary-validation gate (R1)** ‚ÄĒ not strict decoding ‚ÄĒ is the mechanism that prevents silent acceptance of a mojibake value.
- **Two distinct mechanisms, not to be conflated**: the human-readable stderr **warning** (CLI-only, R9) and the machine-readable **corruption signal** (read by gate-deciding consumers, R7/R8).
- Field-precedence: read the override target from `.to` only (verified against `bin/.events-registry.md` and the writers); remove `state_cli`'s dead field-named fallback.
- **"Agreement" is defined over identical log CONTENT and over EFFECTIVE values after default projection** ‚ÄĒ `state_cli` emits the raw accumulator dict while `read_tier`/`read_criticality`/`refine` apply their absent‚Üídefault projection; the R12 test compares effective values, not raw return shapes. The `@lru_cache` stat-key cache-coherency edge is explicitly out of this agreement scope (the test uses fixed fixtures with no concurrent mutation).
- `state_cli` output never emits `null` post-fix ‚ÄĒ always the accumulator dict (possibly empty) plus, when relevant, the corruption signal field.
- Dual-source mirror coupling: editing `common.py` / `state_cli.py` / `skills/**` (canonical) regenerates the `plugins/cortex-core/` mirrors via the pre-commit drift hook; build the plugin and commit canonical + mirror together.
- The bash+jq oracle is gone (`bin/cortex-lifecycle-state` is a pure Python wrapper), so the torn-line fixture is re-founded on the `common.py`-agreement contract, not jq parity.

## Proposed ADR

### Proposed ADR: 0010-lifecycle-state-reads-tolerate-torn-lines-and-fail-safe

**Context.** `events.log` is an append-only JSONL log whose tier/criticality state is read by three independent reducers and feeds the spec-phase adversarial gates and the overnight post-merge review gate. The readers diverged: `state_cli` collapsed to `null` (silently degrading the gate to `simple` and skipping it) on any torn line, while `common.py`/`refine.py` tolerantly skipped but raised on non-UTF-8. In-repo precedent points both ways ‚ÄĒ `project.md`'s "Graceful partial failure" / "Historical compatibility shim pattern" favor read-side tolerance, while `pipeline.md:42` chose fail-loud for unparseable ordering metadata. The dominant real trigger (an LLM emitting a malformed `lifecycle_start`) makes the tier *unknowable*, not merely degraded.

**Decision.** Lifecycle-state reads tolerate torn/malformed/out-of-vocabulary lines (skip-and-continue to the last valid value) through one shared, vocabulary-validating reducer used by all three sites. When corruption makes the tier unknowable, the gate-deciding consumers **fail safe toward review** ‚ÄĒ a feature whose state cannot be classified is reviewed, not skipped. A CLI-only stderr warning surfaces skipped lines to humans; a separate machine-readable corruption signal drives the fail-safe at the gate-deciding consumers only.

**Trade-off.** Pure tolerance (skip-and-default) was rejected because, for the dominant trigger, it leaves the gate silently skipping the exact complex/critical feature that most needs review ‚ÄĒ the original bug, relabelled. Fail-loud was rejected because the overnight runner has no operator at the gate and cannot repair a historical torn line. Fail-safe-toward-review threads both: the gate fires on un-classifiable corruption (closing the headline bug for the dominant trigger) while recoverable logs read normally. The cost is a scoped corruption signal that the gate-deciding consumers read ‚ÄĒ a deliberate, narrow third state (not a generic one): its false-positive surface is bounded to logs where a line genuinely failed to parse and no tier survived. Reversing this contract back to silent-default would re-break the gate, which is why it is recorded.
