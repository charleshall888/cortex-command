# Plan: cortex-lifecycle-state-collapses-to-null

## Overview
Extract one shared, tolerant, vocabulary-validating reducer (`reduce_lifecycle_state`) into `cortex_command/common.py` and rewire all three hand-rolled reducers (`state_cli.py`, `common.py`'s `read_tier`/`read_criticality`, `refine.py`'s `_reduce_current_state`) to call it, so the readers agree by construction; then add a machine-readable corruption signal ‚Äî true when corruption makes the tier **or criticality** unknowable ‚Äî that gate-deciding consumers (the overnight review-decision sites, spec/plan-phase gate prose) fail safe toward review on, a CLI-only stderr warning for skipped lines, and a multi-reader agreement-test matrix.

## Outline

### Phase 1: Unify reducers (tasks: 1, 2, 3, 4)
**Goal**: One shared tolerant reducer in `common.py` consumed by all three reader sites; torn-line/encoding/field-precedence/vocabulary reconciled; `state_cli`'s reduce-to-null removed.
**Checkpoint**: `just test` green; `cortex-lifecycle-state` on the torn-line fixture emits the last-valid accumulator (never `null`); `grep -c "return None" cortex_command/lifecycle/state_cli.py` = 0.

### Phase 2: Gate fail-safe + observability + test re-founding (tasks: 5, 6, 7, 8, 9, 10)
**Goal**: Corruption that makes the tier or criticality unknowable fires the review gates instead of silently skipping them; skipped lines are warned on the CLI; the agreement contract is pinned by a multi-reader test matrix.
**Checkpoint**: `pytest -k "corrupt or non_utf8 or requires_review_corrupt" ` green; torn-`lifecycle_start`-only log ‚Üí overnight review decision True and `{"corrupted":true}` on CLI stdout; agreement matrix test passes across all seven axes.
