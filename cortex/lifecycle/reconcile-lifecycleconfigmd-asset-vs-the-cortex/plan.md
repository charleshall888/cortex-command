## Overview
Reconcile the skills asset `lifecycle.config.md` **up** to the init template by adding the missing 11-line `backlog:` block (regenerating the plugin mirror in the same commit), add a CI parity test that byte-compares the two files' frontmatter regions plus asserts the load-bearing option lines are present, record the decision as ADR-0017, then correct the three schema-enumerating docs to stop misciting/omitting the schema and name the asset as the single scaffolded-schema referent. ### Task 3: Add the frontmatter byte-slice parity test with positive-content and sentinel self-tests
- **Files**: `tests/test_lifecycle_config_parity.py`
- **What**: Add a pytest that byte-compares the asset's and init template's frontmatter regions, asserts the asset region carries the load-bearing option lines, and includes two in-memory sentinel self-tests proving the gate fails on (a) single-file divergence and (b) convergent option-line loss. (R3, R4; cites ADR-0017 for R5)
- **Depends on**: [1, 2]
- **Complexity**: simple
- **Context**: Model on `tests/test_dual_source_reference_parity.py` (`REPO_ROOT = Path(__file__).resolve().parent.parent`, `Path.read_bytes()`, a pure `assert_*` helper, named-diagnostic asserts, an in-memory sentinel that never writes disk — note that file supplies only a byte-parity helper, so the option-presence helper below is new and must be authored, not copied). Two file constants: `ASSET = REPO_ROOT / "skills/lifecycle/assets/lifecycle.config.md"` and `TEMPLATE = REPO_ROOT / "cortex_command/init/templates/cortex/lifecycle.config.md"`. A `_frontmatter_region(raw: bytes) -> bytes` helper slices the bytes **between** the opening `---` delimiter line and the closing `---` delimiter line of the raw bytes (locate both `---` markers in the raw bytes directly). **Do NOT** route the comparison through `cortex_command/lifecycle_config.py` `_extract_frontmatter_text` — its `splitlines()`+`"\n".join()` normalizes line endings and would mask CRLF/trailing-newline drift (spec Technical Constraints; R3). **One shared pure helper `_assert_options_present(region: bytes)`** (raises `AssertionError` naming the missing line) is the single source of the positive-content check and **must be called by both test 2 and test 4** — do not inline a second membership check in test 4, or its sentinel proves nothing about the production gate. `_assert_options_present` checks **line-integrity, not bare-token presence**: assert each of these specific line-substrings appears in the region — `backend: cortex-backlog`, `# backend: github-issues`, `# backend: jira`, `# backend: none`, `# instructions:`, plus the documenting-comment lines `# Freeform prose hint` and `harden in #318`. Using `# backend: none` (not bare `none`) closes the generic-substring hole; including the two comment-prose lines (esp. the `#318` cross-ref) closes the convergent-comment-loss hole a 5-token check leaves open. Required test functions (4, all `def test_`): (1) `test_frontmatter_byte_parity` — asserts `_frontmatter_region(ASSET.read_bytes()) == _frontmatter_region(TEMPLATE.read_bytes())`; (2) `test_asset_frontmatter_carries_options` — calls `_assert_options_present(_frontmatter_region(ASSET.read_bytes()))` and asserts the region is non-empty; (3) `test_divergence_sentinel` — mutate one region's bytes in memory and assert the byte-compare reports mismatch; (4) `test_convergent_loss_sentinel` — delete one required option line (e.g. `# backend: github-issues`) from **both** in-memory regions, then assert BOTH (i) `_assert_options_present` raises on the mutated region AND (ii) `_frontmatter_region` byte-parity still **passes** on the two mutated regions — the second assertion is what proves this is the residual case a pure two-file diff misses. Header comment cites `ADR-0017`. Auto-collected via `pyproject.toml` `testpaths`; runs under `just test`.
- **Verification**: `.venv/bin/pytest tests/test_lifecycle_config_parity.py -q` exits 0 AND `grep -c 'def test_' tests/test_lifecycle_config_parity.py` ≥ 4 AND `grep -c '_assert_options_present' tests/test_lifecycle_config_parity.py` ≥ 3 (one def + a call in each of test 2 and test 4 — guards against test 4 re-inlining) AND `grep -rl '0017' tests/test_lifecycle_config_parity.py` is non-empty — pass if all hold.

## Risks
- **Trunk dual-source coupling**: Task 1 edits a canonical `skills/` source whose plugin mirror must be regenerated and committed in the same commit (pre-commit Phase 4 blocks drift). On a `main`/trunk checkout with concurrent sessions, commit with an explicit pathspec so another session's staged files don't leak in, and stage the regenerated mirror alongside the asset. If dispatched into a worktree, do not run `pip/uv install -e` there (it would repoint the parent `.venv` editable pth); the parity test only reads files, so no reinstall is needed.
- **Frontmatter slice correctness**: the gate must slice the raw frontmatter bytes directly and must NOT reuse `_extract_frontmatter_text` (line-ending normalization would mask CRLF/trailing-newline drift). This is the single most load-bearing implementation detail; the divergence + convergent-loss sentinels exist to prove the gate actually fails.
- **R9 ↔ R6 tension**: R6 requires setup.md to state a corrected surface while R9 prefers pointing at the asset over re-enumerating. The boundary is now partly mechanical, partly judged: Task 4 forces the asset-path pointer into setup.md (mechanical, binds R9's "name the asset"), but whether a corrected enumeration is "concise" versus a "divergent full re-listing" is not grep-decidable and is left to Task 7's operator confirmation (b). This is a genuine residual judgment surface — flagged here, not overclaimed as fully resolved by the decomposition.
- **Verification binds to location, not file-wide token presence**: the docs-task checks (Tasks 4–6) are region-scoped (awk fence/section slices) or use tokens absent from each file today (`backlog.backend`, the asset path, in-section `synthesizer_overnight_enabled`/`branch-mode`). A bare whole-file `grep -c 'backlog'` is inert — those tokens already appear 5/14/4 times incidentally — so a passing check must reflect the actual edit, not a pre-existing mention.
