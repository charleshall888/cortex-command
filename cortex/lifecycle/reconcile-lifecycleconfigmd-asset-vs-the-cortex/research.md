# Research: Reconcile lifecycle.config.md asset vs the cortex-init template (drift, no sync gate)

**Clarified intent (scope anchor):** Eliminate the silent drift between the two `lifecycle.config.md` sources so the config users are pointed at always carries the #317 `backlog:` backend block, and add a drift gate so the sources can't silently diverge again. The reconcile-vs-collapse mechanism was the headline open question; research substantially resolves it (see Tradeoffs + Adversarial → **reconcile-up + frontmatter byte-slice parity test**).

**The three files (confirmed by byte diff):**
- **A** = `skills/lifecycle/assets/lifecycle.config.md` (29 lines) — the skills asset. **Missing** the 11-line `backlog:` block. Docs point users here.
- **A-mirror** = `plugins/cortex-core/skills/lifecycle/assets/lifecycle.config.md` — byte-identical auto-generated mirror of A (regenerated by `just build-plugin`; never hand-edited).
- **B** = `cortex_command/init/templates/cortex/lifecycle.config.md` (40 lines) — the init scaffold template `cortex init` drops verbatim. **Has** the `backlog:` block.

A and B differ in **exactly two spots**: (1) B's frontmatter has the 11-line `backlog:` block (lines 18–28) A lacks — the bug; (2) A's body intro (line 22) appends "Copy this file to your project root as `cortex/lifecycle.config.md` and customize." which B omits — an intended, asset-only difference (A is a copy-template, B is auto-placed). Note: A↔B frontmatter is byte-identical for lines 1–17; the *only* frontmatter delta is the backlog block, and the intro sentence is in the **body, outside the frontmatter**.

## Codebase Analysis

- **Files that change** (reconcile-up branch): `skills/lifecycle/assets/lifecycle.config.md` (add the `backlog:` block) → `plugins/cortex-core/skills/lifecycle/assets/lifecycle.config.md` regenerates via `just build-plugin` (stage in the same commit — pre-commit drift hook blocks otherwise); a new `tests/test_*_parity.py`; docs pointer updates.
- **A↔A-mirror pipeline (fully tooled):** `justfile:575` `BUILD_OUTPUT_PLUGINS := "cortex-core cortex-overnight cortex-backlog"`; `build-plugin` (justfile:589) rsyncs the **entire** `skills/lifecycle/` tree (`rsync -a --delete`, justfile:617) so `assets/lifecycle.config.md` is mirrored automatically (no per-file enumeration). `.githooks/pre-commit` Phase 2 flags a build when a staged path matches `skills/*`; Phase 3 runs `just build-plugin`; Phase 4 `git diff --quiet plugins/$p/` fails the commit on mirror drift. Enabled via `just setup-githooks`.
- **A↔B relationship: fully independent, no tooling** (`grep -c templates justfile` = 0; no `bin/` or pre-commit trigger references `init/templates`). **This independence is the root cause** — they can drift silently. A new gate must explicitly read both worlds.
- **Runtime usage:** the skills asset A (and A-mirror) is **never parsed at runtime** — cited only in skill *prompts* (`skills/lifecycle/SKILL.md:30`, `references/specify.md`, `plan.md`, `complete.md`, `skills/critical-review/SKILL.md`, `skills/morning-review/`). The file users consume is the project-root `cortex/lifecycle.config.md` dropped from **B**, read by `cortex_command/lifecycle_config.py`. So **B is the de-facto live source; A is documentation**.
- **New test:** `tests/test_lifecycle_config_parity.py`, modeled on `tests/test_plugin_mirror_parity.py` (`REPO_ROOT = Path(__file__).resolve().parent.parent`, named-diagnostic asserts). Must tolerate the intended intro-sentence body difference while catching the backend-block divergence.
- **Contradiction resolved:** the Codebase pass initially read A↔A-mirror as guarded *only* by the pre-commit hook (because `test_plugin_mirror_parity.py`'s hardcoded list omits the asset). The Drift-Gate pass (which read both tests) found `test_dual_source_reference_parity.py`'s `assets/*.md` glob **does** cover A↔A-mirror in CI. Resolution: **A↔A-mirror is CI-gated** (glob test) + pre-commit; the only true gap is **A↔B**.

## Web Research

Three architectures, in rough order of robustness:
1. **Single canonical source → generate the other** (with a "DO NOT EDIT — generated from X" header) + CI regenerate-and-diff (e.g. Kubernetes `hack/verify-codegen.sh`, Go golden-file `-update` discipline, GraphQL-Codegen `--check`, `git diff --exit-code`). Removes drift by construction.
2. **Shared base + thin override** (transclusion / include / partial templating) — the one intended difference is the only thing separate; the rest physically can't diverge.
3. **Two hand-maintained files + a semantic-parity test** — most flexible, most brittle; the test must encode the intended difference.
- **Anti-patterns:** the tolerated-difference allowlist accretes into Sandi Metz's "Wrong Abstraction"; Google's "Change-Detector Tests Considered Harmful" — a test mirroring the implementation adds negative value. False-negative risk: as the allowlist grows, coverage shrinks silently. Mitigations: prefer generated/derived (nothing to allowlist); allowlist the *narrowest* unit; pair the failing diff with the exact remediation command; CI runs *without* the `-update` flag.
- **Net web steer:** for "one intended difference," prefer generated/transcluded (choices 1–2). Fall back to a semantic-parity test only if the files genuinely can't share a source — and keep the tolerance to the single narrowest unit. (Adversarial review below shows the cortex-specific facts push toward a *fourth* option — reconcile-up + frontmatter byte-slice parity — that sidesteps both generation and an allowlist.)

## Requirements & Constraints

- **`cortex/requirements/backlog.md:30`** (FR "Configurable backend in the cortex init config") governs **the init-scaffolded artifact specifically**: "`cortex/lifecycle.config.md` (the doc `cortex init` scaffolds) gains a `backlog:` block." This requirement is **already satisfied by B**; the stale **A** is the gap — so #335 is hygiene/correctness + a durable gate, **not** a requirement-gap closure.
- **backlog.md:21, :104** ("established dual-source pattern, drift-checked at pre-commit") scope to the **cortex-backlog plugin's skill-surface mirror** — the precedent #335 points at, but its own scope is the plugin mirror, not the A↔B pair. The existing dual-source gate does **not** cover the A↔B divergence.
- **backlog.md:120** (edge case): an absent `backlog:` block resolves to `cortex-backlog` with zero behavior change — so a gate enforces **schema/doc correctness, not mandatory presence** in every copy.
- **`project.md` Solution horizon + Complexity:** "Complexity must earn its place… when in doubt, simpler wins." The durable version is justified only if the simpler fix is *known* to need redoing. The schema is **demonstrably still moving** (#318 hardens external backends; advisory keys slated for activation) — that is the named "applies in multiple known places" condition that argues for *some* gate over a bare reconcile.
- **CLAUDE.md "structural over prose-only enforcement"** + the **two-mode gate pattern** (`--staged`/`--audit`, project.md:98) are the documented shapes for a new gate. A **structural test gate sidesteps the MUST-escalation policy** entirely (it adds no imperative prose) — favoring a test over prose enforcement.
- **Out of scope per CLAUDE.md:** editing `skills/` is lifecycle-gated; commit via `/cortex-core:commit`.
- Note: `cortex/requirements/glossary.md` (listed under project.md `## Global Context`) is **absent on disk** — recorded as `glossary.md (skipped: file absent)` per the load-requirements protocol.

## Tradeoffs & Alternatives

- **Alt A — Reconcile + semantic gate:** add the block to A; keep A and B hand-maintained; parity test on the parsed `backlog` subtree, tolerating the intro sentence. *Pro:* both files stay directly editable; fits the parity-test culture. *Con:* dual-maintenance burden survives (the gate only *detects*); **parsed-dict comparison is blind to comment loss** (see Adversarial #2) — fatal for this ticket.
- **Alt B — Collapse, generate A from B (`sync-install-guard` shape):** B canonical; generate A via a deterministic transform (inject the "Copy this file…" sentence) + `--check` pre-commit. *Pro:* makes this drift class structurally impossible; reuses a proven recipe. *Con:* cross-channel dependency inversion (CLI package drives plugin source); the **"GENERATED — DO NOT EDIT" header is self-contradictory on a copy-target file** (see Adversarial #4–5).
- **Alt C — Collapse, A becomes a thin pointer to B / "run `cortex init`":** *Pro:* removes the duplication entirely; no gate needed. *Con:* **strands plugin-only users** who installed cortex-core via `/plugin install` and never installed the CLI — they have neither B nor `cortex init` on disk, and A-as-schema-reference was their only field list.
- **Alt D — Reconcile only, no gate:** cheapest. *Con:* does nothing to prevent recurrence; the ticket explicitly asks for a gate and the schema is still moving — weighted against.
- **Existing precedent found:** `just sync-install-guard` already does generate-one-from-the-other (extract body → prepend GENERATED header → `--check` byte-identity, pre-commit wired). Relevant to Alt B but rejected below.
- **Recommended (post-adversarial): reconcile-up (add the block to A) + a frontmatter *byte-slice* parity test** — a fourth option that combines Alt A's "keep both editable" with byte-level (not parsed-dict) comparison of the frontmatter region. See Adversarial #7 and Drift-Gate findings.

## Drift-Gate & Parity-Harness Findings

- **No existing test compares A↔B** — genuine gap. A↔A-mirror **is** gated by `tests/test_dual_source_reference_parity.py` (canonical-vs-plugin-mirror, **strict byte-equality** via `assert_byte_parity`, `assets/*.md` glob, includes a sentinel-mutation self-test) and the pre-commit build-plugin hook. B is not a plugin mirror, so that gate can't cover A↔B.
- **`tests/test_parity_contract.py` is the design precedent** for a tolerant gate: a **closed, named** `TOLERANCE_CATEGORIES` frozenset, unknown name raises `ValueError` (fails loud), byte-identical by default, every escape named and per-stream. JSON/CLI-oriented, but the *discipline* is the model.
- **Production primitive for these files:** `cortex_command/lifecycle_config.py._extract_frontmatter_text(text)` (line-scans between the two `---` delimiters) + `yaml.safe_load` — the same parser the runtime uses.
- **Recommended technique (refined by Adversarial #7): frontmatter byte-slice parity.** Extract A's and B's frontmatter regions (between `---` delimiters) and compare **byte-for-byte**. This tolerates the body "Copy this file" sentence **for free** (it is outside the frontmatter) AND catches comment drift inside the `backlog:` block (which `yaml.safe_load` discards). After adding the block to A, A-frontmatter == B-frontmatter byte-identically — green and meaningful. Add a **sentinel-mutation test** (mutate one file's frontmatter in memory, assert the comparison raises) per `test_dual_source_reference_parity.py`.
- **Location/run:** `tests/test_*_parity.py` (28 such files); auto-collected via `pyproject.toml` `testpaths`; run by `just test` (`.venv/bin/pytest tests/ -q`, justfile:524). **Optional commit-time gate:** add a `.githooks/pre-commit` Phase + `just check-*` recipe triggered when either config path is staged — but pytest-alone is lower-friction and consistent with the existing CI-layer-behind-pre-commit precedent.

## Init-Scaffold & Hash-Coupling Findings

- **B is dropped byte-for-byte verbatim** — `scaffold()` does `content = template.read_text(...)` then `atomic_write(dest, content)` (`scaffold.py:312-313`), **no substitution**. The ticket's Edges suspicion that B "may carry init-time placeholders" is **false for this file** → a byte-mirror of B is fully possible.
- **`_HASH_INPUT_TEMPLATES`** (`scaffold.py:67-73`) lists `"cortex/lifecycle.config.md"`; `_compute_init_artifacts_hash()` normalizes (BOM-strip, CRLF→LF, single trailing newline) then sha256s, stored in each repo's `.cortex-init` marker. **No committed golden-digest test exists** (grep-confirmed); `tests/test_init_artifacts_hash_inputs.py` only checks determinism, `v1:` shape, and tuple-vs-disk *coverage*.
- **Per-branch impact:** reconcile-up edits **A only** → init hash untouched, no `.cortex-init` churn. Editing **B** changes the digest → see Adversarial #1 (one-time drift report into initialized repos). The `v1:` prefix is bumped only for input-set/algorithm changes, not content edits. **No release/auto-update coupling** (`bin/cortex-auto-bump-version` does not read the init hash).
- **Latent test gap:** collapsing *down* (stripping B's block to match A) would silently regress #317 — `test_init_backlog_scaffold.py` only asserts `resolve_backlog_backend == "cortex-backlog"`, which is the **default even when the block is absent** (`lifecycle_config.py:112` `_BACKLOG_BACKEND_DEFAULT`). A collapse-down would pass tests while degrading the scaffold. → **reconcile UP, never down.**

  - `docs/overnight-operations.md:706,708` — calls the asset "the template — source of truth for the field list" (strongest miscitation). Its own `:717` rule says "do not enumerate fields in more than one doc" — already violated by setup.md.

## Adversarial Review

1. **Editing B is NOT consequence-free** (corrects the "cheap" framing). B is in `_HASH_INPUT_TEMPLATES`; `cortex init --ensure` (wired in-session via `CORTEX_AUTO_ENSURE` / `lifecycle/init_ensure.py`) compares the marker hash to the installed hash and emits a one-time drift report into **every initialized repo's lifecycle session** on mismatch. Recoverable, not blocking — but a recurring cross-repo side effect. Reconcile-up edits A, not B, so it doesn't trip this; **B-gen (B canonical) pays it on every future field add.**
2. **A-sem (parsed-dict gate) is nearly vacuous.** In B's block the only parseable key is `backend: cortex-backlog`; `github-issues`/`jira`/`none`/`instructions:` are all **comments**, which `yaml.safe_load` discards. So `safe_load(A)["backlog"] == safe_load(B)["backlog"]` passes the instant A has `backend: cortex-backlog` — **even if A drops every documenting comment**, which is exactly the "missing backend section" the ticket names. → reject parsed-dict; use byte-slice.
3. **More than two content surfaces; the live ones a two-file gate won't touch:** the repo's own project-root `cortex/lifecycle.config.md` is heavily drifted from B (carries a consumed `branch-mode: prompt` + an embedded Branch Mode doc body) — proving "init drops B verbatim" holds only *at init time*, never after. Docs inline their own stale YAML (overnight.md, setup.md). A↔B parity does nothing for these — **the docs problem is separable from the A↔B gate.** Also: the archived `plugins/cortex-interactive/...` mirror is **dead** (renamed cortex-core) — any hardcoded mirror-path list in a new gate will rot; prefer glob/derived paths.
4. **B-gen creates a cross-channel dependency inversion with no precedent** — A (plugin-source channel) driven by B (CLI channel), chaining B→A→mirror with order dependence; couples two independent release channels. A durability cost, not a win.
5. **"GENERATED — DO NOT EDIT" is self-contradictory on A** — docs tell users to *copy and edit* A. The header reaches users on the copy path. C-pointer's escape (point docs at `cortex init`) strands plugin-only users who never installed the CLI.
6. **CLAIM 2 (low blast radius) and CLAIM 7 (schema moving) are reconcilable:** low blast radius caps how *expensive* the gate should be; schema-still-moving justifies *some* cheap gate. They collide only if CLAIM 7 is read as license for the heavyweight B-gen pipeline — over-engineered for a zero-runtime-blast-radius doc-completeness gap. **A gate, yes; a build pipeline, no.**
7. **Frontmatter byte-slice parity strictly dominates A-sem** at the same zero-allowlist cost: tolerates the body sentence for free (outside frontmatter) AND catches comment drift in the block. **Recommended gate.**
8. **Future non-default-safe field:** a gate must then also assert *presence*, not just A≡B (two files can agree and both be wrong). Out of scope for this default-safe field — do not pre-build.

**Adversarial recommended disposition:** reject B-gen and A-sem-as-specified; **adopt reconcile-up (add the block to A) + a frontmatter byte-slice parity test**; treat the docs fix as a **separate, scoped** effort that keeps A named as the plugin-only schema reference.
