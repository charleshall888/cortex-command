## Problem Statement
The skill-value audit (epic #347 → #357) scored 32 provisional-tail trim candidates across five `cortex/requirements/` area files (backlog.md ×11, pipeline.md ×6, observability.md ×6, remote-access.md ×5, multi-agent.md ×4) but never adversarially verified them. This batch verifies each candidate against its inline keep-guidance and its pin evidence, then applies the confirmed-non-load-bearing trims — reclaiming ~3.3k weighted tokens of selectively-loaded requirements prose whose retention has a real, measured attention cost, without dropping requirements context that shapes agent behavior. Because these files are selectively loaded (not test-anchored code), there is no test backstop, so verification cannot rest on a self-authored "looks safe" reason — every applied trim must cite an independent checkable signal (Req 2), and three cheap mechanical gates catch the silent-regression modes editorial review misses: H2-heading loss (G1), a normative line deleted inside a surviving section (G2), and stale cross-file line anchors (Req 7).

## Phases
- **Phase 1: Verify & capture** — Per file, confirm each candidate's span is non-load-bearing against its inline keep-guidance and pins; correct the two drifted claims; plan the four coordinated same-text clusters; record every candidate's `(file,id)` outcome (survives/refuted + signal + reason) to a structured outcomes file.
- **Phase 2: Apply, refresh & gate** — Apply surviving trims one file at a time (one commit per file), refresh live cross-file line-anchor citers, and enforce the H2-set and merge-sink mechanical gates before each commit.

## Edge Cases
- **A trim's file rarely loads**: realized cost is low, but the trim still applies if the span is non-load-bearing — editorial mode weighs load-frequency for prioritization, not as a veto.
- **A candidate is refuted on verification**: no diff is produced; its refuted verdict is recorded in verify-outcomes.md (Req 8). This suppresses re-proposal only once the verdict reaches `master_candidates.json` — via the OQ3 direct-write option or the Req 10 tracked reconciliation; recording it in the lifecycle artifact alone does not (demonstrated by #353). This is why Req 10 installs a mechanism, not a note.
- **Two candidates target the same text** (the four clusters, Req 5): applying independently would leave the later one editing text the earlier already removed, or re-duplicating consolidated content — hence single coordinated edits. The backlog s3/s5/s6/s7 cluster is the sharp case: s6's enum and s7's decisions do **not** live in the AC sink, so a naive "fold all four into AC" would drop them.
- **A normative line is deleted inside a preserved H2** (the dominant COMPRESS case): G1 (H2-set) passes because the heading is intact, so G2 (normative-line survival, Req 6) is the gate that must catch it — the removed line is shown preserved elsewhere or classified informative-only.
- **A citer points at a line that gets deleted/merged** (e.g. `research`/code citing `backlog.md:100` when the AC sink is rewritten): there is no post-trim line; Req 7 repoints it to the surviving canonical location or converts it to a durable heading reference rather than mapping to shifted content.
- **A live citer uses a non-colon form** (L-prefix ``pipeline.md`` L143`` or range `:87-95`): Req 7's enumeration matches all three forms; a single `\.md:[0-9]+` grep would miss the L-prefix citer and the range tail, so completeness is asserted by a post-batch re-run showing no stale citer.
- **MERGE_DEDUP moves content across H2 sections** (e.g. backlog s3/s5 FR → AC sink): permitted for content, but G1 must still pass and G2 must confirm the merged statement literally survives at its named canonical home — which is *not* always the AC sink (s6's enum stays at FR L52, s7's decisions at FR L60 / OQ L129).

## Technical Constraints
- **Load-requirements selection path**: all 5 files are in project.md's `## Conditional Loading` selective bucket, gated by case-insensitive substring match of trigger phrase against the feature's `index.md` `tags:`. None are always-loaded.
- **No test backstop**: `tests/test_requirements_skill_e2e.py` validates the requirements *writer's* output in `tmp_path`, never the committed `cortex/requirements/*.md` — so Req 6's H2 gate and Req 7's anchor grep are the only regression guards.
- **Ledger keying**: `id` is non-unique across files (12 distinct ids over 32 candidates); `(file,id)` is unique — all outcome/reconciliation keying uses `(file,id)`.
- **Named must-keep spans** (do not sever): backlog.md ADR-0016/ADR-0019 back-pointers + the "`cortex-backlog` (not `local`)" naming rule; pipeline.md fixed-architectural-constraint statements, state transitions, and the "Post-Merge Review" heading (cited as "the contract" by ADR-0015 + docs/internals/pipeline.md); observability.md "Install-mutation invocations" maintainer contract; multi-agent.md the cited `anthropics/claude-code#39886`.
- **Editorial precedent**: #351 (complete) established the "audit found what is duplicated, not what the user wants to keep saying in their own voice — user reviews at spec/PR" posture; #353 established one-commit-per-file grain.
- **Verification hygiene**: recall-first then precision — do not cut in the same pass meant to preserve; review the whole file, not the span in isolation (cross-section instruction-bleed risk).
