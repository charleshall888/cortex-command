# Research: cli-served-lifecycle-state-machine

Replace model-as-router skill navigation with a CLI-served lifecycle state machine: `cortex-lifecycle-next --feature X` emits {state, instruction fragment, pause spec, advance contract}; `cortex-lifecycle-advance` records completion and emits events. Sized complex/critical; 9 core research agents + 1 adversarial agent + a 4-angle critical review (7 ratified fix-invalidating objections applied below).

## Research Questions

1. **Fragment distribution — plugin markdown + caller-passed `--skill-dir` vs wheel-embedded, with ADR-0002 skew implications?** → **Answered, with a mandatory prerequisite**: *neither flavor is safe unaided*. cortex-core has zero version-skew protection today — `CLI_PIN`, the MCP-gated reinstall, and the SessionStart reinstall hooks all live only in cortex-overnight, by twice-reaffirmed scope decision [cortex/lifecycle/trigger-cortex-cli-reinstall-at-sessionstart/spec.md:60; cortex/lifecycle/extend-cortex-cli-version-sync-hook/spec.md:21,113], and cortex-core-only installs have no automatic CLI install path at all [docs/setup.md:28]. No `--skill-dir`-style argv precedent exists anywhere; the SP001/SP002 lint covers Read/prompt paths only, not argv or Python string literals [cortex_command/lint/skill_path.py:32-48]. A **protocol handshake ("step 0", design in RQ5)** is a prerequisite for either flavor — and it *guards* the skew surface (detect-and-halt with remediation), it does not *close* it: the payload flavor has no auto-remediation and cortex-core-only installs have no automatic reinstall path. The selector-vs-server choice itself remains open — see Open Questions — because it is really the structural-guarantee question, not a token-cost question.
2. **What does `cortex-lifecycle-next` emit, and do one-line next-state previews suffice?** → **Answered**: the envelope generalizes proven shapes — `resolve.py`'s `{state, next}` directive [cortex_command/lifecycle/resolve.py:59-74,192-209], `complete_route.py`'s 12-route `{route, terminal, continue_to}` [cortex_command/lifecycle/complete_route.py:34-48], and `branch_decision.py`'s `{skip|resolved|prompt}` pause-vs-advance split [cortex_command/lifecycle/branch_decision.py:99-132]. No fragment in the corpus needs more than one-hop lookahead — every cross-phase dependency is either a state-read at current-phase entry or the SKILL.md-mandated "Next" transition line [skills/lifecycle/SKILL.md:74; skills/lifecycle/references/implement.md:111,117]. **Caveat (adversarial)**: that evidence comes from a world where the model holds the whole map; the machine should carry an optional path-overview field for resume reporting and "what's left?" questions [skills/lifecycle/SKILL.md:48].
3. **Overnight `--mode` pause-resolution policy?** → **Answered, re-scoped**: overnight's pause substitutes are not pause *responses* but *different transitions* — refined-status precondition instead of spec approval [cortex_command/overnight/backlog.py:42], deferral files instead of questions [cortex_command/overnight/deferral.py:1-4], brain triage instead of failure prompts, auto-merge instead of merge-wait. The headless orchestrator can never reach a pausing state, so every pause spec needs an overnight policy arm. The honest claim is **"shared state+event core, two policy layers"**, not "orchestrator-round.md retires." A written overnight transition table — quantifying what fraction of orchestrator-round.md is machine-ownable sequencing (round filter, dependency gate, criticality lookup, envelope-extraction mechanics, event emission) vs judgment prose that stays (plan format contract, synthesis criteria — must-stay-prose either way) — is a prerequisite for funding the unification step, and its go/no-go is a separate decision from steps 0–2 (see RQ5).
4. **Golden-replay verification harness design?** → **Answered, redesigned after critical review**. The originally-recommended arms — (a) terminal-state replay over the 325-log / 10,534-line historical corpus, (c) forward decision-capture — cover the **pure-reducer slice, which the extraction keeps unchanged and which already has an 8-case golden-replay parity suite** [tests/fixtures/cortex-lifecycle-state/README.md]. They are worth having (S effort each; they catch split-brain-reader divergence, the class of the fixed `cortex-lifecycle-state` null-collapse [cortex/lifecycle/cortex-lifecycle-state-collapses-to-null/research.md:11-35]) but they are **not the extraction's safety net**: historical logs were written by the old system, so read-side replay never executes the new write path, and decision-capture's goldens are emitted by the code under suspicion — a wrong day-one decision is recorded as correct. Day-one confidence must come from arms that exercise the new surfaces directly: **(d) write-side verb tests** that run `advance` in a sandbox and assert emitted event order against the gate check (the wave-2 `test_finalize.py` retarget pattern — this, not historical order-assertions, is the F1 detector); **(e) a CI-diffed golden of the `describe` verb's rendered transition table plus per-state envelope tests** pinning the state→fragment and state→pause-spec mappings the extraction creates (no historical event records fragment reads; `feature_paused` carries zero payload fields [cortex_command/lifecycle_event.py:231-279]); **(f) environment-fixture guard tests** staging lock files and probe states against `next`'s envelope (guards evaluate live state that never appears in events.log); **(g) a status-projection differential sweep** running the retained scan_lifecycle mismatch detector pre/post cutover and requiring the new projector to reproduce or shrink the existing mismatch set (historical frontmatter is a known-inconsistent oracle). The complete-route slice's gh/git inputs were never recorded — historical replay is structurally impossible there [cortex_command/lifecycle/complete_route.py:79-243]; decision-capture covers it from version N onward only. Per-event diff (b) stays M–L, scoped to pure-reducer transitions.
5. **Migration sequence with standalone value per step?** → **Answered, amended twice** (adversarial pass, then critical review): step 0 (protocol handshake) → 1 (selector verb) → 2 (events-to-advance, with the pause-kind discriminant in the schema from day one) → 3 (pauses-as-data runtime) → 4 (overnight core-sharing) → 5 (entry-point configs). Step-2-as-foundation matches the #330→#331/#332 precedent [cortex/backlog/336-offload-deterministic-lifecycle-mechanics-to-cli-verbs.md:17-25]. Critical-review amendments to step 0's design, all binding: **(i) carrier** — the protocol version is stamped into the *existing* verbs' payloads (`resolve`, `enter`, `complete-route`) at step 0, before the new verbs exist, and the SKILL.md loop treats a missing field *and* verb-invocation failure (command-not-found / wrapper exit 2) as skew with a remediation message — otherwise the fresh-plugin + stale-wheel pairing at the step-1 boundary fails in the shell before any assertion can run; **(ii) bidirectionality** — the loop passes its own expected protocol as an argv flag (`--protocol N`) that the wheel asserts; a legacy plugin passes no flag and the wheel treats absence as legacy-protocol, keeping legacy verbs serviceable — because the stale-plugin + fresh-wheel direction is the one the repo's own machinery manufactures (auto-release bumps the wheel per merge [.github/workflows/auto-release.yml]; the cortex-core plugin waits for a manual `/plugin` update); **(iii) tolerance semantics** — the protocol version is a small integer decoupled from release versions, bumped only on breaking contract changes, asserted as a compatibility range — exact-equality against per-merge release tags would halt every consumer on every wheel-only release; **(iv) coexistence policy** — absorbed verbs (`cortex-lifecycle-resolve`, per-event `cortex-lifecycle-event` subcommands) stay callable until a protocol-floor bump, following the events-registry 30-day-grandfather discipline, and `advance` carries idempotency/dedup keys so a skewed consumer running both paths cannot double-emit (schema-additive but count-corrupting, which decision-capture would then pin as golden). Each step must additionally be additive-only against events.log or it forces a major bump + coordinated plugin release.
6. **In-flight lifecycle compatibility?** → **Answered**: the migration-sentinel pattern is established (derive as-if-migrated state from historical shapes, validate against the real corpus [cortex/lifecycle/auto-progress-lifecycle-phases-when-no-blockers/spec.md:25]); readers-tolerate-all-historical-shapes is a codified constraint [cortex/requirements/project.md:37]. Only 2 in-flight lifecycles exist locally — but consumer repos have uninspectable corpora and receive partial step sequences via auto-release, so **"safe to land in an arbitrary consumer repo mid-lifecycle" is a per-step acceptance criterion**, discharged per step by the RQ5 handshake + coexistence machinery, not assumed. The tolerant-reader discipline extends to **consumer config files**, not just events.log: step 5's axis rename must alias old `lifecycle.config.md` field names with a deprecation warning — a silent reinterpretation under opposite floors yields a wrong default, not an error.
7. **Fold in #370?** → **Answered**: bug confirmed at [cortex_command/lifecycle/resolve.py:183] — the `new` branch checks `dir_exists` on the raw token and never consults the `backlog["lifecycle_slug"]` it already holds; the `mode=="resume"` branch has the same blindness [resolve.py:161-169]. "Resolver becomes machine core" = slug as the canonical identity key, with numeric ID/title/UUID as input normalization before state lookup. **But the fix + a verb-level regression test should land now, independent of this epic** — porting an untested fix is how dropped-literal bugs happened last wave.

## Codebase Analysis

- **The proposal is a generalization of a proven, mid-transition pattern, not greenfield.** 26 lifecycle verbs exist [pyproject.toml:43-94]; SKILL.md is already a resolve→enter→phase-table loop [skills/lifecycle/SKILL.md:26-70]; `complete.md` routes almost entirely off `complete_route`'s JSON [skills/lifecycle/references/complete.md:19]. Wave-2's own closeout deferred exactly this next lever: "an operator decision at the gate" beyond clause-parity trimming [cortex/lifecycle/lifecycle-corpus-trim-wave-2/route-table-after.md:103-111].
- **Fragment corpus**: 21 files, 65,938 bytes in skills/lifecycle/references/ plus ~39K in skills/refine/references/. Grading uses a **provenance criterion** (sharpened by critical review): a gate is *machine-verifiable* only when its evidence is CLI-produced or CLI-checkable without model authorship — prior advance-emitted events under flock, guard-verb exit codes, file-existence facts the machine itself established. Model-authored artifacts (plan.md checkboxes, review.md verdicts) and model-relayed human utterances (spec approval) are **attestations in substance even when a file exists to read** — CLI verification of them confirms shape, not truth. Machine-ownable *sequencing* (routing tables, gate matrices, guard composition): backlog-writeback.md, wontfix.md, critical-review-gate skip matrix, criticality-matrix behavior table, refine-delegation's event/gate sequencing. Recite-only routing (served, never verified — gh/git/tool-call outcomes): complete.md's PR-state routing, worktree-entry.md's EnterWorktree call (ADR-0008 requires the structural two-mode split; a runtime soft-gate was explicitly rejected [cortex/adr/0008:20,39]). Must-stay-prose: plan.md authoring rules, orchestrator checklists, reviewer/builder prompt templates, failure-handling judgment.
- **Pause inventory**: 10 test-enforced pauses [skills/lifecycle/references/kept-pauses.md:5-14] + 3 sites the parity regex misses (implement.md:76 retry/skip/abort; complete-first-run.md:11 test-command ask; concurrent-sessions.md:5 resume pick). Three kinds need a discriminant: AskUserQuestion-site, phase-exit wait, config-conditional (suppression predicate already exposed by `cortex-lifecycle-branch-decision`), **plus a fourth class critical review surfaced: relayed-consent gates (spec approval, merge consent) — the judgment is the human's, not the model's, and they are the highest-stakes pauses**. kept-pauses.md is tests-only with zero runtime consumers [kept-pauses.md:3] — the parity test *is* the current enforcement, so retiring the pin class requires a successor contract, not just a generated registry: (a) a generated parity check asserting every pause-requiring state's envelope carries its pause spec, and (b) a runtime tooth — `advance` for a pause-gated transition **refuses unless a recorded pause event exists** (the `approval_checkpoint_responded` closed-vocabulary pattern from discovery [cortex_command/discovery.py:419-430] is the in-tree precedent). Residual trust remains: a model could fabricate the pause event; the event is at least durable and auditable. The parity net must be fixed on the current corpus before it's lifted into pause specs.
- **Events substrate**: append-only per-feature events.log under flock+O_APPEND [cortex_command/lifecycle_event.py:75-119]; `_EVENT_SUBCOMMANDS` table owns field contracts [lifecycle_event.py:231-279]; ADR-0020 closes the hand-written exception class at 3. `detect_lifecycle_phase` is a **hybrid** artifact-presence + event machine [cortex_command/common.py:239-393] — pure event replay cannot reproduce it without point-in-time artifact snapshots.
- **Status projection**: three independent writers (refine SKILL.md prose → `refined` [skills/refine/SKILL.md:94]; finalize.py:159 → `complete`; wontfix_cli.py:139-146 → `wontfix`); `update_item` accepts arbitrary `--status` with no enum validation; scan_lifecycle.py:166-200's mismatch detector exists because they disagree in practice. **Adversarial correction**: the machine cannot "own" a projection into github-issues/jira — external write-back is deliberately best-effort (ADR-0016) and a human closing an issue in the GitHub UI is legitimate and invisible. Owned projection scopes to the cortex-backlog backend; the mismatch detector is retained as the reconciliation surface **and enrolled in the verification harness as the pre/post-cutover differential sweep (RQ4 arm g)**.
- **Guards**: interactive lock (8-row liveness table, interactive_lock.py:387-448), overnight-active probe (bash sidecar + deliberately-duplicated Python reimplementation with KEEP-IN-SYNC comments [cortex_command/lifecycle/prepare_worktree.py:16-29]), branch decision — all already verbs; prose only sequences them, and `prepare_worktree.py` already composes guard→lock→create. Guard evaluation reads live environment state that never appears in events.log — guard misfire is invisible to event replay, hence RQ4 arm (f).
- **Overnight duplication surface**: orchestrator-round.md re-implements plan.md §1a and competing-plans.md §1b semantics by explicit citation [cortex_command/overnight/prompts/orchestrator-round.md:241-303,412-413], pinned only by the hand-maintained heading-existence test tests/test_skill_section_citations.py. The scheduler (launchd/lock/spawn) vs per-feature sequencing (orchestrator.py/feature_executor.py/outcome_router.py) boundary is already docstring-enforced. Note the cited §1a/§1b duplication is judgment prose that stays prose either way — the machine-ownable content in orchestrator-round.md is its sequencing scaffolding (round filter, dependency gate, criticality lookup, envelope extraction, event emission), whose actual volume the RQ3 transition table must quantify before step 4 is funded.
- **Point fixes that don't need the machine** (solution-horizon discipline — credit them separately): plan.py:145-151 `_TERMINAL` missing the wont-do variants (one-line shared-import fix); stale common.py:175-176 comment (backlog.py already imports the shared set [cortex_command/overnight/backlog.py:28]); vestigial `SYNTHESIZER_VALIDATED_{INTERACTIVE,OVERNIGHT}` event names defined but never emitted [cortex_command/overnight/events.py:85-86].
- **Test lattice**: FILE_EVENTS pins exact per-file event-invocation counts across 8 files [tests/test_lifecycle_event_roundtrip.py:123-154]; ~5 markdown-anchored tests (kept-pauses, invocation-grammar, step-v-ordering, EnterWorktree-callsites, dual-source cases) retire-and-retarget to verb-level tests per the wave-2 precedent (test_complete_index_sync_gate.py → test_finalize.py). **W003 parity-orphan fires for every verb whose only wiring is deleted reference prose** [cortex_command/parity_check.py:70-84] — needs allowlist rows or scanner extension. ## Web & Documentation Research

- **Pause-as-data reference shapes**: AG-UI's Interrupt type (`id`, `reason`, `message`, `responseSchema`, `expiresAt`, resume-payload validation, idempotent resubmission) is the closest external analog to the pause spec (https://docs.ag-ui.com/concepts/interrupts); Mastra pairs a `suspendSchema` with a `resumeSchema` per step. LangGraph's `interrupt()`/`Command(resume=...)` warns: code before a pause re-executes on resume, so pre-pause side effects must be idempotent.
- **Idempotency is the recurring universal constraint**: LangGraph, Temporal, Google ADK ("tools run at least once and may run more than once"), and event-stream analyses all converge — a retried `advance` with real side effects needs dedup/idempotency keys; separate the append-only decision log from derived state. (Now load-bearing in RQ5's coexistence policy.)
- **Schema evolution**: Temporal's `GetVersion()` gates are the battle-tested precedent for evolving the machine definition while old logs stay replayable; event-sourcing best practice is additive-only fields, prefer new event types over mutating shapes — matching the repo's own tolerant-reader doctrine.
- **Replay verification**: deterministic-replay literature requires structured (parseable) decision emission, "fail loudly" on out-of-trace operations, and captured metadata; free-prose decisions are not replay-verifiable (https://www.sakurasky.com/blog/missing-primitives-for-trustworthy-ai-part-8/).
- **The forcing constraint is real and platform-level**: anthropics/claude-code#34592 (primary source) — AskUserQuestion is absent from all sub-agent tool contexts across versions, with the community workaround being exactly a file-relay-to-main-loop pattern.
- **Progressive disclosure caution**: SkillRouter (arXiv 2603.22455) — routing on name/description alone dropped Hit@1 by 31–44pp; the router needs body content. cortex's design sidesteps this (code, not an LLM, routes) **only if** the state table encodes enough precondition logic that the model never has to re-infer "what state am I really in" from prose.
- **No prior art found for the exact next/advance CLI-served pattern** in aider/goose/opencode/BMAD/Spec Kit — flagged as not-found rather than confirmed-absent. MCP's resource_link-vs-embedded-content guidance maps onto the selector/server tradeoff.

## Architecture

### Pieces

- **Transition table** — the single wheel-side source of truth for states, gates, guard preconditions, pause specs, and parameters (criticality × tier × backend × config overrides); the thing `describe` renders, parity tests generate from, and the harness's envelope tests pin.
- **Identity resolver** — slug-canonical resolution absorbing #370; numeric ID/title/UUID normalize to the lifecycle slug before any state lookup.
- **next verb** — read-only server: resolves identity, reduces events.log, evaluates guard preconditions, and emits {state, fragment ref-or-content, pause spec, advance contract, optional path overview, protocol version}.
- **advance verb** — the write-side transition executor. Its F1 defense is **structural composition**: side effects execute inside the verb, behind its own gate check, in fixed source order (the `wontfix_cli`/`enter` needs-decision pattern) — not advance-time verification of externally-performed work. It carries idempotency keys, verifies machine-verifiable gates (CLI-produced evidence only), records attestation-mode and relayed-consent transitions as durable events with rationale, refuses pause-gated transitions without a recorded pause event, emits additive-only events through the ADR-0020 contract, and owns the in-repo status projection (cortex-backlog backend only).
- **Pause-spec registry** — the four-kind discriminant (question / phase-exit / conditional / relayed-consent) with per-kind data (options, resume semantics, suppression predicate), replacing kept-pauses.md as generated output, with the successor enforcement contract (generated parity + advance's recorded-pause-event requirement).
- **Fragment corpus** — the judgment-dense What/Why prose that stays prose; distribution flavor (plugin-path selector vs wheel-served content) is the open structural decision.
- **Protocol handshake** — a small protocol integer, decoupled from release versions, stamped into existing-verb payloads at step 0 and passed bidirectionally (`--protocol N` argv from the loop; payload field from the wheel), compat-range asserted on both sides, with verb-invocation failure treated as skew. **Guards** cortex-core's unprotected skew surface (detect-and-halt with remediation); closing it (auto-remediation) requires reversing #235/#263 — an open decision.
- **Interactive policy layer** — the thin SKILL.md next/advance loop in the main conversation, the only place pauses render (realistic size 2.5–4K with a line-item budget, not ~1K).
- **Overnight policy layer** — the pause-resolution arms (defer/precondition/triage/auto-merge) consuming the same core; replaces orchestrator-round.md's sequencing scaffolding (volume to be quantified by the RQ3 transition table) but not its judgment content or policy role.
- **Verification harness** — day-one arms: write-side advance order tests, describe-verb transition-table golden + per-state envelope tests, environment-fixture guard tests, status-projection differential sweep. Supplementary arms: historical terminal-state replay (reducer slice), forward decision-capture (version-N+1 regressions), per-event diff (deferred).
- **describe verb** — generated markdown rendering of the transition table for authoring ergonomics *and* the harness's CI-diffed golden (generate_index.py pattern, not the hand-authored events-registry pattern).
- **Escape hatches** — phase override, criticality/complexity override, wontfix, cancel: kept as sibling verbs outside the transition table, as today.

### How they connect

The transition table is the machine; everything else is a view or an actor on it. `next` is the pure read path: identity resolver → events.log reduction (the existing `reduce_lifecycle_state`/`detect_lifecycle_phase` primitives remain the derivation layer) → guard evaluation → envelope emission, always exit 0, protocol version stamped. The interactive loop acts on the envelope in the main conversation — reading the fragment (selector) or receiving it inline (server), rendering any pause spec via AskUserQuestion, then calling `advance`, which executes the transition's side effects inside its own gate-checked body, requires recorded pause events for pause-gated transitions, emits the transition event, and updates the in-repo status projection. events.log remains the sole durable state (no state.json — #190's multi-writer rejection and ADR-0004's frontmatter-sub-state rejection both bind); the overnight layer consumes the identical next/advance core but resolves each pause spec through its policy arms instead of rendering it, and keeps the fleet scheduler untouched above. The harness closes the loop from both sides: write-side verb tests and envelope goldens pin the new surfaces at cutover; historical replay pins the reducer; decision-capture makes future transitions (including the gh/git-dependent complete-route slice) regression-detectable from version N onward. Boundaries: the machine owns sequencing and machine-verifiable gates; attestation and relayed-consent gates are recorded, auditable, and honestly labeled as trust-carrying; fragments own judgment; escape-hatch verbs mutate state outside the table; external backlog backends receive best-effort write-back, never ownership.

## Decision Records

**events.log stays the only durable state.** #190 rejected a state.json because the single-writer assumption is empirically false (morning-review is a third writer class) [cortex/lifecycle/promote-lifecycle-state-out-of-eventslog-full-reads/spec.md:26]; ADR-0004 rejected frontmatter sub-state and log rewrites. The machine is a pure transition function over file-derived state (ADR-0001-compatible), never a daemon or side-store.

**The machine owns sequencing and machine-verifiable gates, never thinking — with the trust boundary stated honestly.** Machine-verifiable means CLI-produced or CLI-checkable evidence (prior advance-emitted events, guard exit codes, machine-established file facts). Model-authored artifacts (plan.md checkboxes, review verdicts) and relayed human utterances (spec approval) are attestations in substance regardless of file shape — CLI-reading them confirms shape, not truth, and the design does not claim otherwise. Routing tables, gate matrices, and guard sequencing are machine-ownable; plan-authoring rules, checklists, prompt templates, and failure judgment stay judgment-dense prose.

**The F1 defense is structural composition plus write-side tests — not gate verification.** F1 is an ordering bug; verification at advance-time checks truth, not ordering, and cannot police side effects performed before advance was invoked. The defense: transitions with side effects execute those side effects *inside* the advance verb, behind its gate check, in fixed source order (generalizing `enter()`'s needs-decision short-circuit and `wontfix_cli`'s function-body-as-invariant), pinned by write-side verb tests that run advance and assert emitted order. Historical order-assertions cannot catch this class — old logs were written by the old system.

**Status projection scopes to the cortex-backlog backend.** The machine becomes the sole lifecycle-driven in-repo status writer (replacing three independent write sites); external backends stay ADR-0016 best-effort; scan_lifecycle's mismatch detector is retained as the reconciliation surface and doubles as the harness's pre/post-cutover differential oracle.

**Overnight is re-scoped from "retires" to "shared core, two policy layers" — and step 4 is a separate go/no-go.** The forcing constraint means the headless orchestrator can never pause; its four substitutes are different transitions, not pause responses. The cited orchestrator-round.md duplication (§1a/§1b) is judgment prose that stays prose either way; the machine-ownable sequencing scaffolding is unquantified until the RQ3 transition table exists. What certainly retires is the heading-citation pin class.
