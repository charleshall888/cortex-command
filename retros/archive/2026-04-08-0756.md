**Problem**: `test_sighup_triggers_cleanup` was investigated as a potential regression from lifecycle changes. Multiple hypothesis-and-test cycles were run (checking python3 call count, timing, settings.json diff, TMPDIR behavior) before finding the root cause: macOS `mktemp` ignores `TMPDIR` env var and always uses the launchd-assigned `/var/folders/...` path, which is sandbox-blocked when runner.sh runs as a subprocess of pytest under the Claude Code sandbox. **Consequence**: Extended diagnosis (~30 minutes of conversation, multiple test runs, diagnostic script writing) for what turned out to be a one-line fix.

**Problem**: `setdefault("TMPDIR", ...)` in the test fixture was written as if it would override TMPDIR, but `setdefault` is a no-op when the key already exists. The comment intent ("set TMPDIR to test's tmp dir") was not matched by the behavior. **Consequence**: The fixture never actually redirected mktemp's temp directory; the sandbox-blocking issue was masked until the sandbox environment became the persistent context.
