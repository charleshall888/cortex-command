# Investigate overnight/morning-review git sync gaps

## Context

After the overnight session `overnight-2026-04-07-0008`, the morning review merged the PR on GitHub (remote), but local main had 15 unpushed commits (bug fixes from retry attempts + the morning review commit itself). This caused a diverged state requiring a manual `git pull --rebase` with 3 merge conflicts — all in `lifecycle/*/plan.md` files where both the local artifact-recording commit and the overnight feature branch had generated the same plans.

## Investigation Areas

### 1. Should the overnight process push local commits before starting?

The overnight runner makes commits to main (artifact recording, bug fixes during retries) but doesn't push them. Research whether the runner should push to origin before creating the integration branch, or at session boundaries. Consider: what if the remote has diverged? What if the push fails?

### 2. Should morning review push after committing?

The morning review commits lifecycle advancement and ticket closures locally but has no push step. After Step 5 (commit) and before Step 6 (PR merge), pushing local main would reduce divergence before the PR merge introduces the overnight feature commits.

### 3. Post-PR-merge sync step

After the PR merges on the remote, local main diverges. Research adding a Step 6b to morning review that fetches, rebases, and pushes. The key design question: how to auto-resolve conflicts in lifecycle artifacts where the overnight version is always authoritative (plan.md files generated by both local artifact recording and the overnight feature branch).

### 4. Dirty working directory handling

Consider what happens when the user has uncommitted changes in the working directory when the overnight process ran or when morning review runs. Research how to detect stale local state, warn the user, and offer safe resolution paths.

### 5. Multiple retry scenario

When the overnight runner is retried multiple times (as happened here), each attempt may leave local commits. Research whether the runner should clean up or consolidate artifact commits from failed attempts before the successful run.
