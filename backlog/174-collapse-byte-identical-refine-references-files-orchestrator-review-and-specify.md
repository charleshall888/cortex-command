# Collapse byte-identical refine/references files (orchestrator-review.md + specify.md → lifecycle canonical)

Delete two near-byte-identical files in `skills/refine/references/` and have refine read the lifecycle copies. Largest single deduplication move in the audit, ~346 lines of pure duplicated text.

## Context from discovery

The audit's pressure-test pass verified by `diff` that:

- `skills/refine/references/orchestrator-review.md` (183 lines) differs from `skills/lifecycle/references/orchestrator-review.md` (184 lines) by **exactly one line** — lifecycle has the `P8 Architectural Pattern` row; refine omits it because refine never reaches the plan phase.
- `skills/refine/references/specify.md` (163 lines) is byte-identical to `skills/lifecycle/references/specify.md` (186 lines) except for the trailing 23 lines (the `### 5. Transition` block + `## Hard Gate` table that refine drops). Body identical.

Refine's `SKILL.md` Step 5 already enumerates the documented adaptations refine applies when reading lifecycle's specify.md. The collapse is mechanically safe.

Audit verified safe by parity tests:
- `tests/test_dual_source_reference_parity.py` discovers pairs via glob — when a canonical file disappears, the parametrized case disappears.
- `justfile build-plugin` uses `rsync -a --delete` which prunes mirror files when canonical files are deleted.

## What to land

- Delete `skills/refine/references/orchestrator-review.md` (canonical)
- Delete `skills/refine/references/specify.md` (canonical)
- Update `skills/refine/SKILL.md` Step 5 (specify invocation) to read from lifecycle canonical (`${CLAUDE_SKILL_DIR}/../lifecycle/references/specify.md`); orchestrator-review is reached transitively via lifecycle's specify.md (no separate refine SKILL.md edit needed for orchestrator-review)
- For specify: refine SKILL.md Step 5 already documents the §5 Transition + Hard Gate skip — keep that adaptation note explicit so the reader knows what differs
- For orchestrator-review: refine never reaches plan phase, so the P8 row is irrelevant for refine's invocations. No adaptation needed; refine just doesn't ever invoke the post-plan checklist.
- Run `just build-plugin` to regenerate mirror; pre-commit hook will reject drift if mirror not staged

## Touch points
