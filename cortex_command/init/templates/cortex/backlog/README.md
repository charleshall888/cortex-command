# Backlog

This directory holds project backlog items as individual markdown files with
YAML frontmatter. Each file is one ticket; `index.md` is an auto-generated
overview regenerated by `cortex` tooling.

## File layout

- `<slug>.md` - one markdown file per backlog item
- `index.md` - generated overview (do not edit by hand)

## Frontmatter schema

The frontmatter schema is owned by the cortex-core plugin. See
`plugins/cortex-core/skills/backlog/references/schema.md` for the authoritative
ticket template, or run `/cortex-backlog:backlog` to create items via the
supported skill (which produces schema-compliant frontmatter automatically).

## Status vocabulary

Canonical values: `backlog | refined | in_progress | implementing | review | complete | abandoned`.

- `backlog` - captured, not yet groomed
- `refined` - run through Clarify -> Research -> Spec, ready for overnight
- `in_progress` - currently being worked on
- `implementing` - mid-implement phase
- `review` - awaiting or in review
- `complete` - shipped and closed
- `abandoned` - withdrawn without being shipped

Legacy values (`open | in-progress | blocked | resolved | wontfix | done | ready`)
are still accepted by the tooling for backward compatibility, but new items
should use the canonical vocabulary above.

## Regenerating the index

Run your project's backlog-index recipe (for example `just backlog-index`) to
regenerate `index.md` from the individual ticket files.
