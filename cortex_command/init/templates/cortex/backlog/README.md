# Backlog

This directory holds project backlog items as individual markdown files with
YAML frontmatter. Each file is one ticket; `index.md` is an auto-generated
overview regenerated by `cortex` tooling.

## File layout

- `<slug>.md` - one markdown file per backlog item
- `index.md` - generated overview (do not edit by hand)

## Frontmatter schema

Each backlog item begins with a YAML frontmatter block:

```yaml
---
id: <short-slug-or-number>
title: <one-line human title>
status: <backlog | ready | refined | in-progress | complete | archived>
epic: <optional epic slug>
size: <S | M | L>
complexity: <simple | medium | complex>
priority: <low | medium | high | critical>
---
```

## Status vocabulary

- `backlog` - captured, not yet groomed
- `ready` - groomed, acceptance criteria clear enough to pick up
- `refined` - run through Clarify -> Research -> Spec, ready for overnight
- `in-progress` - currently being worked on
- `complete` - shipped and closed
- `archived` - withdrawn without being shipped

## Regenerating the index

Run your project's backlog-index recipe (for example `just backlog-index`) to
regenerate `index.md` from the individual ticket files.
