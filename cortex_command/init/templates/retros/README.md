# Retros

Dated, problem-only session retrospectives. Each file captures what went
wrong in a session - user corrections, mistakes, things missed, wrong
approaches - each paired with its consequence. Retros do NOT capture what
worked or accomplishments; the commit log already records those.

## Filename convention

```
YYYY-MM-DD-HHMM[-optional-context-tag].md
```

Examples:

- `2026-04-02-1629.md`
- `2026-04-07-0938-lifecycle-037-and-skill-improvements.md`

Create retros by running the `/cortex-interactive:retro` skill at the end of a session.
